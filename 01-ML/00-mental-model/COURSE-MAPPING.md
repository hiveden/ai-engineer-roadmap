## 3. ⚠️ 学习路径的真实位置（最容易搞混的事）

**直播课进度** 和 **仓库进度** 是**两条独立的时间线**：

| 维度 | Alex 当前位置 |
|---|---|
| 直播课位置 | **day-04，Python 语法阶段（装饰器）** |
| 距离课内开讲 ML day-01 | 还有 ~20 天（要先过完 Python 阶段） |
| 仓库 ML 第 1 层认知层 | ~40-50%（特征/标签/监督学习已 Level 2-3 通关） |
| 仓库 ML 章节 01-08 | 0 / 9（认知层结束才开始进算法主题） |

**关键事实**：Alex 在课堂里是 day-04 Python，但在仓库里**已经领先于课堂 ~20+ 天**进入了 ML 的概念阶段。

**为什么领先**：Alex 是 7 年全栈，Python 装饰器对他是"复习 30 秒"，课堂节奏被卡住。仓库是他的**超车道**——跳过 Python 阶段直接学 ML 工程化思维。

**为什么这是合理的**：
- Alex 的转型目标是 **AI Agent 工程师**，不是算法岗就业
- 课程节奏服务"零基础就业班"，Alex 不是这个画像
- 真正的护城河（跨语言部署、ML 在系统中的位置）在原课程里几乎没有，必须靠仓库补
- 课堂的算法 API 和面试套路在仓库里也不会教，**两边互补**

## 4. 当直播课进入 ML 阶段后的策略

将来某一天直播课进到 ML day-01（KNN），Alex 会同时面对两份"day-01"内容——课堂的 + 仓库的。**不要让两边互相绑架**：

| 在课堂上抓 | 在仓库里抓 |
|---|---|
| sklearn API 的调用细节 | "为什么这个算法 work" 的直觉 |
| 调参套路、超参数实操 | 算法 ↔ 业务场景的配对 |
| 算法岗面试常考点 | 跨语言落地（ONNX/PMML/PFA） |
| 同班同学的提问角度 | LLM 时代下的位置（哪些被替代、哪些没被替代） |
| 项目实战 demo 的执行经验 | 系统架构层的迁移决策 |

**冲突处理**：
- 课堂讲法和仓库讲法不一致 → **以仓库为准**（仓库的视角是工程化迁移，课堂的视角是知识传授）
- 课堂进度比仓库快 → **不追**，让仓库继续按 active recall 节奏走
- 仓库进度比课堂快 → **保持**，仓库本来就是超车道
