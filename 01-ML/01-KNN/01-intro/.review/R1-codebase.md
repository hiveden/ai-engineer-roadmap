# KNN K 值调优 Demo 量化分析报告

## 1. 数据几何约束分析

### 结论
**"下红"是几何必然，不是 bug**。

### 定量证据
- 样本分布：5 绿（喜欢）/ 6 红（不喜欢）
- 按 lead (y 轴) 分层统计：
  - lead < 8：绿 = 0, 红 = 3（哥斯拉 7.0, 寒战 6.5, 喜羊羊 4.0）
  - lead ∈ [8, 8.7)：绿 = 2, 红 = 2
  - lead ≥ 8.7：绿 = 3, 红 = 1

**关键发现**：决策边界自然在 lead ≈ 8 附近。下半区（lead < 8）完全被红样本占据，所以任何 k 值都会在该区产生红色预测。这与 KNN 的几何本质一致——没有近邻绿样本就无法预测绿。

---

## 2. 网格分辨率与 Voronoi 单元

### 结论
**当前分辨率足够，不是分辨率导致孤岛缺失**。

### 定量分析

#### 步长计算错误
```
代码第 113-114 行：
  step_x = (10.5 - 4) / n_grid      # ❌ 错算
  步长应该 = (10.5 - 4) / (n_grid - 1)  # ✓ np.linspace 的实际步长
```

对于 n_grid = 60：
- 代码计算：step_x = 0.108333
- 实际步长：step_x = 0.110169
- 缝隙：0.001836（相对误差 1.695%）

#### Voronoi 单元宽度
每个红样本的 Voronoi 单元宽度（= 最近异类样本距离 / 2）：

| 样本 | 坐标 | 最近绿样本 | 距离 | Voronoi 宽 | vs step_x |
|------|------|----------|------|-----------|----------|
| 哥斯拉 | (6.4, 7.0) | 阿凡达 | 2.052 | 1.026 | >> 0.110 |
| 长津湖 | (7.4, 8.0) | 阿凡达 | 0.640 | 0.320 | >> 0.110 |
| 寒战 | (7.2, 6.5) | 沙丘2 | 1.985 | 0.992 | >> 0.110 |
| 喜羊羊 | (5.5, 4.0) | 沙丘2 | 5.000 | 2.500 | >> 0.110 |
| **烂片高分A** | (8.2, 9.0) | 流浪地球2 | 0.100 | **0.050** | **< 0.110** |
| 烂片高分B | (8.0, 8.7) | 阿凡达 | 0.283 | 0.141 | >> 0.110 |

**关键发现**：
- 只有"烂片高分A"的 Voronoi 宽度小于步长（0.050 < 0.110）
- 其他红样本的单元宽度都远大于步长，理论上应该在 k=1 时形成孤岛并被当前分辨率显示
- 步长缝隙（1.695%）不足以隐藏任何大于 0.110 的孤岛

**结论**：k=1 时理论上应该能看到红色孤岛（除烂片高分A外），但用户反馈看不到。这不是分辨率问题，需要检查可视化逻辑。

## 3. K 参数空间扫描

### 扫描结果
三个分辨率下，对 k ∈ {1, 3, 5, 7, 9, 11} 统计绿色像素占比：

| n_grid | k=1 | k=3 | k=5 | k=7 | k=9 | k=11 |
|--------|-----|-----|-----|-----|-----|------|
| 60 | 36.6% | 31.3% | 26.6% | 31.1% | 35.3% | **0.0%** |
| 120 | 36.5% | 31.1% | 26.4% | 31.0% | 35.2% | **0.0%** |
| 200 | 36.4% | 31.0% | 26.3% | 30.9% | 35.1% | **0.0%** |

### 关键观察

1. **k 变化幅度**（n_grid=60 为例）：
   - k=1→3：绿% 下降 5.3%（过拟合收缩）
   - k=3→5：绿% 下降 4.8%
   - k=5→7：绿% 上升 4.6%（奇偶数交替）
   - k=9→11：绿% 下降 35.3%（**关键突变**）

2. **k=11 全红现象是真实的**：
   - 所有 3,600 格点（n_grid=60）全部预测红色
   - 原因：11 个样本中红色 6 票 > 绿色 5 票，任何点都无法翻转

3. **分辨率稳定性**：n_grid 从 60 增至 200，绿%曲线完全平行
   - Δ(k=1 绿%)：36.6% - 36.4% = 0.2%（可忽略）
   - 说明 n_grid=60 已足够表现 k 的影响

### 结论
**k 确实会改变边界形状，但效果有限**：
- k ∈ [1,9] 内绿% 波动范围仅 36.6% - 26.3% = 10.3%
- 用户主要看到的是"下红"（来自数据几何），而非"k 改变导致的边界移动"
- k=11 虽然文案提到，但由于 slider 上限为 9，用户无法体验

## 5. step_x / step_y 计算缝隙

### 定量分析

对 n_grid = 60：

| 轴 | 范围 | 代码计算 | np.linspace 实际 | 缝隙 | 相对误差 |
|---|------|---------|-----------------|------|---------|
| X (rating) | [4, 10.5] | 0.108333 | 0.110169 | 0.001836 | 1.695% |
| Y (lead) | [3, 11] | 0.133333 | 0.135593 | 0.002260 | 1.695% |

### Altair mark_rect 的影响

Altair 矩形规范：
```
矩形范围 = [x, x_end) × [y, y_end)
           = [x, x+step_x) × [y, y+step_y)
```

问题：
- x/y 来自 `np.linspace(..., n_grid)`，间距 = 0.110169
- 但 step_x 代码给的是 0.108333
- → **矩形宽度 < 相邻矩形间距** → 出现 1-2 像素细缝

### 覆盖率
```
应覆盖范围: 6.5 (x轴) × 8 (y轴)
代码给的:  60 × 0.108333 = 6.5 ✓ (x轴完全覆盖)
```

虽然数值上覆盖 100%，但由于步长不匹配，Altair 会在矩形间产生微小间隙（取决于浏览器渲染）。

### 结论
这是**代码缺陷但影响微弱**：
- 间隙宽度仅 1-2 像素，对决策区域视觉影响很小
- 不足以隐藏任何决策边界，更不足以导致"k 改变无效"的假象

## 6. 平票偏向分析

### 投票逻辑
代码第 78 行 和 第 120 行使用：
```python
if (y[_top_k] == 1).sum() > k / 2:  # 严格大于
    pred = 1
else:
    pred = 0
```

### 平票统计（n_grid=60）

| k | 平票格点数 | 占比 | 偏向 |
|---|----------|------|------|
| 2 | 932 | 25.9% | 红（因 1 > 2/2=1.0 是 False） |
| 4 | 846 | 23.5% | 红（因 2 > 4/2=2.0 是 False） |
| 6 | 435 | 12.1% | 红（因 3 > 6/2=3.0 是 False） |
| 8 | 547 | 15.2% | 红 |

### 偶数 k 的影响
```
k=偶数时出现平票 (绿票 = k/2) 的条件：
  - 严格 '>': green_count > k/2 → False → 预测红
  - 使用 '>=': green_count >= k/2 → True → 预测绿
```

对于 k=2：
```
1票绿 vs 1票红 → 严格 '>' 判断 1 > 1.0 = False → 预测红
如果改为 '>='，判断 1 >= 1.0 = True → 预测绿
```

### 结论
**这是一个设计选择，不是 bug**：
- 使用 `>` 在平票时倾向保守（预测不喜欢）
- Slider 只提供奇数 k [1,3,5,7,9]，不包括 k=2,4,6,8
- **在当前配置下，平票问题不会在用户界面出现**
- 如果扩展 k 范围到偶数，建议改为 `>=` 或 weighted voting

## 核心洞察与教学建议

用户主诉的现象 **既不是 bug，也不是代码问题，而是数据几何的自然结果**。

```
绿样本分布：
  lead >= 8: ████ (5个)
            - 流浪地球2, 阿凡达, 泰坦尼克, 复联4, 沙丘2
            - 全部高分 + 高吸引度

红样本分布：
  lead < 8:  ███ (3个)
            - 哥斯拉, 寒战, 喜羊羊
            - 低吸引度偏好者集聚区
  
  lead >= 8: ██ (3个)
            - 长津湖, 烂片高分A/B
            - 噪声/异常值
```

→ **没有近邻样本就无法改变预测，这是 KNN 的约束，不是参数问题**。

建议：
1. 在演示前加数据分布直方图，让用户看到 lead 的两极分化
2. 修复 B1（扩展 slider）让用户看到 k=11 的全红现象
3. 在文案中明确指出"下红区是数据的特性，不是 k 改变能解决的"
