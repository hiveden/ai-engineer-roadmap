# 工作流程：分类 / 回归

## 底稿

### 分类流程

1. 计算未知样本到每一个训练样本的距离
2. 将训练样本根据距离大小升序排列
3. 取出距离最近的 K 个训练样本
4. 进行多数表决，统计 K 个样本中哪个类别的样本个数最多
5. 将未知的样本归属到出现次数最多的类别

### 回归流程

1. 计算未知样本到每一个训练样本的距离
2. 将训练样本根据距离大小升序排列
3. 取出距离最近的 K 个训练样本
4. 把这个 K 个样本的目标值计算其平均值
5. 作为将未知的样本预测的值

## 总结

### 1. KNN 概念（K Nearest Neighbor）

一个样本最相似的 k 个样本中的大多数属于某一个类别，则该样本也属于这个类别。

### 2. KNN 分类流程

1. 计算未知样本到每一个训练样本的距离
2. 将训练样本根据距离大小升序排列
3. 取出距离最近的 K 个训练样本
4. 进行多数表决，统计 K 个样本中哪个类别的样本个数最多
5. 将未知的样本归属到出现次数最多的类别

### 3. KNN 回归流程

1. 计算未知样本到每一个训练样本的距离
2. 将训练样本根据距离大小升序排列
3. 取出距离最近的 K 个训练样本
4. 把这个 K 个样本的目标值计算其平均值
5. 将未知的样本预测的值了

### 4. K 值的选择

- K 值过小：过拟合
- K 值过大：欠拟合

## 直觉：五步法 = 完整流水线

前面 02 / 03 / 04 学的所有钩子，在这里被串成一条流水线：

```
数据 → 选 K → 算距离 → 排序 → 投票/平均 → 预测
```

**这就是 KNN 的全部**。分类用投票，回归用平均，只差最后一步。

## 分类 vs 回归：KNN 能做的两件事

"分类"和"回归"是 ML 里两大基本任务类型，**区别只在输出长什么样**：

| 任务 | 输出是什么 | 例子 |
|---|---|---|
| **分类** classification | **离散类别**（从有限标签里选一个）| "喜欢 / 不喜欢"、"猫 / 狗 / 鸟"、"垃圾邮件 / 正常邮件" |
| **回归** regression | **连续数值**（一个具体的数字）| "房价"、"明天股价"、"你给电影打几分（1-5 星）" |

简单判断：

- 你想预测的东西是 **从有限选项里挑一个** → **分类**
- 你想预测的东西是 **一个具体数字** → **回归**

KNN 同一思想能做这两件事——区别只在"找完 K 个邻居后怎么用":

- 邻居都是"喜欢 / 不喜欢"标签 → **投票**（哪类邻居多就预测哪类）→ 分类
- 邻居都是数字（如打了几分）→ **平均**（K 个邻居数字的均值）→ 回归

下面分别走一遍完整 5 步流程。

## 分类完整走一遍（豆瓣主线）

延续 [02 数据集](./02-接近程度.md)：9 部已知 + 流浪地球 3 `(8.5, 9.5)`，预测你会不会喜欢。选 **K=5**。

| 步 | 动作 | 豆瓣主线落地 |
|---|---|---|
| 1 | 计算距离 | 算流浪地球 3 到 9 部已知电影的欧式距离 |
| 2 | 升序排序 | 复仇者(0.32) → 泰坦尼克(0.50) → 流浪地球 2(0.54) → … |
| 3 | 取 Top K | K=5：复仇者 / 泰坦尼克 / 流浪地球 2 / 阿凡达 / 沙丘 2 |
| 4 | 多数表决 | 5 喜欢 + 0 不喜欢 |
| 5 | 输出预测 | **喜欢** |

> 完整计算过程见 [02 练习段](./02-接近程度.md)。

## 回归完整走一遍（预测打几分）

如果不是预测"喜欢/不喜欢"（分类），而是预测 **"你给流浪地球 3 打几分"**（数值），就用回归版。

数据要求改：历史标签从"喜欢/不喜欢"换成"具体打过几星"（数值）。

| 步 | 动作 | 豆瓣主线落地 |
|---|---|---|
| 1 | 计算距离 | 同分类 |
| 2 | 升序排序 | 同分类 |
| 3 | 取 Top K | 同分类（K=5） |
| **4** | **平均** | 假设 5 部最近邻你打过的分：4.5 / 4.0 / 4.3 / 4.0 / 4.5 → 平均 ≈ **4.26** |
| 5 | 输出预测 | **你给流浪地球 3 打约 4.3 分** |

## 五步对比：只差第 4 步

把分类和回归的 5 步并排放：

| 步 | 分类 | 回归 |
|---|---|---|
| 1 | 算距离 | 算距离 |
| 2 | 升序排序 | 升序排序 |
| 3 | 取 K 个邻居 | 取 K 个邻居 |
| **4** | **多数表决**：K 个邻居中票数最多的类别胜出 | **平均**：K 个邻居数值的均值 |
| 5 | 输出**类别** | 输出**数值** |

前 3 步完全相同（找最近的 K 个邻居），第 4 步分叉（投票还是平均），第 5 步输出（标签还是数字）。

→ KNN **核心一个、变种两个**。所以 sklearn 提供两个 API（`KNeighborsClassifier` / `KNeighborsRegressor`），见 [第 2 章 02-api](../../02-api/)。
