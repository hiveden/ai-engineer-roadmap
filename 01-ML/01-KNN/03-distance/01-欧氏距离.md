# 欧氏距离 Euclidean Distance

> **★ 掌握级别**

## 课程原文

欧氏距离 Euclidean Distance

---

## 公式

$$d(a, b) = \sqrt{\sum_{i=1}^{n}(a_i - b_i)^2}$$

每多一维就加一项平方，最后开方 —— 即**勾股定理 N 维推广**。

## 直觉

**两点之间的直线距离** —— 勾股定理在 N 维的推广。

## 几何形态：单位圆是圆

**单位圆 unit ball** —— 到原点距离恰好为 1 的所有点形成的形状。

欧氏距离下，方程 $x^2 + y^2 = 1$ —— **这正是平面圆**：

```
      ●●●●●●●
    ●●       ●●
   ●           ●
  ●             ●
  ●      +      ●     (+ = 原点)
  ●             ●
   ●           ●
    ●●       ●●
      ●●●●●●●
```

→ 等距点在 2 维是圆，3 维是球面，N 维是 **超球面 hypersphere**。这就是为什么"欧氏 = 直线距离" —— 沿半径就是直线。

## L2 范数视角

欧氏距离 = **L2 距离** = 两点差的 **L2 范数**：

$$\|x\|_2 = \sqrt{\sum_i x_i^2}$$

> 向量 / 范数 / 距离三者的关系，见 [`00-向量与范数`](./00-向量与范数.md)（前置）。

## 实算（豆瓣主线复习）

流浪地球 3 (8.5, 9.5) 到流浪地球 2 (8.3, 9.0)：

$$d = \sqrt{(8.5-8.3)^2 + (9.5-9.0)^2} = \sqrt{0.04 + 0.25} \approx 0.54$$

## 工程场景

- **KNN 默认距离**：sklearn `metric='euclidean'`（实际通过 `metric='minkowski'` + `p=2` 实现）
- **特征量纲对齐 + 各维度同等重要** 时最合适
- **经典应用**：图像相似度（像素向量）、文本 embedding 相似度、推荐系统（用户特征向量）

## 与其他距离的关系

| 距离 | 路径直觉 | 别名 |
|---|---|---|
| **欧氏**（本节）| 直接拉直线 | L2 / Minkowski p=2 |
| 曼哈顿 | 按网格走（下一节）| L1 |
| 切比雪夫 | 取最大维度差 | L∞ |
| 闵可夫斯基 | 通用参数化 | 含上面三种 |

## 关键启示

KNN 默认用欧氏，**绝大多数场景用它就够**。后续三种是特殊场景的优化或统一形式。
