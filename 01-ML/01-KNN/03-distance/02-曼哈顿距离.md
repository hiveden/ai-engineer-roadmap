# 曼哈顿距离 Manhattan Distance

> **★ 掌握级别**

## 课程原文

曼哈顿距离 Manhattan Distance

---

## 公式

$$d(a, b) = \sum_{i=1}^{n} \lvert a_i - b_i \rvert$$

每个维度差的 **绝对值之和**（不平方、不开方）。

## 直觉：出租车走法

想象你在曼哈顿（纽约市）街道走路 —— 街道是网格状的，你不能斜穿楼房。从 A 到 B：

- 欧氏距离：直接拉直线（穿楼）
- **曼哈顿距离：按网格一段一段走** → 横走 N 米 + 竖走 M 米

所以也叫 **"出租车距离" Taxicab distance** 或 **L1 距离**。

## 几何形态：单位圆是菱形

曼哈顿距离下，方程 $\lvert x \rvert + \lvert y \rvert = 1$ —— 4 条直线段拼成 **菱形 diamond**：

```
        ● (0, 1)
       / \
      /   \
     /     \
    /       \
(-1,0)●——+——●(1,0)
    \       /
     \     /
      \   /
       \ /
        ● (0, -1)
```

**为什么是菱形?** 横走 0.6 + 竖走 0.4 = 1，横走 1.0 + 竖走 0 = 1，**总和恒为 1** 划出 4 个顶点的菱形。这正是"出租车走法"的几何边界。

## L1 范数视角

曼哈顿距离也叫 **L1 距离**，对应 **L1 范数**：

$$\|x\|_1 = \sum_i \lvert x_i \rvert$$

→ 各维度绝对值之和。

## 实算（豆瓣主线对比）

流浪地球 3 (8.5, 9.5) 到流浪地球 2 (8.3, 9.0)：

$$d = \lvert 8.5-8.3 \rvert + \lvert 9.5-9.0 \rvert = 0.2 + 0.5 = 0.7$$

对比欧氏 ≈ 0.54 → **曼哈顿距离更长**（按网格走比直线远）。

## 工程场景

- **特征独立、量纲不同**：每个维度物理意义不同时（如"年龄+收入"），曼哈顿避免某一维度被平方放大支配
- **稀疏数据**：文本 / 用户行为向量（大量 0 值），曼哈顿对差异更敏感
- **棋盘 / 网格 / 地图导航**：实际 routing 走法
- **sklearn**：`metric='manhattan'`

## 与欧氏对比

| 维度 | 欧氏 | 曼哈顿 |
|---|---|---|
| 公式 | $\sqrt{\sum (a_i-b_i)^2}$ | $\sum \lvert a_i-b_i \rvert$ |
| 路径 | 拉直线 | 走网格 |
| 别名 | L2 / Minkowski p=2 | L1 / Taxicab / Minkowski p=1 |
| 异常值敏感性 | 平方后放大 → 敏感 | 取绝对值 → 不那么敏感 |

## 关键启示

曼哈顿是 KNN 第二常见的距离公式 —— 适合 **特征独立、稀疏数据、网格场景**。判断口诀:**有平方放大就用欧氏，无平方放大就用曼哈顿**。
