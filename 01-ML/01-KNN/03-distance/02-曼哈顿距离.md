# 曼哈顿距离 Manhattan Distance

> ★ 掌握级别

## 课程原文

曼哈顿距离 Manhattan Distance

---

## KNN 视角：什么时候不用欧氏用它？

如果 KNN 数据集里的特征是 **独立计数**（比如"打斗镜头数 / 接吻镜头数"），欧氏距离会用平方放大维度差异——但你可能希望两个特征**公平相加**，各自独立贡献。

**曼哈顿距离** = 各维度差的绝对值相加，**不平方放大**。在这种场景下 KNN 找邻居更合理。

## 直觉：出租车走法

想象你在曼哈顿（纽约市）街道走路 —— 街道是网格状，你不能斜穿楼房。从 A 到 B：

- 欧氏：直接拉直线（穿楼）
- **曼哈顿**：按网格 → 横走 N 米 + 竖走 M 米

也叫 **"出租车距离" Taxicab distance**。

## 公式

$$d(a, b) = \sum_{i=1}^{n} \lvert a_i - b_i \rvert$$

各维度差的绝对值之和（不平方、不开方）。

## 实算

流浪地球 3 (8.5, 9.5) 到流浪地球 2 (8.3, 9.0)：

$$d = \lvert 8.5-8.3 \rvert + \lvert 9.5-9.0 \rvert = 0.2 + 0.5 = 0.7$$

对比欧氏 ≈ 0.54 → **曼哈顿距离更长**（按网格走比直线远）。

### 几何示意：KNN 用曼哈顿（网格走）

```
       主演
       9.5 ┤  ●(8.5, 9.5) 流浪地球 3
           │  │
           │  │ ← 竖走 0.5
           │  │
       9.0 ┤  ●━━━━━●(8.3, 9.0) 流浪地球 2
           │     横走 0.2
           └─────────────────► 评分
              8.3   8.5

距离 = |0.2| + |0.5| = 0.7
```

→ 同两点，欧氏拉直线（0.54）vs 曼哈顿走网格（0.7）。

## KNN 何时用曼哈顿

- **特征独立 / 量纲不同**：每维物理意义不同（如"年龄+收入"），曼哈顿避免某维平方放大独裁
- **稀疏数据**：文本 / 用户行为向量（大量 0），曼哈顿对差异更敏感
- **异常值多**：取绝对值不平方，KNN 受异常值影响更小
- **sklearn**：`metric='manhattan'` 或 `metric='minkowski', p=1`

## 与欧氏对比（KNN 视角）

| 维度 | 欧氏（KNN 默认）| 曼哈顿 |
|---|---|---|
| 公式 | $\sqrt{\sum (a_i-b_i)^2}$ | $\sum \lvert a_i-b_i \rvert$ |
| 路径 | 拉直线 | 走网格 |
| 异常值 | 平方放大 → 敏感 | 取绝对值 → 不敏感 |
| 典型 KNN 场景 | 量纲对齐的连续特征 | 特征独立 / 稀疏数据 |

## 关键启示

KNN 第二常见的距离公式 —— 适合 **特征独立、稀疏数据**。判断口诀：**不希望某维度平方放大独裁就用曼哈顿**。
