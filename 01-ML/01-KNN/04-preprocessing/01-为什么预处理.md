# 为什么做归一化和标准化

> ★ 掌握级别

## 课程原文

特征的单位或者大小相差较大，或者某特征的方差相比其他的特征要大出几个数量级，容易影响（支配）预测结果，使得一些模型（算法）无法学习到其它的特征。

比如 KNN 欧式距离的计算。

---

## 直觉：量纲不齐 → 距离被一个特征支配

KNN 用欧式距离找邻居，**距离对每个特征的"敏感度"取决于该特征的数值范围**。

如果两个特征量纲差异大：

- 特征 1：范围 `[0, 10]`（如评分）
- 特征 2：范围 `[0, 200]`（如时长，分钟）

那么"时长差 50" 在距离上的贡献远远超过"评分差 5"——本质上 KNN 只在用时长分类，评分几乎被无视。

## 量纲问题示例

假设两部电影：

- A：(评分=8.5, 时长=180)
- B：(评分=8.0, 时长=130)

欧式距离：

$$d = \sqrt{(8.5-8.0)^2 + (180-130)^2} = \sqrt{0.25 + 2500} \approx 50.0$$

**时长差²（2500）是评分差²（0.25）的 10000 倍** —— 距离几乎完全等于时长差，评分对结果毫无贡献。

### 几何示意：特征空间被一个特征拉长

```
未缩放（评分 0-10 vs 时长 0-200）：

时长 ↑
 200 ┤  ●  ●     ●        ← 空间在时长方向被严重拉长
     │              ← 各点在评分方向几乎贴在一起
 100 ┤  ●  ●     ●
     │
   0 ┤  ●  ●     ●
     └──────────────────► 评分
        0    5      10

→ KNN 量距离 ≈ 量时长（评分被无视）
```

```
缩放后（两特征都映射到 [0, 1]）：

时长' ↑
 1.0 ┤   ●     ●        ●
     │
     │
 0.5 ┤      ●  ●  ●
     │
     │
 0.0 ┤   ●        ●
     └────────────────► 评分'
       0.0   0.5   1.0

→ 两特征对距离贡献变得可比
```

| 距离对比 | 计算 | 结果 |
|---|---|---|
| 原始(8.5,180) vs (8.0,130) | $\sqrt{0.25 + 2500}$ | $\approx 50.0$（时长支配 99.99%）|
| 归一化(0.85,0.90) vs (0.80,0.65) | $\sqrt{0.0025 + 0.0625}$ | $\approx 0.255$（两特征公平）|

## 解决方案：把所有特征拉到同一量纲

两种主流做法：

| 方法 | 思路 | 主场 |
|---|---|---|
| **归一化** Min-Max | 把每个特征压缩到固定区间 `[0, 1]` | [`02-归一化`](./02-归一化.md) |
| **标准化** Z-score | 把每个特征转换为均值 0、标准差 1 的分布 | [`03-标准化`](./03-标准化.md) |

→ 两种方法都让 **每个特征对距离的贡献变得可比**，避免某个量纲大的特征独裁。
