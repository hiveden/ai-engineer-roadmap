## 底稿

### PPT

> 【知道】交叉验证

是一种模型评估方法，利用分割后的数据集将训练集划分为 n 份，拿一份做验证集、其他 n－1 份做训练集。

- **交叉验证法原理**：将数据集划分为 cv=4 份

1. 第一次：把第一份数据做验证集，其他数据做训练
2. 第二次：把第二份数据做验证集，其他数据做训练
3. …… 以此类推，总共训练 4 次，评估 4 次。
4. 使用训练集 + 验证集多次评估模型，取平均值做交叉验证为模型得分。

   更换模型超参数，比如 KNN 的 K（邻居数），再做交叉验证。
5. 若 k=5 模型得分最好，再使用全部训练集（训练集 + 验证集）对 k=5 模型再训练一遍（即正常训练，不要遗忘），再使用独立测试集对 k=5 模型做评估

交叉验证法，是划分数据集的一种方法，目的就是为了得到更加准确可信的模型评分。

## 直觉

03 章 `train_test_split` 切了一刀：80% 训练、20% 测试。问题是——**这一刀是随机的**。

> `random_state` 是 sklearn 的"随机数种子"参数。给同一个数（比如 20），切出来的训练/测试集每次都一样（可复现）；换个数（42、7…），切法就完全不同。它**不影响模型**，只影响"哪些样本进训练、哪些进测试"。

同一个 k=5，不同 `random_state` 的真实评分（鸢尾花，sklearn 实测）：

| random_state | 7 | 20 | 0 | 42 | 1 | 100 |
|---|---|---|---|---|---|---|
| k=5 score | **0.9000** | 0.9667 | 0.9667 | 1.0000 | 1.0000 | 1.0000 |

跨度从 0.90 到 1.00 ——**10 个百分点**。测试集才 30 条，**多答错 1 道题就跳 3.3 个百分点**，运气最差的 seed=7 比运气好的多错了 3 道——这是抽样的运气，不是模型的本事。

**交叉验证（Cross Validation，简称 CV）** 的思路：把训练集切成 n 份，每份轮流当一次验证集，跑 n 次得 n 个评分，取平均。

> 类比月考：判一个学生水平，看 1 次月考会抖、看 4 次月考取均分才稳。CV 就是数据版的"看 4 次月考"。

#### 为什么是 $1/\sqrt{n}$ 而不是 $1/n$？

体重秤单次称有 ±1 kg 噪声，多称几次取平均，误差怎么缩？

| 称几次 | 平均后误差 | 缩小倍数 |
|---|---|---|
| 1 次 | ±1.00 kg | 1× |
| 4 次 | ±0.50 kg | **2 倍**（不是 4 倍） |
| 16 次 | ±0.25 kg | **4 倍**（不是 16 倍） |
| 100 次 | ±0.10 kg | 10 倍 |

**反直觉点**：你以为称 4 次误差就缩 4 倍，其实只缩 2 倍。**想把误差减半，得把次数翻 4 倍**。

为什么慢这么多？因为单次的抖动有正有负——这次秤显示高了 +0.8 kg，下次可能低了 −0.5 kg，**平均时正负部分相消**。如果 4 次抖动都恰好同方向（全部 +1），平均还是 +1（误差完全没缩）；但 4 次同向是小概率事件，更常见的是有正有负、相消大半、剩下一点。

数学上的精确公式：

$$\text{平均后的误差} = \frac{\text{单次误差}}{\sqrt{n}}$$

代回上面的体重秤表格验证：

| n | $\pm 1 / \sqrt{n}$ | 对得上吗 |
|---|---|---|
| 4 | $\pm 1 / 2 = \pm 0.50$ kg | ✓ |
| 16 | $\pm 1 / 4 = \pm 0.25$ kg | ✓ |
| 100 | $\pm 1 / 10 = \pm 0.10$ kg | ✓ |

记忆点：$\sqrt{4} = 2$（翻 4 倍 → 误差减半）；$\sqrt{100} = 10$（翻 100 倍 → 误差减到 1/10）。

落到 CV：把"称体重"换成"评模型"，把"单次称的 ±1 kg"换成"单切分的评分抖动"。

| 评估方式 | 评分抖动估计 |
|---|---|
| 单切分（n=1） | ±10 个百分点（前面 6 个种子 0.90~1.00 实测） |
| cv=4 | $\pm 10 / \sqrt{4} = \pm 5$ 个百分点 |
| cv=16 | $\pm 10 / \sqrt{16} = \pm 2.5$ 个百分点 |

副产品比"更稳"还重要：**每条样本都当过一次验证集**——单切分里训练集那 120 条数据对评分没投过票，cv=4 里它们全投过，没有漏网的"运气分"。

### 三集职责：CV 切训练集内部，测试集独立

CV 切的是**训练集内部**，测试集从头到尾不掺和。一定要分清三个集合各自的职责：

```
全部数据
│
├── 训练集 ──→ CV 在内部反复切 n 折，调超参用
│       ┌──────┬──────┬──────┬──────┐
│       │ 折 1 │ 折 2 │ 折 3 │ 折 4 │  每轮换 1 折当验证折，
│       └──────┴──────┴──────┴──────┘  其余 n−1 折当训练折
│
└── 测试集 ──→ 全程不动，最后一锤定音验收
```

| 集合 | 出场次数 | 用途 |
|---|---|---|
| 训练折（n−1 份） | n 次 | 在每折里"上课" |
| 验证折（1 份） | n 次 | 在每折里"小测"，给超参打分 |
| 测试集 | 1 次 | 选定最优超参 + 全量重训后，**最终验收** |

**为什么测试集要藏到最后？** 如果你拿测试集来挑 k，那测试集就被"摸过"了——它的评分就不再代表"模型见到陌生数据的真实表现"。CV 的存在就是给你一个**不动测试集**的渠道来调超参。

## 数学方法：cv=4 走查

设训练集 100 条，cv=4 → 每折 25 条。

```
        ├──── 训练集 100 条 ────┤
        ┌─────┬─────┬─────┬─────┐
fold 1: │ Val │  Tr │  Tr │  Tr │ → 训练 → 验证 → 准确率₁
fold 2: │  Tr │ Val │  Tr │  Tr │ → 训练 → 验证 → 准确率₂
fold 3: │  Tr │  Tr │ Val │  Tr │ → 训练 → 验证 → 准确率₃
fold 4: │  Tr │  Tr │  Tr │ Val │ → 训练 → 验证 → 准确率₄
        └─────┴─────┴─────┴─────┘
                                  ↓
                         平均 = k=5 这组超参的 CV 得分
```

单组超参（比如 k=5）的 CV 得分 = 4 折准确率的平均：

$$\text{CV score} = \frac{1}{n} \sum_{i=1}^{n} \text{accuracy}_i$$

**多组超参就外层循环**：把这套四折流程对 `k=1, 3, 5, 7, 9, 11` 各跑一遍，得 6 个 CV 得分，挑最高的那个 k——这就是和[网格搜索](./02-网格搜索.md)的接口。

**选定 k 之后必须全量重训**：用最佳 k（比如 k=5）+ **全部训练集**（训练折 + 验证折= 100 条）重训一次模型，最后用从头没动过的测试集做一次性验收。

为什么不能直接拿 CV 里某一折的模型？因为 CV 每折的模型只见过 75 条数据，不是最终要交付的模型。最终模型应该吃满 100 条训练数据。

## 代码落地：sklearn `cross_val_score`

```python
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.neighbors import KNeighborsClassifier

X, y = load_iris(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=20
)

# CV 只在训练集内部切，测试集这里不出现
scores = cross_val_score(
    KNeighborsClassifier(n_neighbors=5),
    X_train, y_train, cv=4
)
print(scores)         # [0.9667, 0.9667, 0.9667, 0.9667]
print(scores.mean())  # 0.9667
```

> 这里 4 折恰好都是 0.9667，是因为鸢尾花数据规整 + 训练折每折 30 条都恰好答对 29 题。换 seed=1 切训练集再 cv=4，会得到 `[0.967, 0.900, 0.967, 0.900]` 平均 0.9333——现实数据通常有抖动，CV 取平均的意义就在这里。

流程动作 ↔ 代码对照：

| 动作 | 代码 |
|---|---|
| 滚动切 n 折 + 训练验证 | `cv=4` 参数自动完成 |
| 取 n 折平均得 CV 得分 | `scores.mean()` |
| 多超参循环挑最优 | 套外层循环 → 让位[网格搜索](./02-网格搜索.md) |
| 选定 k 后全量重训 + 测试集验收 | `KNeighborsClassifier(k).fit(X_train, y_train).score(X_test, y_test)` |

## 关键启示

| 维度 | 单次 train/test 切分 | 交叉验证 cv=n |
|---|---|---|
| 评分来源 | 1 个测试集打 1 次分 | n 个验证折打 n 次分取平均 |
| 抖动幅度 | 大（$\sigma$） | 小（约 $\sigma/\sqrt{n}$） |
| 样本利用 | 测试集那部分没投过票 | 训练集每条都当过验证 |
| 测试集用途 | 既调参又验收（污染风险） | 全程不动，最后一次验收 |
| 适用场景 | 数据极多、评分波动小 | 数据有限、要可信评分 |

CV 解决"评分可信"，调参组合还得靠[网格搜索](./02-网格搜索.md)——两者合体才是完整的超参选择方案。
