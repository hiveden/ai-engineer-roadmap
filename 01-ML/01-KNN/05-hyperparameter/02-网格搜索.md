利用 KNN 算法对鸢尾花分类 – 交叉验证网格搜索：

```python
"""
交叉验证解释：
    概述：交叉验证是一种更完善、可信度更高的一种模型评估方法。
    思路：把数据集分 N 份，每次都去 1 份当做测试集，其他当做训练集，然后给模型评分
    然后再用下一份当做测试集，其他当做训练集（皇帝轮流坐，今天到你家）。最后计算所有的
    均值，当模型最终评分。
    思路是：
        把数据分成 N 份，例如数据分成 4 份，4 折交叉验证
            第 1 次：把第 1 份数据作为验证集（测试集），其他当做训练集，训练模型，模型预测，获取：准确率→准确率 1
            第 2 次：把第 2 份数据作为验证集（测试集），其他当做训练集，训练模型,模型预测，获取：准确率→准确率 2
            第 3 次：把第 3 份数据作为验证集（测试集），其他当做训练集，训练模型，模型预测，获取：准确率→准确率 3
            第 4 次：把第 4 份数据作为验证集（测试集），其他当做训练集，训练模型，模型预测，获取：准确率→准确率 4
            然后计算上述 4 次的准确率的平均值，作为模型最终准确率
            假设第 4 次最好（准确率最高），则：用全部数据（训练集+测试集）训练模型，再用第 4 次的测试集对模型进行测试
        目的：
        为了让模型最终结果更加准确
        好处：相比单一切分的训练集和测试集获取的评分，经过交叉验证可信度更高

网格搜索：
    概述：机器学习内置 API，一般结合交叉验证使用，通过 API 实现找到最优超参数
    目的：网格搜索 + 交叉验证：让模型变得更强
"""

from sklearn.datasets import load_iris   # 鸢尾花数据集
from sklearn.model_selection import train_test_split, GridSearchCV   # 训练和测试数据集分割   寻找最优超参（网格搜索+交叉验证）
from sklearn.preprocessing import StandardScaler   # 数据预处理  数据标准化
from sklearn.neighbors import KNeighborsClassifier   # 模型，KNN 分类器
from sklearn.metrics import accuracy_score   # 模型评估，计算模型的准确率
from sklearn.pipeline import Pipeline

# 加载鸢尾花数据集
mydataset = load_iris()

# 数据预处理，切分数据集，比例 4:1
# 参 1：数据集    参 2：目标值    参 3：测试集所占比例（随机抽）    参 4：随机数种子
# 返回值：训练集特征，测试集特征，训练集标签，测试集标签
x_train, x_test, y_train, y_test = train_test_split(
    mydataset.data, mydataset.target, test_size=0.3, random_state=20
)
x_train_, x_test_ = x_train.copy(), x_test.copy()

# 定义创建标准化对象
transfer = StandardScaler()

''' 标准做法，利用管道工具
    在交叉验证的每一折内部，独立进行标准化
'''
pipe = Pipeline([
    ('std', transfer),                      # 标准化工具
    ('knn2', KNeighborsClassifier())        # 分类器定义  不写超参，通过检索决定
])

# 参数搜索空间
param_dict = {'knn2__n_neighbors': [1, 3, 5, 7, 9, 11]}
# knn2__n_neighbors  knn2 是你的模型名字，后面必须 2 个下划线，之后 n_neighbors 就是该模型固有参数

# 3. 使用网格搜索（此时传入管道）
# 4.3 创建 网格搜索对象，寻找最优参数（交叉验证 + 网格搜索）
# 参 1：分类器模型对象（管道）   参 2：超参的可能出现的值     参 3：每一组进行交叉验证的次数 cross validation
# 返回值：经过处理后的模型对象
estimator = GridSearchCV(pipe, param_dict, cv=4)
estimator.fit(x_train, y_train)

# 4.5 打印最优参数组合
print(estimator.best_score_)      # 最优准确率             0.9333333333333333
print(estimator.best_params_)     # 最优参数组合           {'n_neighbors': 7}
print(estimator.best_estimator_)  # 获得最优模型，用于后期推理   KNeighborsClassifier(n_neighbors=7)

# 具体的交叉验证中间结果，可不查看
# 每种参数组合都经过 4 次验证（split0 ～ split3），每次交叉验证有 6 个值（k 从 1~11），然后计算平均、标准差等统计量。
print(estimator.cv_results_)
print('=' * 20)

# 确定了最佳模型，开启正式训练流程：
# 利用所有数据进行训练，先对数据集进行标准化
# 数据集预处理-数据标准化     fit + transform  记录均值方差   针对训练集
x_train_ = transfer.fit_transform(x_train_)

# 让测试集的均值和方差，转换测试集数据；仅仅 transform   针对测试集
x_test_ = transfer.transform(x_test_)

# 5 模型评估
# 5.1 "重新定义"最佳 模型 最优超参
estimator = KNeighborsClassifier(n_neighbors=7)
# 或者 estimator = estimator.best_estimator_

# 5.2 使用所有训练集数据重新进行训练
estimator.fit(x_train_, y_train)
# 5.3 模型推理
y_predict = estimator.predict(x_test_)
# 5.4 模型评估
myscore = accuracy_score(y_test, y_predict)   # 用了全部训练集，因此准确率会更高。
print(myscore)
```

## 直觉

01 章末尾说过：选完 k=5 算出 CV 得分后，要换 `k=1, 3, 7, 9, 11` 各跑一次 CV，挑得分最高的 k——这就是网格搜索。名字唬人，本质是**穷举**。

> 类比试衣服：一件件试，每件再各角度照镜子定夺。每件衣服 = 一组超参，照镜子 = 该组超参的 CV 评分。

「网格」的含义：把多个超参的候选值摆成表格，每个格子是一组组合。

```
1 维网格（单超参 k 取 6 个值）：
  ┌────┬────┬────┬────┬────┬─────┐
  │k=1 │k=3 │k=5 │k=7 │k=9 │k=11 │   → 6 个格子
  └────┴────┴────┴────┴────┴─────┘

2 维网格（k 6 个 × metric 2 个）：
                k=1   k=3   k=5   k=7   k=9   k=11
  euclidean      ·     ·     ·     ·     ·     ·
  manhattan      ·     ·     ·     ·     ·     ·   → 12 个格子
```

每个格子要做一次完整 CV → **总训练次数 = 格子数 × 折数**。

### 维度爆炸：为什么"超参越多越难"

| 候选超参 | 格子数 | × cv=4 | 总训练次数 |
|---|---|---|---|
| k 6 个值 | 6 | × 4 | **24** |
| k 6 + metric 2 | 12 | × 4 | **48** |
| k 6 + metric 2 + weights 2 | 24 | × 4 | **96** |
| 三维各 10 个值 + cv=10 | 1000 | × 10 | **10000** |

线性加超参 → 指数级增长训练次数。这就是"超空间网格搜索（暴力破解）越难"的具象——网格搜索是**带成本的工具**，不是免费午餐。

应对（不展开，记名字即可）：
- **粗调 + 细调** 两段策略：先在大跨度稀疏值上找区域，再在该区域密集采样
- `RandomizedSearchCV`：随机采样而非穷举
- 贝叶斯优化：用历史评分指导下一次采哪里

---

## 数学方法：组合数公式

设有 $d$ 个超参，第 $i$ 个超参有 $c_i$ 个候选值，CV 折数为 $n$：

$$\text{总训练次数} = n \times \prod_{i=1}^{d} c_i$$

代入鸢尾花例子（单超参 $k$ 取 6 个值，cv=4）：

$$4 \times 6 = 24 \text{ 次训练}$$

加一维 metric 取 2 个值：$4 \times 6 \times 2 = 48$ 次。每加一维超参乘上去——这就是"维度爆炸"的数学根源。

## 代码落地：sklearn `GridSearchCV` + Pipeline

底稿代码已是完整 demo（鸢尾花 + 标准化 + KNN）。讲解层精读三个关键点。

### 1. Pipeline 双下划线命名机制

```python
pipe = Pipeline([
    ('std', StandardScaler()),
    ('knn2', KNeighborsClassifier())
])
param_dict = {'knn2__n_neighbors': [1, 3, 5, 7, 9, 11]}
```

`knn2__n_neighbors` 拆开看：

| 段 | 含义 |
|---|---|
| `knn2` | Pipeline 里给 KNN 步骤起的名字（任意命名） |
| `__` | 双下划线，sklearn 约定的"嵌套寻址"分隔符 |
| `n_neighbors` | KNN 自己的参数名 |

这个机制让网格搜索精确告诉 sklearn"我要调的是 `knn2` 这一步的 `n_neighbors`"——一个 Pipeline 可能有多个步骤都有同名参数，双下划线消除歧义。

### 2. 为什么标准化要包在 Pipeline 里

如果在 CV 之前就 `fit_transform(X_train)`，那 CV 切出来的"训练折"和"验证折"会**共享同一组均值/方差**——验证折提前"摸过"了它本不该见到的统计量，造成轻微数据泄漏。

把标准化包进 Pipeline 后：每折独立 `fit_transform` 训练折、用训练折的均值方差 `transform` 验证折——干净、无泄漏。（标准化原理本身见 [`../04b-scaling/03-标准化.md`](../04b-scaling/03-标准化.md)。）

### 3. 三件套：取最优结果

```python
estimator.best_score_      # 最优 CV 得分
estimator.best_params_     # 最优超参组合（字典）
estimator.best_estimator_  # 最优模型（已 fit 好的 Pipeline 对象）
```

底稿代码用真实 sklearn 跑出来：

```
best_score_   = 0.9811
best_params_  = {'knn2__n_neighbors': 3}
test set acc  = 0.9111
```

→ k=3 是最优；用全量训练集重训后在 30 条独立测试集上 91.1%。

实战用法：拿 `best_params_` 写报告 / 拿 `best_estimator_` 直接 `predict`，不用再手动重训。

## 关键启示

| 维度 | 单切分 | CV（01 章） | CV + 网格搜索（本章） |
|---|---|---|---|
| 评分可信 | × 抖动 ±10pp | ✓ 抖动 $\pm 10/\sqrt{n}$ pp | ✓ |
| 超参选择 | 手调 | 手调 | **自动穷举** |
| 训练次数 | 1 | $n$（折数） | $n \times \prod c_i$ |
| 边界 | 数据少时不可信 | 缺超参组合机制 | 超空间维度爆炸 |

CV + 网格搜索是 KNN（以及其他经典模型）调参的**完整解决方案**，但要警惕维度爆炸的成本。
