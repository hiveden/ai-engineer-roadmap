**实际 UI**（按这个写录屏稿）：
- **数据切换**：02 是 9 部真实电影分类，03 是 126 部分类，04 切到 **80 部新合成数据 + 连续目标**
  - 完成度 ≈ 5·rating + 5·lead − 30 + N(0, 8)，clip 到 [0, 100]
  - 目标：观看完成度（0-100%），不再是离散标签
- **4 个控件**：k 滑块（1-50, step=1）+ 加权开关 + 新电影评分 + 新电影主演
- **决策面**：60×50 网格 mark_rect + viridis 色阶（0-100%，蓝→紫→黄）+ 训练点（颜色 = 完成度，与背景同色阶）+ 红菱形新电影（与训练点 viridis 色阶区分）+ 金边 top-k + 红虚线 k 半径圆
- **状态卡**：k 小⚠️/适中✓/k 大⚠️ + **MAE / RMSE / R²** 三指标（不是准确率！）
- **预测卡**：预测完成度 X.X% + top-k 邻居完成度 min/max/标准差
- **残差散点图**：横轴真实完成度、纵轴预测完成度、对角虚线 = 完美预测、点颜色 = 残差（红/蓝双向）

**4 个关键档位**（demo 自带的教学结构）：

| k 档位 | 现象 | 教学要点 |
|---|---|---|
| **k=1** | 决策面斑驳，每格颜色 = 最近训练点完成度 | 高方差：跟着噪声 |
| **k=7** ≈√80 | 决策面平滑，保留局部结构（左下冷 / 右上暖）| 拐点 |
| **k=20** | 决策面进一步光滑 | 偏差略增、方差大降 |
| **k=80** | 决策面变成**单一颜色 = 全局均值（≈50%）** | 极端欠拟合 |

**states 操作清单**（草拟，正稿前实跑 demo 校准）：

```yaml
states:
  1. intro     — 静帧介绍 UI（重点：决策面是 viridis 连续色阶，不是双色块）
  2. k7        — k=7, weighted=off, rating=7.5, lead=8.0
                 expect: 预测完成度 ~50-60% · 决策面平滑 · MAE 适中
  3. corner-hi — k=7, rating=10.5, lead=11.5（右上暖区）
                 expect: 预测 ~80-90%
  4. corner-lo — k=7, rating=3.5, lead=3.5（左下冷区）
                 expect: 预测 ~5-15%
  5. k1        — k=1, weighted=off, rating=7.5, lead=8.0
                 expect: 决策面斑驳 · MAE 飙升 · 状态卡=高方差⚠️
  6. k80       — k=80, weighted=off
                 expect: 决策面单一颜色 ≈ 50% · R² 接近 0 · 状态卡=高偏差⚠️
  7. residual  — 滚动到残差散点图，hover 几个偏离对角线远的点
                 expect: tooltip 显示残差值
```
