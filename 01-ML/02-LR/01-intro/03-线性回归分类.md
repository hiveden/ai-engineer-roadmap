### 笔记

> 【知道】线性回归分类

- **一元线性回归**：y = kx + b，目标值只与一个因变量有关系
- **多元线性回归**：目标值与多个因变量有关系

---

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

一句话：**一元 = 一根旋钮（k）+ 偏置（b）；多元 = 多根旋钮（w₁, w₂, ...）+ 偏置**。

教学意义上分一元/多元，工程上没区别——多元就是把 $kx$ 换成 $\mathbf{w}^T \mathbf{x}$，公式更简洁。

### 一元 vs 多元的本质区别

| 维度 | 一元 | 多元 |
|---|---|---|
| **公式** | $y = kx + b$ | $y = \mathbf{w}^T \mathbf{x} + b$ |
| **几何** | 2D 平面上一条**直线** | n+1 维空间一个**超平面** |
| **参数** | 2 个（k, b） | n+1 个（n 个 w + 1 个 b） |
| **可视化** | 散点 + 拟合直线（人眼可读） | 高维不可视；2D 投影只能看片面 |
| **正规方程** | 闭式公式 $k = \frac{\sum (x_i - \bar{x})(y_i - \bar{y})}{\sum (x_i - \bar{x})^2}$ | 矩阵公式 $\mathbf{w} = (X^T X)^{-1} X^T \mathbf{y}$ |
| **典型例** | 身高 → 体重 | 房价 = 面积+位置+楼层+朝向+... |

### 多项式线性回归（PPT slide 11 补充）

#### **【场景】** 多项式从哪里来：业务关系本身就是弯的

很多业务关系**画出来不是直线**——直接用 $y = kx + b$ 拟合必然失败。常见的弯曲场景：

| 业务/物理场景 | 关系直觉 | 弯曲类型 |
|---|---|---|
| **汽车油耗 vs 车速** | 低速怠速浪费、中速最省、高速风阻飙升 | **U 形抛物线**（二次） |
| **风阻 vs 车速** | $F_\text{风阻} = \frac{1}{2}\rho v^2 C_d A$，与速度的**平方**成正比 | **平方关系** |
| **自由落体距离 vs 时间** | 物理定律 $s = \frac{1}{2}gt^2$ | **平方关系** |
| **房价 vs 面积** | 100 平和 200 平不是简单 2 倍——大房有豪宅溢价或刚需折扣 | 弯曲（二次或更高） |
| **学习熟练度 vs 练习时间** | 先快后慢、边际递减 | **平方根 / 对数** |
| **广告投入 vs 销量** | 投 100 万 ≠ 投 10 万的 10 倍销量（饱和效应） | **平方根 / 对数** |

→ **多项式从哪里来？** 不是凭空造的，是**业务关系本来就弯曲**，特征工程师把这种弯曲"显式化"——把单一特征 `x` 升级为 `[x, x², x³]`（或 `[x, √x, log x]`），喂给 LR。

→ **为什么不直接换非线性模型（树 / 神经网络）？** 因为多项式 LR 保留了 LR 的优势：**可解释（看权重大小）/ 训练快 / 凸优化全局最优**。能用多项式 LR 解决的，就别上重武器。

> **抓一个最直观的**：油耗 vs 车速是 U 形——直线拟合会得到一条"平均斜率"，对低速和高速都不准。$\text{油耗} \approx a \cdot v^2 + b \cdot v + c$ 才能描述"低速浪费、中速最省、高速风阻"。本节后面所有公式 / 几何 / 代码都以这个 U 形为脑中画面。

**视觉锚**：5 辆车实测油耗 vs 车速（典型数据：v=30/50/80/100/120 km/h，油耗 ≈ 8.1/6.5/5.6/6.0/7.2 L/100km）。

```
   油耗 (L/100km)
        ↑
      9 │ ●                                       ← 直线 LR (虚线·)：
        │  ·\·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ●     斜率 ≈ 0，水平线
      8 │     ╲                              ╱│     哪儿都不准 ✗
        │      ╲                            ╱ │
      7 │        ●                       ●   │  ← 抛物线 LR (实线─)：
        │         ╲                    ╱     │     y = 0.001v² − 0.16v + 12
      6 │           ╲                ╱       │     贴合 U 形 ✓
        │             ●           ●          │
      5 │              ╲       ╱             │  ← 80 km/h 最省（U 形底 ≈ 5.6）
        │                ●─●                  │
      4 ┼───────────────────────────────────→ 车速 (km/h)
            30      60      80      100    120
```

→ 直线（虚线）= "1 个 w + 1 个 b"，**没法弯**；抛物线（实线）= "2 个 w（管 v 和 v²）+ 1 个 b"，多了 v² 这个特征，**能弯**。下面看为什么"能弯"还叫线性。

#### 先扫盲「多项式」（polynomial）

| 概念 | 定义 | 例 |
|---|---|---|
| 多项式 | 由变量的若干**幂次项**加和组成的表达式 | $a + bx + cx^2 + dx^3$ |
| 次数 degree | 最高项的指数 | 上例 degree = 3（三次多项式）|
| 一次多项式 | degree = 1，就是直线 $a + bx$ | LR 一元 $y = kx + b$ 就是它 |
| 高次多项式 | degree ≥ 2，曲线弯弯绕绕 | $y = x^2 - 4x + 5$ 是抛物线（degree = 2）|

> 中学数学的"二次函数"$ax^2 + bx + c$ 就是 degree = 2 的多项式。「多项式」只是把这个概念推广到任意 degree。

#### **【困惑】** 抛物线为什么还叫"线性"回归？

回到开头那个问题：$y = w_1 x + w_2 x^2 + w_3 x^3 + b$ 画出来明明是**曲线**，凭什么叫"线性"？

#### 直觉 1 · 「线性」说的是混合方式，不是看图

LR 模型干的事可以拆成两件，"线性"只管其中一件：

| 步骤 | 内容 | 谁决定 |
|---|---|---|
| **1. 准备配料** | 输入用什么：$x$、$x^2$、$\log x$、$\sin x$ ... | **特征工程师**（你）|
| **2. 混合配料** | 每种配料乘一个系数 $w$，加起来 + $b$ | **LR 模型** |

→ **"线性"指第 2 步**——不管给什么配料，LR 都只会用「加权求和」一种动作。配料本身可以任意弯曲（$x^2$ 抛物线、$\sin x$ 波浪），不影响"混合方式"是线性的。

#### **【数学】** 直觉 2 · 换名字 trick（最快理解的方式）

把 $x^2$ 重命名——管它叫 $z$。模型瞬间变成：

$$y = 1 \cdot x + 0.5 \cdot z + 2$$

→ 这是标准的"对 (x, z) 线性"的方程。**"非线性"被藏到了 $z = x^2$ 这一步预处理里**，**模型本身仍然是线性的**。

#### **【数学】** 直觉 3 · 几何画面（升维展平）

**视觉锚**：用 5 个样本（$x=1,2,3,4,5$，$y=0.5x^2+x+2$）演示。

```
1D 视角 (x, y)：肉眼看到弯（抛物线）

  y                              5 个样本：
  ↑                ●             (1,  3.5)
20│                               (2,  6.0)
  │             ●                 (3,  9.5)
15│                               (4, 14.0)
  │          ●                    (5, 19.5)
10│
  │       ●                       形状：抛物线（弯）
 5│    ●                          直线 LR 拟合不上
  │  ●
  └─────────────────→ x
    1  2  3  4  5

升维一招：让 z = x² 当成新坐标轴 → 1D 特征 → 2D 特征

  z = x²                           5 个 (x, z) 点：
   25│              ●              (1,  1)
     │                              (2,  4)
   16│           ●                  (3,  9)
     │                              (4, 16)
    9│        ●                     (5, 25)
     │
    4│     ●                       z 是 x 的"影子"——
    1│  ●                          点必然落在 z=x² 曲线上
     └─────────────→ x             （z 不独立于 x）
      1  2  3  4  5

把 y 加进来 → 三维 (x, z, y) 空间里 5 个点 **共面** ✓

  每个点升级为三元组 (x, x², y)：
   (1,  1,  3.5)
   (2,  4,  6.0)            ← 验证 y = 1·x + 0.5·z + 2：
   (3,  9,  9.5)             (1)+(0.5·1)+2 = 3.5  ✓
   (4, 16, 14.0)             (2)+(0.5·4)+2 = 6.0  ✓
   (5, 25, 19.5)             (3)+(0.5·9)+2 = 9.5  ✓
                            
  所有 5 个点都满足 y = 1·x + 0.5·z + 2 → 在 (x, z, y) 空间是个平面！
  LR 学的就是这个平面（线性）—— 投影回 1D 看才是"弯"。
```

**对照表**：

| 空间 | 方程 | 形状 |
|---|---|---|
| 原始 $(x, y)$ —— 1 维特征 | $y = 0.5 x^2 + x + 2$ | **抛物线**（弯）|
| 增广 $(x, z, y)$ —— 2 维特征，$z = x^2$ | $y = x + 0.5 z + 2$ | **平面**（直！）|

→ 在"展平后"的高维空间里 LR 就是**平的**（标准线性）；只是我们画图时把它压回 $x$ 一维看起来又弯了。这叫**升维展平 / 降维弯曲**。

#### **【数学】** 精炼答案

**仍然是线性回归**——原因 3 条：

- "线性"指的是**对权重 w 线性**（每个 $w_i$ 一次幂）
- 对**特征 x** 可以是任何形式：$x^2, \sqrt{x}, \log x, \sin x$ 都行
- 把 $x^2, x^3$ 视为**新特征** $z_2, z_3$，原模型变成 $y = w_1 z_1 + w_2 z_2 + w_3 z_3 + b$ —— 标准线性回归

#### **【代码】** sklearn 落地：两个工具串起来

直觉讲完，看 sklearn 怎么实现这套"先做配料，再混合"的流程——需要两个工具：

| sklearn API | 是什么 | 干什么 |
|---|---|---|
| **`PolynomialFeatures`**（多项式特征生成器） | `sklearn.preprocessing` 模块里的**预处理器**（不是模型）| 输入 1 维 $x$，输出 $[1, x, x^2, \ldots, x^d]$，d 由 `degree` 控制 |
| **`Pipeline`**（流水线） | `sklearn.pipeline` 模块里的**串联器** | 把"预处理 + 模型"打包成一个对象，`fit/predict` 自动按顺序走 |

**`PolynomialFeatures` 具体例**：

```
PolynomialFeatures(degree=2)：  输入 [3]  →  输出 [1, 3, 9]
PolynomialFeatures(degree=3)：  输入 [3]  →  输出 [1, 3, 9, 27]
```

**完整代码**：

```python
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

# Pipeline = 把"先做配料"和"再混合"串成一根流水线
model = Pipeline([
    ("poly", PolynomialFeatures(degree=3)),  # 步骤 1 配料：x → [1, x, x², x³]
    ("lr",   LinearRegression()),            # 步骤 2 混合：标准 LR 训练
])
model.fit(X, y)        # 自动按顺序：先 poly.fit_transform → 再 lr.fit
model.predict(X_new)   # 预测时也自动：先 poly.transform → 再 lr.predict
```

**为什么用 Pipeline 而不是手动两步**：训练 / 预测时把 `PolynomialFeatures` 和 `LinearRegression` 当成一个对象用，避免忘了对测试集也做同样的预处理（最常见的坑：`X_train` 用了 `degree=3`，`X_test` 直接喂原始 1 维数据 → 维度对不上 → 报错）。

→ 所以"我能拟合非线性数据吗？" 答案是 **能**——通过特征工程把非线性变换喂进 LR。这是 [`07a-overfit`](../07a-overfit/) 章节的核心：**多项式次数太高 → 过拟合**。

### 不是所有 "y = f(x)" 都是线性回归

反例：

| 模型 | 是不是线性回归 | 理由 |
|---|---|---|
| $y = ax + b$ | ✓ | 一元 LR |
| $y = ax^2 + bx + c$ | ✓ | 多项式 LR（对 a, b, c 线性） |
| $y = a \sin x + b \cos x + c$ | ✓ | 三角特征 LR（对 a, b, c 线性） |
| $y = e^{ax + b}$ | ✗ | 对 a, b **指数** 不线性（需要换成 logistic 回归 / 对数变换）|
| $y = \frac{1}{1 + e^{-(ax+b)}}$ | ✗ | sigmoid，是 [`03-LogReg`](../../03-LogReg/) 的范畴 |

→ "线性回归"的核心判据：**模型表达式对所有未知参数（w 和 b）只出现一次幂、加减组合**。
