### 笔记

> 【知道】应用场景

线性回归是最古老、最简单也最实用的回归算法，**只要因变量与自变量有近似线性关系**就能用：

- **房价预测**：面积 / 房龄 / 学区评分 → 房价（多元 LR 经典场景）
- **销售预测**：广告投放 / 价格 / 季节因子 → 销售额
- **股票/经济指标短期预测**：GDP / 利率 / 失业率等线性回归建模
- **生物医学**：剂量 → 反应程度（剂量-效应曲线）
- **工程**：温度 / 湿度 → 设备能耗
- **特征工程基线**：任何回归任务**先用 LR 跑一遍**做 baseline，比这条线还差的复杂模型不值得上

**何时不要用 LR**：
- 因变量是离散类别（用 LogReg / 树）
- 关系明显非线性且无法用多项式特征矫正（用树 / 神经网络）
- 强多重共线性导致权重不稳（用 Ridge / Lasso，见 07b）

### 直觉

一句话：**只要因变量 y 与自变量 x 有"差不多的直线关系"，LR 都能用** —— 哪怕只是大致线性。

|---|---|---|---|

| **物理实验** | 标定测量 | 电流 / 长度 / 时间 | 电压 / 体积 / 距离 |

### LR 在 ML 项目中的角色

**永远先用 LR 跑一遍 baseline**：

```
项目启动 → 用 LR 跑通完整 pipeline（数据→训练→评估→预测）
        → 拿 LR 的评估分数当作基线
        → 上更复杂模型(树 / NN / 集成)
        → 复杂模型的分数必须显著优于 LR，否则不值得上
```

> 「评估分数」用什么具体指标？回归任务常用的有 **MSE / RMSE / MAE / R²**——主场在 [`05-metrics/`](../05-metrics/)，本章只点名不展开。这里只需记住：先训 LR、记下分数、再用复杂模型对比。

**为什么**：

1. LR 训练快（毫秒级），可以快速验证数据 pipeline
2. LR 可解释，能告诉你哪些特征有效（看权重大小）
3. LR 是凸优化，没有局部最优陷阱
4. LR 是 baseline，证明你的复杂模型确实"学到了非线性"——而非数据本身就线性可分

### 何时不要用 LR

| 反例场景 | 推荐替代 |
|---|---|
| 因变量是**离散类别**（喜欢 / 不喜欢、猫 / 狗 / 鸟）| 逻辑回归（二分类）/ Softmax（多分类）/ 决策树 |
| **明显非线性**且无法用多项式特征矫正（如时序周期、复杂图像）| 决策树 / 神经网络 / 时序模型 |
| 强**多重共线性**（特征间高度相关，如 房价 = 面积 + 套内面积 + 公摊面积）| 岭回归 Ridge / Lasso（见 [`07b-regularization`](../07b-regularization/)）|
| 异常点很多且无法清洗 | Huber 回归 / 树模型 / 鲁棒回归 |
| 数据非平稳（分布随时间漂移）| 在线学习 / 增量更新模型 |

### 跨语言落地视角

LR 训练后**只需要保存权重向量 w 和偏置 b**——可以在任何语言里复现预测：

```python
# Python 训练
model.coef_       # array([w1, w2, ...])
model.intercept_  # b
```

```go
// Go 推理（无需任何 ML 库）
func predict(x []float64) float64 {
    sum := bias
    for i, xi := range x {
        sum += weights[i] * xi
    }
    return sum
}
```

```javascript
// JavaScript 浏览器端推理
const predict = (x) => x.reduce((sum, xi, i) => sum + weights[i] * xi, bias)
```

→ 这是 LR 工程化的优势：**模型小、推理快、跨语言无成本**。和深度学习几百 MB 的模型形成对比，LR 完全可以在边端 / 浏览器 / 嵌入式设备上跑。
