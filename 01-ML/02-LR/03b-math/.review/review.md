# 03b-math Design Review

## 总评 [部分 PASS]

整体方向对——A（残差方块）+ B1/B2（1D 抛物线切片）符合手册"先 1D 后 2D"的渐进原则，预设也覆盖了完美/欠拟合/远距/反向的对比锚点。但有 **2 个硬伤**必修：(1) `frame` 滑块当前在本章无任何用途，违反"单通道输入"和反模式"滑块语义双重/无用控件"；(2) 预设与手动滑块的互斥/同步关系没说清楚，存在状态漂移风险。修掉这两点即可 PASS。

## 维度 1 · 5 秒测试 [⚠️]

design 第 104-110 行给的 3 个判断点本身合理，但视觉设计有几处削弱了 5 秒理解：

- **判断点 1（红方块含义）站得住**：A 视图同时给"红粗线（当前）+ 绿虚线（最优）+ 红方块"，方块大小 ∝ err² 的语义有 size 编码做支撑。✓
- **判断点 2（滑块作用）有风险**：design 第 38 行手动滑块和第 37 行预设选择器并列摆放，5 秒内非熟人**分不清谁是主控**。手册 line 42 推荐"模式徽章"明确当前状态——design 第 130 行虽然提到了"🔵 手动模式"徽章，但布局图（第 119-128 行）里没体现，容易在实现时丢掉。
- **判断点 3（最优解位置）有歧义**：A 视图绿虚线 + B1/B2 绿钻石**同色但不同形**，5 秒内非熟人不一定建立"它们指的是同一个最优"的连接。建议统一图例或在标题里标注。
- **标题问题**：design 第 116 行主标题"MSE 损失函数最小化 · k、b 怎么影响拟合效果"OK，但 4 个子图标题（第 124-127 行）只写"A 散点+方块 / B1 Loss vs k / B2 Loss vs b / 指标面板"——**B1/B2 是术语缩写，新手第一次见看不懂**。必修改成自解释标题。

## 维度 2 · 单通道输入 [✗]

**这是本设计最大问题。**

- **`frame` 滑块（第 76 行）当前完全无用**：design 自己注明"保留给未来 GD"，但本章是 03b-math（损失函数 + 数学复习），**不涉及梯度下降**（GD 是 04b 的内容，见 README 第 264-265 行的指引）。手册反模式明确警告"滑块语义双重 / 一上来塞 8 种 mark"——放一个本章用不上的滑块，新手会困惑"帧是什么、我应该拖它吗"。**必修：本章删掉 frame 滑块**，未来 04b 的 demo 再加。
- **预设 vs 手动滑块的关系没说清**：design 第 67 行只写"预设模式时滑块隐藏"，但第 119-122 行 UI 草图把滑块和预设并列展示。需要明确：
  - 选预设 → 滑块**隐藏**还是**禁用**还是**同步显示预设值**？
  - 从预设切回"✋ 手动" → 滑块停留在预设最后一次的值，还是回到初始 0.5/-2.0？
  - 拖滑块时是否自动切到"✋ 手动"？
  必修：明确这三个状态转移规则，否则 P3 实现会随意发挥导致用户困惑。
- **k、b 滑块本身 OK**：语义单一（直接对应数学公式 y=kx+b 的两个参数），范围合理。✓

## 维度 3 · 多视图 [✓]

- **模式选择对**：A + B1 + B2 = 1D 渐进，符合手册"建直觉用 1D 抛物线（B1/B2）"的阶段定位。本章没选 C（等高线）和 D（3D 曲面）是对的——03b 是数学复习章，到 1D 抛物线刚够，2D/3D 留给 04a/04b 升维。✓
- **联动逻辑一致**：design 第 67 行声明滑块改动 → A/B1/B2 同步重算，符合"单一来源"。✓
- **数据范围基本统一**：k ∈ [-2, 6]、b ∈ [-4, 6]——但 B1 视图扫 k 时 b 取 cur_b、B2 扫 b 时 k 取 cur_k（design 第 60 行），这里有个**坐标尺度细节**：当用户调到极端值（如 b=6）时，B2 的抛物线谷底是否还在 [-4, 6] 视野内？design 没回答这个边界。软建议 P3 实现时打印一下不同预设下 B1/B2 的 y 轴范围确认不会爆。
- **绿钻石/绿虚线最优解**：A 视图绿虚线 + B1/B2 绿钻石的语义统一可视化做得好。✓

## 维度 4 · 反模式 [⚠️]

逐条对照手册第 110-117 行：

- ✓ 没有一上来 3D 曲面（本章只到 1D，正确）
- ✗ **滑块语义双重**：`frame` 滑块当前章节无意义=类似的"无效控件"问题（见维度 2）
- ✓ 给了 6 个 preset，不是纯自由探索
- ✓ 单图没塞 8 种 mark：A 视图有散点 + 当前线 + 最优线 + 方块共 4 种，可控
- ⚠️ **中文 + matplotlib 字体**：design 第 30 行布局图说要用 altair（design 第 29 行没显式声明工具链，但参考 v2 范本和手册推荐 1D 切片用 altair）——altair 本身字体没坑，但**指标面板**和 mo.md 里的中文如果落到 matplotlib 渲染就会方块雪崩。必修：P3 实现时所有图表强制 altair（或明确声明哪些用 matplotlib 并配字体）。
- ✓ 没有 `mo.state + run_button` 反模式：design 第 67 行声明纯 reactive（滑块直接驱动）

## 给 P3 开发的硬性要求（必修）

1. **删除 `frame` 滑块**：本章 03b 不涉及 GD，frame 完全无用。design 第 76 行的"保留给未来"等到 04b 再加。控件表只保留 `preset`、`k`、`b` 三项。
2. **明确预设/手动状态转移**：
   - 选预设 → k、b 滑块**显式同步到预设值**（不是隐藏）+ 显示禁用态（半透明 + 不可拖），让用户看到"预设把参数定到这里"。
   - 用户拖动滑块 → preset dropdown **自动切回 "✋ 手动"**，避免显示不一致。
   - 切到"✋ 手动" → 滑块解除禁用，保留刚才预设的值（不重置）。
3. **子图标题改成自解释**：
   - "A 散点+方块" → "数据点 + 当前拟合 + 残差平方（红方块越大误差越大）"
   - "B1 Loss vs k" → "固定 b={cur_b:.2f}，loss 随 k 变化（绿钻=最优 k）"
   - "B2 Loss vs b" → "固定 k={cur_k:.2f}，loss 随 b 变化（绿钻=最优 b）"
4. **指标面板必含**：当前 k、当前 b、当前 Loss、最优 k\*、最优 b\*、最优 Loss\*、当前/最优比值。让用户随时看到"我现在离最优还差多远"。
5. **统一中文字体**：所有图表优先 altair（自带字体处理 OK）；如必须用 matplotlib（如指标条形图），强制 `plt.rcParams["font.sans-serif"]=["PingFang SC","Heiti SC","Arial Unicode MS"]`。
6. **模式徽章必须实现**：design 第 130 行的"🔵 手动模式 / 🟡 预设模式"徽章不能在实现时丢掉——必须在主面板顶部用 mo.md 的彩色 span 渲染，5 秒识别当前状态。

## 给 P3 的软性建议（可选）

1. **数据点数量考虑加到 8-10 个**：design 第 87 行 5 个点对应 PPT slide 24 的算例，忠实没问题。但 5 个点的红方块在 A 视图里数量太少，"方块阵列"的视觉冲击会弱。可以考虑扩到 8-10 个点（保留同样的 w_true=0.97, b_true=-95），让方块密集感强一点。
2. **绿色虚线 + 绿钻石加 hover tooltip**：altair 原生支持，hover 时弹出"最优 k\*=0.97, b\*=-95.0, loss\*=1.42"。增强"绿是特别"的认知锚点。
3. **抛物线下方加一段 LaTeX 公式**：mo.md 渲染当前的 $L(k, b) = \frac{1}{n}\sum (y_i - kx_i - b)^2$，把数学和图形锚定。配合 README 03-导数.md / 04-偏导.md 的复习一致。
4. **配色合规检查**：design 第 133 行红/绿/蓝/紫四色 OK，但红+绿对色弱用户不友好。软建议绿钻石改成"绿钻+黑边框"，红方块改成"红方块+斜纹填充"。本课程不强求 a11y，记录一下即可。
5. **预设标签精简**：design 第 96-102 行的 emoji + 长描述（如"📍 欠拟合 k 偏小"）在 dropdown 里可能截断。考虑改短："✓ 完美 / 📉 k 偏小 / 📈 b 偏大 / 🚀 远距 / 🔄 反向"。
