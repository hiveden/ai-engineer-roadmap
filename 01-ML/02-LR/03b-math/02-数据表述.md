## ━━━━━━━━ 底稿 ━━━━━━━━

### PPT

> Slide 31 · 基础数学 – 数据表述

- **标量** scalar：一个独立存在的数，只有大小没有方向
- **向量** vector：一列顺序排列的元素，**默认列向量**，有大小和方向（模和夹角）
  - 例：张三的数理化成绩 [70, 80, 90]
- **矩阵** matrix：二维数组（m 行 n 列）
  - 例：张三、李四的数理化成绩 [[70,80,90],[75,85,95]]
  - m = 行数（样本数），n = 列数（特征数）
- **张量** Tensor：基于向量和矩阵的推广，更高阶数组
  - 例：2 个 3×4 矩阵 / 3 个 2×4 / 4 个 2×3 等

```
标量    向量              矩阵                  张量（3 阶）
 7    ┌70┐         ┌70  80  90┐         ┌─┐ ┌─┐ ┌─┐
      │80│         │75  85  95│         │ │ │ │ │ │
      └90┘         └          ┘         └─┘ └─┘ └─┘
                                       多个矩阵叠起来
```

## ━━━━━━━━ 讲解 ━━━━━━━━

### 生活锚 · 一份成绩单的 4 种装法

想象学校期末出成绩——同样是「数」，**装法不同，能回答的问题就不同**：

| 装法 | 长什么样 | 能回答什么 |
|---|---|---|
| **一个数** | 张三数学：**85** | 「张三数学考多少？」 |
| **一行数** | 张三：[**85**, 80, 90]（数、英、理）| 「张三这学期偏科吗？」 |
| **一张表** | 全班 30 人 × 3 科的成绩表 | 「数学全班平均多少？」 |
| **一摞表** | 全校 6 个班 × 30 人 × 3 科 | 「哪个班最强？」 |

这四种装法在数学里有正式名字：**标量 scalar / 向量 vector / 矩阵 matrix / 张量 tensor**——**每升一级，多一个维度**。

> 上一节我们用 5 个人的身高体重算 LR，那 5 个身高 [160, 166, 172, 174, 180] 已经是个**向量**，整个 (身高, 体重) 数据集已经是个 **5×2 的矩阵**——只是当时没点名。本节给这些日常数据**贴上标签**，下面 03-导数 / 04-偏导 / 05-向量 / 06-矩阵 才好引用。

### 业务问题 · 数据要怎么塞进电脑

ML 模型不直接吃「学生」「电影」「房子」——它只吃**数字数组**。所以拿到任何数据，工程师第一件事就是：

```
   原始数据 ──► 决定装成几维 ──► 选 标量/向量/矩阵/张量 ──► 喂给模型
   "5 个学生身高体重"        "2 维"                "5×2 矩阵"
```

装错形状就等于把方插头怼圆插孔——`sklearn` 报 `ValueError: Expected 2D array`、`numpy` 抛 `shapes not aligned`、`PyTorch` 嫌 `dim mismatch`，全是这一步没对齐。本节就把这把"插头规格"讲明白。

### 阶数全景图 · 维度逐级 +1

ASCII 把 4 个概念按维度从小到大排开：

```
  ┌──────────┬─────────────┬───────────────────┬─────────────────────┐
  │ 标量      │ 向量          │ 矩阵                │ 张量（3 阶起）       │
  │ scalar   │ vector       │ matrix            │ tensor               │
  │ 0 维      │ 1 维          │ 2 维                │ 3+ 维                │
  ├──────────┼─────────────┼───────────────────┼─────────────────────┤
  │          │  ┌──┐        │  ┌──────────────┐  │  ┌──┐ ┌──┐ ┌──┐       │
  │          │  │70│        │  │ 70  80  90  │  │  │··│ │··│ │··│       │
  │   85     │  │80│        │  │ 75  85  95  │  │  │··│ │··│ │··│       │
  │          │  │90│        │  └──────────────┘  │  └──┘ └──┘ └──┘       │
  │          │  └──┘        │                   │  3 张矩阵叠起来        │
  ├──────────┼─────────────┼───────────────────┼─────────────────────┤
  │ 一个分数 │ 张三 3 科     │ 张三+李四 3 科       │ 6 个班全部成绩         │
  │ 形状 ()  │ 形状 (3,)    │ 形状 (2, 3)        │ 形状 (6, 30, 3)      │
  └──────────┴─────────────┴───────────────────┴─────────────────────┘
       0 维          1 维             2 维                ≥ 3 维
       维度 +1       维度 +1          维度 +1             …
```

**记忆**：**点 → 线 → 面 → 体**——几何上升一级，数据装一级。

### 主线案例 · 5 样本身高体重的矩阵长相

把 [`01-损失函数.md`](./01-损失函数.md) 的 5 个学生数据**贴上标签**——这是后续 04a / 04b / 05 / 06 都会反复出现的同一组数：

| i | 身高 $x_i$ | 体重 $y_i$ |
|---|---|---|
| 1 | 160 | 56.3 |
| 2 | 166 | 60.6 |
| 3 | 172 | 65.1 |
| 4 | 174 | 68.5 |
| 5 | 180 | 75.0 |

**写成向量 / 矩阵**（**默认列向量**——竖着放）：

$$
\mathbf{x} \;=\; \begin{bmatrix}160\\166\\172\\174\\180\end{bmatrix}_{5\times 1}
\quad\quad
\mathbf{y} \;=\; \begin{bmatrix}56.3\\60.6\\65.1\\68.5\\75.0\end{bmatrix}_{5\times 1}
\quad\quad
X \;=\; \begin{bmatrix}160 & 56.3\\166 & 60.6\\172 & 65.1\\174 & 68.5\\180 & 75.0\end{bmatrix}_{5\times 2}
$$

横着写省纸时用「转置」记号：$\mathbf{x} = [160, 166, 172, 174, 180]^T$、$\mathbf{y} = [56.3, 60.6, 65.1, 68.5, 75.0]^T$——上标 $T$ 的意思就是"把横着的扳成竖着的"。

**ASCII 看清矩阵 X 的结构**：

```
        特征 1: 身高    特征 2: 体重
        ────────────   ────────────
  i=1     160              56.3        ← 学生 1（一行 = 一个样本）
  i=2     166              60.6        ← 学生 2
  i=3     172              65.1        ← 学生 3
  i=4     174              68.5        ← 学生 4
  i=5     180              75.0        ← 学生 5
        ────────────   ────────────
        ↑              ↑
        一列 = 一个特征  一列 = 一个特征

  矩阵形状：(m, n) = (5 行, 2 列) = (5 样本, 2 特征)
                    ───────  ───────
                     m = 5    n = 2

  小明 176 cm 是新样本，未来要预测的就是给一个新行，吐出对应 ŷ。
```

**ML 形状铁律**——**「行样本，列特征」**，全章节贯通：

- 矩阵 $X \in \mathbb{R}^{m\times n}$：m 行 = m 个样本，n 列 = n 个特征
- 标签 $\mathbf{y} \in \mathbb{R}^{m\times 1}$：列向量，每行一个样本的真实值
- 单个样本 $\mathbf{x}^{(i)} \in \mathbb{R}^{n\times 1}$：取一行 → 转置成列向量

> 后续 [`05-向量.md`](./05-向量.md) 把 $\mathbf{w}^T\mathbf{x}$ 点积讲透；[`06-矩阵.md`](./06-矩阵.md) 把 $X\mathbf{w}$ 批量预测讲透。本节先记下"5×2 矩阵"这个形状，后面接得上。

### 数学符号扫盲

首次出现的几个符号，提前打预防针——避免后面公式里突然蹦出来不认识：

| 符号 | 读法 | 直觉 | 例子 |
|---|---|---|---|
| $\mathbf{x}$（粗体小写）| "向量 x" | 一列数（列向量），不是单个数 | $\mathbf{x} = [160, 166, 172, 174, 180]^T$ |
| $X$（大写）| "矩阵 X" | 二维表，多行多列 | 上面 5×2 的身高体重表 |
| $\mathbf{x}^T$ | "x 转置" transpose | 把列扳成行（或反过来） | $[1,2,3]^T = \begin{bmatrix}1\\2\\3\end{bmatrix}$ |
| $\mathbb{R}^{m\times n}$ | "实数 m 行 n 列" | 形状记号——告诉你"装几个数" | $X \in \mathbb{R}^{5\times 2}$ 表示 X 是 5×2 矩阵 |
| $X_{i,j}$ | "X 第 i 行第 j 列" | 取矩阵里的一个元素 | $X_{1,1} = 160$（第 1 行第 1 列）|
| $\mathbf{x}^{(i)}$ | "第 i 个样本" | 上标加括号——区分样本编号 vs 幂 | $\mathbf{x}^{(2)} = [166, 60.6]^T$ |

> 上标 $T$ 不是 $T$ 次方！是"转置"的英文 transpose 首字母。$x^2$ 是 x 的平方，$\mathbf{x}^T$ 是 x 转置——一看粗体一看符号位置就能区分。

### 正例 vs 反例 · 形状对了 vs 不对

**正例**（形状对得上 → 模型能跑）：

```python
import numpy as np
X = np.array([[160, 56.3], [166, 60.6], [172, 65.1], [174, 68.5], [180, 75.0]])
X.shape   # (5, 2)  ✓  5 个样本、2 个特征
```

**反例**（形状错位 → sklearn 直接拒收）：

```python
X_wrong = np.array([160, 166, 172, 174, 180])     # 形状 (5,)，1 维
X_wrong.shape  # (5,)
# model.fit(X_wrong, y)
# → ValueError: Expected 2D array, got 1D array instead.
#   Reshape your data using array.reshape(-1, 1) ...

X_fixed = X_wrong.reshape(-1, 1)   # 变成 (5, 1)，2 维 ✓
```

**这就是为什么要分清「向量 (5,) vs 矩阵 (5, 1)」**——人眼看着都是 5 个数，但 sklearn 要 2 维矩阵。`reshape(-1, 1)` 是新手最常踩的坑修复动作。

### 张量 · 不止图像视频，但 LR 用不上

PPT 给的张量例子是「2 个 3×4 矩阵 / 3 个 2×4」——3 阶张量就是把多个矩阵摞起来。生活中最常见的：

- **彩色图片**：`(高, 宽, 3)`——3 个通道（R, G, B）各一张矩阵
- **一段视频**：`(帧数, 高, 宽, 3)`——4 阶张量，多一维时间
- **一批图片**：`(批量, 高, 宽, 3)`——4 阶张量，多一维 batch

```
   3 阶张量 = 3 张矩阵摞起来
   
   ┌──────────┐   R 通道（2D 矩阵）
   │ ........  │
   │ ........  │
   └──────────┘
        ┌──────────┐   G 通道
        │ ........  │
        │ ........  │
        └──────────┘
             ┌──────────┐   B 通道
             │ ........  │
             │ ........  │
             └──────────┘
```

> **LR 用不到张量**——LR 是"行样本、列特征"的 2D 矩阵世界。张量是 CNN / RNN / Transformer 的主场（`PyTorch` 的 `Tensor`、`TensorFlow` 的 `tf.Tensor`），本章只点名不展开，到深度学习章节才正式登场。

### 常见混淆 / 坑

- **「张量」两种含义**：数学物理的张量是带变换规则的几何对象；ML 里的张量就是「多维数组」——简化理解，不要被物理课吓到。
- **`(n,)` vs `(n, 1)`**：前者 1D 向量，后者 2D 列向量。`numpy` 的 1D array 既不是行也不是列——做矩阵乘时 broadcast 自动救你，但严格场景（sklearn 标签、PyTorch 模型）要 `reshape(-1, 1)`。
- **行向量 vs 列向量**：数学和 ML **默认列向量**——所以 $\mathbf{w}^T\mathbf{x}$（行 × 列）才能得到一个标量，预测公式里 $T$ 不是装饰是必需。
