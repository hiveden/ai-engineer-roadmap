## 1. 教学目标

- **核心**：从"沿坡下山"的**几何直觉**出发，看清梯度下降如何一步步逼近最优解
- **关键参数**：学习率（步长）太小→龟速；太大→振荡/飞出；恰好→稳定收敛
- **多维理解**：
  - 一维切片（固定一参，扫另一参）→ "滚球"落谷底的直觉
  - 二维等高线（俯视地形） → 看完整碗形 + 白线轨迹
  - 三维曲面（可拖拽视角） → 立体几何感
  - 业务图（数据点 + 拟合线 + 残差方块） → 回到 MSE 本质

## 4. 状态计算逻辑

### 统一 GD 计算源（单一数据流）

```
用户输入（预设 or 手动）
    ↓
确定起点 (w0, b0) + 学习率 lr_use
    ↓
前向跑 30 步 GD：
    traj = [(w0, b0)]
    cw, cb = w0, b0
    for i in range(30):
        yp = cw * x + cb
        err = yp - y
        grad_w = 2 * mean(err * x)
        grad_b = 2 * mean(err)
        cw -= lr_use * grad_w
        cb -= lr_use * grad_b
        cw = float(np.clip(cw, -1e6, 1e6))  # 防 NaN/inf
        cb = float(np.clip(cb, -1e6, 1e6))
        traj.append((cw, cb))
    ↓
按帧 f 截断：history = traj[:f+1]
    ↓
当前位置 = (traj[f][0], traj[f][1])
当前 loss = MSE(yp, y)
    ↓
所有视图更新
```

### 关键计算

#### 当前最优解（闭式解，一次计算）
```python
w_opt = sum((x - x_mean) * (y - y_mean)) / sum((x - x_mean)²)
b_opt = y_mean - w_opt * x_mean
loss_opt = mean((w_opt*x + b_opt - y)²)
```

#### 当前 Loss
```python
y_pred_cur = cur_w * x + cur_b
cur_loss = mean((y_pred_cur - y)²)
```

#### 1D 切片（B1/B2）
- B1：固定 `b=cur_b`，扫 `w ∈ [-2, 6]` 100 点，计算各点 loss
- B2：固定 `w=cur_w`，扫 `b ∈ [-4, 6]` 100 点，计算各点 loss

#### 2D 等高线网格（C）
```python
ws_grid = linspace(-2, 6, 80)
bs_grid = linspace(-4, 6, 80)
W, B = meshgrid(ws_grid, bs_grid)
L[i,j] = mean((W[i,j]*x + B[i,j] - y)²)
```

#### 3D 网格（D）
```python
ws_3d = linspace(-2, 6, 50)
bs_3d = linspace(-4, 6, 50)
# 同 C，但分辨率略低（50 vs 80）以加速
```

## 5. 数据生成

### 合成数据集
```python
N = 20
x = linspace(-3, 3, N)
w_true = 2.0
b_true = 1.0
y = w_true * x + b_true + normal(0, σ=1.2, size=N)
```

### 数值稳定性
- **Clip 防护**：每步 GD 后 `np.clip(val, -1e6, 1e6)`，避免发散导致 NaN/inf
- **Log 等高线**：`levels = exp(linspace(log(L.min()+0.1), log(L.max()), 18))`，防止 log(0)
- **3D 投影**：轨迹点裁剪至网格范围，避免 plotly 坐标爆炸

## 7. 预设方案详解

| 序号 | 名称 | w0 | b0 | lr | 教学要点 |
|------|------|----|----|----|---------|
| 1 | ✓ 完美 | 4.0 | 4.0 | 0.08 | 恰好的学习率 + 合理起点 → 平稳收敛，3~5 步抵达最优 |
| 2 | 🐢 龟速 | 4.0 | 4.0 | 0.005 | lr 太小 → 每步移动极小，需 30+ 步才能接近最优，形成"之字路" |
| 3 | 🌊 振荡 | -1.0 | -3.0 | 0.27 | lr 太大 → 轨迹来回摇晃（overshoot），无法稳定收敛 |
| 4 | 💥 飞出去 | -1.0 | -3.0 | 0.5 | lr 极大 → GD 参数发散，loss 跳向无穷，曲线看不见（必须用 clip） |
| 5 | 🎯 远距离 | 5.5 | -3.5 | 0.05 | 起点离最优很远 + 学习率稍小 → 长旅程，可观察完整下山过程 |
| 6 | （手动） | 0.5 | -2.0 | 0.05 | 用户自由拖滑块，体验学习率的影响 |

   - "白线是 GD 从我设的起点走到谷底的脚印 → 随处改 w/b 滑块，白线重算"

   - "拖帧滑块 0→30 → 看 GD 慢动作爬山，每一步在哪"

## 9. 核心交互玩法（≤5 条）

1. **改参数看轨迹**：拖 w/b/lr 滑块 → 四个视图同时联动，白线（C/D）立刻重算轨迹
2. **预设对比**：dropdown 一键切换 6 种典型场景，直观感受学习率的影响
3. **慢动作回放**：拖帧滑块 0→30，看 GD 每一步在地形图和 3D 的确切位置
4. **1D + 2D + 3D 渐进升维**：B → C → D，从"滚球"到"地形"再到"立体几何"
5. **损失值反馈**：loss 颜色实时变化（红→橙→绿），提示收敛程度

## 12. 防坑清单

- ✅ **Clip 防护**：GD 每步后 `np.clip(val, -1e6, 1e6)`，避免发散导致 plotly/matplotlib 爆炸
- ✅ **Log 等高线**：`levels = exp(linspace(log(L.min()+0.1), ...))`，防止 log(0) 错误
- ✅ **网格分辨率**：C=80×80, D=50×50（足够显示特征，不卡）
- ✅ **轨迹裁剪**：3D 中 `np.clip(hist, [-2,-4], [6,6])`，防止超出网格范围
- ✅ **单一数据源**：预设/手动逻辑统一，无重复计算
- ✅ **可变性监听**：每个滑块改动→所有视图立即刷新，无滞后（纯 reactive，不用 state）
- ✅ **中文字体**：matplotlib 头部配置 `plt.rcParams["font.sans-serif"] = ["PingFang SC", ...]`
- ✅ **Altair width 固定**：不用 `"container"`，用 px 值保证布局稳定

## 14. 总结

本 demo 是 02-LR 第 4 章的**核心可视化**，将"梯度下降"这一抽象的迭代算法拆解为四个递进的几何视图：
- **A**（业务图）：回到数据和误差的本质
- **B1/B2**（1D 切片）：建立单变量的"滚球"直觉
- **C**（2D 等高线）：看完整的"碗"形和轨迹
- **D**（3D 曲面）：立体感受参数空间的几何形状

通过**6 个预设**快速演示不同学习率的后果（龟速/振荡/飞出/合适/长征），**帧滑块**支持慢动作回放，**模式徽章**实时反馈状态，整体形成"5 秒看懂梯度下降"的教学目标。
