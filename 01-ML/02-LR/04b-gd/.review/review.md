## 总评

**PASS · 1 项必修**

design 几乎是 v2 范本的等比映射 + 章节叙事（"沿坡下山" / 学习率三态 / BGD-SGD-MBGD 铺垫），落地难度低、风险点已基本被 v2 趟过一遍。

但 design 把章节定位放成 02-LR ⭐核心 demo，目前**没有任何针对 04b 章特有内容的增量**——无 BGD/SGD/MBGD 三种算法的视觉区分，无"收敛判据"指示，无"梯度向量"显式可视化（章节 README 大量篇幅讲梯度的"上升最快方向 + 负号"，但 demo 里没出现箭头）。这不是 fail，但作为核心章 demo 略显平淡。

## 维度 1 · 5 秒测试

**结论**：A 业务图先出场 ✅，路径 A→B→C→D 渐进 ✅。但"梯度下降在做什么"和"loss 最优解"的 5 秒测试**几乎完全相同**——v2 是讲"loss 形状"的，design 是讲"GD 算法"的，两者教学目标不同但视觉表达没区分。

具体问题：
- design §8 的 5 秒路径第 2 步是"拖 w/b 滑块 → 红点位置变 → 方块大小变"——这是 v2 的"loss 直觉"路径，**不是 GD 算法路径**。GD 的核心是"自动迭代"，5 秒测试应该让用户看到"白线一帧帧长出来"，而不是"我手动拖滑块"。
- 没有任何视觉元素直接传达"负梯度方向 = 下一步走向"。章节 README 的 Slide 67-68 反复强调"梯度反方向 / 加负号"，demo 里这是隐性的（藏在白线轨迹里）。

**5 秒测试能 get 到"loss 长什么样"，但 get 到"GD 在做什么"要 10 秒以上。** 不必修，但建议（见末尾）。

## 维度 2 · 单通道输入

预设 + 滑块互斥逻辑 ✅ 清晰：`preset.value is None` 时启用手动滑块，否则忽略。这套机制 v2 已验证。

**问题**：

| 项 | 状态 | 备注 |
|---|---|---|
| 6 个预设语义 | ✅ | 完美/龟速/振荡/飞出/远征/手动，命名清晰带 emoji |
| 帧滑块 0-30 | ⚠️ | "0=起点, 30=终点" 文案OK，但 30 步对"龟速"预设（lr=0.005）远不够收敛——用户拖到 30 看到白线还在山腰，会误以为算法不收敛，应在文案或 mode tag 里提示"30 步未必收敛" |

| lr 滑块语义 | ✅ | 仅在手动模式生效，预设携带自己的 lr |

**学习率 / 起点 / 帧三者交互**：design §4 数据流图明确（输入→GD→帧截断→视图），单一来源，没有双重语义。✅

**额外检查**（design 自己引入的风险）：

- **log levels**：`exp(linspace(log(L.min()+0.1), ...))` —— "飞出去"预设会让 L.min() 巨大，加 0.1 没意义。v2 同写法，实测能跑（因为最优点附近 loss 仍小），但 design 没说明。**非必修**
- **3D 轨迹 clip**：硬编码 `np.clip(hist, [-2,-4], [6,6])` 与网格范围耦合，design §4 网格 ws=[-2,6]/bs=[-4,6] 一致，OK
- **"飞出去"预设的视觉表现**：design §13 测试清单写"loss 跳向无穷，点超出网格（被 clip，曲面无法显示该段）"——这意味着 lr=0.5 这个预设**用户看不到任何动作**（白线被 clip 到边界一动不动）。教学价值打折。**建议改预设 lr 到 0.4 边缘振荡**，或者保留 0.5 但在 mode tag 里加红色"⚠ 已发散" 文字提示

## 必修项（M）

**M1 · "飞出去 lr=0.5" 预设的失败状态可见性**

按 design §13 测试清单，"飞出去"会被 clip 到网格边界，**用户视觉上只看到白线静止贴在边界**，分不清是收敛了还是发散了。这违反 5 秒测试原则。

至少二选一：
- (a) 把预设 lr 调到 0.4 左右，让发散过程可见（前 5 步剧烈摆动后才被 clip）
- (b) 在 mode tag 里检测发散（`np.any(np.abs(traj) > 1e3)`）并显式渲染 `<span style="color:red">⚠ 已发散</span>`

不修这一项，教学时学生看到"飞出去"预设会困惑——design 自己列的核心痛点（学习率太大）反而是最弱的可视化点。

## 建议项（S，可选）

**S1 · 给"GD 算法核心"加 1 个独有视觉符号**
当前 demo 复用 v2 的所有视图，作为 02-LR 章 4b 核心 demo 略平。加一个**梯度向量箭头**（在 C 图当前点画 `quiver` 指向负梯度方向、长度 ∝ lr·|grad|）即可。一行 matplotlib 代码，把 README 里反复强调的"负梯度方向"显性化。

**S2 · 加 "loss vs step" 小折线图替代 B2**
B1+B2 两张 1D 切片信息冗余（讲的都是"loss 是抛物线"）。把 B2 换成 `loss_history` 折线图（横轴 step 0-30，纵轴 cur_loss），4 种典型预设的曲线形状立刻分辨：
- 完美 = 单调下降快收敛
- 龟速 = 单调下降但慢
- 振荡 = 锯齿
- 飞出 = 一根冲天柱

教学价值远高于"扫 b 抛物线"。

**S3 · 帧滑块默认值改为 0**
当前默认 30（终点），用户打开就看到完整轨迹，损失了"看 GD 一帧帧走"的戏剧性。改 0 + 加自动播放按钮（如不想引入 state，仅改默认值即可）。

**S4 · "30 步" 写死的暴露**
design 全文 30 步固定。"龟速"预设 30 步远未收敛，"完美"预设 5 步就到了。建议加一个 `max_steps` 滑块（10/30/100/300）作为高级开关，或在每个预设里指定不同的步数。

**S5 · 收敛判据指示器**
README 提到"两次差距小于阈值"为收敛判据。可在 mode tag 旁加 `已收敛 ✓` / `未收敛 ✗` 徽章（`abs(traj[-1] - traj[-2]) < 1e-4`）。3 行代码强化章节核心概念。

## 落地风险扫描

| 项 | 风险 | 缓解 |
|---|---|---|
| `np.clip` 后 GD 仍不稳 | "飞出去"预设第 1-3 步可能产生 inf 才被 clip，clip 后梯度还是大 → 来回弹边界 | v2 已验证可跑，无需改 |
| matplotlib + plotly 混用渲染抖动 | C 图（mpl figure）每次 reactive 重绘，配合 D 图（plotly）刷新，可能闪烁 | 范本已实测，可接受 |
| 80×80 contourf + 50×50 surface | 每次滑块拖动重算 6400 + 2500 个 loss 点 | N=20 数据，向量化下毫秒级，不卡 |
| Altair 5 mark 叠加（A 图） | mark_square + mark_circle 同位置 size 编码可能重叠误导 | v2 已通过；保持 opacity=0.35 即可 |
