# 梯度下降算法分类

> 维度：算法变体
> 知识点级别：【了解】BGD 全样本 / SGD 1 样本 / MBGD 小批量；现代深度学习默认 MBGD
> 章节底稿全文见 [`README.md`](./README.md)（PPT slide 75-76, 80 题 4）

## ━━━━━━━━ 底稿 ━━━━━━━━

### PPT

#### Slide 75 · 梯度下降法分类

- **全梯度下降算法 FGD**（Full Gradient Descent）：每次迭代时，使用**全部样本**的梯度值
  - 有 m 个样本，求梯度时用了所有 m 个样本
- **随机梯度下降算法 SGD**：每次迭代时，**随机选择并使用一个样本**的梯度值
- **小批量梯度下降算法 mini-batch**：每次迭代时，随机选择并使用**小批量的样本**梯度值
  - 从 m 个样本中，选择 x 个样本进行迭代（1 < x < m）

```python
from sklearn.linear_model import SGDRegressor
# 若 batch_size=1，则变成了 SG；若 batch_size=n，则变成了 FGD
```

> 概念：**总量 / 批次大小 / 批次数**

#### Slide 76 · 三种梯度下降方法的对比

**全梯度下降算法 FGD**：
- 由于使用全部数据集，数据间的随机性、差异性被抹平
- 每个参数能够学到一个**整体性的更新方向**
- 但**训练速度较慢**
- **GPU 塞不下**

**随机梯度下降算法 SGD**：
- **简单、高效、不稳定**
- SG 每次只使用一个样本迭代
- 关键在于：样本之间的差异性会让模型参数的更新方向出现**随机性**

**小批量梯度下降算法 mini-batch**：
- 结合了 SG 的**胆大**和 FG 的**心细**，表现也正好居于 SG 和 FG 二者之间
- 目前使用最多，正是因为它**避开了 FG 运算效率低成本大和 SG 收敛效果不稳定的缺点**

> **目前使用较多的是：小批量梯度下降**

### 笔记

> 【了解】梯度下降算法分类

按"每次迭代用多少样本算梯度"分三类：

| 算法 | 每次用样本数 | 优点 | 缺点 | sklearn |
|---|---|---|---|---|
| **BGD** 批量梯度下降 | **全部 n 个** | 梯度准、收敛稳定 | 每步慢、大数据卡内存 | — |
| **SGD** 随机梯度下降 | **1 个**（随机抽） | 每步极快、能跳出局部最优 | 噪声大、震荡 | `SGDRegressor` |
| **MBGD** 小批量梯度下降 | **一批 batch_size 个**（如 32/64） | 兼顾速度和稳定 | 多 1 个超参 batch_size | 深度学习默认 |

**直觉**：BGD = 看完整本书再下结论；SGD = 看一句话就下结论；MBGD = 看一段。

**现代实践**：深度学习几乎都用 MBGD，batch_size 32-512；传统 ML 数据量小时用 BGD。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

一句话：**BGD = 看完整本书再下结论；SGD = 看一句话就拍板；MBGD = 看一段，最常用**。

三者的算法骨架完全相同，区别只在「算梯度时塞多少样本进去」。

### 三类对比表

| 维度 | BGD（全量）| SGD（单样本）| MBGD（小批量）|
|---|---|---|---|
| **每步样本数** | n（全部）| 1（随机抽）| batch_size（如 32 / 64 / 256）|
| **梯度方向** | 精确（真实梯度）| 噪声大（单样本估计）| 较准（无偏估计 + 低方差）|
| **每步速度** | 慢（要遍历整个数据集）| 极快 | 中等 |
| **每 epoch 迭代次数** | 1 | n | n / batch_size |
| **收敛轨迹** | 平滑、单调下降 | 抖动剧烈、震荡前进 | 介于两者之间 |
| **能否跳出局部最优** | 弱（凸问题没差；非凸难逃）| 强（噪声反而是优点）| 中 |
| **GPU 利用率** | 内存塞不下 → 跑不动 | 利用率低（数据传输瓶颈）| **最优**（一批正好喂满显存）|
| **典型场景** | 数据量小 / 教学 | 在线学习 / 数据流 | **深度学习默认**、传统 ML 大数据 |

### 收敛轨迹对比（loss-迭代图）

### sklearn 的实现

```python
from sklearn.linear_model import SGDRegressor

# 名字叫 SGDRegressor，但实际通过参数控制三种模式：
# - 默认逐样本更新（接近 SGD）
# - partial_fit 每次喂一个 mini-batch → MBGD
# - 一次性 fit 全量、shuffle=False → 接近 BGD（不严格）
model = SGDRegressor(
    loss="squared_error",
    learning_rate="constant",
    eta0=0.01,                  # 学习率 α
    max_iter=1000,
    tol=1e-3,
)
model.fit(X, y)
```

> 名字陷阱：`SGDRegressor` 不严格等于 1 样本 SGD，它是「以 SGD 为底、可调 batch」的家族实现。深度学习框架（PyTorch / TensorFlow）才是 MBGD 的主战场，通过 `DataLoader(batch_size=...)` 控制。

### 深度学习为什么默认 MBGD

四个工程理由：

1. **显存约束**：模型参数动辄上 GB，再把 100 万张图片全塞进去算梯度，GPU 直接 OOM。BGD 在 DL 里**根本跑不动**。
2. **GPU 并行的甜蜜点**：单样本 SGD 的矩阵乘是细长条，GPU 利用率不到 10%；batch_size = 32 起，GPU 才能吃饱。
3. **梯度噪声反而是 regularization**：MBGD 的小批量噪声让模型更难记住训练集 → 抗过拟合（和 [`07b-regularization/`](../07b-regularization/) 的 L1/L2 一起作用）。
4. **batch_size 的甜区**：32-512 是经验值。Bert / ViT 那种大模型用更大 batch + 更高学习率（linear scaling rule）。

经典口诀：**batch_size 翻倍 → 学习率翻倍**（近似规律，对应 SGD 噪声方差变化）。

### batch_size 的工程取舍

| batch_size | 优点 | 缺点 |
|---|---|---|
| **小（8-32）** | 噪声大、抗过拟合好、内存友好 | 训练时间长、GPU 利用率不满 |
| **中（64-256）**| 收敛稳定 + GPU 吃满，主流选择 | 平衡，没明显短板 |
| **大（1024+）**| 单 step 快、易并行 | 泛化差、学习率难调、易卡平坦区 |

**小结**：现代 ML / DL 选 MBGD，batch_size 拍 32 或 64 起步，按显存往上调。SGD 在「在线学习」（数据流式到来，不能存历史）场景仍有用。BGD 几乎只在教学和小数据演示出现。
