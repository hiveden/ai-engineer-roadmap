#### Slide 67 · 梯度下降法 – 机器学习的核心优化方法

- 顾名思义：**沿着梯度下降的方向求解 loss 函数极小值**，从而获得模型最优的参数
- 举个例子：**坡度"最陡"下山法**，直至山底
- 一个模型的优化过程：不断改进的模型参数，使得模型预测更准确
- 下坡路 = 梯度的反方向！！
- 求最佳斜率：**一步到位（正规方程） vs 步步为营（梯度下降）**

> **备注**：当 W₀ 发现应该变大，则 loss 下降。

#### Slide 68 · 梯度下降法 – 梯度和学习率

- **什么是梯度（gradient / grad）**：
  - 单变量函数：某一点切线斜率（导数，有正负），方向是函数增长最快的方向
  - 多变量函数：某点的偏导数向量
- **梯度下降公式**（PPT 原图）：$\mathbf{w} \leftarrow \mathbf{w} - \alpha \nabla L$
- **三点说明**：
  1. 需要循环迭代求当前点的梯度，更新权重参数
  2. **α: 学习率（步长）** 不能太大也不能太小。机器学习中：**0.001 ~ 0.01**
  3. 梯度是上升最快的方向，需要下降最快的方向，所以加负号

#### Slide 69 · 求解过程 – 单变量梯度下降举例

目标：最小化 $f(\theta) = \theta^2$，导数 $f'(\theta) = 2\theta$，初值 θ=1，α=0.4

| 步 | θ | 更新 |
|---|---|---|
| 1 | 1 | — |
| 2 | 1 − 0.4·(2·1) = **0.2** | θ 发生了更新 |
| 3 | 0.2 − 0.4·(2·0.2) = **0.04** | |
| 4 | 0.04 − 0.4·(2·0.04) = **0.008** | |
| 5 | 0.008 − 0.4·(2·0.008) = **0.0016** | |
| ... | 极其接近最优值 0 | J(θ) 接近最小 |

> **小结**：经过四次运算，即走了四步，基本抵达了函数的最低点。

> PPT 原图：多元线性回归损失函数对 w 求偏导得到梯度向量 → 应用更新公式 $\mathbf{w} \leftarrow \mathbf{w} - \alpha \nabla L$。

#### Slide 74 · 复习：梯度下降法 – 求解过程

- **梯度下降优化过程**：
  1. 给定初始位置、步长（学习率）
  2. 计算该点当前的梯度的负方向
  3. 向该负方向移动步长
  4. 重复 2-3 步直至**收敛**（两次差距小于阈值 / 达到指定迭代次数）
- 步长决定了在梯度下降迭代过程中，每一步沿梯度负方向前进的长度
- **学习率太小**：下降速度会慢
- **学习率太大**：容易错过最低点、产生震荡、甚至**梯度爆炸**
- **为什么梯度要乘以负号**：梯度方向是函数在此点上升最快的方向；需要朝下降最快的方向走（负梯度方向），所以加负号

#### Slide 75 · 梯度下降法分类

- **全梯度下降算法 FGD**（Full Gradient Descent）：每次迭代使用**全部样本**的梯度值
- **随机梯度下降算法 SGD**：每次迭代**随机选择一个样本**的梯度值
- **小批量梯度下降算法 mini-batch**：每次迭代**随机选择小批量样本**（1 < x < m）

```python
from sklearn.linear_model import SGDRegressor
# 若 batch_size=1，则变成 SGD；若 batch_size=n，则变成 FGD
```

> 概念：**总量 / 批次大小 / 批次数**

#### Slide 76 · 三种梯度下降方法的对比

| 方法 | 特点 |
|---|---|
| **FGD** | 由于使用全部数据集，数据间的随机性、差异性被抹平；每个参数能够学到一个**整体性的更新方向**；但**训练速度较慢**，GPU 塞不下 |
| **SGD** | **简单、高效、不稳定**。每次只使用一个样本迭代，样本之间差异性会让模型参数更新方向出现随机性 |
| **mini-batch** | 结合了 SG 的胆大和 FG 的心细，表现介于两者之间。**目前使用最多**，避开了 FG 效率低成本大和 SG 收敛不稳定的缺点 |

> **目前使用较多的是：小批量梯度下降**

#### Slide 77 · 梯度下降法 vs 正规方程

| 维度 | 梯度下降 | 正规方程 |
|---|---|---|
| 学习率 | 需要选择 | 不需要 |
| 求解方式 | 迭代 | 一步出解 |
| 特征数量 | 较大可使用 | 较小可使用 |
| 应用场景 | 更普适，迭代方式适合**嘈杂、大数据**应用场景 | 适合小数据、特征明确的情况 |

> **注意**：梯度下降在各种损失函数（目标函数）求解中大量使用。深度学习中更是如此，深度学习模型参数很轻松就上亿，**只能通过迭代的方式求最优解**。

#### Slide 78 · 本节小结

1. 梯度概念：单变量函数 = 导数；多变量函数 = 偏导数。**梯度方向是上升最快的方向**
2. 梯度下降法：沿负梯度方向迭代
3. 三种分类：FGD（全部样本）/ SGD（一个样本）/ mini-batch（小批量样本）

> 04 · 梯度下降算法

> 【掌握】梯度下降算法思想

什么是梯度下降法？

- 求解函数极值还有更通用的方法就是梯度下降法。顾名思义：沿着梯度下降的方向求解极小值
- 举个例子：坡度最陡下山法

**坡度最陡下山法**：

- 输入：初始化位置 S；每步距离为 a。输出：从位置 S 到达山底
- 步骤 1：令初始化位置为山的任意位置 S
- 步骤 2：在当前位置环顾四周，如果四周都比 S 高，返回 S；否则执行步骤 3
- 步骤 3：在当前位置环顾四周，寻找坡度最陡的方向，令其为 x 方向
- 步骤 4：沿着 x 方向往下走，长度为 a，到达新的位置 S'
- 步骤 5：在 S' 位置环顾四周，如果四周都比 S' 高，则返回 S'。否则转到步骤 3

小结：通过循环迭代的方法不断更新位置 S（相当于不断更新权重参数 w）。

最终找到最优解。这个方法可用来求损失函数最优解，比正规方程更通用。

```
梯度下降过程就和下山场景类似
可微分的损失函数，代表着一座山
寻找的函数的最小值，也就是山底
```

> 【理解】银行信贷案例

**场景**：银行根据客户**收入**预测**贷款额度**。

数据（n 个样本）：

| $x$ 收入(万) | $y$ 额度(万) |
|---|---|
| 4.0 | 20 |
| 8.0 | 50 |
| 5.0 | 30 |
| ... | ... |

**模型**：$\hat{y} = w x + b$，损失 $L(w, b) = \frac{1}{n} \sum (y_i - w x_i - b)^2$。

**梯度下降迭代**：

$$w \leftarrow w - \alpha \cdot \frac{\partial L}{\partial w}, \qquad b \leftarrow b - \alpha \cdot \frac{\partial L}{\partial b}$$

其中：

$$\frac{\partial L}{\partial w} = -\frac{2}{n} \sum x_i (y_i - w x_i - b), \qquad \frac{\partial L}{\partial b} = -\frac{2}{n} \sum (y_i - w x_i - b)$$

**关键超参 学习率 $\alpha$**：
- 太小：收敛慢，迭代次数爆炸
- 太大：来回震荡，甚至发散
- 经验值：0.001 ~ 0.1，配合标准化后的特征

**收敛判据**：$\lvert L_{t} - L_{t-1} \rvert < \epsilon$ 或达到最大迭代数。

> 【了解】梯度下降算法分类

按"每次迭代用多少样本算梯度"分三类：

| 算法 | 每次用样本数 | 优点 | 缺点 | sklearn |
|---|---|---|---|---|
| **BGD** 批量梯度下降 | **全部 n 个** | 梯度准、收敛稳定 | 每步慢、大数据卡内存 | — |
| **SGD** 随机梯度下降 | **1 个**（随机抽） | 每步极快、能跳出局部最优 | 噪声大、震荡 | `SGDRegressor` |
| **MBGD** 小批量梯度下降 | **一批 batch_size 个**（如 32/64） | 兼顾速度和稳定 | 多 1 个超参 batch_size | 深度学习默认 |

**直觉**：BGD = 看完整本书再下结论；SGD = 看一句话就下结论；MBGD = 看一段。
**现代实践**：深度学习几乎都用 MBGD，batch_size 32-512；传统 ML 数据量小时用 BGD。

> 【理解】正规方程和梯度下降算法的对比

| 维度 | 正规方程 | 梯度下降 |
|---|---|---|
| **求解方式** | 一步闭式解 $(X^T X)^{-1} X^T \mathbf{y}$ | 迭代逼近 |
| **学习率 α** | 不需要 | 必须调（敏感超参） |
| **特征标准化** | 可有可无 | **强烈建议**（量纲悬殊会让 GD 震荡） |
| **特征数** | $< 10^4$（受 $O(n^3)$ 求逆限制） | 任意（$O(n)$ 每步） |
| **样本数** | 受内存限制（X 要放下） | 任意（SGD/MBGD 可流式） |
| **是否可用** | $X^T X$ 必须可逆 | 总能跑（可能不收敛） |
| **适用算法** | 仅 LR（很少几个有闭式解） | LR / LogReg / NN / 几乎所有 |
| **典型场景** | 教学 / 小数据 / 特征明确独立 | 大数据 / 深度学习 / 一切现代 ML |

**Tech Lead 视角**：正规方程像"小学数学"——能直接列方程的题。梯度下降是"工程通用解"，
深度学习时代基本不会回头用正规方程。学正规方程的目的是**理解 LR 的本质是凸优化、有唯一解**，
理解后再去看 GD 的迭代为什么收敛、为什么收敛到全局最优（凸函数性质）。
