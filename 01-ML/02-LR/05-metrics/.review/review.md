# Demo 设计评审：metric-vs-outlier.py

> 评审对象：`.review/design.md` | 章节目标：【知道】MAE/MSE/RMSE 公式，【了解】对异常点敏感度差异
> 评审框架：5 秒测试 / 单通道输入 / 多视图破抽象 / 反模式

**结论先行**：方向对，核心隐喻（拽红星 → 看数字爆炸 + 线被拽歪）扣住了 slide 86-87 的认知痛点。但**柱图视觉对比方案有重大缺陷**（共享 y 轴时 MSE 直接顶天 → MAE/RMSE 视觉变零），4 个预设的差异度也不够暴力。需修 3 项硬伤，否则 5 秒测试不过。

---

## 一、5 秒测试 ❌（必修）

> 测试问法：用户打开 demo 不看任何文字，5 秒内能不能说出"MSE 对异常点最敏感"？

**当前设计的失败点**：

1. **三个柱图分列三栏（MAE | MSE | RMSE 各占一柱）**——design.md 中-左 / 中-右 / 右的布局，每个柱图用自己的 y 轴。如果 y 轴各自 auto-scale，三个柱看起来"都很高"，倍数差被坐标轴消化掉了。**视觉震撼归零**。
2. 数字面板写 `1.3x / 14x / 3.7x`——这是文字阅读，不是视觉冲击。5 秒测试只看图形高度比例。
3. 四列横排（散点 + 三柱）会把每个图压得很窄，柱子高度差被横向挤压稀释。

**必修方案**：

- **三个指标合并到同一个柱图，共享 y 轴**（横轴：MAE / MSE / RMSE 三个 category；每个 category 内 baseline 灰柱 + 当前蓝柱并列）。这样 MSE 一柱顶天、MAE/RMSE 几乎贴底——20x 倍数差异**用柱高直接砸脸**。
- 倍数标签用**大字号 + 红色徽章**贴在柱顶（`14x` 不要藏在数字面板里）。
- 布局改为**两行**：第一行散点 + 拟合线（占满宽度），第二行单个合并柱图（占满宽度）。横向拉满才能让柱高差异最大化。

**可选**：y 轴加切换按钮（线性 / log）。线性是默认（要的就是爆炸感），log 给"想看清小柱"的用户。

## 二、单通道输入 ⚠️（必修一处）

> 原则：滑块=唯一起点，预设是它的派生。

**当前问题**：

- design.md 写 `preset` dropdown + `x_outlier` / `y_outlier` slider 同时存在，但**没说清谁是主**。
  - 如果选预设后 slider 不跟着变 → 用户拖 slider 时 dropdown 还显示"极端"，状态不一致。
  - 如果选预设后 slider 自动跳 → 需要 `mo.state` 或者 dropdown 直接改 slider 的 default value（marimo 反应式里这是个坑）。

**必修方案**（参考 `_marimo-math-guide.md` 的"预设 + 帧 slider · 统一来源"片段）：

```python
# 单一来源原则
if preset.value == "手动":
    x_out, y_out = x_slider.value, y_slider.value
else:
    x_out, y_out = PRESET_TABLE[preset.value]  # 预设直接出值，slider 此刻被忽略
```

下游所有计算只读 `x_out, y_out`，不读 slider/preset 本身。在 UI 上加个**模式徽章**：当前是"手动"还是"预设：极端"，避免用户拖 slider 没反应时困惑。

**双异常点预设（design.md 第 144 行）**：和单异常点的数据维度不一致（一个 vs 两个红星），破坏了"拖一个点看一组数字"的核心隐喻。**建议从 v1 删掉**，挪到进阶扩展（第 277 行已经列了，重复）。

## 三、多视图破抽象 ✅（基本到位，1 项可选增强）

当前设计的视图：
1. **散点 + 拟合线**（业务图）—— 看异常点把线"拽歪"
2. **三个柱图**（量化图）—— 看数字差异
3. **callout 信息面板**（叙事）—— 解释"为什么"

这三层 = 业务直觉 + 量化对比 + 文字归因，结构合理。

**可选增强**（不阻塞 PASS）：

- 散点图里**加红色虚线连接异常点 → 拟合线上对应预测点**，让"误差长度"看得见（即手册"残差方块"模式的简化版）。slide 87 强调的就是"异常点的误差被平方"，把这条误差线画出来 + 标注 `err=15` 比纯柱图更直观。
- **可选超级增强**：散点图叠**淡红色平方残差方块**（`mark_square`，size 编码 err²）。异常点处会冒出一个巨大的红方块，正常点的方块小到看不见——这个视觉本身就解释了"为什么 MSE 爆炸"。手册 §复用片段第一段直接能粘。

## 四、反模式检查

| 反模式 | 是否触发 | 说明 |
|---|---|---|
| 一上来 3D | ✅ 没有 | 全 2D，符合主题 |
| 滑块语义双重 | ⚠️ 见单通道输入 | preset/slider 主从关系不清 |
| 只给自由探索无 preset | ✅ 4 预设 | 但预设差异度不够（见下） |
| 一图塞 8 种 mark | ✅ 没有 | 散点图最多 4 种元素 |
| 中文标题 matplotlib | — | 全 Altair，无此风险 |
| `mo.state + run_button` 做实时反馈 | ✅ 没有 | 纯 reactive |

**预设差异度问题**（必修）：

design.md 第 142-145 行的预设：

| 预设 | y_out | 离拟合线 | MSE 倍数 |
|---|---|---|---|
| 轻微 | 14 | ±2 | ~2x |
| 中等 | 22 | ±8 | ~5x |
| 极端 | 28 | ±15 | ~20x+ |

`x=5` 处拟合线 y ≈ 15，所以"轻微 y=14"几乎贴在拟合线上 → MSE 倍数 ~2x 是乐观估计，实际可能 < 1.5x，**学生看不出差别**。

**必修**：把"轻微"换成"明显偏离但不爆炸"——例如 `(5, 18)` 偏离 +3，MSE ~3-4x。预设序列要呈**指数级递进**：
- 无异常 (1x baseline)
- 轻微 (3x)
- 中等 (8x)
- 极端 (25x+)

每一档跳一个数量级，对比才有戏剧性。

## 五、技术实现备注

design.md 第 95-98 行有 bug（不阻塞设计评审，实现时改）：

```python
X_current = np.vstack([x_base, [x_outlier]])    # ❌ x_base 是 1D，vstack 维度不对
y_current = np.vstack([y_base, [y_outlier]])    # ❌ 同上
```

应是 `np.append(x_base, x_outlier).reshape(-1, 1)` 或 `np.concatenate`。`LinearRegression.fit(X, y)` 要求 X 是 2D `(n, 1)`，y 是 1D `(n,)`。

## 必修 vs 可选 总结

### 必修（3 项，否则 5 秒测试不过）

1. **柱图合并共享 y 轴**：三指标并列在一张图里，共享线性 y 轴。倍数标签贴柱顶（大字号 + 颜色编码）。删掉"四列横排"的散点 + 三独立柱方案。
2. **单通道输入归一**：preset 和 slider 走"单一来源"模式，加模式徽章。删除"双异常点"预设（挪到 P2 扩展）。
3. **预设倍数指数递进**：轻微 → 中等 → 极端 的 MSE 倍数差应跨越数量级（3x / 8x / 25x），不要 2x / 5x / 20x（前两档差异不够）。

### 可选（提分项）

4. 散点图加红色虚线 = 误差长度可视化（slide 87 的字面图解）。
5. 散点图叠平方残差方块（手册标配模式 1），异常点处冒巨大红方块——直接视觉化"平方"。
6. y 轴加 线性/log 切换。

### 实现期再说（非设计问题）

7. 修 `np.vstack` 维度 bug。
