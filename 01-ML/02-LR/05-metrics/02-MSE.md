# 均方误差 MSE

> 级别：【知道】
> 维度：概念
> 章节底稿见 [`README.md`](./README.md)（PPT slide 83-86）

## 公式

$$\text{MSE} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

n 为样本数量，$y_i$ 真实值，$\hat{y}_i$ 预测值。

**Mean Squared Error**：每个样本误差平方后求平均。

---

## 直觉

一句话：**MSE = MAE 但把误差先平方** —— 平方做了两件大事。

## 平方的两个作用

### 作用 1：总是正数（不抵消）

如果直接用 $\sum (y - \hat{y})$，正负误差会**互相抵消**：

| 样本 | 误差 |
|---|---|
| 1 | +5 |
| 2 | -5 |
| 平均 | **0**（看起来完美！） |

但模型其实**两次都错了 5**。平方后变 25 + 25 = 50，平均 25 —— 真实暴露了误差。

> 当然取绝对值（MAE 的做法）也能解决这个问题。但平方还有第二个作用 ↓

### 作用 2：大误差被放大

| 误差 | 平方后 |
|---|---|
| 1 | 1 |
| 10 | 100 |
| 100 | **10000** |

**一个误差 100 的离群点 = 一万个误差 1 的常规点**。

业务上的意义：**MSE 强烈偏向「避免大错」**。如果业务允许 100 个小错但不能有 1 个大错（医疗 / 自动驾驶刹车距离 / 风控授信额度），MSE 是天然的指标。

## LR 训练的 loss 就是 MSE

为什么 LR 训练用 MSE 而不用 MAE？

| 性质 | MSE | MAE |
|---|---|---|
| **可导** | 处处可导（$2(y-\hat y) \cdot (-1)$）| **0 点不可导**（绝对值尖角） |
| **凸性** | 严格凸 → 全局唯一最优 | 凸但不严格 → 可能多解 |
| **解析解** | 有正规方程 $w = (X^T X)^{-1} X^T y$ | 无解析解 |
| **梯度下降** | 平滑下降 | 0 点附近震荡 |

→ **MSE 是为优化设计的，MAE 是为评估设计的**。

> 详细推导见 [`03b-math/01-损失函数.md`](../03b-math/01-损失函数.md) 和 [`04-梯度下降/`](../04-梯度下降/)。

## 单位问题（MSE 的硬伤）

预测房价（元），MSE = 25000000 元² 。**没人能解释「元的平方」是什么单位**。

→ MSE 适合训练时优化、模型间相对比较，**不适合直接报告给业务方**。开根号变 RMSE 就解决了，见 [`03-RMSE.md`](./03-RMSE.md)。

## sklearn API

```python
from sklearn.metrics import mean_squared_error

mse = mean_squared_error(y_test, y_predict)
# 越小越好，但数值通常很大（被平方了）
```

## 异常点放大效应（数值演示）

误差 [1, 3]：

$$\text{MSE} = \frac{1^2 + 3^2}{2} = \frac{1 + 9}{2} = 5$$

加一个离群点 [1, 3, 100]：

$$\text{MSE} = \frac{1 + 9 + 10000}{3} \approx 3336.7$$

变化倍数 = 3336.7 / 5 ≈ **667 倍**。

> 对比 MAE 同情境只变 17 倍 —— **MSE 对离群点的敏感度比 MAE 高一个数量级以上**。

## 何时用 MSE

| 场景 | 理由 |
|---|---|
| **训练 loss**（必选） | 可导 + 凸 |
| 数据干净，想强烈惩罚大误差 | 平方放大效应是 feature 不是 bug |
| 模型间相对比较（同一数据集） | 数值大不影响排序 |

## 何时**不要**用 MSE

- **报告业务方**：单位是 y² 没法解释，改用 RMSE。
- **数据有未清洗的异常点**：训练会被异常点拽偏，改用 MAE 或 Huber。
