# 三种指标比较

> 维度：概念
> 知识点级别：【了解】MAE / MSE / RMSE 在场景下的差异；工业三铁律：训练用 MSE / 报告用 RMSE / 异常多换 MAE
> 章节底稿全文见 [`README.md`](./README.md)（PPT slide 85, 86, 88）

## ━━━━━━━━ 底稿 ━━━━━━━━

### PPT

> Slide 85 · 三种指标对比（场景 1：噪声小）

- RMSE 的计算公式中有一个平方项，因此大的误差将被平方，会增加 RMSE 的值，**大多数情况下 RMSE > MAE**
- 在数据点弥散度小时，**MAE 和 RMSE 非常接近**
- 对误差的**平方级惩罚**

> 用直线 y = 2x + 5 拟合数据时观察。

> Slide 86 · 三种指标对比（场景 2：含异常点）

- 对比第一张图，数据点弥散度变大，**所有指标都变大了**
- **RMSE 几乎达到 MAE 值的两倍**——RMSE 对异常点更为敏感
- **MSE 对异常点特别敏感**，几乎爆炸（正常数据和异常数据的误差共同决定了该误差，但**异常数据的贡献占了大头**！！！）
- 对误差的平方级**乘法**（PPT 原文 typo，应为"惩罚"——同 slide 85 表述）

> Slide 88 · 综合结论

| 指标 | 反映 | 对大误差 | 对异常点 |
|---|---|---|---|
| **MAE** | "真实"的平均误差 | 不敏感 | 不敏感 |
| **RMSE** | 放大大误差点的影响 | 敏感 | 敏感 |

- 一般使用 **MAE 和 RMSE** 这两个指标
- 都能反映出预测值和真实值之间的误差

### 笔记

> 【了解】三种指标的比较

我们绘制了一条直线 **y = 2x + 5** 用来拟合 **y = 2x + 5 + e** 这些数据点，其中 e 为噪声。

| 指标 | 单位 | 对离群值 | sklearn API | 默认用 |
|---|---|---|---|---|
| **MAE** | 同 y | **不敏感**（线性增长） | `mean_absolute_error` | 异常值多时 |
| **MSE** | y² | **极敏感**（平方放大） | `mean_squared_error` | LR 训练损失 |
| **RMSE** | 同 y | 敏感（开方后仍保留差异） | `root_mean_squared_error`（或 `np.sqrt(mse)`） | 报告评估时 |

**关键差异**（一句话）：RMSE 平方放大大误差，所以**对离群点敏感**；MAE 线性求和，离群点影响有限。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 一句话

**MAE 老实 / MSE 放大 / RMSE 是 MSE 开根号回到正常单位**。

### 全维度对比表

| 维度 | MAE | MSE | RMSE |
|---|---|---|---|
| **公式** | $\frac{1}{n}\sum \lvert e_i \rvert$ | $\frac{1}{n}\sum e_i^2$ | $\sqrt{\frac{1}{n}\sum e_i^2}$ |
| **单位** | 同 y | y² | 同 y |
| **可解释性** | ★★★ 最直观 | ★ 不直观 | ★★ 直观 |
| **对离群点** | 不敏感（线性） | **极敏感**（平方） | 敏感（平方但开根号） |
| **可导** | 0 点不可导 | 处处可导（凸）| 可导但 0 点梯度爆炸 |
| **能否当 loss** | 不推荐 | **首选** | 不推荐 |
| **sklearn API** | `mean_absolute_error` | `mean_squared_error` | `root_mean_squared_error` |
| **典型用途** | 异常点多时报告 | LR 训练 loss | **默认报告指标** |

> $e_i = y_i - \hat{y}_i$ 是单个样本误差。

### 数值例子（场景 1：常规误差）

预测误差为 [1, 3]：

| 指标 | 计算 | 结果 |
|---|---|---|
| MAE | (1 + 3) / 2 | **2** |
| MSE | (1 + 9) / 2 | **5** |
| RMSE | √5 | **2.24** |

观察：**RMSE > MAE**（2.24 > 2），这是数据点弥散度小时的常态 —— 但**两者非常接近**。

### 数值例子（场景 2：含异常点）

误差变为 [1, 3, 100]（多了一个离群点）：

| 指标 | 计算 | 结果 | 相对场景 1 |
|---|---|---|---|
| MAE | (1 + 3 + 100) / 3 | **34.7** | × 17 |
| MSE | (1 + 9 + 10000) / 3 | **3336.7** | × 667 |
| RMSE | √3336.7 | **57.8** | × 25.8 |

→ **RMSE 几乎是 MAE 的两倍**（57.8 vs 34.7），符合 PPT slide 86 的结论。

### 异常点放大效应（更真实的实验）

跑 100 个样本，正常误差均匀分布在 [-1, 1]：

| 异常点比例 | MAE | MSE | RMSE |
|---|---|---|---|
| 0%（无异常） | 0.50 | 0.33 | 0.58 |
| 3%（3 个 ±10 异常）| **0.70** | 1.0 | 1.0 |
| 加大异常到 ±30 | 1.4 | **28.4** | 5.3 |

观察：异常幅度从 ±10 → ±30，MSE 从 1.0 暴涨到 28.4（**28 倍**），MAE 只从 0.7 → 1.4（**2 倍**）。

→ 这就是为什么数据脏的时候**绝对不能用 MSE 训练**：异常点的梯度会主导优化方向，模型偏离正常数据分布。详见 [`07a-overfit/`](../07a-overfit/) 中训练曲线 U 形的形成机制。

> 想自己跑这个实验：[`demos/metric-vs-outlier.py`](./demos/metric-vs-outlier.py)

### 决策树：选哪个？

```
异常点多吗？
├─ 是 + 已知是噪声 → MAE（评估）+ Huber loss（训练）
└─ 否
    ├─ 训练 loss → MSE（必须，可导且凸）
    └─ 报告评估 → RMSE（默认） / MAE（如果业务方要「平均偏差」）
```

工业三大铁律：

1. **训练用 MSE**（不可议）
2. **报告用 RMSE**（默认）
3. **异常多换 MAE**（鲁棒）

### 补充：决策系数 R²（PPT 没讲但工程常用）

虽然不在本章 PPT 中，但实际工作避不开：

$$R^2 = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}$$

含义：**模型相对「瞎猜均值」好多少**。

| R² 取值 | 含义 |
|---|---|
| = 1 | 完美预测 |
| = 0 | 跟瞎猜均值一样 |
| < 0 | **比瞎猜均值还差**（模型完全失败）|

```python
from sklearn.metrics import r2_score
r2 = r2_score(y_test, y_predict)
```

→ **R² 是无量纲的相对指标**，跨数据集可比；RMSE 是绝对数值，跨数据集不可比。两者经常一起报告：「RMSE = 5000 元，R² = 0.85」。
