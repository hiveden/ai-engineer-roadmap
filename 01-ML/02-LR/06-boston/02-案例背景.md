## ━━━━━━━━ 讲解 ━━━━━━━━

### 历史背景

**Boston Housing Dataset** 是 1978 年由 Harrison & Rubinfeld 发表的一个经典回归数据集，506 条样本，13 个特征，目标值是房价中位数（MEDV，单位千美元）。从 90 年代起几乎所有 ML 教材都用它演示线性回归——直到 2020 年。

### 为什么被废弃

数据集中包含一个名为 `B` 的特征：

> `B = 1000(Bk - 0.63)²`，其中 Bk 是该街区黑人居民比例

这个字段把种族变量直接编码进特征空间，且公式背后假设了「黑人比例对房价有 U 形影响」——被认定有伦理问题。

**时间线**：

- 2020-07：sklearn 0.24 在 `load_boston` 上加 `DeprecationWarning`
- 2022-12：sklearn **1.2** 正式移除 `load_boston`，调用直接报 `ImportError`
- 现在的 1.4+ 版本只能从 OpenML 镜像或原始 CMU 仓库手动下载

```python
# sklearn >= 1.2 直接报错
from sklearn.datasets import load_boston
# ImportError: `load_boston` has been removed from scikit-learn since version 1.2.
```

### 替代方案：California Housing

sklearn 官方推荐的替代是 **加州房价数据集** `fetch_california_housing`：

| 属性 | Boston | California |
|---|---|---|
| 样本数 | 506 | 20640 |
| 特征数 | 13 | 8 |
| 目标值 | MEDV（千美元） | MedHouseVal（10 万美元） |
| 来源 | 1978 波士顿调查 | 1990 美国人口普查 |
| 伦理问题 | 含种族变量 B | 无 |

数据规模大得多（4 万倍样本），更接近真实生产场景。本章后续所有代码用 California 演示，但**学习目标和训练流程完全一致**。

### California 8 字段（实际使用）

| 简称 | 含义 |
|---|---|
| MedInc | 街区收入中位数 |
| HouseAge | 房屋平均房龄 |
| AveRooms | 平均房间数 |
| AveBedrms | 平均卧室数 |
| Population | 街区人口 |
| AveOccup | 平均居住人数（人/户） |
| Latitude | 纬度 |
| Longitude | 经度 |
| **MedHouseVal** | 目标：房价中位数（10 万美元） |

> 注意 California 的目标值单位是 **10 万美元**——后面看到 MSE ≈ 0.55 不要惊讶，它是 (10 万美元)² 量级的，开根号后约 7.4 万美元 RMSE。

### 给学习者的提醒

PPT 里的 `load_boston` 调用**今天会直接报错**，不要去硬装老版本 sklearn。本章范本代码全部用 California，把它当作「Boston 的精神继承者」即可。流程、API、评估指标完全一样——案例的目的是讲 LR 工程流程，不是讲 Boston 这个具体数据。
