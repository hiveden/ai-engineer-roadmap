## ━━━━━━━━ 讲解 ━━━━━━━━

### 标准 6 步 pipeline

任何监督学习项目，骨架都是这 6 步——和 [`01-KNN/04c-iris-case`](../../01-KNN/04c-iris-case/) 里的鸢尾花流程**完全一致**，只是换了估计器。

```
1. load    →  加载数据
2. split   →  训练/测试拆分
3. scale   →  标准化（特征工程）
4. fit     →  模型训练
5. eval    →  性能评估
6. predict →  对新样本预测
```

### 第 1 步 · load

```python
from sklearn.datasets import fetch_california_housing

data = fetch_california_housing()
X, y = data.data, data.target
print(X.shape, y.shape)   # (20640, 8) (20640,)
print(data.feature_names) # ['MedInc', 'HouseAge', ...]
```

首次调用会从网络下载（约 400 KB），缓存在 `~/scikit_learn_data/`。代理环境记得设 `HTTPS_PROXY`。

### 第 2 步 · split

```python
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=22,
)
```

**注意：回归任务不用 `stratify`**。

> 对比 KNN 鸢尾花案例里的 `stratify=mydataset.target`——那是分类任务，分层抽样保证训练/测试集里每个类别比例一致。
>
> 回归的目标值是连续的，不存在「类别」这个概念，强行 stratify 会报错。如果担心目标分布偏斜，可以手动按 y 的分桶来分层（用 `pd.qcut`），但 90% 场景默认随机切就够。

### 第 3 步 · scale（关键）

```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s = scaler.transform(X_test)   # 注意：测试集只 transform，不 fit
```

**为什么标准化对线性回归必要**——要分两种情况看：

| 求解方式 | 是否必须 | 原因 |
|---|---|---|
| 正规方程 `LinearRegression` | 不强制（结果一样） | SVD 求解不依赖量纲，但数值稳定性更好 |
| 梯度下降 `SGDRegressor` | **必须** | 不同特征量纲差几个数量级时，损失函数等高线极扁，梯度方向偏离最优解，学习率难调 |

California 数据里 `Population` 在数千量级，`AveBedrms` 在 1 附近——差 4 个量级。GD 不标准化基本不收敛。详见 [`04b-scaling/`](../../01-KNN/04b-scaling/) 的等高线可视化。

> 坑点：**先 split 再 scale**，不能反过来。如果先 scale 再 split，测试集的均值方差泄漏到训练集，违反「测试集对模型透明」的原则——这是初学者最常踩的数据泄漏（data leakage）坑之一。

### 第 4 步 · fit

```python
from sklearn.linear_model import LinearRegression

model = LinearRegression()
model.fit(X_train_s, y_train)
```

LinearRegression 在 California 数据上 fit 一次大约 50ms。SGDRegressor 大约 200ms（要迭代 1000 轮）。

### 第 5 步 · eval

```python
from sklearn.metrics import mean_squared_error, r2_score

y_pred = model.predict(X_test_s)
mse = mean_squared_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)
print(f"MSE={mse:.3f}  R²={r2:.3f}")
# MSE=0.555  R²=0.596
```

指标怎么读、典型范围多少 → 见 [`04-性能评估.md`](./04-性能评估.md)。

### 第 6 步 · predict

```python
import numpy as np

# 一个虚构的「街区」：8 个特征值
new_x = np.array([[8.3, 41, 6.9, 1.0, 322, 2.5, 37.88, -122.23]])
new_x_s = scaler.transform(new_x)   # 用同一个 scaler！
y_hat = model.predict(new_x_s)
print(f"预测房价: {y_hat[0]*100000:.0f} 美元")
```

> 关键：**预测时必须复用训练时的 scaler**。用新数据重新 fit 一个 scaler，等于把模型放在了一个完全陌生的坐标系里——预测必然乱。生产环境一般把 scaler 和 model 一起 pickle 保存。

### 流程图总览

```
原始数据 X, y
    │
    ├──[split]──→ X_train, y_train  ──[fit_transform]──→ X_train_s ──┐
    │                                                                 │
    └──→ X_test,  y_test  ──[transform 用上面的 scaler]──→ X_test_s   │
                                                                      │
                                                  ┌───────────────────┘
                                                  ▼
                                       LinearRegression.fit()
                                                  │
                                                  ▼
                                       coef_, intercept_
                                                  │
                                                  ▼
                                  predict(X_test_s) → eval（MSE / R²）
```
