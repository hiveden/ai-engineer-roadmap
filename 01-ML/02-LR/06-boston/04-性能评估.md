## ━━━━━━━━ 讲解 ━━━━━━━━

四个回归指标的定义和数学已在 [`05-metrics/`](../05-metrics/) 讲过，本节聚焦「在 California 房价上**典型值是多少**、**怎么读**」。

### 四个指标速查

| 指标 | 公式（直觉） | 单位 | 越大/越小越好 |
|---|---|---|---|
| MSE | 误差平方均值 | 目标值² | 越小越好 |
| RMSE | √MSE | 目标值 | 越小越好 |
| MAE | 误差绝对值均值 | 目标值 | 越小越好 |
| R² | 1 − 残差平方和 / 总方差 | 无量纲 | 越接近 1 越好 |

### California 上的实测基线

用 PPT 标准 6 步 pipeline（`random_state=22`，test_size=0.2，标准化），在 California 数据上跑 LinearRegression 的典型输出：

```
正规方程: MSE=0.555  MAE=0.533  RMSE=0.745  R²=0.596
SGD:      MSE=0.557  MAE=0.534  RMSE=0.746  R²=0.594
```

**翻译成人话**（目标值单位是 10 万美元）：

- **MSE = 0.555** → 误差²平均 0.555 × (10 万)² = 5.55 × 10⁹ 美元²。看着吓人，但平方量纲本来就大，**不要直接读 MSE 的数字**，看相对趋势就行。
- **RMSE = 0.745** → 平均误差约 7.45 万美元。这个数字最直观，「模型预测一个街区房价，平均偏 7.5 万」。
- **MAE = 0.533** → 平均绝对误差 5.33 万美元。比 RMSE 小，因为 RMSE 被大误差样本拉高（平方放大）。
- **R² = 0.596** → 模型解释了 60% 的房价方差。剩下 40% 是 LR 抓不到的非线性关系（地理位置 × 人口的交互、收入分布的非线性等）。

### 指标怎么选

| 场景 | 选哪个 | 理由 |
|---|---|---|
| 给老板汇报「平均偏多少钱」 | RMSE 或 MAE | 单位和目标值一致，直观 |
| 论文/比赛对比模型 | MSE 或 R² | sklearn 默认、社区惯例 |
| 数据有明显异常值，不想被拉偏 | MAE | 不放大大误差 |
| 想知道「比瞎猜好多少」 | R² | R²=0 等于「猜均值」基线，R²<0 说明模型反而比瞎猜还差 |
| 损失函数本身 | MSE（平方可导） | RMSE 不光滑，不直接做损失函数 |

### R² 是什么意思

```
R² = 1 - SS_res / SS_tot
   = 1 - Σ(y_true - y_pred)² / Σ(y_true - y_mean)²
```

- R² = 1 → 完美预测
- R² = 0 → 模型 = 「永远猜均值」（最弱基线）
- R² < 0 → 模型比「永远猜均值」还差，**多半是数据泄漏的反面：标准化忘了 transform 测试集**之类的 bug
- R² ≈ 0.6 → 中等水平。California LR 的 0.596 是行业标准基线——如果你的复杂模型（XGBoost / NN）在 California 上做不到 R² > 0.7，说明没真正学到非线性

### 怎么用基线指导调优

```
LR 基线：       RMSE = 0.745   R² = 0.596
        ↓
加多项式特征：   RMSE = 0.71    R² = 0.62   ← 略好，证明确实有非线性
        ↓
随机森林：       RMSE = 0.51    R² = 0.81   ← 大幅提升，值得上
        ↓
XGBoost：        RMSE = 0.46    R² = 0.84   ← 边际收益递减
```

这就是为什么[第 1 章应用场景](../01-intro/04-应用场景.md)说「永远先用 LR 跑 baseline」——没有基线的指标都是耍流氓。

### 评估代码片段

```python
from sklearn.metrics import (
    mean_squared_error, mean_absolute_error, r2_score,
)
import numpy as np

y_pred = model.predict(X_test_s)

mse = mean_squared_error(y_test, y_pred)
mae = mean_absolute_error(y_test, y_pred)
rmse = np.sqrt(mse)
r2 = r2_score(y_test, y_pred)

print(f"MSE  = {mse:.3f}")
print(f"MAE  = {mae:.3f}")
print(f"RMSE = {rmse:.3f}")
print(f"R²   = {r2:.3f}")

# 也可以直接用 model.score（默认返回 R²）
print(f"model.score = {model.score(X_test_s, y_test):.3f}")
```

> 提醒：sklearn 1.4+ 也有 `mean_squared_error(..., squared=False)` 直接返回 RMSE，但参数名在 1.6 改成了 `root_mean_squared_error` 单独 API。版本切换时注意。
