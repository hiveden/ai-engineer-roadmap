# 欠拟合与过拟合

> 维度：概念
> 知识点级别：【理解】欠拟合 = 模型太简单 / 过拟合 = 模型太复杂；找 U 形曲线最低点（bias-variance 拐点）

> Slide 101 · 欠拟合与过拟合概念复习

- **欠拟合**：模型在训练集上表现不好，在测试集上也表现不好。**模型过于简单**
  - 此时模型还没学到规律，训练和测试集 LOSS 均不下降
- **过拟合**：模型在训练集上表现好，在测试集上表现不好。**模型过于复杂**
  - 训练集表现优异，测试集表现拉胯，LOSS 很高
- 欠拟合在训练集和测试集上的误差都较大
- 过拟合在训练集上误差较小，而测试集上误差较大

> Slide 102 · 训练曲线

随着训练迭代次数（横坐标是时间或训练轮数），训练集 / 测试集 loss 曲线分化是过拟合的典型信号。

### 笔记

> 【理解】欠拟合与过拟合

**过拟合**：一个假设**在训练数据上能够获得比其他假设更好的拟合，但是在测试数据集上却不能很好地拟合数据**（体现在准确率下降），此时认为这个假设出现了过拟合的现象。（模型过于复杂）

**欠拟合**：一个假设**在训练数据上不能获得更好的拟合，并且在测试数据集上也不能很好地拟合数据**，此时认为这个假设出现了欠拟合的现象。（模型过于简单）

| 维度 | 欠拟合 underfit | 良好 good fit | 过拟合 overfit |
|---|---|---|---|
| **模型复杂度** | 太简单 | 适中 | 太复杂 |
| **训练误差** | 大 | 小 | **极小**（接近 0） |
| **测试误差** | 大 | 小 | **大**（远大于训练） |
| **典型表现** | 直线拟合曲线数据 | 多项式 2-3 次 | 多项式 10+ 次抖动剧烈 |
| **bias / variance** | high bias | balanced | high variance |

**判断方法**：画"训练误差 vs 测试误差随模型复杂度变化"曲线（learning curve）：
- 训练 ↓、测试 ↑ → 过拟合区
- 训练 ↑、测试 ↑ → 欠拟合区
- 两者都低且接近 → 甜蜜点

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

一句话：**欠拟合（underfit）= 模型太笨学不会；过拟合（overfit）= 模型太较真，把噪声也当规律背下来了**。

考试类比：

- **欠拟合**：复习不够，平时作业（训练集）和考试（测试集）都不及格
- **良好拟合（good fit）**：理解了知识点，作业 90、考试 88
- **过拟合**：把作业答案一字不差「背」下来了，作业 100，但考试换个数字就不会 → 87

机器学习要的不是「训练误差最小」，而是**泛化误差**（generalization error）小，即在没见过的数据上也表现好。

### 形象记忆

欠拟合 = **直**得没道理，过拟合 = **抖**得没道理。

### bias-variance 拐点

模型期望误差可以做经典分解（[`05-metrics/`](../05-metrics/) MSE 概念延伸）：

$$\text{Error} = \underbrace{\text{Bias}^2}_{\text{偏差：模型本身就不对}} + \underbrace{\text{Variance}}_{\text{方差：换一批数据就抖}} + \underbrace{\sigma^2}_{\text{不可约噪声}}$$

- **bias**（偏差）：模型平均预测值与真实值的距离 —— 简单模型 bias 高（拟合不上）
- **variance**（方差）：模型在不同训练集上预测值的波动 —— 复杂模型 variance 高（噪声当信号）

随**模型复杂度**横轴变化，两条曲线呈剪刀状：

```
误差
 |
 |   测试误差（U 形）
 |   \                          /
 |    \                        /
 |     \____               ___/
 |          \____      ___/   ← 拐点 = 甜蜜点（sweet spot）
 |               \____/
 |
 |                  \____ 训练误差（单调下降）
 +---------------------------- 模型复杂度
   欠拟合区  |  合适  |  过拟合区
   high bias       high variance
```

**拐点（U 形最低点）= 偏差和方差的最佳折中**。模型选择本质就是找这个拐点（→ 后续 [`07b-regularization/`](../07b-regularization/) 用正则化主动控制）。

### 与 KNN 的 k 调优类比

回顾 [`01-KNN/01-intro/`](../../../01-KNN/01-intro/)：KNN 的超参 k 就是同一个故事的另一个版本。

| 状态 | KNN | 多项式 LR |
|---|---|---|
| **过拟合** | k = 1（只看最近一个邻居，被噪声点带跑） | degree = 10（高次项过多） |
| **甜蜜点** | k 适中（如 5-10） | degree = 2-3 |
| **欠拟合** | k = N（看全体，永远预测多数类 / 全局均值） | degree = 1（直线） |

→ 一个统一的认知：**模型自由度（capacity）越高，过拟合风险越大**。所有有监督学习模型都要在「太死板」和「太放飞」之间调参。

### 怎么判断当前在哪个区？

唯一的工程做法：**train_test_split**（→ [`02-代码认识过拟合.md`](./02-代码认识过拟合.md) 实操）。

| 训练误差 | 测试误差 | 结论 |
|---|---|---|
| 大 | 大 | 欠拟合 → 加复杂度 |
| 小 | 小且接近训练 | good fit |
| **极小** | **远大于训练** | 过拟合 → 减复杂度 / 加正则 / 加数据 |

只看训练误差判断不了 —— 它对欠拟合敏感，对过拟合**毫无信号**（训练误差还在降呢）。这就是为什么必须留出测试集。
