# 原因与解决办法

> 维度：方法论
> 知识点级别：【理解】欠拟合加信息（特征/模型容量）/ 过拟合减容量+加数据+加约束；正则化是过拟合主流杀器
> 章节底稿全文见 [`README.md`](./README.md)（PPT slide 107-108）

## ━━━━━━━━ 底稿 ━━━━━━━━

### PPT

> Slide 107 · 欠拟合 – 出现原因和解决方案

**欠拟合出现的原因**：学习到数据的特征过少。

**解决办法**（从数据、模型角度）：

- **数据角度**：添加更多有关联的特征（可能当前的特征与标签没有关联性，比如从"姓名"来预测"寿命"）
  - 有时出现欠拟合是因为特征项不够导致的
  - "组合"、"泛化"、"相关性"三类特征是特征添加的重要手段
- **模型角度**：换复杂的模型，比如用多项式特征项强化拟合能力
  - 模型过于简单时的常用套路，例如将线性模型通过添加二次项或三次项使模型泛化能力更强

> Slide 108 · 过拟合 – 出现原因和解决方案

**过拟合出现的原因**：原始特征过多，存在一些嘈杂特征，模型过于复杂是因为模型尝试去兼顾各个测试数据点。

**解决办法**：

- **重新清洗数据**：对于过多异常点数据、数据不纯的地方再处理
- **增大训练数据的总量**：
  - 增加数据多样性，避免模型钻牛角尖学到脏特征
  - 深度学习：数据增强（图像镜像、亮度、清晰度等等，生成式 AI）
  - 模型剪枝
- **正则化**：解决模型过拟合的方法，在机器学习、深度学习中大量使用 → 详见 [`07b-regularization/`](../07b-regularization/)

### 笔记

> 【理解】原因以及解决办法

**欠拟合产生原因**：学习到数据的特征过少。

**解决办法**：

1. **添加其他特征项**，有时出现欠拟合是因为特征项不够导致的，可以添加其他特征项来解决
2. **添加多项式特征**，模型过于简单时的常用套路，例如将线性模型通过添加二次项或三次项使模型泛化能力更强

**过拟合产生原因**：原始特征过多，存在一些嘈杂特征，模型过于复杂是因为模型尝试去兼顾所有测试样本。

**解决办法**：

1. 重新清洗数据，导致过拟合的一个原因有可能是数据不纯，如果出现了过拟合就需要重新清洗数据
2. 增大数据的训练量，还有一个原因就是我们用于训练的数据量太小导致的，训练数据占总数据的比例过小
3. **正则化**
4. 减少特征维度

## ━━━━━━━━ 讲解 ━━━━━━━━

前置：[`01-欠拟合与过拟合.md`](./01-欠拟合与过拟合.md) 概念 / [`02-代码认识过拟合.md`](./02-代码认识过拟合.md) 实操

### 速查表

| | 欠拟合 underfit | 过拟合 overfit |
|---|---|---|
| **核心原因** | 模型容量 < 数据复杂度 | 模型容量 > 数据信息量 |
| **直接表征** | 训练 + 测试 误差都大 | 训练误差极小，测试误差大 |
| **应对方向** | **加** 信息（特征 / 模型容量） | **减** 容量 / **加** 数据 / **加** 约束 |

### 欠拟合：原因 + 解决

#### 原因

**学到的特征过少 / 模型本身太简单**，具体两类：

1. **特征不够**：当前特征与标签关联弱（极端例子：用「姓名」预测「寿命」）
2. **模型太死板**：用线性模型去拟合曲线（→ [`02-代码认识过拟合.md`](./02-代码认识过拟合.md) 阶段 1）

#### 解决（数据角度）

**加特征** —— 让模型有信息可学：

- **组合特征**：面积 × 楼层、收入 / 家庭人数
- **泛化特征**：日期 → 星期几 / 是否周末 / 是否节假日
- **相关性特征**：业务知识引入新字段（房价加「学区评分」）
- **多项式特征**：把 $x$ 扩展为 $x, x^2, x^3$（sklearn 的 `PolynomialFeatures`，→ [`02-代码认识过拟合.md`](./02-代码认识过拟合.md) 阶段 2）

#### 解决（模型角度）

- 线性模型不行 → 上**树模型**（决策树 / 随机森林 / GBDT）
- 树模型不够 → 上**神经网络**
- 直接的：**加深 / 加宽**当前模型（DL 增加层数 / 宽度）

工程口诀：**欠拟合先怀疑特征工程，再怀疑模型选型**。

### 过拟合：原因 + 解决

#### 原因

**原始特征过多 / 含嘈杂特征 / 模型试图兼顾每一个数据点**，具体三类：

1. **特征过多**：维度灾难（→ [`02-代码认识过拟合.md`](./02-代码认识过拟合.md) 阶段 3 的 degree=10）
2. **数据不纯**：异常点 / 标注错误 / 噪声大，模型把噪声也当规律学了
3. **数据量小 vs 参数量大**：100 个样本喂 1 万参数模型，必然死记硬背

#### 解决（按工程优先级排）

##### 1. 重新清洗数据 —— 性价比最高

去异常点、修正错标、统一编码 —— **数据问题永远是首要怀疑对象**。

##### 2. 增大训练数据量

数据多了，模型钻牛角尖的边际收益就低：

- **直接采集**：能加就加
- **数据增强（data augmentation）**：图像翻转 / 裁剪 / 调亮度；文本同义词替换；音频变速
- **生成式合成**：GAN / 扩散模型造样本（DL 时代主流）

##### 3. 减少特征 / 模型剪枝

- 特征选择（filter / wrapper / embedded 方法）
- 树模型剪枝（max_depth、min_samples_leaf）
- DL 网络裁剪（pruning）

##### 4. 正则化（regularization）—— 主流杀器

不直接砍特征，而是在损失函数里加惩罚项，让模型**自己**把无用权重压小：

$$\text{Loss}_{\text{new}} = \text{MSE} + \lambda \cdot \|\mathbf{w}\|$$

详见 [`07b-regularization/`](../07b-regularization/)（L1 / L2 / ElasticNet）。

##### 5. 早停（early stopping）

训练曲线（→ [`04b-gd/`](../04b-gd/) GD 迭代过程）上看到 **测试 / 验证 loss 开始反弹** 那一刻就停 —— 不让模型继续往过拟合区走：

```
loss
 |          ___ 验证 loss（拐点反弹）
 |        /    \
 |       /      \___
 |      /
 |     /         ___ 训练 loss（继续降）
 |    /        /
 |___/________/____ epoch
            ↑
         在这里停
```

### 工程视角：DL 里的标配组合

深度学习时代，过拟合是日常对手，标配套路：

| 工具 | 本质 | 对应本章 |
|---|---|---|
| **数据增强** | 增大有效样本量 | 解决方案 #2 |
| **Dropout** | 训练时随机失活神经元 → 强迫网络不依赖少数路径 | 隐式正则 |
| **BatchNorm** | 标准化激活值 → 平滑 loss landscape | 间接正则 |
| **L2 weight decay** | 权重衰减 = L2 正则 | → [`07b-regularization/`](../07b-regularization/) |
| **Early stopping** | 监控验证集 loss | 解决方案 #5 |

→ 一句话总结：**所有「让模型不那么自由」的招都是反过拟合招**。

### 上线前的最后检查

回到工程：上线前用 train / val / test 三分（不是两分）：

- **train**：训练
- **validation**：调超参（degree / λ / 学习率 ...）
- **test**：**只在最终评估时用一次**，不能用来挑模型，否则 test 集就被「污染」成了 val 集

→ 这是 [`05-metrics/`](../05-metrics/) 章节会强调的工程纪律。
