## 总评

**结论：PASS（必修 3 项）**

整体设计质量高于平均线。三视图分工清晰（A 业务对比 / B 权重主舞台 / C λ-MSE U 形），与 07a 共享数据 + 端口相邻是漂亮的"对照学习"安排。L1 红框零柱是真正的 5 秒"恍然大悟"信号，slide 114 的核心论点被钉死。

**最大亮点**：

1. **B 视图选位**——把权重柱图（而非散点拟合）放中间作为主舞台，视觉冲击对应教学目标，符合"L1 砍 L2 压"心智的最强落点。
2. **A 视图三线同框**——无正则线作为不变锚点，L1/L2 随 λ 变动；这种"变量 + 锚点"的对比模式比"切换查看"更省脑。
3. **数据复用 07a**——同 seed 同 split 同次数，避免引入新认知负担。

**最大隐忧**：B 视图 ASCII 草图里 10 列 × 3 行的并排柱，在 width=520 实际渲染下挤到一起的可能性高（每列约 50px，含组间距）；红框是 demo 的命门，渲染细节决定成败。

## 维度 1：5 秒测试（一眼看懂）

| 视图 | 5 秒任务 | 评估 |
|---|---|---|
| A · 散点+三条线 | "无正则疯狂抖，L1/L2 贴真值" | OK，前提是默认 λ=0.1 能拉开差距（设计文档第 7 节判断点 1 已自检） |
| B · 权重柱 | "L1 后 6~8 根是红框零柱，L2 全部矮但非零" | **风险点**：见下文必修 1 |
| C · λ-MSE U 形 | "测试线先降后升，红点指示当前位置" | OK，log scale + 红点 + 垂直参考线足够 |

**5 秒测试通过的关键**：B 视图的红框必须比柱子本身更显眼，否则用户第 3 秒会盯着橙色柱（即便它们高度=0）而不是红框。

## 维度 2：单通道输入（来源唯一）

设计已识别该原则（4.3 节"单一数据源原则"），但**preset 与 slider 共存时的实际行为需要明确**：

- 文档说"选预设 → 锁定 log₁₀(λ) 值（覆盖滑块），徽章显示预设模式"。
- 但 marimo 的 reactive 模型里，`log_lambda.value` 是滑块值，不会因 preset 选择而改变 UI 上的滑块位置（除非用 `mo.state` 反向写入，那又违反 guide 的"不要 state + reactive 混用"建议）。

**实际行为**：cell 4 里写 `cur_lambda = preset.value if preset.value is not None else 10**log_lambda.value`——这是正确的"派生量"模式，但用户拖滑块时**滑块本身**没失效，会产生"我拖了但没反应"的困惑（因为 preset 没切回 ✋ 手动）。

**04b-gd 范本怎么处理的**：predict 按预设时滑块仍生效，只是显示预设来源——这与本设计一致。但 04b 是**从滑块切预设是单向**（选预设后用户期待"看预设"），本 demo 强调"对比 4 档预设跨度"，需要用户频繁在 ✋ 手动 ↔ 4 档 间切换。

→ 见**必修 2**。

## 维度 3：多视图破抽象

**符合度：高**。

- **A**（业务）+ **B**（参数空间）+ **C**（超参扫描）三个不同抽象层——这正是 guide 教学叙事模板"抓眼球 → 建直觉 → 升维"的标准三段。
- 联动逻辑：λ 滑块统一驱动 A/B/C 三视图，C 额外受 toggle 控制——单一通道扇出三视图，干净。
- 颜色语义一致：红=无正则、橙=L1、绿=L2、跨 A/B 视图保持，符合"心智一致"原则。

**小瑕疵**：C 视图当前展示**只一种**正则的 U 形（受 toggle 控制），这违反了"多视图同时破抽象"的精神。设计文档 4.2 自己也承认"两条 U 形叠在一起会乱"——但其实 L1/L2 的 U 形谷底位置很可能差一个数量级，叠在同一图上是**有教学价值的对比**（L1 通常谷底 λ 更小）。

→ 见**建议 1**。

## λ 对数滑块专项评估

**跨度 1e-4 → 1e2（6 个数量级）合理性**：

- 对 Lasso：λ < 1e-3 时 zero_count ≈ 0（没效果），λ > 10 时 zero_count = 10（全砍光）；甜蜜点通常在 0.01~1。**有效区间约 3 个数量级**，扫 6 个数量级是合理的"包络"，让用户看到两端极限。
- 对 Ridge：相同分析，甜蜜点常在 0.1~10。

**用户能理解 log scale 吗**：

- log₁₀(λ) 滑块（-4 到 2）比 λ 直接滑块（1e-4 到 1e2）友好得多——线性滑块在 [1e-4, 1e2] 上 99% 的位置都对应"几乎无正则"，调不到。
- 但**普通用户不一定看懂 log₁₀(λ)** 这个标签——会下意识当成"取值 -4 到 2 的线性参数"。
- 当前补救：滑块旁同步显示 `(λ = 0.1000)`——OK 但不够强。

→ 见**建议 2**。

## 三视图联动清晰度

| 用户操作 | A | B | C |
|---|---|---|---|
| 拖 log_lambda 滑块 | L1/L2 两条线变（红线不变）| L1/L2 两行变（红行不变）| 红点位置移动（曲线不变）|
| 切 preset | 同上（λ 跳变）| 同上 | 同上 |
| 切 L1/L2 toggle | **不变** | **不变** | 曲线数据源切换（红点位置不变）|

联动矩阵清晰。但 toggle 不影响 A/B 是个**反直觉点**：用户切到"L2"时可能期待 A/B 高亮 L2 行/线。

→ 见**建议 3**。

## L1 红框零柱视觉信号强度

**当前设计**：红色虚线框（`stroke='#dc2626'`，`strokeDash=[3,2]`，`fillOpacity=0`）+ "0" 文字标签。

**评估**：

- 虚线框颜色（深红 #dc2626）与无正则线颜色（亮红 #ef4444）相近，**可能被误读为"和无正则同类"**。L1 主色橙色 #f97316 和零标记红色 #dc2626 也容易混淆。
- `fillOpacity=0` 的空心红框在白底上偏弱；如果 L1 行其他柱子（非零的 w₁、w₂）也是深色，红框会被挤掉。

→ 见**必修 1**。

## 4 档预设跨度评估

| 预设 | log₁₀(λ) | 预期 zero_count | 实际跨度评估 |
|---|---|---|---|
| 0️⃣ 无正则 | -4.0 (λ=1e-4) | 0~1 | OK，复现 07a |
| 🌱 弱 | -3.0 (λ=1e-3) | 2~4 | 跨度仅 1 个数量级，与"无正则"差异不够鲜明 |
| ✓ 适中 | -1.0 (λ=0.1) | 6~8 | OK，标志性甜蜜点 |
| 💪 强 | 1.0 (λ=10) | 9~10 | OK，明显欠拟合 |

"无正则" → "弱" 只跨 10×，B 视图上很可能两档看起来一样（都是"高次项还在"）。建议"弱"改 log₁₀(λ) = -2（λ=0.01），与"无正则"差 100×，与"适中"差 10×，分布更均匀。

→ 见**建议 4**。

## 必修项（3）

### 必修 1：B 视图零柱视觉信号需加强

**问题**：当前红色虚线空心框 + 颜色与无正则红 (#ef4444) 混淆 + 空心 fillOpacity=0 在白底上偏弱。

**修复方向（任选其一）**：

- **方案 A（推荐）**：换色——零标记用紫色或黑色虚线框（如 `#1f2937` 或 `#7c3aed`），脱离"红色家族"避免与无正则线撞色；同时把 stroke width 提到 1.5px。
- **方案 B**：保留红框但加深背景——`mark_rect(fill='#fee2e2', fillOpacity=0.5)` 给零柱位置一个浅红底色块（"被划掉的格子"感），框 + 底色双重信号。
- **方案 C**：在零柱顶部加 ✗ 或 ⌀ 符号（mark_text），符号 > 文字"0"在 5 秒识别上更快。

**验收**：5 秒测试时第一眼能识别 L1 后段是"特殊状态"（不是低值柱），而不是"和高位柱差不多颜色"。

### 必修 2：preset / slider 切换的徽章语义需明确

**问题**：用户选预设后再拖滑块，cur_lambda 仍取 preset.value（按设计 cell 4 逻辑），滑块"看似无效"。

**修复方向**：

- 用户拖动 `log_lambda` 时，徽章应明确提示"预设已锁定，切回 ✋ 手动方可生效"——或者更激进：拖滑块自动把 preset 切回 ✋ 手动（需要 mo.state 单向写入，违反 guide，**不推荐**）。
- 推荐**徽章文案补强**：当 preset.value 非 None 时，徽章除了显示"预设模式 · λ=X"外，加一行小字 "（手动滑块已锁定，选 ✋ 手动 解锁）"。

**验收**：用户在 preset=适中 状态下拖滑块 5 秒内能意识到"我在锁定模式"。

### 必修 3：StandardScaler 后权重柱图的语义需在 B 视图标注

**问题**：设计文档第 5 节正确指出权重是**标准化空间下**的，不做 inverse_transform。但 B 视图标题"权重对比"会让学生误以为是原始空间权重——他们记笔记时可能写错（"x³ 的权重 = 0.3"），而原始空间权重完全不是这个数。

**修复方向**：

- B 视图标题或副标题加一行小字："（标准化空间下系数，仅用于稀疏性对比，非原始 x^k 系数）"。
- 或在 B 视图右下角加一个 ⓘ 提示气泡。

**验收**：学生看完不会把柱高直接当原始多项式系数报出来。

## 建议项（5，可选）

### 建议 1：C 视图把 L1 + L2 两条 U 形叠在同一图

把 toggle 改为"highlight"语义而非"切换数据源"：两种 U 形都画（淡色 + 选中色加粗），让用户**看见 L1 和 L2 的甜蜜点 λ 不在同一位置**——这是 PPT slide 117"工程偏 L2"建议的重要数据支撑。

### 建议 2：log_lambda 滑块标签 + 数值显示加强

把 label 从 `log₁₀(λ)` 改为 `正则强度 log₁₀(λ)`，并在滑块下方常驻显示三段：`λ = 0.1000  ⬆ 强 ⬇ 弱  数量级跨度=6`，让对数概念落地。

### 建议 3：toggle 切换时 A/B 视图同步高亮

切到"L1"时 A 图 L2 线降低 opacity 到 0.3、B 图 L2 行降低 opacity——保持"对比始终在场"但视觉焦点跟随 toggle。当前设计 A/B 完全不响应 toggle，是干净但可能"信息不响应"。

### 建议 4：4 档预设的"弱"档调成 log₁₀(λ) = -2

`-3 → -1` 跨 100×，但 `-3 → -4` 仅 10× 看起来差不多。建议改为：
- 0️⃣ 无正则：-4
- 🌱 弱：**-2**（λ=0.01，约 zero_count=3~5）
- ✓ 适中：-1（λ=0.1，约 zero_count=6~8）
- 💪 强：1（λ=10）

让 4 档每档 zero_count 拉开 ≥ 2 的差距，B 视图变化最显著。

### 建议 5：与 07a-overfit 数据/pipeline 一致性自检

确认 07a 也用了 `PolynomialFeatures(include_bias=False)` + `StandardScaler`，如果不一致，把 07a 的 pipeline 加上（07a 那边的修改属于跨章对齐），让两章数据曲线 1:1 对照。

## 防坑清单（追加）

设计文档第 11 节已经很全，再补 2 条：

- ⚠️ **Lasso 在小 λ 下不稳定**：`alpha=1e-4` 时 Lasso 实际几乎等于 LinearRegression，但 sklearn 会发 ConvergenceWarning。`max_iter=20000` 已设但仍可能发，考虑 `import warnings; warnings.filterwarnings('ignore', category=ConvergenceWarning)` 或 cell 内捕获。
- ⚠️ **PolynomialFeatures(include_bias=False) 后 coef_ shape 是 (10,)**：design 第 5 节假设正确，但 LinearRegression / Lasso / Ridge 都需要这个一致才能并排画——单元测试时打印一次三个 shape 确认相等。
