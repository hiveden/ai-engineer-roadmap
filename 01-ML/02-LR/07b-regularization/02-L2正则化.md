# L2 正则化（Ridge）

> 维度：数学 + 代码
> 知识点级别：【理解】L2 = 权重平方和的惩罚；按比例收缩，权重逼近 0 但不等于 0；工程默认抗过拟合方法
> 章节底稿全文见 [`README.md`](./README.md)（PPT slide 112, 115）

## ━━━━━━━━ 底稿 ━━━━━━━━

### PPT

> Slide 112 · L2 正则化

- **λ 叫做惩罚系数**，该值越大则权重调整的幅度就越大
- L2 正则化：在损失函数中添加 L2 正则化项 → $\lambda (w_1^2 + w_2^2 + \cdots + w_n^2)$
- L2 正则化会使得权重**趋向于 0，一般不等于 0**

> **备注**：类比**橡皮筋**——把权重往 0 拉但不会拉到 0。

> Slide 115 · L2 实现 – Ridge 回归

```python
from sklearn.linear_model import Ridge

def dm05_模型过拟合_L2正则化():
    np.random.seed(666)
    x = np.random.uniform(-3, 3, size=100)
    y = 0.5 * x**2 + x + 2 + np.random.normal(0, 1, size=100)

    estimator = Ridge(alpha=0.005, normalize=True)

    X = x.reshape(-1, 1)
    X3 = np.hstack([X, X**2, X**3, X**4, X**5, X**6, X**7, X**8, X**9, X**10])
    estimator.fit(X3, y)
    print('estimator.coef_', estimator.coef_)

    y_predict = estimator.predict(X3)
    myret = mean_squared_error(y, y_predict)
    print('myret-->', myret)

    plt.scatter(x, y)
    plt.plot(np.sort(x), y_predict[np.argsort(x)], color='r')
    plt.show()
```

> Ridge 线性回归 L2 正则——**不会将系数变为 0**，但是对高次方项系数影响较大。
>
> **工程开发中 L1、L2 使用建议**：一般倾向使用 **L2** 正则。

### 笔记

> 【理解】L2 正则化

- 假设 $L(W)$ 是未加正则项的损失，$\lambda$ 是一个超参，控制正则化项的大小
- 则最终的损失函数：$L = L(W) + \lambda \cdot \sum_{i=1}^{n} w_i^2$

**作用**：主要用来防止模型过拟合，可以减小特征的权重。

**优点**：越小的参数说明模型越简单，越简单的模型则越不容易产生过拟合现象。

**梯度下降更新规则**：

$$w_i \leftarrow w_i - \alpha \cdot \frac{\partial L(W)}{\partial w_i} - 2 \alpha \lambda \cdot w_i = (1 - 2\alpha\lambda) w_i - \alpha \cdot \frac{\partial L(W)}{\partial w_i}$$

注意 $(1 - 2\alpha\lambda) w_i$：每次更新都把 $w_i$**乘一个 < 1 的因子**（按比例收缩），所以权重越来越小但**不会直接归零**——这就是 L2 与 L1 的本质区别。

Ridge 回归 API：`from sklearn.linear_model import Ridge`

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

一句话：**L2 给每个权重绑根橡皮筋，往 0 拉但永远拉不到 0**——权重越大被拉得越狠（按比例）。

### 几何直觉：为什么圆形不稀疏

```
        w2
         |
        ●●●          ← L2 约束区域（圆形，处处光滑）
       ●   ●
  ----●--+--●----- w1
       ●   ●
        ●●●
         |
```

椭圆从外面收缩，与圆**相切的点几乎不会落在坐标轴上**——光滑曲面切到光滑曲面，切点是「中间」位置。

→ $w_i$ 都被压得很小，但**没有一个等于 0**。

对比 [`01-L1正则化.md`](./01-L1正则化.md) 的菱形：尖角 vs 光滑，这就是稀疏 vs 不稀疏的根本原因。

### 梯度更新规则：按比例收缩

更新公式（对比 [`04b-gd/`](../04b-gd/)）：

$$w_i \leftarrow w_i - \alpha \cdot \frac{\partial L(W)}{\partial w_i} - 2\alpha\lambda \cdot w_i$$

整理一下，把 $w_i$ 提出来：

$$w_i \leftarrow (1 - 2\alpha\lambda) \cdot w_i - \alpha \cdot \frac{\partial L(W)}{\partial w_i}$$

**关键看 $(1 - 2\alpha\lambda)$**：这是一个**小于 1 的常数因子**。每次更新都把 $w_i$ 乘以它一次。

| 当前 $w_i$ | 惩罚项做的事 |
|---|---|
| 大（如 10） | 乘 0.99 后变 9.9 → **减得多** |
| 小（如 0.01） | 乘 0.99 后变 0.0099 → **减得少** |
| 0 | 乘 0.99 还是 0 |

→ 这是「**按比例收缩**」：$w_i = 10 \to 9.9 \to 9.8 \to \cdots \to 0.001 \to 0.00099 \to \cdots$

**为什么不会归零？** 比例收缩是**指数衰减**——永远逼近 0 但触不到。数学上叫渐近线（asymptote）。

### 橡皮筋类比

把每个权重想象成一根**橡皮筋绑在 0 点**：

- 权重越往大处跑（绝对值越大），橡皮筋拉力越强（$2\lambda w_i$ 与 $w_i$ 成正比）
- 拉力把权重拽回 0，但橡皮筋永远不会把它「拉死」在 0 上
- 拉力强弱靠 $\lambda$ 调节：$\lambda$ 大 = 橡皮筋紧 = 权重压得狠

效果：模型不会把宝全押在某个特征上（避免「极端权重」），所有权重分布更均匀 → 对噪声更鲁棒。

### L1 vs L2 一句话总结

| | L1（Lasso） | L2（Ridge） |
|---|---|---|
| 惩罚 | $\lambda \sum \lvert w_i \rvert$ | $\lambda \sum w_i^2$ |
| 几何 | 菱形（尖角） | 圆形（光滑） |
| 收缩方式 | 等量减常数 | 按比例乘小于 1 的因子 |
| 解 | **稀疏**（部分 = 0） | 不稀疏（都很小但 ≠ 0） |
| 用途 | 特征选择 | 通用抗过拟合（**默认选这个**） |

> 工程开发中**默认用 L2**（Ridge）。需要特征选择 / 高维稀疏场景才上 L1（Lasso）。两者混合是 ElasticNet——见 [`03-正则化案例.md`](./03-正则化案例.md) 末尾。
