# demo-02 · 数据流图（示范版）

> 这是一份**示范**——下次你闭卷自己画时，照这个 template：
> 每一步都标出 **输入 / 动作 / 输出 / shape**。
> shape = 数据形状（几行几列），ML 调试最核心的信息。

---

## 全景 · 4 步串起来

```
┌──────────────────────────────────────────────────────────┐
│ Step 1 · load + EDA（加载 + 观察）                         │
│                                                          │
│ 输入：  sklearn.datasets 内置的乳腺癌数据                    │
│ 动作：  load_breast_cancer() → 拿到 Bunch 对象             │
│        包成 DataFrame，加 Diagnosis 列                     │
│ 输出：  df  shape = (569, 31)                             │
│        └─ 569 行（每行一条检测） × 30 特征 + 1 标签列       │
│                                                          │
│ 侧产物（打印看）：                                          │
│   df.shape            → (569, 31)                        │
│   value_counts()      → 良性 357 / 恶性 212               │
└──────────────────────────┬───────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ Step 2 · split（切分）                                    │
│                                                          │
│ 输入：  df  (569, 31)                                     │
│ 动作：  ① X = df.drop("Diagnosis", axis=1) → (569, 30)    │
│        ② y = df["Diagnosis"]               → (569,)      │
│        ③ train_test_split(X, y, test_size=0.2)           │
│                                                          │
│ 输出：  ┌─ X_train  (455, 30)   y_train  (455,)          │
│        └─ X_test   (114, 30)   y_test   (114,)          │
│                                                          │
│ 注：工业标准切 3 份（train + val + test），demo 跳过调参 → 2 份 │
└──────────────────────────┬───────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ Step 3 · train（训练）                                    │
│                                                          │
│ 输入：  X_train (455, 30) + y_train (455,)               │
│ 动作：  ① model = LogisticRegression(max_iter=10000)     │
│           └─ 构造模型，传超参                              │
│        ② model.fit(X_train, y_train)                    │
│           └─ 迭代优化，学出参数                            │
│                                                          │
│ 输出：  训练好的 model，内部带：                            │
│        ├─ model.coef_       shape (1, 30)  权重          │
│        └─ model.intercept_  shape (1,)     偏置           │
│                                                          │
│ 侧产物（抽 5 条眼观）：                                      │
│   model.predict(X_test[:5])        → 类别数组 (5,)       │
│   model.predict_proba(X_test[:5])  → 概率矩阵 (5, 2)     │
└──────────────────────────┬───────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ Step 4 · evaluate（评估）                                 │
│                                                          │
│ 输入：  训练好的 model + X_test (114, 30) + y_test (114,)│
│ 动作：  ① y_pred = model.predict(X_test)  → (114,)       │
│        ② 把 y_test 和 y_pred 送进各种评估函数对比：        │
│                                                          │
│           accuracy_score(y_test, y_pred)                 │
│             → 0.956（109/114）                           │
│                                                          │
│           classification_report(y_test, y_pred)          │
│             → 每类 precision / recall / f1               │
│                                                          │
│           confusion_matrix(y_test, y_pred)               │
│             → [[39, 4], [1, 70]]                         │
│                                                          │
│ 输出：  各项评估指标（整套评估报告）                         │
└──────────────────────────────────────────────────────────┘
```

## 闭卷画图时的 template（你下次自己画照这个）

每一步一个盒子，盒子里固定四行：

```
┌──────────────────────────────┐
│ Step N · 动作名（中英对照）    │
│                              │
│ 输入：  <shape + 来自哪一步>   │
│ 动作：  <做了什么动作 / API>    │
│ 输出：  <shape + 去到哪一步>   │
└──────────────────────────────┘
```

**4 个盒子用箭头 ↓ 串起来**，末尾补侧产物（眼观 / 打印）。

---

## 为什么必须标 shape

ML 最常见的错误：**"shape 不对"**——输入给错维度、预测出来的形状不符合预期、评估函数对不上。

**养成"每个变量 shape 是多少"的肌肉记忆**，比死记 API 重要 10 倍。

以后做 demo-03 / demo-04 时，**第一眼看 shape，再看 API 名字**。
