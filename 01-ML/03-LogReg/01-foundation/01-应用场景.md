### 笔记

> 【了解】应用场景

逻辑回归（Logistic Regression, LR）是解决**二分类**问题的利器。名字带"回归"，本质是分类——线性回归输出连续值，LR 把它压到 [0, 1] 当作概率，再按阈值切类。

典型场景：

- 广告点击率预估（CTR）
- 是否流失 / 是否欺诈 / 是否垃圾邮件
- 信用评分卡（金融风控）
- 疾病诊断阳性 / 阴性

工程视角：LR 训练快、可解释（权重 = 特征重要性）、易上线（一行点积 + sigmoid），是工业界最常见的 baseline。

---

## ━━━━━━━━ 讲解 ━━━━━━━━

业务想要的是一个**类别 + 概率**——比如"这封邮件是垃圾，概率 91%"。怎么算出来？

第一步先算一个**分数**。把每个特征乘以一个权重 weight（sklearn 里叫 `coef_`）再加起来：比如邮件里"中奖"这个词出现 → 加 3 分，发件人是陌生人 → 加 1.5 分，正文长度正常 → 减 0.8 分……全部加完得到一个分数，记作 $z$。这就是**线性回归 linear regression**那一步。

但 $z$ 是个普通的数字，可能是 $-4.2$，也可能是 $7.8$，没有上下限。**概率天然要在 0 到 1 之间**（0% 到 100%），所以这分数不能直接当概率读。

第二步就是把 $z$ **压到 0 和 1 之间**——这个"压缩"动作由 sigmoid 函数 sigmoid function 完成（下一节主场）。压完得到一个值 $p$，比如 $\sigma(2.3) \approx 0.91$，这就能读成"91% 概率是正类"。

第三步看 $p$ 跟 0.5 比，过线就判正类，否则判负类。

```
特征 x ─┐
        │   ① 线性回归         ② sigmoid 挤压       ③ 阈值
        ├──────────►  z  ──────────►  p ∈ (0, 1) ──────►  类别 {0, 1}
权重 w ─┘  z = w·x + b       p = σ(z)            p ≥ 0.5 ?
         任意实数              0 到 1 之间          输出类别
```

三段解耦各管一件事：①线性层决定**特征怎么加权**（LR 可解释性的来源——权重直接就是特征贡献），②sigmoid 层把分数**翻译成概率**（下一节展开），③阈值层决定**业务侧怎么切**（默认 0.5）。

> 后续章节：sigmoid 在 02 节展开；权重 $w$ 怎么学出来在第 2 章 · 原理。本节止于"为什么 LR 长这样"。

**这一节的关键启示**：LR = 线性回归算分 → sigmoid 读概率 → 阈值切类，三段解耦。

| 故事里的元素 | 术语名 | 主场 |
|---|---|---|
| 把特征加权求和的"分数" $z$ | 线性组合 / logit | 02 节 |
