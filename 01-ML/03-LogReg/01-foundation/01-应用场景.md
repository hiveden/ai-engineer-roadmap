### 笔记

> 【了解】应用场景

逻辑回归（Logistic Regression, LR）是解决**二分类**问题的利器。名字带"回归"，本质是分类——线性回归输出连续值，LR 把它压到 [0, 1] 当作概率，再按阈值切类。

典型场景：

- 广告点击率预估（CTR）
- 是否流失 / 是否欺诈 / 是否垃圾邮件
- 信用评分卡（金融风控）
- 疾病诊断阳性 / 阴性

工程视角：LR 训练快、可解释（权重 = 特征重要性）、易上线（一行点积 + sigmoid），是工业界最常见的 baseline。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 生活锚 · 天气预报为什么不只说"下不下雨"

打开手机看天气，没人只想看"明天下雨"或"明天不下雨"——大家都要看到那行小字："**明天降雨概率 70%**"。

为什么？因为光说"下"，你不知道该不该带伞、该不该改约：

- **70% 会下** → 果断带伞、户外活动改期
- **30% 会下** → 赌一把，顺手带把折叠伞
- **5% 会下** → 不管它，省事

**类别（下 / 不下）+ 概率（70%）+ 业务动作（带不带伞）**——天气预报天然把这三件事捆在一起给你。LR 干的就是这件事，只是场景换成邮件、客户、贷款申请。

### 业务想要的不是"是 / 否"

做反垃圾邮件的产品经理拿来一份邮件名单，问你："这封到底是不是垃圾？"

最简单的回答只有"是"或"否"两个字。但业务拿到"是"就够了吗？不够——他们会立刻追问：

- "你**多有把握**？50% 还是 95%？"
- "把握小的先放收件箱别拦截，免得误伤"
- "把握 90% 以上的直接扔垃圾箱，省审核人力"

所以业务真正想要的不是一个标签，而是**类别 + 概率**——"这封邮件是垃圾，概率 91%"。光说"是垃圾"信息量太低，业务侧没法据此分级处理。

这就像**信用评分卡**：先把你各项资料折算成一个总分，再把总分翻译成"违约概率 12%"，最后切一刀判"批 / 不批"。LR 干的就是这三件事——**算分 → 翻译成概率 → 按阈值切类**。

### 第一步：算分（线性那一步）

把每个特征乘以一个权重 weight（sklearn 里叫 `coef_`）再加起来。还是邮件例子：

| 特征 | 权重 |
|---|---|
| "中奖"出现 | +1.5 |
| 发件人陌生 | +1.2 |
| 正文长度正常 | -0.4 |

全部加完得到 $z = 1.5 + 1.2 - 0.4 = 2.3$。这个分数有个学名叫 **logit**（log-odds，详细含义在下一节展开），数学上写作：

$$z = w_1 x_1 + w_2 x_2 + \dots + b$$

**这就是线性回归那一步**——你在 LR 章已经熟了 $y = \mathbf{w} \cdot \mathbf{x} + b$，LogReg 没换公式，只是把输出改名叫 $z$、当成中间产物，**后面还要再过一道挤压**。

### 第二步：把分数翻译成概率（sigmoid 那一步）

但 $z$ 是个普通实数——可能 $-4.2$，可能 $7.8$，没有上下限。**概率天然要在 0 到 1 之间**（0% 到 100%），所以 $z$ 不能直接当概率读。

反例感受一下：$z = 7.8$ 直接当概率？"780% 是垃圾邮件"——荒谬。$z = -4.2$？"负 420% 概率"——更荒谬。所以必须把 $z$ **挤到 (0, 1) 区间**。

这个挤压动作由 sigmoid 函数 sigmoid function 完成（下一节主场）。挤完得到 $p = \sigma(z)$，本例 $\sigma(2.3) \approx 0.91$，就能读成"91% 概率是垃圾"。

### 第三步：切类（阈值那一步）

看 $p$ 跟 0.5 比，过线判正类，否则判负类。0.5 这条分界线叫**决策阈值 decision threshold**——业务侧可以调：金融风控宁可错杀，阈值上抬到 0.7；广告召回想多投，阈值压到 0.3。默认就是 0.5。

```
特征 x ─┐
        │   ① 线性回归         ② sigmoid 挤压       ③ 阈值
        ├──────────►  z  ──────────►  p ∈ (0, 1) ──────►  类别 {0, 1}
权重 w ─┘  z = w·x + b       p = σ(z)            p ≥ 0.5 ?
         任意实数              0 到 1 之间          输出类别
```

三段解耦各管一件事：①线性层决定**特征怎么加权**（LR 可解释性的来源——权重直接就是特征贡献），②sigmoid 层把分数**翻译成概率**（下一节展开），③阈值层决定**业务侧怎么切**。

> 后续章节：sigmoid 在 02 节展开；权重 $w$ 怎么学出来在第 2 章 · 原理。本节止于"为什么 LR 长这样"。

**这一节的关键启示**：LR = 线性回归算分 → sigmoid 读概率 → 阈值切类，三段解耦。

### 术语

| 故事里的元素 | 术语名 | 主场 |
|---|---|---|
| 把特征加权求和的"分数" $z$ | 线性组合 / logit | 02 节 |
| 把分数压到 0–1 的函数 $\sigma$ | sigmoid 函数 sigmoid function | 02 节 |
| 切类用的 0.5 | 决策阈值 decision threshold | 03-api / 04c |
| 给每个特征加多少分 | 权重 weight / 系数 coefficient | 02 章原理 |
