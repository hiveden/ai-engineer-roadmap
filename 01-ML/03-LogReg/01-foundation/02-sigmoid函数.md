### 笔记

> 【知道】sigmoid 函数

把任意实数 $z \in (-\infty, +\infty)$ 压到 $(0, 1)$ 区间，作为概率使用。LR 的输出层就是它。

**定义**：

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

**关键性质**：

| 性质 | 公式 / 描述 |
|---|---|
| 值域 | $\sigma(z) \in (0, 1)$ |
| 中点 | $\sigma(0) = 0.5$ |
| 单调性 | 严格单调递增 |
| 对称性 | $\sigma(-z) = 1 - \sigma(z)$ |
| 导数 | $\sigma^\prime(z) = \sigma(z)(1 - \sigma(z))$ |

**工程接口**：

```python
import numpy as np
def sigmoid(z): return 1.0 / (1.0 + np.exp(-z))
# scipy 版（数值稳定）
from scipy.special import expit  # expit(z) == sigmoid(z)
```

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题

线性模型给你一个分数 $z = w^\top x + b$，值域是整条实数轴。要做二分类，你想说"这个用户点击的概率是 0.83"——但 $z$ 可能是 -47.2，可能是 +193.6，没法当概率。

朴素方案有两个，都不行：

1. **clip 到 [0, 1]**：$\max(0, \min(1, z))$。问题是大段输入被压成 0 或 1，梯度直接消失，模型学不动；而且 0.0 / 1.0 这种"绝对置信"让后续阈值调整、概率校准全废。
2. **线性归一化**：依赖整批数据的 min/max，新样本一来就溢出，在线推理直接崩。

要的是一个**单调、光滑、值域 (0, 1)、处处可导**的挤压函数（squashing function）。中间过渡平滑，两头渐近不撞硬边界。sigmoid 函数 sigmoid function（也叫 logistic function / 激活函数 activation function）是满足这组约束的经典选择，并且——后面 03/04 章会看到——它和"对数几率（log-odds）= 线性"的概率假设、和伯努利分布的 MLE 推导，整套数学链路对得上。这里先把函数本身吃透。

**反例**：如果不挤压、直接拿 $z$ 当"概率"——$z = 12.5$ 该读 1250%？$z = -3$ 是 -300%？业务侧根本无法定阈值，也没法跟另一个模型的概率对齐做集成。sigmoid 把范围锁死在 $(0, 1)$ 才让概率语义成立。

### 数学方法

#### 数学符号扫盲

| 符号 | 读法 | 直觉 | 例子 |
|---|---|---|---|
| $\sigma$ | sigma | 希腊字母 σ，约定俗成的 sigmoid 函数名 | $\sigma(0) = 0.5$ |
| $e$ | 自然底 e / Euler's number | 自然指数底，约 2.71828，微积分里求导最顺手的底 | $e^0 = 1$，$e^1 \approx 2.718$ |
| $\sigma'$ | sigma prime | $\sigma$ 对自变量的一阶导数 | $\sigma'(0) = 0.25$ |

#### 定义与性质

回看 5 条性质，每条都不是装饰，是后续推导的钩子：

- **值域 (0, 1)**：开区间，永远取不到 0 和 1。这避免了 $\log 0$ 在交叉熵里炸锅。
- **中点 $\sigma(0) = 0.5$**：决策边界天然落在 $z = 0$，也就是 $w^\top x + b = 0$ 这条超平面。阈值 0.5 不是拍脑袋，是几何对称。
- **单调递增**：$z$ 越大，预测概率越高。"线性分数高 → 正类概率大"这条直觉守住了。
- **对称性 $\sigma(-z) = 1 - \sigma(z)$**：把符号翻转等价于"正负类概率互换"。03 章把 LR 写成 $P(y=1) = \sigma(z)$、$P(y=0) = 1 - \sigma(z) = \sigma(-z)$ 时直接复用这条。
- **导数 $\sigma'(z) = \sigma(z)(1 - \sigma(z))$**：神来之笔。导数能用函数自身表达，不用再算一次 exp。后向传播里前向算出 $\sigma(z)$ 顺手就有梯度，省一倍计算。最大值在 $z = 0$ 处取 0.25——这也是为什么深层堆 sigmoid 会梯度消失（每层最多打 0.25 折）。

#### 实算

把"压到 (0, 1)"具象化，手算四个点：

| $z$ | $e^{-z}$ | $\sigma(z) = 1/(1+e^{-z})$ |
|---|---|---|
| -1 | $\approx 2.718$ | $1/3.718 \approx 0.269$ |
| 0  | 1            | $1/2 = 0.5$ |
| 1  | $\approx 0.368$ | $1/1.368 \approx 0.731$ |
| 2  | $\approx 0.135$ | $1/1.135 \approx 0.881$ |

观察：$z$ 从 -1 到 +1 跨度 2，$\sigma$ 从 0.27 跳到 0.73，**中段斜率最大**（梯度大、学得快）；$z = 2$ 已经到 0.88，再往上爬得很慢——这就是"饱和区"。

#### 几何直觉（ASCII 图）

```
   σ(z)
  1.0 ┤- - - - - - - - - - - - - - ──────  ← 上渐近线 y=1
      │                       ╱
  0.88┤                   ╱
      │                ╱
  0.73┤             ╱
      │          ╱
  0.5 ┤───────●─────────────────────────  ← 中点 σ(0)=0.5
      │     ╱
  0.27┤  ╱
      │╱
  0.0 ┤────────────────────────────────  ← 下渐近线 y=0

```

S 形（"sigmoid" 词源）：两端贴着 0 和 1 渐近线，中段斜率陡，关于 (0, 0.5) 中心对称。

### 代码落地

底稿给了两版实现，差异不在结果，在**数值稳定性 numerical stability**。

```python
import numpy as np
from scipy.special import expit

z = -1000.0
np.exp(-z)   # exp(1000) → overflow，得到 inf，结果 1/(1+inf) = 0（侥幸正确但伴随 RuntimeWarning）
expit(z)     # 0.0，无警告
```

- 朴素版 `1.0 / (1.0 + np.exp(-z))`：$z$ 是大负数时 $-z$ 是大正数，$e^{-z}$ 直接 overflow。
- 工业做法是分支：$z \geq 0$ 用原式，$z < 0$ 改写为 $e^z / (1 + e^z)$，两边 exp 的指数都 $\leq 0$ 不会爆。
- `scipy.special.expit` 是 C 实现，内部已做这种分支并处理边界，CPU 上比 numpy 闭式表达更快也更稳。生产代码直接调它，自己写只在面试 / 教学场景。

**这一节的关键启示**：sigmoid 一招把无界分数翻译成概率，导数还能用自身表达——LR / 神经元都靠它。
