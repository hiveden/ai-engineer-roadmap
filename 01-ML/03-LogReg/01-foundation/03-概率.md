### 笔记

> 【理解】概率

**概率**（probability）：随机事件发生可能性的度量，取值 $P \in [0, 1]$。

**联合概率**：多个独立事件同时发生，等于各自概率相乘。

$$P(A \cap B) = P(A) \cdot P(B) \quad (\text{当 } A, B \text{ 独立})$$

**条件概率**：在事件 $B$ 已发生的前提下 $A$ 发生的概率。

$$P(A \mid B) = \frac{P(A \cap B)}{P(B)}$$

LR 中的输出 $\sigma(w^\top x)$ 直接解释为 $P(y=1 \mid x; w)$——给定特征 $x$ 和参数 $w$ 时正类的概率。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 生活锚 · 天气预报 70% 到底什么意思

天气 App 又说"明天降雨概率 **70%**"——这 70% 在说什么？

直觉上有三种读法，**只有一种是对的**：

- ❌ "明天 70% 部分时间在下雨"——错。再下也只是几小时的事
- ❌ "模型对下雨这事儿 70% 把握"——模糊，没法对账
- ✅ **"过去 100 个像明天这样的日子（气压、湿度、风向都接近），有 70 天下了雨"**

第三种读法叫**频率含义** frequency interpretation——它能跟现实**对账**：把今年所有标"70%"的日子拉出来，到年底一统计，下雨的应该接近 70 天。这种数才配叫"概率"，差一点就只是"分数"或"把握"。

LR 输出的"违约概率 12%"想达到的就是这种口径——拉来 100 个像他这样的客户，平均 12 个真的违约。下面把这件事掰开。

### 业务问题

01 / 02 节最后都给了一个数——"违约概率 12%"。但 12% 这个数到底是什么意思？拿给业务同事用，三种常见解读，差一点就用错：

| 解读 | 业务画面 | 能不能落地 |
|---|---|---|
| **分数 score** | "这个客户分高、那个分低，按分排队" | 只能比大小——A 客户 30、B 客户 60，**不代表 B 违约可能是 A 的两倍** |
| **把握 confidence** | "模型对自己挺有信心" | 模糊词，对不上账——"有信心"到底是 70% 还是 99%？阈值怎么定？ |
| **概率 probability** | "拉来 100 个像他这样的客户，平均会有 12 个违约" | 真能落地——能定阈值、能融合、能事后核账 |

LR 输出的**就是**第三种——一个真正有"频率含义"的数。这点直接给业务三件武器：

- **阈值能按业务调**：风控宁可错杀，把违约阈值压到 0.3；广告召回想多投，阈值抬到 0.7。前提是这个数真是概率，否则调它没意义。
- **多模型能融合**：A 模型说 0.3、B 模型说 0.5，平均一下读 0.4——只有概率才能这样加。
- **事后能核账**：把所有"违约概率 70%"的客户拉出来，看实际违约率是不是真的接近 70%。对账对得上，模型才算靠谱。

但要让"12%"当真是概率而不是"看起来像概率的数"，得先把概率本身两个最基本的概念立起来——**联合概率**和**条件概率**。这是本节的活儿。

### 【数学】数学方法

#### 数学符号扫盲

| 符号 | 读法 | 直觉 | 例子 |
|---|---|---|---|
| $P(\cdot)$ | "P of ..." | 一个事件发生的可能性，落在 [0, 1] | $P(\text{抛硬币正}) = 0.5$ |
| $\cap$ | "交"（and / 同时） | 两个事件**都**发生 | $P(\text{下雨} \cap \text{堵车})$ |
| $\mid$ | "given"（在...条件下） | 已知右边发生时，左边的概率 | $P(\text{堵车} \mid \text{下雨}) = 0.8$ |
| `;` | "参数化于" | 分号后是参数（不是随机变量） | $P(y \mid x; w)$：给定 $x$，参数为 $w$ |

竖线 $\mid$ 和分号 `;` 的区别是 LR 公式里最常踩的坑：竖线右边是**已观测的随机变量**，分号右边是**模型的参数**。统计学上前者是条件概率，后者只是说"这个分布由 $w$ 决定"。频率派立场下 $w$ 不是随机变量，所以不写在竖线右边。

#### 联合概率

两个事件**独立**时（一个发不发生不影响另一个），联合概率是乘积：

$$P(A \cap B) = P(A) \cdot P(B)$$

独立性是个**假设**，不是免费午餐。"今天下雨"和"明天下雨"显然不独立，写成乘积就错了。LR 训练时假设 N 个样本独立同分布（i.i.d.），整个训练集的联合概率才能写成 $\prod_i P(y_i \mid x_i; w)$——这是 04 节似然函数的起点。

#### 条件概率

竖线右边是已观测的"前提"，分母是前提自己的概率：

$$P(A \mid B) = \frac{P(A \cap B)}{P(B)}$$

直觉：把样本空间从"全宇宙"缩小到"$B$ 已经发生的世界"，再问 $A$ 在这个小世界里占多大比例。

#### 文氏图直觉（ASCII）

```
       全样本空间 Ω
   +-------------------------+
   |                         |
   |   .-----.   .-----.     |
   |  /       \ /       \    |
   | |    A    X    B    |   |
   |  \       / \       /    |
   |   '-----'   '-----'     |
   |             ^           |
   |          A ∩ B          |
   +-------------------------+

P(A ∩ B) = 中间交叠区面积 / Ω 总面积
P(A | B) = 中间交叠区面积 / B 圆面积   ← 分母换成了 B
```

公式上看 $P(A \mid B) = P(A \cap B) / P(B)$，几何上看就是把"参考面积"从整个 Ω 换成 B 圆——同一块交叠区，分母不同，比值不同。

#### 实算

**独立联合概率（抛硬币）**：公平硬币 $P(\text{正}) = 0.5$，独立连抛 3 次都正：

$$P(\text{正} \cap \text{正} \cap \text{正}) = 0.5 \times 0.5 \times 0.5 = 0.125$$

**条件概率（盒子摸球）**：盒子里 3 红 2 蓝共 5 球。

- $P(\text{红}) = 3/5 = 0.6$
- 已经摸出一个红球**不放回**，再摸一次：剩 2 红 2 蓝，$P(\text{第二次红} \mid \text{第一次红}) = 2/4 = 0.5$
- 联合：$P(\text{两次都红}) = P(\text{第一次红}) \cdot P(\text{第二次红} \mid \text{第一次红}) = 0.6 \times 0.5 = 0.3$

注意第二个公式：两次摸球**不独立**（不放回改变了样本空间），所以联合概率不能写成 $0.6 \times 0.6$，必须用条件概率展开。这是独立性假设失效时的标准处理方式。

**反例 · 独立性假设的代价**：抽两张扑克牌，问"两张都是 A"。如果按"独立"算 $P = \frac{4}{52} \times \frac{4}{52} \approx 0.59\%$ —— 错。第一张抽走后牌堆变了，正确算法是 $\frac{4}{52} \times \frac{3}{51} \approx 0.45\%$。**独立 ≠ 同分布**，独立要求"前事件结果不改变后事件概率"。LR 假设样本之间独立 → 似然才能写成连乘 → 训练才有解。这个假设若塌（如时序数据），LR 直接失准。

### LR 的连接

LR 的预测输出 $\hat{y} = \sigma(w^\top x + b)$ 是个落在 (0, 1) 的数。把它**直接**解释为条件概率：

$$P(y=1 \mid x; w) = \sigma(w^\top x + b)$$

读法："给定输入特征 $x$、模型参数 $w$ 已确定的前提下，这个样本属于正类的概率。" 三件事被一次性表达：

- 竖线左边 $y=1$：我们关心的事件（是正类）
- 竖线右边 $x$：已观测的前提（这个具体样本的特征向量）
- 分号右边 $w$：参数化这个概率分布的旋钮（训练好了就固定）

互补事件直接由 sigmoid 的对称性给出：

$$P(y=0 \mid x; w) = 1 - \sigma(w^\top x + b)$$

到这里 LR 完成了"线性打分 → 概率"的语义转换。下一步的问题——"$w$ 怎么定才让这些概率最贴合训练数据"——是 04 节 MLE 的主场，本节不展开。

**这一节的关键启示**：LR 的输出不是分数，是 $P(y=1 \mid x; w)$ —— 一个有概率语义的数。
