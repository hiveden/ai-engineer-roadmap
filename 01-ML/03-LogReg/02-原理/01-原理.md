### 笔记

> 【理解】原理

**Logistic Regression**：分类模型。把线性回归的输出 $z = wx + b$ 喂给 sigmoid，得到 $(0, 1)$ 区间的概率值。

**两步走**：

1. 线性模型按特征重要性算一个分 $z = wx + b$
2. sigmoid 把 $z$ 压到 $(0,1)$ 当概率；阈值（默认 0.5）切成 0/1 两类

**假设函数**：

$$h_w(x) = \sigma(w^\top x + b) = \frac{1}{1 + e^{-(w^\top x + b)}}$$

**Sigmoid**：

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

性质：$z \to +\infty$ 趋近 1；$z \to -\infty$ 趋近 0；$z = 0$ 时为 0.5。所以"线性回归 + 阈值切分" = 二分类。

**决策**：$h_w(x) \geq 0.5 \Rightarrow \hat{y} = 1$，否则 $\hat{y} = 0$。等价于 $w^\top x + b \geq 0$，即决策边界是 $w^\top x + b = 0$（线性超平面）。

**代码 API**：

```python
from sklearn.linear_model import LogisticRegression
clf = LogisticRegression()
clf.fit(X_train, y_train)
clf.predict(X_test)        # 返回 0/1 标签（阈值 0.5）
clf.predict_proba(X_test)  # 返回 [P(y=0), P(y=1)] 概率
```

## ━━━━━━━━ 讲解 ━━━━━━━━

### 生活锚 · 工资条三步算到手

想象你拿到一张计件工资条——做的每件事都有不同汇率，最后再加底薪：

- 装 1 件 A 产品 +7 元，装 1 件 B 产品 -3 元（返工扣的）
- 这个月做了 A 产品 20 件、B 产品 10 件，加底薪 100 元
- **总分** 210 元——这一步只是把"每件事 × 汇率"加起来再加底薪，没有上下限

可这个 210 没法直接当"老板对你满意吗"的答案——下个月可能算出 -47，下下个月算出 +1024。HR 想要的是一句人话："这个月表现 88 分，达标"。所以工资条之后还要加两步：**翻译成一个 0-100 分的满意度，再卡个 60 分线决定要不要发奖金**。

LR 干的就是同一条流水线——算分、翻译、卡线，三步走。下面把这条流水线掰开。

### 业务问题

线性回归 linear regression 输出一个连续的实数分。要做二分类——比如"用户会不会点击 / 邮件是不是垃圾邮件"——这个分本身没法直接当答案：它可能是 -47.2，可能是 +193.6，业务侧只想要"会 / 不会"两个标签，外加一个 0 到 1 之间的概率值用来排序、定阈值、做集成。

逻辑回归 logistic regression（也叫 LR / logit model）就是干这件事的最小拼装：**线性算分 → sigmoid 压成概率 → 阈值切类**。三步走，每一步都有现成零件。

> 别被名字骗了：「回归」是历史包袱（输出 logits 是回归量），实际任务是**分类**。sklearn 把它放在 `linear_model` 模块也是这个原因——参数化形式还是线性的，但任务变了。

### 【数学】数学方法

#### 数学符号扫盲

| 符号 | 读法 | 直觉 | 例子 |
|---|---|---|---|
| $w$ | weight 向量 | 每个特征的权重，sklearn 里叫 `coef_` | $w = [0.7, -0.3]$ |
| $b$ | bias / intercept | 偏置 / 截距，sklearn 里叫 `intercept_` | $b = 0.1$ |
| $w^\top x$ | w 转置乘 x | 向量点积 $\sum_i w_i x_i$ | $[0.7, -0.3] \cdot [2, 1] = 1.1$ |
| $z$ | logits | 线性分，sigmoid 之前的原始值 | $z = w^\top x + b$ |
| $h_w(x)$ | h sub w of x | 假设函数 hypothesis function，模型对 $x$ 的预测概率 | $h_w(x) = 0.83$ |
| $\hat{y}$ | y hat | 预测标签（0 或 1） | $\hat{y} = 1$ |

#### 三部曲拆解

**步骤 1 · 算 logits**：$z = w^\top x + b$

和线性回归一模一样的式子。$w$ 是各特征权重向量，$b$ 是偏置。$z$ 的范围是整条实数轴 $(-\infty, +\infty)$。

**步骤 2 · sigmoid 压成概率**：$h_w(x) = \sigma(z) = \dfrac{1}{1 + e^{-z}}$

sigmoid 把 $z$ 单调地映射到 $(0, 1)$，正好可以当 $P(y=1 \mid x)$ 用。

> sigmoid 本身的性质（值域 / 中点 / 导数 / 数值稳定）在 [`../01-foundation/02-sigmoid函数.md`](../01-foundation/02-sigmoid函数.md) 是主场，本节只用三条结论：$\sigma(0) = 0.5$、单调递增、$\sigma(z) \in (0, 1)$。

**步骤 3 · 阈值切类**：$\hat{y} = \mathbb{1}[h_w(x) \geq \tau]$，默认阈值 $\tau = 0.5$。

#### 假设函数实算

设 $w = [0.7, -0.3]$，$b = 0.1$，新样本 $x = [2, 1]$：

| 步骤 | 计算 | 结果 |
|---|---|---|
| logits | $z = 0.7 \times 2 + (-0.3) \times 1 + 0.1$ | $z = 1.2$ |
| 概率 | $h_w(x) = 1 / (1 + e^{-1.2}) = 1/(1 + 0.301)$ | $\approx 0.768$ |
| 标签 | $0.768 \geq 0.5$ | $\hat{y} = 1$ |

#### 决策边界 decision boundary

$\hat{y} = 1$ 等价于 $\sigma(z) \geq 0.5$，因为 sigmoid 单调，再等价于 $z \geq 0$，即：

$$w^\top x + b = 0$$

二维特征下这是一条**直线**，三维是**平面**，更高维是**超平面 hyperplane**。LR 的决策面只能是线性的——这是它和决策树 / 核方法的本质区别，下面用反例点出来。

ASCII 示意（二维，类别 0 / 1）：

```
   x2
    │       1   1   1
    │      ╱
    │   1╱   1
    │   ╱  1
    │  ╱       ← 决策边界：w₁x₁ + w₂x₂ + b = 0
    │ ╱  0
    │╱ 0   0
    └──────────── x1
       0    0
```

#### 阈值不一定是 0.5

底稿 PPT 例子用阈值 0.6——业务里调阈值很常见：

- **召回优先**（漏判代价大，如疾病筛查）：阈值调低（0.3），多预测为正类
- **精度优先**（误判代价大，如反欺诈拦截）：阈值调高（0.8），少误伤

`predict()` 默认用 0.5；要自定义阈值，从 `predict_proba()` 取 $P(y=1)$ 列自己比较。

#### 反例：非线性可分场景

LR 的决策面是直线 / 超平面。如果两类数据呈"同心圆"分布——内圈是 0、外圈是 1——LR 怎么转都画不出一个圆把它们分开，准确率会停在 50% 附近。这种场景要么手工加多项式特征 $(x_1^2, x_2^2, x_1 x_2)$ 把数据映射到高维，要么换决策树 / SVM 核方法 / 神经网络。

**判断信号**：训练集准确率怎么调超参都上不去 → 大概率是 LR 表达力不够，不是优化问题。

### 【代码】代码落地

底稿给的三行就是 sklearn 全套接口：

```python
from sklearn.linear_model import LogisticRegression
clf = LogisticRegression()
clf.fit(X_train, y_train)            # 训练：学 w 和 b
clf.predict(X_test)                  # 0/1 标签（默认阈值 0.5）
clf.predict_proba(X_test)            # 概率：每行 [P(y=0), P(y=1)]
clf.coef_, clf.intercept_            # 学到的 w 向量和 b 标量
```

> `LogisticRegression(...)` 的全部参数（penalty / C / solver / class_weight / max_iter）在 [`../03-api/`](../03-api/) 是主场，本节只用默认配置打通最小流程。

**自定义阈值**示例：

```python
proba = clf.predict_proba(X_test)[:, 1]   # 取 P(y=1)
y_pred = (proba >= 0.3).astype(int)        # 阈值改 0.3
```

**这一节的关键启示**：线性算分 + sigmoid 压概率 + 阈值切类，LR 三步走。
