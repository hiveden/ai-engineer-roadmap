**代码骨架**：

```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

# 1. 获取数据
column_names = ['Sample code number', 'Clump Thickness', 'Uniformity of Cell Size',
                'Uniformity of Cell Shape', 'Marginal Adhesion', 'Single Epithelial Cell Size',
                'Bare Nuclei', 'Bland Chromatin', 'Normal Nucleoli', 'Mitoses', 'Class']
data = pd.read_csv('breast-cancer-wisconsin.data', names=column_names)

# 2.1 缺失值处理：'?' → NaN → dropna
data = data.replace(to_replace='?', value=np.nan)
data = data.dropna()

# 2.2 特征值 / 目标值
x = data.iloc[:, 1:-1]   # 去掉 id 和 label
y = data['Class']

# 2.3 分割数据
x_train, x_test, y_train, y_test = train_test_split(x, y, test_size=0.2, random_state=22)

# 3. 标准化（量纲不一致 → LR 对尺度敏感）
transfer = StandardScaler()
x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)

# 4. 逻辑回归
estimator = LogisticRegression()
estimator.fit(x_train, y_train)

# 5. 模型评估
y_pred = estimator.predict(x_test)
print('预测值：', y_pred)
print('准确率：', estimator.score(x_test, y_test))
print('w:', estimator.coef_)
print('b:', estimator.intercept_)
```

**坑点提醒**：

- `?` 是字符串，`replace` 后整列 `dtype` 仍可能是 `object`，必要时 `astype(float)`
- 类别不均衡场景下，光看 `score`（准确率）会被误导 → 下一章学精确率 / 召回率 / F1

## ━━━━━━━━ 讲解 ━━━━━━━━

### 生活锚 · 流水线体检的五道关

去医院做一次入职体检，整套流程是固定的五道关，错一道后面全废：

1. **挂号取条码**：拿到一张贴纸，把你和后面所有化验单绑在一起
2. **采血**：抽血护士把不合格的样本（溶血 / 凝块）当场剔掉，送进下一道
3. **化验机校准**：机器开机先跑一管标准液，把读数零点对齐，否则报告全偏
4. **跑样本**：合格血样依次进机器出读数
5. **医生判读**：拿到读数表，结合参考区间下结论——正常 / 复查 / 转诊

少一步都不行：没校准 → 数值偏移；没剔废样 → 机器卡死；没判读 → 一堆数字业务用不上。

下面这个癌症分类案例就是把这五道关搬到代码里：取数据 → 清洗废样 → 标准化校准 → 跑模型 → 看准确率。整章 LR 流水线一次走通。

### 业务问题

医院给你一个 Excel：699 个肿瘤样本，每个样本 9 个医学指标（细胞团厚度 / 细胞大小均匀性 / 染色质等）+ 1 个标签（2 = 良性 benign / 4 = 恶性 malignant）。任务：给一个新样本的 9 个指标，输出是否恶性。

二分类、特征都是数值、样本量小——LR 教科书级场景。

### 【代码】完整可运行代码

```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

# 1. 获取数据
column_names = ['Sample code number', 'Clump Thickness', 'Uniformity of Cell Size',
                'Uniformity of Cell Shape', 'Marginal Adhesion', 'Single Epithelial Cell Size',
                'Bare Nuclei', 'Bland Chromatin', 'Normal Nucleoli', 'Mitoses', 'Class']
data = pd.read_csv('breast-cancer-wisconsin.data', names=column_names)

# 2.1 缺失值处理：'?' 是字符串占位，先转 NaN 再 dropna
data = data.replace(to_replace='?', value=np.nan)
data = data.dropna()

# 2.2 特征值 / 目标值
x = data.iloc[:, 1:-1]          # 去掉 id 列和 label 列，剩 9 个特征
y = data['Class']               # 标签：2 / 4

# 2.3 分割数据
x_train, x_test, y_train, y_test = train_test_split(
    x, y, test_size=0.2, random_state=22)

# 3. 特征工程：标准化（z-score）
transfer = StandardScaler()
x_train = transfer.fit_transform(x_train)   # fit 只用训练集，避免数据泄露 data leakage
x_test  = transfer.transform(x_test)        # 测试集只 transform

# 4. 机器学习（逻辑回归）
estimator = LogisticRegression(solver='lbfgs', max_iter=1000)
estimator.fit(x_train, y_train)

# 5. 模型评估
y_pred = estimator.predict(x_test)
print('预测值:', y_pred[:10])
print('准确率:', estimator.score(x_test, y_test))     # 通常 0.96 ~ 0.98
print('w (9 个特征系数):', estimator.coef_)
print('b:', estimator.intercept_)
print('classes_:', estimator.classes_)               # [2 4]，恶性 4 是正例
```

跑通后 `accuracy` 一般在 0.96 ~ 0.98 区间（受 `random_state` 影响）。

### 【代码】反例：跳过两步会怎样

#### 反例 1：不做 `dropna` 直接 fit

```python
# data 里还有 16 行 'Bare Nuclei' = '?'
estimator.fit(x_train, y_train)
# ValueError: could not convert string to float: '?'
```

`?` 是字符串，sklearn 把整列推断成 `dtype=object`，`fit` 第一步往 numpy float 转就崩。**必须先 `replace('?', np.nan)` 再 `dropna()`**。

#### 反例 2：不做标准化直接 fit

```python
# 注释掉 StandardScaler 三行
estimator = LogisticRegression(solver='lbfgs', max_iter=100)
estimator.fit(x_train, y_train)
# ConvergenceWarning: lbfgs failed to converge (max_iter reached)
print(estimator.coef_)
# 系数被量纲大的特征独裁，绝对值差几个数量级
```

9 个特征都在 1~10 区间，量纲差距其实不大，问题没有"厘米 vs 米"那么夸张，但 `lbfgs` 在未标准化数据上仍然容易触发收敛警告 / 系数不稳定。**标准化 standardization 是 LR 的默认前处理**，不是可选项。

### 【代码】关键 API 对照（链回 01）

| 这里用了 | 上一节讲过的点 |
|---|---|
| `LogisticRegression(solver='lbfgs', max_iter=1000)` | `solver` 选规模 / 默认 `lbfgs` 仅支持 `l2` |
| `estimator.coef_` / `intercept_` | 训练后的 `w` / `b` |
| `estimator.score` | 默认准确率 accuracy |
| `estimator.classes_` | 确认正例方向（少数类是正例） |

### 常见坑

- **`?` 残留 dtype**：`replace('?', np.nan)` 后 `data.dtypes` 看 `Bare Nuclei` 还是 `object`，原因是 numpy 不会自动把 NaN 列推回 float。`dropna` 后再 `data['Bare Nuclei'] = data['Bare Nuclei'].astype(float)` 显式转换更安全
- **`fit_transform` 只用训练集**：测试集用 `transform`，不要再 `fit`——否则训练集和测试集用了不同的均值方差，等同信息泄露
- **`random_state` 不固定 → 准确率漂移**：同一段代码不同人跑出 0.94 和 0.98 不是 bug，是 `train_test_split` 没固定种子
- **正例方向**：`classes_ = [2, 4]`，sklearn 按少数类（恶性 4 占 ~34%）当正例。`predict_proba` 第二列是"恶性概率"，画 ROC 时别取错列
- **`max_iter` 默认 100 太小**：`lbfgs` 在中等规模特征上常需要 500~1000，加大不影响效果只影响时间

### 准确率 0.97 够不够？

医疗场景下"准确率"是个陷阱：

- 假设 100 个样本里 65 良性 35 恶性，**全猜良性**也有 65% 准确率
- 漏诊（恶性判良性）和误诊（良性判恶性）的代价天差地别——前者死人，后者多做一次活检
- 必须看混淆矩阵 confusion matrix / 召回率 recall / F1 / ROC-AUC

→ 评估指标主场在 [`../04-classification-eval/`](../04-classification-eval/)，本节到 `score` 截止。

**这一节的关键启示**：LR 落地八成时间在数据清洗和标准化，模型本身就两行。
