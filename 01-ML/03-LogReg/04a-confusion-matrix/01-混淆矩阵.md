> 【理解】混淆矩阵

混淆矩阵作用在测试集上，把"真实标签 × 预测标签"的二维交叉计数列出来。二分类下 4 个格子，多分类下 N×N。

**命名规则**：第二个字母（P/N）= 预测的类别；第一个字母（T/F）= 预测对了没有。所以 TP = 预测为正且预测对了；FP = 预测为正但预测错了（实际是负）。

**4 个核心概念**（约定正例 = 我们关心的少数类，例如恶性肿瘤、欺诈交易）：

- **TP**（True Positive，真正例）：实际正 + 预测正 → 命中
- **FN**（False Negative，伪反例）：实际正 + 预测负 → 漏报（病人被判为健康）
- **FP**（False Positive，伪正例）：实际负 + 预测正 → 误报（健康人被判为病人）
- **TN**（True Negative，真反例）：实际负 + 预测负 → 正确放过

**矩阵布局**（行 = 真实，列 = 预测）：

|              | 预测 正例 | 预测 负例 |
|--------------|----------|----------|
| **真实 正例** | TP        | FN        |
| **真实 负例** | FP        | TN        |

**恒等式**：`TP + FN + FP + TN = 测试集总样本数`。

**例子**：6 个恶性 + 4 个良性，正例 = 恶性。

| 模型 | TP | FN | FP | TN | 解读 |
|------|----|----|----|----|------|
| A    | 3  | 3  | 0  | 4  | 不敢报恶性 → 漏诊一半 |
| B    | 6  | 0  | 3  | 1  | 宁可错杀 → 恶性全抓但误报 3 个良性 |

> 同一个准确率 `(TP+TN)/总数` 可能掩盖完全不同的错误结构 —— 这就是后面要引入 Precision / Recall / F1 的原因。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题

之前章节算分类质量只用一个数：准确率 accuracy = 预测对了的样本数 / 总样本数。简单、直观，但盖不住一类硬骨头——**类别不平衡 class imbalance**。

举个反例。某种癌症在体检人群里的患病率是 1%。我训练一个"模型"，规则是：**永远输出"健康"**。在 100 人测试集上，99 个真健康全猜对，1 个真癌症漏掉，准确率 99%。指标好看，业务上是灾难——所有病人 100% 漏诊。

**只看一个总数（准确率）丢掉了"错在哪类"的信息**。需要把「真实标签 × 预测标签」二维交叉摊开看：哪些是该报没报的（漏诊），哪些是不该报却报了的（误报）。这张交叉计数表叫**混淆矩阵 confusion matrix**——名字来自"看看模型把哪些真实类别混淆 confuse 成了哪些预测类别"。

### 数学方法

#### 数学符号扫盲

混淆矩阵的 4 个格子有一套两字母命名法，吃透命名规则比死记 4 个名字管用。

| 符号 | 读法 | 直觉 | 例子 |
|---|---|---|---|
| **T** / **F** | True / False | 第一字母 = 预测**对错**（T 对 / F 错） | TP 的 T = 这次预测对了 |
| **P** / **N** | Positive / Negative | 第二字母 = **预测的**类别（P 正 / N 负） | TP 的 P = 模型这次预测的是正例 |
| **TP** | True Positive，真正例 | 预测正 + 对了 → 实际也是正 → 命中 | 病人被判病人 |
| **FN** | False Negative，伪反例 | 预测负 + 错了 → 实际是正 → 漏报 | 病人被判健康（漏诊） |
| **FP** | False Positive，伪正例 | 预测正 + 错了 → 实际是负 → 误报 | 健康人被判病人（误诊） |
| **TN** | True Negative，真反例 | 预测负 + 对了 → 实际也是负 → 正确放过 | 健康人被判健康 |

口诀：**第二字母看预测，第一字母看对错**。读 FN 时先看 N——模型预测是负；再看 F——预测错了，所以实际是正。FN = 把正例错判成了负例 = 漏报。

约定：**正例 = 我们关心的少数类 / 高代价类**——癌症诊断里正例是恶性，欺诈检测里正例是欺诈，缺陷检测里正例是缺陷。这个约定决定了哪类错误叫"漏报"哪类叫"误报"。

#### 矩阵布局

把 4 个格子按"行 = 真实，列 = 预测"摆开：

|              | **预测 正例** | **预测 负例** |
|--------------|----------|----------|
| **真实 正例** | TP（命中）| FN（漏报）|
| **真实 负例** | FP（误报）| TN（正确放过）|

**对角线**（TP + TN）= 预测对的样本数；**反对角线**（FN + FP）= 预测错的样本数。

**恒等式**：`TP + FN + FP + TN = 测试集总样本数`——每个测试样本必然落进且只落进一个格子。准确率 = 对角线 / 总数 = `(TP + TN) / (TP + FN + FP + TN)`。

> 注意：sklearn 的 `confusion_matrix` 默认行列顺序是按 `labels` 参数给的顺序排，**不是固定"正例在前"**。后面代码段会踩这个坑。

多分类时矩阵从 2×2 扩成 N×N，行列含义不变（行真实、列预测），对角线还是预测对的格子。本节聚焦二分类。

#### 实算（PPT 例子）

题面（PPT slide 34）：测试集 10 个样本，6 个恶性肿瘤 + 4 个良性肿瘤，正例 = 恶性。

**模型 A**：预测对了 3 个恶性、4 个良性。

- 6 个恶性里预测对了 3 个 → 这 3 个是 **TP = 3**（实际正、预测正）。
- 6 个恶性里剩下的 3 个被错判成了良性 → **FN = 3**（实际正、预测负 → 漏诊）。
- 4 个良性里预测对了 4 个 → **TN = 4**（实际负、预测负）。
- 4 个良性里被错判成恶性的：4 - 4 = 0 → **FP = 0**（实际负、预测正 → 误诊）。
- 校验：3 + 3 + 0 + 4 = 10 ✓

**模型 B**：预测对了 6 个恶性、1 个良性。

- 6 个恶性全部预测对 → **TP = 6**，**FN = 0**（一个不漏）。
- 4 个良性只对了 1 个，剩 3 个被错判成恶性 → **TN = 1**，**FP = 3**。
- 校验：6 + 0 + 3 + 1 = 10 ✓

汇总：

| 模型 | TP | FN | FP | TN | 准确率 = (TP+TN)/10 |
|------|----|----|----|----|---|
| A | 3 | 3 | 0 | 4 | **70%** |
| B | 6 | 0 | 3 | 1 | **70%** |

两个模型准确率**完全相同**，但错误结构截然不同——见下面反例段。

### 代码落地

底稿代码用 `sklearn.metrics.confusion_matrix`，关键参数和坑点：

```python
from sklearn.metrics import confusion_matrix
import pandas as pd

y_true = ["恶性"]*6 + ["良性"]*4
y_pred_A = ["恶性"]*3 + ["良性"]*7  # 模型 A：3 恶 + 7 良
labels = ["恶性", "良性"]            # 显式指定行列顺序：正例在前

cm = confusion_matrix(y_true, y_pred_A, labels=labels)
print(pd.DataFrame(cm, index=["真实 恶性", "真实 良性"],
                       columns=["预测 恶性", "预测 良性"]))
#            预测 恶性  预测 良性
# 真实 恶性        3        3      ← TP=3, FN=3
# 真实 良性        0        4      ← FP=0, TN=4
```

**两个常见坑**：

1. **`labels` 必须显式给** —— 不传时 sklearn 按字母序排（"良性" < "恶性" 在中文里不一定符合直觉），结果矩阵的行列顺序会和你脑里的"正例在前"不一致，TP / FN 位置全错位。
2. **正例约定要和业务对齐** —— `labels=["恶性", "良性"]` 表示恶性放第 0 行第 0 列，TP 就在 `cm[0, 0]`。如果误把良性当正例，整个混淆矩阵的 4 个数没变，但语义上的"漏诊 / 误诊"就反了。

跨语言落地视角：模型训练完只需把 `y_true` 和 `y_pred` 两列存成 csv / parquet，Go / Java / Node 任何语言都能现场算 4 个数（就是两层循环计数），不依赖 sklearn 运行时。混淆矩阵是**评估侧**的纯统计，不进生产推理路径。

### 反例：同准确率掩盖的错误结构

回到上面 PPT 例子，模型 A 和模型 B 准确率都是 70%，单看这个数会以为两个模型一样好。摊开混淆矩阵立刻看出问题：

| 模型 | TP | FN | FP | TN | 错误模式 | 业务后果（癌症诊断场景）|
|---|---|---|---|---|---|---|
| A | 3 | **3** | 0 | 4 | 不敢报恶性 → 漏诊一半 | 3 个真癌症被告知健康，错过治疗窗口 |
| B | 6 | 0 | **3** | 1 | 宁可错杀 → 全抓但误报 3 个 | 3 个健康人被吓一跳，复检排除 |

**漏诊（FN）和误诊（FP）的代价天差地别**——癌症场景一个 FN 是人命，一个 FP 是一次复检。"准确率 70%" 这一个数把这件事完全抹平了。混淆矩阵把 4 个数分开摆出来，业务方才能据此选模型、调阈值。

下一节 04b 用 Precision / Recall / F1 三个比例，从不同视角把这 4 个数拆成可比的指标。

**这一节的关键启示**：一个准确率盖不住两种错，混淆矩阵把漏诊和误诊摊开。
