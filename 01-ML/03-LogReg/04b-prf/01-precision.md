精确率(Precision) - 举个例子

已知：样本集10样本，有 6 个恶性肿瘤样本，4 个良性肿瘤样本，我们假设恶性肿瘤为正例

模型B：预测对了 6 个恶性肿瘤样本，1个良性肿瘤样本
- 真正例 TP 为：6
- 伪反例 FN 为：0
- 伪正例 FP 为：3
- 真反例 TN：1
- 精准度：6/(6+3) = 67%

模型A：预测对了 3 个恶性肿瘤样本，4 个良性肿瘤样本
- 真正例 TP 为：3
- 伪反例 FN 为：3
- 伪正例 FP 为：0
- 真反例 TN：4
- 精准度：3 / (3+0) = 100%

> **Notes**：类比 举例：样本极少 统计误差大：比如 预测中国男女比例

from sklearn.metrics import accuracy_score
from sklearn.metrics import precision_score

def dm01_精度Precision():

    y_true = ["恶性", "恶性", "恶性", "恶性", "恶性", "恶性", "良性", "良性", "良性", "良性"]

    result = precision_score(y_true, y_pred2, pos_label="恶性")

### 笔记

> 【掌握】Precision（精确率）

**精确率**（Precision，又称查准率）：模型预测为正例的样本中，真正是正例的比例。回答"**报警的有多准**"。

$$
Precision = \frac{TP}{TP + FP}
$$

**直觉**：分母是"模型说是正例"的全部，分子是"说对了的"。FP（假正例 / 误报）越多，精确率越低。

**例**：6 个恶性 + 4 个良性，正例 = 恶性。
- 模型 A：抓出 3 个全对 → $P = 3/(3+0) = 100\%$
- 模型 B：抓出 9 个里 6 个对 → $P = 6/(6+3) \approx 67\%$

**适用场景**：误报代价高的业务——垃圾邮件过滤（误杀正常邮件）、推送广告（打扰用户）、刑事定罪（宁可放过不可错杀）。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题

延续 04a 的 TP/FP/FN/TN 四分格——精确率只看"模型按下报警按钮的那一列"。

业务场景：你做一个肿瘤筛查模型，把患者分成"恶性"和"良性"两类。模型每次报"恶性"时，医生就要安排穿刺活检——一次穿刺意味着患者要承受痛苦、花钱、还要等 3 天结果。**模型每报一次警，都是在花真实成本**。

老板的提问换种说法：「**你按下警报的那 9 次，里头到底有几次是真的？**」——这就是精确率。

反过来想，如果模型瞎报，9 次有 6 次真、3 次假——医生白做 3 次活检——业务方会立刻砍掉这个模型，不管它召回多高。

### 数学方法

#### 数学符号扫盲

| 符号 | 读法 | 直觉 |
|---|---|---|
| TP | true positive | 真正例：模型说"恶性"，实际"恶性" |
| FP | false positive | 伪正例 / 误报：模型说"恶性"，实际"良性" |

#### 公式 + 直觉

$$
Precision = \frac{TP}{TP + FP}
$$

分母 $TP + FP$ = "**模型预测为正的全部**"（不管对错，凡是按了警报的）。
分子 $TP$ = 这堆里**真的是正的**。

一句话：**报警里的真警报占比**。FP 越多，分子不变分母涨，精确率掉。

#### 实算（PPT 6 + 4 例）

10 个样本：6 恶性 + 4 良性，正例 = 恶性。

**模型 A**：保守派，只敢报 3 次，全报对了。
- 6 个恶性里 → 模型 A 报对 3 个，剩 3 个被划为良性
- 4 个良性里 → 模型 A 全部判良性

| | 实际恶性 | 实际良性 |
|---|---|---|
| **预测恶性** | TP = 3 | FP = 0 |
| **预测良性** | FN = 3 | TN = 4 |

$$
P_A = \frac{3}{3 + 0} = 100\%
$$

**模型 B**：激进派，9 次报恶性、1 次良性。
- 6 个恶性 → 全抓出（6 TP）
- 4 个良性 → 错杀 3 个（3 FP），1 个判对（1 TN）

| | 实际恶性 | 实际良性 |
|---|---|---|
| **预测恶性** | TP = 6 | FP = 3 |
| **预测良性** | FN = 0 | TN = 1 |

$$
P_B = \frac{6}{6 + 3} \approx 67\%
$$

模型 A 报 3 中 3，**精确率 100%**；模型 B 报 9 中 6，**精确率 67%**。光看精确率，A 完胜。

### 适用场景 + 反例

**适用**：误报代价 ≫ 漏报代价时。

| 业务 | 误报代价 |
|---|---|
| 垃圾邮件过滤 | 把客户的合同邮件丢进垃圾箱 |
| 推送广告 / 召回触达 | 打扰用户、损耗信任 |
| 刑事定罪 | 冤枉好人 |
| 自动驾驶刹停决策 | 误刹追尾 |

**反例**（精确率的盲区）：模型 A 精确率 100% 看着完美——但它**漏掉了 6 个恶性里的 3 个**（FN = 3）。这 3 个患者带着癌细胞回家了。

把"反例"做到极端：再造一个**模型 C**，10 个样本里**只敢挑最有把握的那 1 个**报恶性，恰好对了。
- TP = 1，FP = 0 → $P_C = 1/(1+0) = 100\%$
- FN = 5（漏掉 5 个恶性）

精确率 100%，召回率 1/6 ≈ 17%。**只看精确率，C 和 A 一样满分**——但 C 漏掉的患者更多。

这就是只看精确率的盲区：**它鼓励模型保守、少报、宁可漏**。要补这个盲区就得引入召回率（下一节），最后用 F1 把两边拉回平衡。

### 代码落地

`sklearn.metrics.precision_score` 一行搞定：

```python
from sklearn.metrics import precision_score

y_true  = ["恶性"]*6 + ["良性"]*4
y_pred_A = ["恶性"]*3 + ["良性"]*7              # 模型 A
y_pred_B = ["恶性"]*9 + ["良性"]*1              # 模型 B

precision_score(y_true, y_pred_A, pos_label="恶性")  # 1.0
precision_score(y_true, y_pred_B, pos_label="恶性")  # 0.6666...
```

**坑点**：

1. **`pos_label` 必须显式指定**。标签不是 0/1 时（这里是中文 "恶性"/"良性"），sklearn 不知道哪边算正例，不传会报错或默认拿字符串字典序的"良性"当正例，结果完全反过来。
2. **样本里完全没有"预测为正"时**（分母为 0），sklearn 默认返回 0 并发 `UndefinedMetricWarning`。生产里建议显式传 `zero_division=0` 或 `1` 自己定。
3. 多分类要传 `average="macro"/"micro"/"weighted"` ——本章不展开，04c 章 `classification_report` 时再讲。

**这一节的关键启示**：精确率守的是"报警别瞎报"，越保守越高，但不管漏掉多少。
