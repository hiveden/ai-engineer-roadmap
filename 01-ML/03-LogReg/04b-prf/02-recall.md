召回率(Recall) – 概念

也叫查全率，指的是预测为真正例样本占所有真实正例样本的比重
例如：恶性肿瘤当做正例样本，则我们想知道模型是否能把所有的恶性肿瘤患者都预测出来。

计算方法：R = TP / (TP + FN)

大白话：所有的正例中，你找到多少？
避免漏报

召回率(Recall) - 举个例子

已知：样本集10样本，有 6 个恶性肿瘤样本，4 个良性肿瘤样本，我们假设恶性肿瘤为正例

模型B：预测对了 6 个恶性肿瘤样本，1个良性肿瘤样本
- 真正例 TP 为：6
- 伪反例 FN 为：0
- 伪正例 FP 为：3
- 真反例 TN：1
- 召回率：6 / (6 + 0) = 100%

模型A：预测对了 3 个恶性肿瘤样本，4 个良性肿瘤样本
- 真正例 TP 为：3
- 伪反例 FN 为：3
- 伪正例 FP 为：0
- 真反例 TN：4
- 召回率：3 / (3 + 3) = 50%

查全率 - score - 举个例子（PPT 原文单行连写）：

```python
# 召回率（查全率）
from sklearn.metrics import recall_score

def dm02_召回率recall():
    # 样本集中共有6个恶性肿瘤样本, 4个良性肿瘤样本
    y_true = ["恶性", "恶性", "恶性", "恶性", "恶性", "恶性",  "良性", "良性", "良性", "良性"]
    # 1. 模型 A: 预测对了3个恶性肿瘤样本, 4个良性肿瘤样本
    y_pred1 = ["恶性", "恶性", "恶性", "良性", "良性", "良性", "良性", "良性", "良性", "良性"]
    result = recall_score(y_true, y_pred1, pos_label="恶性")
    print("模型A召回率:", result)
    # 2. 模型 B: 预测对了6个恶性肿瘤样本, 1个良性肿瘤样本
    y_pred2= ["恶性", "恶性", "恶性", "恶性", "恶性", "恶性","恶性", "恶性", "恶性", "良性"]
    result = recall_score(y_true, y_pred2, pos_label="恶性")
    print("模型B召回率:", result)
```

### 笔记

> 【掌握】Recall（召回率）

**召回率**（Recall，又称查全率、TPR）：所有真实正例样本中，被模型成功预测出来的比例。回答"**该抓的抓到了多少**"。

$$
Recall = \frac{TP}{TP + FN}
$$

**直觉**：分母是"真实的全部正例"，分子是"被抓到的"。FN（伪反例 / 漏报）越多，召回率越低。

**例**：6 个恶性肿瘤为正例。
- 模型 A：只识别出 3 个恶性 → $R = 3/(3+3) = 50\%$（漏掉 3 个）
- 模型 B：6 个恶性全部识别出 → $R = 6/(6+0) = 100\%$

**适用场景**：漏报代价高的业务——癌症 / 疾病筛查、风控反欺诈、安防监控（宁可错杀不可放过）。

**Precision vs Recall 取舍**：调高分类阈值 → 精确率↑召回率↓；调低阈值 → 召回率↑精确率↓。鱼和熊掌不可兼得。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题

延续 04a 的 TP/FP/FN/TN 四分格——召回率只看"实际患者那一行"。

换 01 节里医生的视角：精确率管"报警别瞎报"，但医生还有另一句话——「**所有真的得了癌的人，我们到底抓到了几个？**」。漏掉一个，就是一条命。

业务场景：6 个恶性 + 4 个良性的体检池。

- 模型只敢报最有把握的 3 个 → 看似精确率 100% 风光，**但还有 3 个癌症患者带着病走出诊所**。
- 这 3 个人就是 FN（伪反例 / 漏报 false negative）。每一个 FN 都是医疗事故。

老板的提问换种说法：「**真患者总共 6 人，我们筛出几个？**」——这就是召回率。

### 数学方法

#### 数学符号扫盲

| 符号 | 读法 | 直觉 |
|---|---|---|
| FN | false negative | 伪反例 / 漏报：实际"恶性"，模型说"良性" |
| TPR | true positive rate | 真正例率，召回率的别名（04c ROC 章主用此名） |

#### 公式 + 直觉

$$
Recall = \frac{TP}{TP + FN}
$$

分母 $TP + FN$ = "**实际所有正例**"（不管模型有没有抓到，反正真就是正）。
分子 $TP$ = 模型**真的抓到的**。

一句话：**真患者里被筛出来的占比**。FN 越多，分母不变（真患者总数固定），分子越小，召回率掉。

#### 跟精确率分母对比

精确率分母看模型"按了几次警报"，召回率分母看世界"实际有几个正例"。**两个分母不一样**——这是 P/R 二者无法压缩成一个的根本原因。

| | 分母 | 看谁的视角 |
|---|---|---|
| 精确率 | TP + FP（按了警报的全部）| **模型**的报警单 |
| 召回率 | TP + FN（实际所有正例）| **世界**的真值 |

#### 实算（PPT 6 + 4 例）

接 01 节：6 恶性 + 4 良性，正例 = 恶性。

**模型 A**：保守派，只敢报 3 个。
- TP = 3（抓对 3 个恶性），FN = 3（剩 3 个恶性被错判良性）

$$
R_A = \frac{3}{3 + 3} = 50\%
$$

——6 个真患者，**漏掉一半**。

**模型 B**：激进派，9 个判恶性。
- TP = 6（6 个恶性全抓），FN = 0

$$
R_B = \frac{6}{6 + 0} = 100\%
$$

——6 个真患者**全部筛出**。

光看召回率，**B 完胜 A**。但回顾 01 节，B 的精确率只有 67%——它误报了 3 个良性。这就是 P 和 R 的取舍。

### 适用场景 + 反例

**适用**：漏报代价 ≫ 误报代价时。

| 业务 | 漏报代价 |
|---|---|
| 癌症 / 重病筛查 | 漏诊 = 死亡 |
| 风控反欺诈 | 漏一笔欺诈 = 真金白银损失 |
| 安防 / 火灾报警 | 漏一次 = 灾难 |
| 司法搜查（前期初筛）| 漏一个嫌疑人 = 跑路 |

口诀：「**宁可错杀，不可放过**」。

**反例**（召回率的盲区）：把所有样本**全判正例**——TP = 6，FN = 0，**召回率 100%**。但精确率 = 6/(6+4) = 60%（4 个良性全是 FP）。

更极端的「全报警 baseline」：模型 D 啥都不学，闭眼输出 "恶性"——10 次全报。
- TP = 6，FN = 0 → $R_D = 100\%$
- FP = 4 → $P_D = 60\%$

**召回率给了它满分**——但这模型完全没用，等于把整个体检中心一锅端去做穿刺。

这就是只看召回率的盲区：**它鼓励模型激进、多报、宁可错杀**。要补这个盲区就得回头看精确率，最后用 F1 把两边拉回平衡（下一节）。

#### 阈值取舍（钩到 03 章 sigmoid）

LR 的 sigmoid 输出概率 $\hat{p} \in (0, 1)$，默认阈值 0.5 才判正。把阈值调一调，P/R 反向走：

| 阈值 | 行为 | 精确率 | 召回率 |
|---|---|---|---|
| 0.9 | 只报最有把握的 | ↑ | ↓ |
| 0.5 | 默认 | 中 | 中 |
| 0.1 | 稍微像就报 | ↓ | ↑ |

——同一个模型，阈值滑一下，P 和 R 此消彼长。这条曲线就是 04c 要讲的 ROC / PR 曲线主场。

### 代码落地

`sklearn.metrics.recall_score` 一行：

```python
from sklearn.metrics import recall_score

y_true  = ["恶性"]*6 + ["良性"]*4
y_pred_A = ["恶性"]*3 + ["良性"]*7              # 模型 A
y_pred_B = ["恶性"]*9 + ["良性"]*1              # 模型 B

recall_score(y_true, y_pred_A, pos_label="恶性")  # 0.5
recall_score(y_true, y_pred_B, pos_label="恶性")  # 1.0
```

**坑点**：

1. **`pos_label` 必须显式指定**——同 01 节，字符串标签不传会反过来或报错。
2. **极端不平衡数据**（如 1% 正例 99% 负例）单看 recall 容易被「全报正」骗——必须配 precision 一起看，或直接用 F1（下一节）。
3. `recall_score` 在多分类下要选 `average`，本章不展开。

**这一节的关键启示**：召回率守的是"该抓别漏抓"，越激进越高，但不管误报多少。
