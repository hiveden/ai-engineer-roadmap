AUC（Area Under the ROC Curve）曲线下面积

ROC曲线的优劣可以通过曲线下的面积（AUC）来衡量，AUC越大表示分类器性能越好。
当AUC=0.5时，表示分类器的性能等同于随机猜测
当AUC=1时，表示分类器的性能完美，能够完全正确地将正负例分类。

真正率即召回率：TPR = TP / (TP + FN)
伪正率：FPR = FP / (FP + TN)

      │ ╭╯  ╌╌╌
  0.0 ┼╌╌╌╌╌╌╌╌╌╌╌╌

> 比如以头发长短判断性别，阈值变化
> AUC=0.5 是随机猜测。比如没有分数和阈值2个概念，对于N个样本，直接随机概率p=0.5预测为正例，那么N个正例中，真正率=0.5N/N；M个假例中，伪正率=0.5M/M，结果横纵坐标都是0.5。
> 现在调整概率阈值p，无论P是多少，横纵坐标相等，且都等于P。
> AUC=1是完美分类器，（0,1）点代表纵坐标分母FN=0，横坐标分子FP=0

### 笔记

> 【知道】ROC 曲线和 AUC 指标

样本不平衡时，单看 Recall / Precision 易被多数类淹没。ROC + AUC 的核心：**抛开单一阈值，直接评估模型对正负样本的整体辨别能力**。

> ROC 曲线

ROC（Receiver Operating Characteristic）曲线把分类器在**所有阈值**下的表现画成一条线。两个轴分别看正负样本：

| 轴 | 含义 | 公式 |
|---|---|---|
| 纵轴 **TPR**（True Positive Rate） | 正样本中被预测为正的概率 = 召回率 Recall | TP / (TP + FN) |
| 横轴 **FPR**（False Positive Rate） | 负样本中被预测为正的概率 = 误报率 | FP / (FP + TN) |

**直觉**：TPR 越高 = 抓正样本越准；FPR 越低 = 误伤负样本越少。理想模型 TPR=1 且 FPR=0，即落在 **(0, 1)** 点。

四个特殊点（横轴 FPR，纵轴 TPR）：

| 点 | 含义 |
|---|---|
| (0, 0) | 全预测为负：正样本全错，负样本全对 |
| (1, 1) | 全预测为正：正样本全对，负样本全错 |
| (0, 1) | 完美分类器 |
| (1, 0) | 全反着来（正负全错） |

对角线 y = x 是**随机猜测**基线（TPR = FPR），曲线必须显著高于对角线才算有判别力。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 生活锚 · 安检门灵敏度旋钮

机场安检门后台有一个灵敏度旋钮，工作人员可以拧：

- **拧到最高**：耳钉、皮带扣、口香糖锡纸都会响——真带刀的肯定漏不掉，但 100 个旅客 80 个被叫去复检，队伍堵到登机口
- **拧到最低**：只有大金属才响——队伍走得飞快，但小刀片可能溜过去
- **拧到中间**：找一个让"真带危险品的人尽量都响"和"普通旅客尽量不被冤枉"都过得去的位置

同一台机器、同一批旅客，旋钮位置一变，"抓到几成危险品"和"误响几成普通人"的成绩单完全不同。可这台机器本身的灵敏程度——能不能把刀和钥匙串区分开——其实是固定的。本节要的就是一根能**抛开旋钮位置、直接评估机器本身有多准**的尺子。

### 业务问题

LR 输出的是概率，要落地成"点击 / 不点击"必须挑一个阈值。04a / 04b 章用的都是默认 0.5——但这只是**一个**阈值下的成绩单，换个阈值精确率召回率全变。

举个广告投放的例子：业务侧今天预算紧，要把误伤率（白白发广告给不感兴趣用户的比例）压到 5% 以下；明天补贴上线，要召回率（成单用户里被识别出来的比例）拉到 95%。同一个模型在两种阈值下表现差到像两个模型——但模型本身的"辨别能力"明明只有一份。

要的是一个**与阈值无关**的指标，把"模型把正负样本拉得多开"这件事单独拎出来评估。受试者工作特征曲线 ROC（Receiver Operating Characteristic curve，源自二战雷达兵区分敌机和飞鸟的研究）就是干这事的：把所有可能的阈值都试一遍，每个阈值出一组 (FPR, TPR)，全部点连起来就是 ROC 曲线。

### 【数学】数学方法

#### TPR 和 FPR：分母锁死，只看一类

混淆矩阵 4 格在 04a 已展开，这里只用钩子：TP / FN / FP / TN。把它们重新分组：

```
正样本这一行：TP + FN（实际为正的总数，固定）
负样本这一行：FP + TN（实际为负的总数，固定）
```

- **TPR = TP / (TP + FN)**：分母**只看正样本**，问"正样本里抓到几成"——和 Recall 一字不差，新名字只是为了和 FPR 配对
- **FPR = FP / (FP + TN)**：分母**只看负样本**，问"负样本里误伤几成"

关键点：TPR 和 FPR **分母分别锁在正、负样本内部**——这就是 ROC 抗不平衡的根源。哪怕 1 万个负样本对 100 个正样本，FPR 也是按 1 万的分母算，不会被多数类绑架（对比：Accuracy 在 99:1 数据上瞎猜负类就有 99% 准确率，毫无信息量）。

#### 数学符号扫盲

| 符号 | 读法 | 直觉 |
|---|---|---|
| TPR | True Positive Rate | 正样本被抓到的比例（= Recall）|
| FPR | False Positive Rate | 负样本被误判为正的比例 |
| TP / FN / FP / TN | true/false positive/negative | 混淆矩阵 4 格，主场 04a |

#### 4 个特殊点的几何意义

横轴 FPR、纵轴 TPR，单位正方形 $[0, 1] \times [0, 1]$ 的四个角各有故事：

```
   TPR
   1 ┤●(0,1) 完美             ●(1,1) 全判正
     │  |\
     │  | \  ← 对角线 y=x：随机猜测
     │  |  \
   0.5┤  |   \
     │  |    \
     │  |     \
   0 ┤●(0,0) 全判负 ── ── ── ●(1,0) 全反
     └──────────────────────→ FPR
        0      0.5      1
```

- **(0, 0)**：阈值拉到 1.0 以上，没人被判正 → 正样本全漏 (TPR=0)、负样本全对 (FPR=0)
- **(1, 1)**：阈值压到 0 以下，全被判正 → 正样本全抓 (TPR=1)、负样本全误伤 (FPR=1)
- **(0, 1) 完美**：正样本一个不漏、负样本一个不误——曲线贴这个角越紧越好
- **(1, 0) 灾难**：正负完全反着来，但**别绝望**——把预测翻过来用就直接到 (0, 1) 完美点

任何模型从 (0, 0) 开始（阈值最高），降低阈值的过程中曲线向右上爬，最终到 (1, 1)（阈值最低）。中间走的轨迹就是 ROC 曲线。

#### 对角线为何是随机基线

随机猜测：不看特征，直接以概率 $p$ 把样本判正。在 $N$ 个正样本里随机判出 $pN$ 个正 → TPR = $p$；在 $M$ 个负样本里随机判出 $pM$ 个正 → FPR = $p$。无论 $p$ 取多少，TPR ≡ FPR，落在对角线上。

**反推**：曲线**贴着对角线** = 模型不比抛硬币强；曲线**在对角线下方** = 比随机还差，把预测取反就到对角线上方。

#### 【拓展】最佳阈值：约登指数

实际部署需要选一个阈值。约登指数 Youden's J statistic：

$$J = \text{TPR} - \text{FPR}$$

几何含义是 ROC 曲线上点到对角线的**垂直距离**——离 (0, 1) 越近、离对角线越远，$J$ 越大。在曲线上扫一遍，$J$ 最大的那个点对应的阈值就是约登准则下的最佳阈值。

业务上还可以按代价加权（漏检贵就压 FN，误报贵就压 FP），约登只是无偏好的默认选法。

**这一节的关键启示**：ROC 把所有阈值的成绩单一次画出，分母锁死正负两边，天然抗不平衡。
