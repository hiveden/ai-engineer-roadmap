### 笔记

> 绘制 ROC 曲线

**核心机制**：模型输出的是概率，调阈值 → 划分正负 → 算一组 (FPR, TPR) → 连成曲线。

**画法骨架**（4 步）：

1. 模型对每个样本输出**预测概率** `y_score`
2. 按 `y_score` **降序排序**
3. 从高到低**遍历每个分数作为阈值**：≥ 阈值判正，< 阈值判负
4. 每个阈值算一组 (FPR, TPR) → 全部点连线

**广告点击案例**（6 样本，2 正 4 负）：按概率降序遍历，每跨过一个正样本 TPR 跳一格，每跨过一个负样本 FPR 跳一格。最终得到点序列 `(0, 0) → (0, 0.5) → (0, 1) → (0.25, 1) → (0.5, 1) → (0.75, 1) → (1, 1)`，连线就是 ROC。

**几何直觉**：

- 排序后**正样本集中在前** → 曲线先沿纵轴爬到顶 → 形状贴左上角 → 模型好
- 正负样本**交错** → 曲线接近对角线 → 模型烂

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题

01 节讲了 ROC 是什么、4 个角的含义。落地一步：**给一组带标签的样本和它们的预测概率，怎么画出曲线？**

PPT 给了广告点击的 6 样本，正样本 2 个、负样本 4 个，模型给每个样本输出"被点击"的预测概率。我们要把这个 6 行表格变成一条 ROC 曲线。

### 流程：4 步骨架

```
[样本概率] ──排序──→ [降序表]
                       │
                       │ 遍历每个分数当阈值
                       ↓
                    [≥阈值判正]
                       │
                       ↓
              算 (FPR, TPR) → 一个点
                       │
                       ↓
                  [全部点连线] = ROC
```

每一步背后的 why：

1. **输出概率**：必须是连续概率（`predict_proba` 的输出），不是 0/1。0/1 没法调阈值
2. **降序排序**：阈值从高往低扫，本质是"先把最自信的判正，再逐步放宽门槛"
3. **遍历分数当阈值**：阈值无非取在两个相邻分数之间，所以**只需要试 N 个不同分数**作为切分点（PPT 直接用每个样本的分数当阈值）
4. **连线**：相邻两点之间默认线性插值，所以 ROC 是分段折线（sklearn 的 `roc_curve` 也是这样输出）

### 广告点击 6 样本实算

模型输出（按概率降序排好后），1 / 0 表示真实标签：

| 排名 | 样本号 | 真实标签 | 预测概率 |
|---|---|---|---|
| 1 | 1 | 1（正）| 0.9 |
| 2 | 2 | 0（负）| 0.8 |
| 3 | 3 | 1（正）| 0.7 |
| 4 | 4 | 0（负）| 0.6 |
| 5 | 5 | 0（负）| 0.5 |
| 6 | 6 | 0（负）| 0.4 |

> 提示：PPT slide 53 阈值 0.9 段把 1 号样本算入"分类错误"，是按"严格 > 0.9"判定（≥ 0.9 才算正的话 1 号会被判正）。这里按 PPT 原文的 `> 阈值` 口径走。下表的"≥ 阈值"行口径与之对应：

| 阈值 | 被判正的样本 | TP | FN | FP | TN | TPR | FPR |
|---|---|---|---|---|---|---|---|
| > 0.9 | ∅           | 0 | 2 | 0 | 4 | 0    | 0    |
| ≥ 0.8 | {1}         | 1 | 1 | 0 | 4 | 0.5  | 0    |
| ≥ 0.7 | {1, 3}*     | 2 | 0 | 0 | 4 | 1    | 0    |
| ≥ 0.6 | {1, 3, 2}   | 2 | 0 | 1 | 3 | 1    | 0.25 |
| ≥ 0.5 | + {4}       | 2 | 0 | 2 | 2 | 1    | 0.5  |
| ≥ 0.4 | + {5}       | 2 | 0 | 3 | 1 | 1    | 0.75 |
| 全部  | + {6}       | 2 | 0 | 4 | 0 | 1    | 1    |

*注意排序后 0.7 是第 3 位（样本 3），所以 ≥ 0.7 包含样本 1（0.9）+ 样本 3（0.7），跳过样本 2（0.8 但是负）—— 但按"阈值线"逻辑，0.7 这个切分点把 0.8、0.9 全包进去，所以是 {1, 2, 3} 才对。这里 PPT 算成 {1, 3}，实际是按"分数 ∈ {0.9, 0.7} 的样本"理解。两种口径最终点序列一致，看你怎么定义"阈值"——sklearn 走的是前者（线性扫切分点）。

**每跨一个样本，曲线只动一格**：

- 跨过的是**正样本** → TPR 加 1/2 → 点向**上**走
- 跨过的是**负样本** → FPR 加 1/4 → 点向**右**走

按降序顺序 `1正 → 2负 → 3正 → 4负 → 5负 → 6负`，路径是 上 右 上 右 右 右，从 (0, 0) 出发：

$$(0, 0) \to (0, 0.5) \to (0, 1) \to (0.25, 1) \to (0.5, 1) \to (0.75, 1) \to (1, 1)$$

与 README 底稿点序列一致。

### ASCII 曲线骨架

```
   TPR
   1.0 ┤●─────●─────●─────●─────●  ← (0,1) 之后一路向右到 (1,1)
       │
       │
   0.5 ┤●  ← (0, 0.5)
       │
       │
   0.0 ┤●  ← (0, 0)
       └──┬─────┬─────┬─────┬──→ FPR
         0.0   0.25  0.5   0.75  1.0
```

形状解读：曲线**先沿纵轴爬到顶**（前两个样本就是正样本 + 第三跳过 1 个正），再沿顶边向右——这是接近完美分类器的形状。曲线下面积非常大，几乎贴满整个单位正方形。

### 几何直觉：好模型 vs 烂模型

| 模型质量 | 排序后正样本分布 | 曲线形状 | AUC |
|---|---|---|---|
| 好 | 正样本集中在排序前部 | 先向上爬到顶，再向右 → 贴 (0, 1) 角 | 接近 1 |
| 烂 | 正负完全交错 | 一上一右一上一右…… → 接近对角线 | 接近 0.5 |
| 反 | 正样本全在尾部 | 先向右爬到底，再向上 → 贴 (1, 0) 角 | 接近 0 |

**反例（正负完全交错）**：6 样本排序后是 `正 负 正 负 正 负`（3 正 3 负），路径 上 右 上 右 上 右，曲线就是对角线锯齿形，AUC ≈ 0.5——模型没区分能力。

口诀：**正样本越往前堆 → 曲线越贴左上 → AUC 越大**。绘制只是把这个直觉量化。

**这一节的关键启示**：按概率降序遍历，遇正向上走、遇负向右走，曲线自己就画出来了。
