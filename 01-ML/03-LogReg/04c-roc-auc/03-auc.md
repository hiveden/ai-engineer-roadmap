### 笔记

> AUC 值

**AUC**（Area Under Curve）= ROC 曲线下面积，把"形状好不好"压成一个 **[0, 1]** 数字。

**解读速查**：

| AUC | 含义 |
|---|---|
| 1.0 | 完美分类器（几乎不存在） |
| 0.5 | 随机猜测，模型无效 |
| < 0.5 | 比随机还差，预测反着用即可 |
| 0.7 ~ 0.85 | 工程实践中常见的"可用"区间 |
| > 0.9 | 强判别力（警惕数据泄漏） |

**概率解释**：随机抽一个正样本和一个负样本，AUC = 模型给正样本打分 > 给负样本打分的概率。

**核心定位**：AUC 评估的是**正负样本的辨别能力**，与阈值选择无关 → 适合不平衡场景。

> 【小结】（slide 57）

ROC 曲线以 TPR / FPR 为 y / x 轴，把模型在不同阈值下的表现以曲线形式展现。AUC 越大表示分类器性能越好；AUC = 0.5 等同随机猜测，AUC = 1 完美分类。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题

01 / 02 节画出了一条 ROC 曲线。但工程要的是**一个能比较的数字**——给两个候选模型，告诉我哪个更好。

逐点比较曲线不现实：模型 A 在低 FPR 段更高，模型 B 在高 FPR 段更高，这种情况两条曲线交叉，光看图说不清。需要把整条曲线压成单值——曲线下面积 AUC（Area Under the ROC Curve，也叫 AUROC）就是这件事的标准答案。

### 几何意义：曲线下面积

AUC = ROC 曲线 + 横轴 + 右边界 围出的面积，范围 $[0, 1]$。

```
  TPR
   1 ┤●●●●●●●●●●●●●●●●●●●●●●  ← 曲线
     │░░░░░░░░░░░░░░░░░░░░░░
     │░░░░░░░░░░░░░░░░░░░░░░  ← 这片阴影面积 = AUC
     │░░░░░░░░░░░░░░░░░░░░░░
   0 ┤●░░░░░░░░░░░░░░░░░░░░░
     └──────────────────────→ FPR
        0                   1
```

完美模型的曲线贴 (0, 1) 角，阴影 = 整个单位正方形 = 1.0；随机模型的曲线就是对角线，阴影 = 三角形 = 0.5；反模型的曲线贴 (1, 0) 角，阴影 = 0。

**02 节广告点击案例的 AUC 实算**：曲线由 7 个点 $(0,0) \to (0,0.5) \to (0,1) \to (0.25,1) \to (0.5,1) \to (0.75,1) \to (1,1)$ 折成。前两段是垂直爬升不贡献面积，后面是水平段在 TPR = 1 高度上覆盖 FPR ∈ [0, 1] → 面积 = 1 × 1 = 1.0。这是个完美分类器（因为 2 个正样本恰好排在最前面）。

### 概率意义：随机抽样比较

AUC 还有一个更直观的等价定义：

> **随机抽一个正样本 $x_+$ 和一个负样本 $x_-$，模型给正样本的打分高于负样本的概率**：
>
> $$\text{AUC} = P(\hat s(x_+) > \hat s(x_-))$$

把模型当裁判：从正堆里抓一个、负堆里抓一个，丢给模型问"哪个更像正"。AUC = 0.85 就是 100 次这种对决里有 85 次猜对。

这个解释说明 AUC 衡量的是**排序质量**——只关心正样本是不是普遍排在负样本前面，不关心具体分数是 0.6 还是 0.96。

### 解读速查表

| AUC | 含义 | 工程动作 |
|---|---|---|
| 1.0  | 完美分类器（几乎不存在）| 怀疑数据泄漏 / 标签穿越 |
| > 0.9 | 强判别力 | 同上，先核 train/test 划分 |
| 0.7 ~ 0.85 | 工程"可用"区间 | 上业务，配合具体阈值调 P/R |
| 0.5 ~ 0.7 | 弱信号 | 加特征 / 换模型 |
| 0.5 | 等同随机猜测 | 模型无效 |
| < 0.5 | 比随机还差 | 把预测取反（$1 - \hat p$）→ AUC = $1 - $ 原值 |

**反例**：训练集 AUC = 0.99 测试集 AUC = 0.62 → 过拟合或数据泄漏，不是模型强。线上指标永远以 hold-out / 时间外验证集为准。

### 阈值无关性：抗不平衡的根源

精确率 / 召回率 / F1 都依赖一个**具体阈值**。同一个模型，阈值 0.3 和 0.7 算出的 P/R/F1 完全不同——评估结果带阈值偏见。

AUC 把所有阈值的表现一次性求和（积分），结果**与阈值无关**。再加上 01 节讲过的"TPR/FPR 分母锁死正负两边"——AUC 在正负 1:99 还是 99:1 都给出可比的数字，这是它在**不平衡分类**（如风控、医疗筛查、广告 CTR）里成为默认指标的原因。

**边界**：AUC 只看排序，不看概率绝对值是否对得上真实比例。要做"概率校准"（calibration，比如把 0.7 真的对应 70% 命中率），AUC 帮不上忙——那是另一套指标（Brier score / calibration plot）的主场，不在本章范围。

**这一节的关键启示**：AUC 把整条 ROC 压成一个数，衡量的是模型把正样本排在负样本前面的能力。
