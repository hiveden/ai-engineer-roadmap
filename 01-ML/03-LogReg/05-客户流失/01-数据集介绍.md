### 笔记

> 【了解】数据集介绍

电信运营商客户行为数据，**预测用户是否流失（churn）**。

- 规模：7043 条 × 21 列
- 目标列：`Churn`（Yes / No 二分类）
- 特征类型混合：
  - **人口属性**：`gender`、`SeniorCitizen`、`Partner`、`Dependents`
  - **业务属性**：`tenure`（在网月数）、`PhoneService`、`MultipleLines`、`InternetService`、`OnlineSecurity`、`OnlineBackup`、`DeviceProtection`、`TechSupport`、`StreamingTV`、`StreamingMovies`
  - **合约/账单**：`Contract`（月付/年付/两年）、`PaperlessBilling`、`PaymentMethod`、`MonthlyCharges`、`TotalCharges`
  - **ID**：`customerID`（建模时丢弃）

业务诉求：找出高流失风险用户，集中资源做留存动作（折扣 / 客服回访 / 套餐升级），而不是给全量用户撒券。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题

电信运营商每月都在掉客户——套餐到期不续、转网、欠费销号。**新客户获取成本（CAC, customer acquisition cost）远高于老客户留存成本**，行业经验大概 5–7 倍。所以与其砸广告拉新，不如提前找出"快要走的那批人"，针对性发券、打回访电话、推套餐升级。

但留存动作本身有成本：一通电话 + 一张折扣券，假设单位代价 20 元。如果对全量 7043 个用户都发，成本 14 万；其中真要流失的只有约 26%（**1869 人**），剩下 5174 人是浪费。模型存在的意义：把这 14 万的预算，集中到最可能流失的那 1869 人头上。

> 反例：如果 PM 直接用规则——"在网月数 < 3 的全发券"——会漏掉那些用了 2 年但最近月付合约 + 涨价的老用户（流失高发段）。规则系统能覆盖的人群边界远窄于模型，且对多特征交互无能为力。

### 数据全景

| 维度 | 说明 |
|---|---|
| 样本数 | 7043 条 |
| 列数 | 21 列（含目标列 `Churn`） |
| 目标 | `Churn` ∈ {Yes, No}，二分类（binary classification） |
| 类别比 | Churn=Yes 约 26.5% / No 约 73.5%，**类别不平衡 class imbalance** |
| 缺失 | `TotalCharges` 有 11 个空白字符串伪缺失（dtype=object 隐藏） |

**4 类特征划分**（按业务语义，不是按 dtype）：

```
人口属性   gender / SeniorCitizen / Partner / Dependents      ── 谁
业务属性   tenure / PhoneService / InternetService / 各类增值服务  ── 用了什么
合约账单   Contract / PaymentMethod / MonthlyCharges / TotalCharges  ── 怎么付
ID         customerID                                          ── 直接 drop
```

`customerID` 是行级唯一字符串，**对模型零信息量但会让 one-hot 编码 one-hot encoding 后膨胀到 7043 列**——必须先扔。

### 业务诉求 → 模型目标的映射

业务想要的不是"准确率 90%"，而是"把要走的人找出来"。映射链路：

```
业务问题  ──→  模型形态  ──→  评估指标
找高流失风险      二分类 LR        Recall on churn=1 优先
                  + 概率排序        AUC 兜底（看排序能力）
                                    Accuracy 仅作参考
```

注意 Recall（召回率）/ Precision（精确率）/ AUC（ROC 曲线下面积 area under ROC curve）的取舍**不在本节展开**——这是 04-评估指标章主场。本节只标记：**流失场景看 churn=1 的 Recall**，因为漏掉一个真要走的用户的代价（流失收入损失）远高于错挽留一个不会走的用户的代价（一张折扣券）。

**这一节的关键启示**：业务诉求决定模型目标，留存的钱要花在刀刃上。
