## A. 输入数据形状

### Q1 `X = [[0], [1], [2], [3]]` 为啥要套两层 `[]`？写成 `[0, 1, 2, 3]` 行不行？
- sklearn 对 X 的 shape 硬规矩是什么？`(n_samples, n_features)` 各代表啥？
- 用数据库表的类比：行 / 列分别对应什么？
- 跑 `[0,1,2,3]` 报什么错？

### Q2 `y = [0, 0, 1, 1]` 为什么是 1D？回归的 `y = [0.1, 0.2, 0.3, 0.4]` 也是 1D，区别仅在哪里？
- y 是 1D 还是 2D 由什么决定？
- 分类 y 和回归 y 的 dtype 差异？
- 多输出回归（同时预测多个目标）时 y 才会变 2D，但这份 demo 不涉及

### Q3 `predict([[4]])` 为啥也要 2D？传 `predict([4])` 会怎样？
- predict 是单条接口还是批量接口？
- `predict([[4], [5]])` 返回的 shape 是什么？
- 这跟 REST API "POST /predict body 永远是数组" 的设计哲学怎么对应？

## B. fit 和 predict 各自做了什么

### Q4 `model.fit(X, y)` 在 KNN 这里到底做了什么？复杂度多少？
- 有没有"训练参数"？
- 跟逻辑回归/线性回归的 fit 在做的事差在哪？

## C. 分类 vs 回归（这份 demo 的核心对比）

### Q6 dm01 和 dm02 在 sklearn API 上具体差哪几个名字？
- 类名差异：`KNeighborsClassifier` vs `KNeighborsRegressor`
- y 类型差异：离散标签 vs 连续值
- 输出差异：predict 返回啥？还有哪个接口分类有、回归没有？

### Q7 "投票"（分类） vs "平均"（回归）分别怎么算？
- dm01 中 k=1，查 `[[4]]`：找到最近邻是哪个？为啥预测是 `[1]`？
- dm02 中 k=2，查 `[[3,11,10]]`：哪两个是最近邻？为啥预测是 `[0.35]`？
- 两个邻居距离不一样时（dm02 就是这样），默认 `weights='uniform'` 是怎么处理的？

## D. 超参 n_neighbors

### Q8 dm01 用 k=1，dm02 用 k=2，这两个值各自的隐患是什么？
- k=1 在分类里直觉上会怎样？（最近那个点决定一切）
- k=2 在分类里会有什么问题？（提示：偶数 + 投票）
- dm02 是回归，k=2 还有平票问题吗？为什么？

### Q9 如果 dm01 改成 k=4，预测 `[[4]]` 会变成什么？

- sklearn 默认怎么破？（按类别索引最小的赢）
