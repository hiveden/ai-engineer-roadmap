**与逻辑回归的不同点 ？？**

女孩相亲的决策树：

```
                    年龄 ≤ 30 ?
                  ┌─────┴─────┐
                 是           否
                  │            │
              长相帅 ?       不见
              ┌──┴──┐
             是    否
              │     │
          收入高?  不见
          ┌──┴──┐
         是    否
          │     │
         见  公务员?
              ┌──┴──┐
             是    否
              │     │
             见    不见
```

每个内部节点 = 特征判断；每条分支 = 判断结果；每片叶子 = "见 / 不见"。

- **推理过程**：对测试样本从根节点一路走到叶子，打上标签
- **训练过程**：从数据里学出"先看哪个特征 / 阈值切在哪 / 叶子打什么标签" —— 特征选择、顺序、分割点

> **Notes**：解决分类和回归问题；给每个男青年打上标签"见还是不见"。
> 有的选择不同的特征分界，有的选择不同的特征顺序，有的干脆用其他特征。
> 逻辑回归是所有特征一起输入，进行分类得分预测；但是这里是特征被依次使用。于是就讲究特征的使用顺序以及分割点。

### 笔记

> 【理解】决策树例子

**Decision Tree**：监督学习算法。本质是 `if-else` 条件分支结构——只是树是从数据里"学"出来的，不是手写的。

**经典例子**：相亲。"年龄 → 长相 → 收入 → 是否公务员" 一路问下来分到"见 / 不见"两类。每个内部节点是一次特征判断，每条分支是判断结果，每个叶子是最终决策。

**与 KNN/LR 的差异**：

- KNN：懒惰学习，预测时才算距离
- LogReg：线性决策边界
- Decision Tree：训练阶段切分特征空间得到一组**轴对齐的矩形区域**；可解释性强，无需标准化

## ━━━━━━━━ 讲解 ━━━━━━━━

### 人脑天然在用决策树

相亲判断用的是 if-else；选餐厅用的是 if-else；医生问诊也是 if-else：

```
"你发烧吗？"  →  是  →  "咳嗽吗？"  →  是  →  可能感冒  →  开药
              →  否  →  "头痛吗？"  →  是  →  可能偏头痛  →  ...
```

这本质上都是**决策树 decision tree**：给一个样本（人 / 餐厅 / 相亲对象），沿着一条问题链走到头，打上一个标签（见 / 不见 / 确诊 / 不确诊）。

机器学习的决策树只干一件事：**把这棵 if-else 树从数据里"学"出来**，而不是靠人手写规则。

### 树的三个组成：对照相亲案例

```
                    年龄 ≤ 30 ?          ← ① 内部节点（internal node）
                  ┌─────┴─────┐            一个特征 + 一个判断条件
                 是           否
                 │            │           ← ② 分支（branch）
             长相帅 ?       不见              判断结果的出口
             ┌──┴──┐
            是    否
            │     │
        收入高?  不见          ← 内部节点继续往下...
        ┌──┴──┐
       是    否
       │     │
      见   公务员?
            ┌──┴──┐
           是    否
           │     │
          见   不见            ← ③ 叶子节点（leaf node）
                                  最终分类结果："见"或"不见"
```

| 组成 | 相亲案例 | ML 术语 |
|---|---|---|
| 内部节点 | "年龄 ≤ 30？" "长相帅？" 等问题 | 一个特征上的判断条件 |
| 分支 | "是" / "否" 两条路 | 判断结果的输出方向 |
| 叶子节点 | 最终拍板："见" / "不见" | 预测标签（分类结果） |

### 推理 vs 训练：两件截然不同的事

拿到一棵树之后，**用树做预测**叫**推理 inference**：

```
推理 = 走树
─────────────────────────────────────────
新相亲对象：年龄 28 岁，帅，月薪 2 万
    ↓
根节点：年龄 ≤ 30？→ 是 → 进左子树
    ↓
下一节点：长相帅？→ 是 → 继续
    ↓
下一节点：收入高？→ 是（月薪 2 万算高）→ 叶子：见
    ↓
结果：见
─────────────────────────────────────────
O(树的深度) 次比较，极快
```

**训练 training** 是另一件事——树从哪来的？

```
训练 = 从数据学树
─────────────────────────────────────────
给定：N 个历史相亲记录（年龄 / 长相 / 收入 / ... + 结果）
需要学：
  ① 先问哪个问题（特征顺序）
  ② 每个问题的切分阈值（年龄切 30 还是 25？）
  ③ 叶子的标签（这堆人里多数是见还是不见）
─────────────────────────────────────────
训练完才能得到上面那棵树
```

**一句话区分**：推理 = 走已有的树；训练 = 从数据里长出树。

### 与 KNN / 逻辑回归的概念对比

三种算法都做分类，但看问题的角度完全不同：

| 维度 | KNN | 逻辑回归 | 决策树 |
|---|---|---|---|
| **核心思路** | 找最近的 K 个邻居投票 | 算一条直线（超平面）得分 | 一系列 if-else 切分 |
| **学习方式** | 懒惰学习（lazy）：不训练，预测时再算 | 急切学习（eager）：训练出一组权重 | 急切学习：训练出树结构 |
| **决策边界形状** | 任意形状（受 K 影响） | 线性超平面 | 轴对齐的矩形区域 |
| **需要特征标准化** | 是（距离敏感） | 是（梯度更稳定） | 否（只比大小，不算距离） |
| **可解释性** | 低（邻居是谁？） | 中（权重有意义） | 高（规则可直读） |

```
二维特征空间里，三种算法的决策边界长什么样：

KNN (K=3)          逻辑回归           决策树
─────────          ─────────          ─────────
 ·  ○ · ○          ·  ○ · ○           ·  ○ │ ○
 · ○   ○           · ○   ○           ·  ○  │ ○
  ─ 不规则 ─        ─ 斜直线 ─          ─── 竖线/横线 ───
○   · ○  ○         ○   · ○  ○        ○  · ─┼─  ○
 ○  · ·            ○  · ·            ○  ·  │ ·
                                      轴对齐矩形
```

决策树的"轴对齐"是双刃剑：XOR 那类斜方向的分界它天然学不好，但"年龄 ≤ 30"这类门槛规则它一刀就切干净。

### 可解释性：业务方能直接审

决策树最大的工程价值不是精度——是**可解释性 interpretability**：

```python
# sklearn 训练完一棵树，可以直接打出规则：
# feature_0 (Age) <= 30.5  →  左子树
# feature_1 (Income) <= 2.0 →  右子树
# ...

# 更直观：导出成文字规则
from sklearn.tree import export_text
print(export_text(clf, feature_names=['年龄', '长相', '收入', '是否公务员']))
```

输出类似：
```
|--- 年龄 <= 30.50
|   |--- 长相帅 <= 0.50
|   |   |--- class: 不见
|   |--- 长相帅 >  0.50
|   |   |--- 收入高 <= 0.50
|   |   |   |--- class: 不见
|   |   |--- 收入高 >  0.50
|   |   |   |--- class: 见
|--- 年龄 >  30.50
|   |--- class: 不见
```

规则对业务方透明：产品、法务、监管都能读。对比 KNN（"因为你和这 3 个人像"）或 NN（"神经网络说的"），决策树的理由人能懂、能审、能争。

这正是金融风控 / 医疗诊断里决策树（或基于它的随机森林）长期不被淘汰的原因。

### 从"直觉排序"到"算法定量"（思考题铺垫）

PPT Slide 9 问：相亲时特征优先级怎么排？

```
直觉答案：年龄 > 长相 > 收入 > 是否公务员

为什么"年龄"排第一？
─────────────────────────────────────────
年龄 ≤ 30 这一刀：
  切完左子树（≤30 岁）：大多数 → 还要继续判断
  切完右子树（>30 岁）：几乎全部 → 不见

→ 年龄一刀把很大一批样本直接送到叶子 → 子集更纯

是否公务员排最后？
  公务员占比小，且见 / 不见都有 → 切完还是混杂 → 区分度低
```

机器学习的答案：**哪个特征切完后子集最纯，就优先用哪个**。怎么度量"纯"？下一章 [`../02-id3/01-信息熵.md`](../02-id3/01-信息熵.md) 用**信息熵 entropy** 把"纯"量化出来。
