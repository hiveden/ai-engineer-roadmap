```
                    年龄 ≤ 30 ?
                  ┌─────┴─────┐
                 是           否
                  │            │
              长相帅 ?       不见
              ┌──┴──┐
             是    否
              │     │

          │     │

              │     │

```

每个内部节点 = 特征判断；每条分支 = 判断结果；每片叶子 = "见 / 不见"。

- **推理过程**：对测试样本从根节点一路走到叶子，打上标签
- **训练过程**：从数据里学出"先看哪个特征 / 阈值切在哪 / 叶子打什么标签" —— 特征选择、顺序、分割点

> 有的选择不同的特征分界，有的选择不同的特征顺序，有的干脆用其他特征。
> 逻辑回归是所有特征一起输入，进行分类得分预测；但是这里是特征被依次使用。于是就讲究特征的使用顺序以及分割点。

### 笔记

> 【理解】决策树例子

决策树算法是一种监督学习算法，英文是 Decision tree。

决策树思想的来源非常朴素，试想每个人的大脑都有类似于 if-else 这样的逻辑判断，这其中的 if 表示的是条件，if 之后的 else 就是一种选择或决策。程序设计中的条件分支结构就是 if-else 结构，最早的决策树就是利用这类结构分割数据的一种分类学习方法。

比如：你母亲要给你介绍男朋友，是这么来对话的：

> 女儿：多大年纪了？
>
> 母亲：26。
>
> 女儿：长的帅不帅？
>
> 母亲：挺帅的。
>
> 女儿：收入高不？
>
> 母亲：不算很高，中等情况。
>
> 女儿：是公务员不？
>
> 母亲：是，在税务局上班呢。
>
> 女儿：那好，我去见见。

于是你在脑袋里面就有了下面这张图（〔图〕images/01.png）。

作为女孩的你在决策过程就是典型的分类树决策。相当于通过年龄、长相、收入和是否公务员对将男人分为两个类别：见和不见。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 一句话思想：把样本扔进一连串"判断题"

设想点外卖场景：打开 app，附近 30 家店列表里每家有"评分 / 距离 / 价格 / 是否能堂食"几个属性——在 ML 里这些叫**特征 feature**（描述样本的可量化信息；可以是数值如评分 4.5、距离 2.5km，也可以是布尔如是否堂食。首次主场 → [`../../01-KNN/01-intro/02-接近程度.md`](../../01-KNN/01-intro/02-接近程度.md)）。最终结果只有"点"或"不点"两类。30 家店混在一起，靠不靠谱不好直接看出来，怎么办？

人脑的本能做法是问一串判断题——"评分 ≥ 4.5 吗？" 评分低于 4.5 的那批几乎都不点，这堆店不用再看；评分够高的还混杂，再挑下一题（距离 / 价格 / ...）继续切。每问一题，剩下的子集就更"齐"一点。一直切到子集足够齐为止。

提炼出来就是决策树的核心思想：

> **面对一堆混在一起的样本，反复挑能切得最齐的判断题切一刀，切到子集里基本是同一类，就停手贴标签。**

```
                问题 1：评分 ≥ 4.5？
              ───────────┬───────────
                ↓                  ↓
        子集 A（点 7 / 不点 3）   子集 B（点 1 / 不点 9）→ 直接贴"不点"
                ↓
          问题 2：距离 ≤ 3km？
          ────────┬────────
              ↓              ↓
       子集 C（点 6/不点 1）   子集 D（点 1/不点 2）
            ...                  ...
```

**反例：选错特征切，等于白切**

设想数据里还混入了一个完全无关的属性——"商家头像背景色"（红 / 黄 / 蓝 / 绿，平台美工随机挑的颜色）。如果硬拿它当切分标准——

```
                问题 1：头像背景色 = 红？
              ───────────┬───────────
                ↓                  ↓
       是（点 5 / 不点 5）   否（点 5 / 不点 5）

       两边和切之前一样混 → 这一刀白费
```

切之前 点:不点 = 10:10，切完左右还是 5:5 / 5:5——纯度没变，等于没切。"头像背景色"由平台美工随手挑，跟好不好吃、卫不卫生、贵不贵都**没有任何因果链**——切它不可能让标签更纯，所以这是一刀典型的"噪声切"。

所以**关键不是"会问"，是每次挑那道把样本切得最齐的问**。"切得齐" = 切完后子集最不混杂（最"纯"）。怎么量化"纯"留给下一章 [`../02-id3/01-信息熵.md`](../02-id3/01-信息熵.md)。

人脑面对"点不点"这类问题，本能就是这套——选餐厅、医生问诊、判断邮件是不是垃圾，都是问题链。机器学习的决策树只干一件事：**把这棵 if-else 树从数据里"学"出来**，而不是靠人手写规则。

### 树的三个组成：把外卖那棵问题树拆开看

上一段画的"评分 → 距离 → 价格"那棵问题树，实际由三种零件拼成——把它们各自命名一下，后续讲解就有共同词汇：

```
                    评分 ≥ 4.5 ?         ← ① 内部节点（internal node）
                  ┌─────┴─────┐            一个特征 + 一个判断条件
                 是           否
                 │            │           ← ② 分支（branch）
              距离≤3km?     不点              判断结果的出口
              ┌──┴──┐
             是    否
             │     │
          价格≤60? 不点          ← 内部节点继续往下...
          ┌──┴──┐
         是    否
         │     │
        点    不点                ← ③ 叶子节点（leaf node）
                                    最终分类结果："点"或"不点"
```

| 组成 | 外卖案例 | ML 术语 | 含义 |
|---|---|---|---|
| ① 内部节点 | "评分 ≥ 4.5？" "距离 ≤ 3km？" 等问题 | internal node | 一个特征上的判断条件 |
| ② 分支 | "是" / "否" 两条路 | branch | 判断结果的输出方向 |
| ③ 叶子节点 | 最终拍板："点" / "不点" | leaf node | 树到头了，给当前样本贴的预测标签（分类结果） |

最顶端那个内部节点（这里是"评分 ≥ 4.5？"）有个特殊名字——**根节点 root node**：树从它开始长，所有推理都从它出发。

> 注：如果数据里混入完全无信号的特征（如"商家头像背景色"这种纯美工属性），训练时算法发现它每次切完都白切，会自动绕开它——训练好的树上**根本不会出现这种节点**。下一章信息熵会精确解释"算法怎么自动识别噪声特征"。

### 几何直觉：在特征空间里"切豆腐"

换个视角——所有样本扔进 N 维特征空间（每个特征一根坐标轴）。

决策树每问一个问题，比如"**评分 ≥ 4.5？**"，几何上等价于：在**评分这根坐标轴**上找到 4.5 这个点，画一条**垂直于评分轴**的切线把空间切两半。问"**距离 ≤ 3km？**"就在距离轴上画一条垂直于距离轴的线。

所以决策树的每一刀必然**平行于某根坐标轴**——这种切法叫**轴对齐 axis-aligned**：刀只能沿坐标轴方向砍，斜的切不出。

```
（2D 空间：评分 × 距离；● = 下单，○ = 不下单）

 ① 切 0 刀（混杂整块）   ② 切 1 刀（评分 ≥ 4.5）  ③ 切 2 刀（再加 距离 ≤ 3km）

距离远                   距离远                    距离远
 ↑                        ↑                         ↑
 │ ○  ●  ○  ●  ○          │ ○  ○ │ ●  ○  ●          │ ○  ○ │ ●  ○  ●
 │ ●  ○  ●  ●  ○          │ ○  ○ │ ●  ●  ○          │ ○  ○ │──●──●──○ ← 距离=3km
 │ ●  ●  ●  ○  ●          │ ○  ○ │ ●  ●  ●          │ ○  ○ │ ●  ●  ●
 └──────────────→ 评分     └──────┴───────→ 评分      └──────┴───────→ 评分
 距离近                          4.5                        4.5
 ●:○ = 7:8 混杂           左 0:6 / 右 7:2            每块内部基本同色
                          一刀切纯了一大半             → 给每块贴一个标签
```

最终空间被切成一堆轴对齐矩形，**每块矩形对应树的一片叶子节点**，贴一个分类标签。下单（●）天然集中在"右下角"——评分高 + 距离近的店；不下单（○）散落在评分低或距离远的区域。

**几何视角顺手解释两件事**：

1. **为什么决策树边界"横平竖直"** —— 决策边界**全是水平线 + 垂直线，不会有斜线或曲线**：

   ```
   决策树能切             决策树切不出
   ─────────             ─────────
      │                        ╲
   ───┼───                      ╲       ⌒
      │                          ╲    ╱   ╲
   水平 / 垂直线           斜线          曲线
   ```

   原因：每问一题只看**一个**特征（"评分 ≥ 4.5？"只看评分一根轴），所以刀只能沿那根坐标轴砍。要画斜线得同时混合两个特征（比如"评分 + 距离 ≤ 5"），决策树天然不干这事。

2. **为什么决策树天然不需要标准化** —— 它只比"评分是否 ≥ 4.5"这种**单特征大小关系**，跟特征量纲无关；不像 KNN 要算距离，尺度一变全乱（KNN 缩放主场 → [`../../01-KNN/04b-scaling/`](../../01-KNN/04b-scaling/)）

**这一节的关键启示**：决策树的全部艺术，就在挑哪一刀切下去。

### 推理 vs 训练：两件截然不同的事

到这里你应该有个疑问：**上面那棵树到底从哪来的？谁规定根节点先问"评分 ≥ 4.5"而不是先问"距离"？阈值为什么是 4.5 不是 4.0？**

这个问题就引出决策树两个截然分开的阶段——**树从数据里"长出来"**叫训练，**用长好的树给新样本打标签**叫推理。

先看相对简单的推理：拿到一棵已经训练好的树，**用它做预测**叫**推理 inference**：

```
推理 = 走树
─────────────────────────────────────────
新店：评分 4.6，距离 2.5km，人均 50 元
    ↓
根节点：评分 ≥ 4.5？→ 是 → 进左子树
    ↓
下一节点：距离 ≤ 3km？→ 是 → 继续
    ↓
下一节点：价格 ≤ 60？→ 是 → 叶子：点
    ↓
结果：点
─────────────────────────────────────────
O(树的深度) 次比较，极快
```

**训练 training** 是另一件事——树从哪来的？

```
训练 = 从数据学树
─────────────────────────────────────────
给定：N 条历史下单记录（评分 / 距离 / 价格 / 是否堂食 + 是否下单）
需要学：
  ① 先问哪个问题（特征顺序）
  ② 每个问题的切分阈值（评分切 4.5 还是 4.0？）
  ③ 叶子的标签（这堆店里多数下单还是不下单）
─────────────────────────────────────────
训练完才能得到上面那棵树
```

**一句话区分**：推理 = 走已有的树；训练 = 从数据里长出树。

### 与 KNN / 逻辑回归的概念对比

三种算法都做分类，但看问题的角度完全不同：

| 维度 | KNN | 逻辑回归 | 决策树 |
|---|---|---|---|
| **核心思路** | 找最近的 K 个邻居投票 | 算一条直线（超平面）得分 | 一系列 if-else 切分 |
| **学习方式** | 懒惰学习（lazy）：不训练，预测时再算 | 急切学习（eager）：训练出一组权重 | 急切学习：训练出树结构 |
| **决策边界形状** | 任意形状（受 K 影响） | 线性超平面 | 轴对齐的矩形区域 |
| **需要特征标准化** | 是（距离敏感） | 是（梯度更稳定） | 否（只比大小，不算距离） |
| **可解释性** | 低（邻居是谁？） | 中（权重有意义） | 高（规则可直读） |

```
二维特征空间里，三种算法的决策边界长什么样：

KNN (K=3)          逻辑回归           决策树
─────────          ─────────          ─────────
 ·  ○ · ○          ·  ○ · ○           ·  ○ │ ○
 · ○   ○           · ○   ○           ·  ○  │ ○
  ─ 不规则 ─        ─ 斜直线 ─          ─── 竖线/横线 ───
○   · ○  ○         ○   · ○  ○        ○  · ─┼─  ○
 ○  · ·            ○  · ·            ○  ·  │ ·
                                      轴对齐矩形
```

决策树的"轴对齐"是双刃剑：XOR 那类斜方向的分界它天然学不好，但"评分 ≥ 4.5"这类门槛规则它一刀就切干净。

### 可解释性：业务方能直接审

决策树最大的工程价值不是精度——是**可解释性 interpretability**：

```python
# sklearn 训练完一棵树，可以直接打出规则：
# feature_0 (Rating) <= 4.5 → 左子树
# feature_1 (Distance) <= 3.0 → 右子树
# ...

# 更直观：导出成文字规则
from sklearn.tree import export_text
print(export_text(clf, feature_names=['评分', '距离', '价格', '是否堂食']))
```

输出类似：
```
|--- 评分 <= 4.50
|   |--- class: 不点
|--- 评分 >  4.50
|   |--- 距离 <= 3.00
|   |   |--- 价格 <= 60.00
|   |   |   |--- class: 点
|   |   |--- 价格 >  60.00
|   |   |   |--- class: 不点
|   |--- 距离 >  3.00
|   |   |--- class: 不点
```

规则对业务方透明：产品、法务、监管都能读。对比 KNN（"因为你和这 3 个人像"）或 NN（"神经网络说的"），决策树的理由人能懂、能审、能争。

### 从"直觉排序"到"算法定量"

回到外卖场景——你点外卖时，特征优先级直觉怎么排？

```
直觉答案：评分 > 距离 > 价格 > 是否堂食
                                  ↑
                          4 个真信号里相对弱者（不是噪声）

为什么"评分"排第一？
─────────────────────────────────────────
评分 ≥ 4.5 这一刀：
  切完左子树（< 4.5）：大多数 → 不点
  切完右子树（≥ 4.5）：还要继续判断

→ 评分一刀把很大一批样本直接送到叶子 → 子集更纯

为什么"是否堂食"在 4 个里排最后？
  「是否堂食」其实是**食安信号**——幽灵外卖问题让它在中国语境
  下成了用户决策的一级关注点。但它的食安信号常被「评分」提前
  吸收：4.8 分的店多半也都有堂食。所以在「评分」已经被切过的
  子集里，它的边际信息量相对小——排第 4，但仍是**真信号**。

→ 这和"白切反例"里的"头像背景色"完全不同：
   头像背景色 = 信息增益 ≈ 0（噪声）
   是否堂食   = 信息增益相对小（弱信号）但 > 0
```

机器学习的答案：**哪个特征切完后子集最纯，就优先用哪个**。怎么度量"纯"？下一章 [`../02-id3/01-信息熵.md`](../02-id3/01-信息熵.md) 用**信息熵 entropy** 把"纯"量化出来。
