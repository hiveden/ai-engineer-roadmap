### PPT

> Slide 5 · 决策树简介

决策树是一种树形结构：**3 个模块**

- 树中每个内部节点表示一个用于判断的特征
- 每个分支代表一个判断结果的输出（≤30 or >30）
- 每个叶子节点代表一种分类结果（对每个样本预测标签）

```
        ┌──────────────┐
        │   年龄 ≤ 30  │ ← ① 内部节点（特征判断）
        └──────┬───────┘
       ≤30 ┌───┴───┐ >30
           │       │      ← ② 分支（判断结果输出）
           ▼       ▼
        ┌─────┐ ┌─────┐
        │ 见  │ │不见 │   ← ③ 叶子节点（分类结果）
        └─────┘ └─────┘
```

> Slide 6 · 决策树（分类）的目标

是希望通过**一系列最合适的特征**、**合适的特征顺序**、**最合适的分割点**将样本进行分类，使得分割后的子类中，样本尽可能单一，减少混杂。从而便于决策。

**决策树执行结果**：标签 / 标签 / 标签 / 标签

```
切分前（混杂）              切分后（尽可能单一）
─────────────              ─────────────────────
                            子类 A：████████ 见
█▓█▓█▓█▓█▓█▓                子类 B：████████ 不见
见 / 不见 混在一起           子类 C：████████ 见
                            子类 D：████████ 不见
H 大（难决策）               H 小（便于决策）
```

剩余样本尽可能单一，减少混杂（避免误认和漏认）。因此特征以及分割点需要权衡。

> Slide 8 · 核心需要决定

决策树的目标：是希望通过一系列最合适的特征、合适的特征顺序、最合适的分割点将样本进行分类，使得分割后的子类中，样本尽可能单一，减少混杂。从而便于决策。

```
   ① 选特征      →    ② 定分割点    →    ③ 剪枝
─────────────       ─────────────       ─────────────
哪个特征切下去      切在哪个值上         砍掉过深 / 过细
让子集最纯？        把样本切两半？       的分支防过拟合
（年龄?长相?）      （年龄 ≤ 30?）       （预剪枝/后剪枝）
   ▼                   ▼                   ▼
信息增益 / 增益率 /   遍历候选阈值        泛化能力 ↑
基尼指数            选最优               训练误差 ↑ 一点
```

**核心需要决定**：

1. 选择什么特征进行分割
2. 确定最佳分割点
3. 通过剪枝防止过拟合

### 笔记

> 【知道】决策树简介

**结构定义**：

- **内部节点**：一个特征上的判断条件
- **分支**：判断结果的输出
- **叶子节点**：一种分类（或回归）结果

**建立过程三步走**：

1. **特征选择**：选取分类能力强的特征（按某种准则，如信息增益 / 信息增益率 / 基尼指数）
2. **决策树生成**：递归用选出的特征切分数据
3. **剪枝**：决策树极易过拟合，用预剪枝 / 后剪枝缓解

**算法族对比预览**（详见后续章节）：

| 算法 | 准则 | 输出 |
|---|---|---|
| ID3 | 信息增益 | 多叉树，仅离散特征 |
| C4.5 | 信息增益率 | 多叉树，支持连续 + 缺失值 |
| CART | 基尼指数 / 平方误差 | 二叉树，分类 + 回归 |

**工程视角**：

- 训练产物是一棵树（节点 + 阈值），可序列化为 JSON / PMML / ONNX
- 推理 = 从根走到叶，复杂度 $O(\log n)$（平衡时）
- 天然处理混合类型特征，无需 one-hot / 标准化

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题：怎么从数据"学"出一棵树

上一节 [`01-例子.md`](./01-例子.md) 回答了"决策树长什么样"。本节回答：**这棵树怎么从数据里学出来？**

问题可以拆成三个子问题：

```
数据集（N 条样本 + 若干特征 + 标签）
         │
         ├─ ① 先问哪个问题？    ← 特征选择
         │      哪个特征切完子集最纯
         │
         ├─ ② 阈值切在哪？      ← 分割点
         │      年龄切 30 还是 25？
         │
         └─ ③ 树何时停止生长？  ← 剪枝
                继续切到只剩 1 个样本？
                还是到某深度就停？
```

三个问题分别对应建树的三步流程。

### 三步建立流程

#### Step 1 · 特征选择（Feature Selection）

每个节点都面临同一个问题："手头这堆样本，用哪个特征切，效果最好？"

"最好" = 切完后子集**尽可能纯**：同一子集里的样本尽量都是同一类，这样叶子就好打标签。

```
切之前（混杂）          用"年龄"切              用"性别"切
─────────────         ─────────────         ─────────────
见  ████ 40%          左（≤30）：             男：
不见 ████████ 60%     见 ████████ 80%        见  ████ 40%
                      不见 ██ 20%            不见 ████████ 60%

                      右（>30）：             女：
                      见 ██ 10%             见 ████████ 80%
                      不见 ████████ 90%      不见 ██ 20%

                      两个子集都偏向一类       两个子集一样杂
                      → 年龄切得更纯           → 性别没什么用
```

**纯度的量化方式**因算法不同而异——这正是三种算法族的分水岭：

| 算法 | 纯度指标 | 计算目标 | 主场 |
|---|---|---|---|
| ID3 | 信息熵 entropy | 信息增益 | [`../02-id3/`](../02-id3/) |
| C4.5 | 信息熵 entropy | 信息增益率（修正 ID3 偏向多值问题） | [`../03-c45/`](../03-c45/) |
| CART | 基尼指数 Gini | 基尼增益 | [`../04-cart/`](../04-cart/) |

本节只需知道：**特征选择 = 找让子集最纯的那个特征**。具体公式各归主场。

#### Step 2 · 树生成（Tree Building）

选好特征之后，递归重复 Step 1：对每个子集再选最优特征继续切，直到触发停止条件。

```
建树过程（以相亲数据为例）

                 全量数据（混杂）
                       │
              Step 1：选"年龄"最优
                  ┌────┴────┐
               ≤30 岁        >30 岁
              （还混杂）      （几乎全不见 → 叶子）
                  │
         Step 1 重复：选"长相"最优
              ┌───┴───┐
             帅        不帅
           （继续）    （全不见 → 叶子）
              │
         Step 1 重复：选"收入"最优
           ┌──┴──┐
          高      低
        （见）  （继续...）
```

停止条件（满足任一即停）：

```
① 子集已纯净   → 直接打叶子标签（所有样本同类）
② 无特征可用   → 打多数类标签（ID3 的特征用完即扔特性）
③ 样本数 < 阈值 → 工程刹车，防过拟合（预剪枝的一种）
```

#### Step 3 · 剪枝（Pruning）

决策树天然贪心：如果不加约束，它会一直切到每个叶子只剩 1 个样本——训练集 100% 准确，测试集一塌糊涂（过拟合 overfitting）。

```
不剪枝（过拟合）              剪枝后（泛化）
─────────────────            ─────────────────
  年龄 ≤ 30？                  年龄 ≤ 30？
  ├─ 长相帅？                  ├─ 长相帅？
  │  ├─ 收入高？               │  ├─ 见
  │  │  ├─ 公务员？            │  └─ 不见
  │  │  │  ├─ ...（10层）     └─ 不见
  │  │  │  └─ ...
深度不限，死记硬背              砍掉细枝，学到规律
```

剪枝两类：

- **预剪枝 pre-pruning**：建树时提前停止（max_depth / min_samples_leaf 等参数）
- **后剪枝 post-pruning**：先长完，再从下往上砍没意义的节点

> 剪枝细节主场在第 7 章（决策树剪枝）。这里只需知道"剪枝是必须的，防过拟合"。

### 算法族预览表：ID3 / C4.5 / CART

三种算法本质都是"贪心递归 + 纯度量化"，差别在于**用什么衡量纯度 + 树的形状**：

```
三种算法的演化关系：

ID3 (1986)
  │  缺陷：偏向多值特征 / 只处理离散 / 无剪枝
  ↓
C4.5 (1993, ID3 的作者 Quinlan 改进)
  │  修复：增益率 + 连续特征 + 缺失值处理
  ↓
CART (1984, Breiman et al.)
  │  路线不同：基尼指数 + 二叉树 + 回归树 + 内置剪枝
  ↓
sklearn 默认实现：CART（二叉，支持分类+回归）
```

| 特性 | ID3 | C4.5 | CART |
|---|---|---|---|
| 树形状 | 多叉 | 多叉 | **二叉** |
| 纯度指标 | 信息增益 | 信息增益率 | 基尼指数 |
| 连续特征 | 不支持 | 支持 | 支持 |
| 缺失值 | 不支持 | 支持 | 支持 |
| 回归任务 | 不支持 | 不支持 | 支持 |
| sklearn 实现 | 无 | 无 | 有（默认） |

后续三章各归主场展开。

### "切完更纯" vs "距离近" 两种世界观

学了 KNN 之后再看决策树，两种算法代表两种截然不同的思维：

```
KNN 世界观                    决策树世界观
─────────────                ─────────────
样本 A 是什么类？             样本 A 是什么类？
→ 找离 A 最近的 K 个邻居      → 问 A 一系列问题
→ 邻居多数是什么类就是什么类   → 一路走到叶子打标签

核心概念：距离（欧氏/曼哈顿）  核心概念：纯度（熵/基尼）
参数：K（邻居数）              参数：树的深度 / 特征选取准则
边界：任意形状（K 影响）       边界：轴对齐矩形
```

**关键启示**：KNN 说"你跟谁近就跟谁一组"；决策树说"你符合哪组的规则就归那组"。前者拼的是距离度量，后者拼的是规则纯度。两种世界观都能解决分类问题，但在不同场景下哪个更好，取决于数据结构。

### 工程视角：训练产物 + 推理效率

```
训练产物：一棵树（序列化友好）

sklearn 树对象
  ├─ tree_.children_left[i]    ← 节点 i 的左孩子索引
  ├─ tree_.children_right[i]   ← 节点 i 的右孩子索引
  ├─ tree_.feature[i]          ← 节点 i 用哪个特征
  ├─ tree_.threshold[i]        ← 节点 i 的阈值
  └─ tree_.value[i]            ← 叶子 i 的类别分布

序列化方式：
  joblib.dump(clf, 'tree.pkl')   ← Python 服务
  export_text / export_graphviz  ← 人工审核
  PMML / ONNX                    ← 跨语言部署（Go/Java/Node）
```

**推理复杂度**：

```
平衡树深度 d，样本数 N：
  训练：O(N × features × d)    每层遍历所有样本 + 所有特征
  推理：O(d) = O(log N)        每次预测只走一条根到叶的路径
         ↑
    极快，轻松 10 万 QPS（不用向量运算）
```

**天然免预处理的工程优势**：

| 预处理步骤 | KNN 需要 | 逻辑回归需要 | 决策树需要 |
|---|---|---|---|
| 特征标准化 | 是 | 推荐 | 否 |
| one-hot 编码类别特征 | 是 | 是 | 否（可直接比较） |
| 缺失值填充 | 是 | 是 | 视算法（CART 有策略） |

对于混合类型特征（数值 + 类别 + 序数混杂），决策树省去大量特征工程步骤。
