#### Slide 4 · 决策树简介 – 生活中的决策树

| 故事元素 | 术语 |
|---|---|
| 相亲对象 | 样本 |
| 依靠的信息 | 特征 |

对每个样本预测一个二分类的类别："见" / "不见"

**与逻辑回归的不同点 ？？**

女孩相亲的决策树：

```
                    年龄 ≤ 30 ?
                  ┌─────┴─────┐
                 是           否
                  │            │
              长相帅 ?       不见
              ┌──┴──┐
             是    否
              │     │
          收入高?  不见
          ┌──┴──┐
         是    否
          │     │
         见  公务员?
              ┌──┴──┐
             是    否
              │     │
             见    不见
```

每个内部节点 = 特征判断；每条分支 = 判断结果；每片叶子 = "见 / 不见"。

- **推理过程**：对测试样本从根节点一路走到叶子，打上标签
- **训练过程**：从数据里学出"先看哪个特征 / 阈值切在哪 / 叶子打什么标签" —— 特征选择、顺序、分割点

> **Notes**：解决分类和回归问题；给每个男青年打上标签"见还是不见"。
> 有的选择不同的特征分界，有的选择不同的特征顺序，有的干脆用其他特征。
> 逻辑回归是所有特征一起输入，进行分类得分预测；但是这里是特征被依次使用。于是就讲究特征的使用顺序以及分割点。

#### Slide 5 · 决策树简介

决策树是一种树形结构：**3 个模块**

- 树中每个内部节点表示一个用于判断的特征
- 每个分支代表一个判断结果的输出（≤30 or >30）
- 每个叶子节点代表一种分类结果（对每个样本预测标签）

```
        ┌──────────────┐
        │   年龄 ≤ 30  │ ← ① 内部节点（特征判断）
        └──────┬───────┘
       ≤30 ┌───┴───┐ >30
           │       │      ← ② 分支（判断结果输出）
           ▼       ▼
        ┌─────┐ ┌─────┐
        │ 见  │ │不见 │   ← ③ 叶子节点（分类结果）
        └─────┘ └─────┘
```

#### Slide 6 · 决策树（分类）的目标

是希望通过**一系列最合适的特征**、**合适的特征顺序**、**最合适的分割点**将样本进行分类，使得分割后的子类中，样本尽可能单一，减少混杂。从而便于决策。

**决策树执行结果**：标签 / 标签 / 标签 / 标签

```
切分前（混杂）              切分后（尽可能单一）
─────────────              ─────────────────────
                            子类 A：████████ 见
█▓█▓█▓█▓█▓█▓                子类 B：████████ 不见
见 / 不见 混在一起           子类 C：████████ 见
                            子类 D：████████ 不见
H 大（难决策）               H 小（便于决策）
```

剩余样本尽可能单一，减少混杂（避免误认和漏认）。因此特征以及分割点需要权衡。

### 笔记

> 01 · 决策树简介

**学习目标**：

1. 理解决策树算法的基本思想
2. 知道构建决策树的步骤

> 【理解】决策树例子

**Decision Tree**：监督学习算法。本质是 `if-else` 条件分支结构——只是树是从数据里"学"出来的，不是手写的。

**经典例子**：相亲。"年龄 → 长相 → 收入 → 是否公务员" 一路问下来分到"见 / 不见"两类。每个内部节点是一次特征判断，每条分支是判断结果，每个叶子是最终决策。

**与 KNN/LR 的差异**：

- KNN：懒惰学习，预测时才算距离
- LogReg：线性决策边界
- Decision Tree：训练阶段切分特征空间得到一组**轴对齐的矩形区域**；可解释性强，无需标准化

> 【知道】决策树简介

**结构定义**：

- **内部节点**：一个特征上的判断条件
- **分支**：判断结果的输出
- **叶子节点**：一种分类（或回归）结果

**建立过程三步走**：

1. **特征选择**：选取分类能力强的特征（按某种准则，如信息增益 / 信息增益率 / 基尼指数）
2. **决策树生成**：递归用选出的特征切分数据
3. **剪枝**：决策树极易过拟合，用预剪枝 / 后剪枝缓解

**算法族对比预览**（详见后续章节）：

| 算法 | 准则 | 输出 |
|---|---|---|
| ID3 | 信息增益 | 多叉树，仅离散特征 |
| C4.5 | 信息增益率 | 多叉树，支持连续 + 缺失值 |
| CART | 基尼指数 / 平方误差 | 二叉树，分类 + 回归 |

**工程视角**：

- 训练产物是一棵树（节点 + 阈值），可序列化为 JSON / PMML / ONNX
- 推理 = 从根走到叶，复杂度 $O(\log n)$（平衡时）
- 天然处理混合类型特征，无需 one-hot / 标准化
