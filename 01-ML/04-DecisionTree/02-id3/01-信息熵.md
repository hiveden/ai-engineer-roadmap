> Slide 14 · 概念 2：信息熵

**信息熵**：代表随机变量的不确定度 / 预测难度的度量

- 熵越大，事件或变量越难以预测，不确定性越高 —— 样本较杂
- 熵越小，事件或变量越容易预测，不确定性越低 —— 样本较纯

数学上：信息熵 = 事件所有信息量的"数学期望"（加权平均）

$$H = -\sum_{i=1}^{k} p_i \log_2 p_i$$

### 笔记

> 【理解】信息熵

**ID3 = Iterative Dichotomiser 3**：基于**信息增益**做特征选择。

**信息熵**（entropy）：随机变量不确定度的度量。

**单事件信息量**（Slide 12-14）：$h(x) = -\log_2 p(x)$。概率越小、信息量越大（"太阳西升"）；信息熵 = 所有事件信息量的数学期望（加权平均）。

$$H = -\sum_{i=1}^{k} p_i \log_2(p_i)$$

**直觉**：

- 熵越大 → 越混乱、越不确定
- 熵越小 → 越纯、越确定
- 极端：单类纯净时 $H=0$；类别均匀时 $H$ 最大

**例子**：

- 占比 $\{1/3, 1/3, 1/3\}$：$H = \log_2 3 \approx 1.585$（最大）
- 占比 $\{1/10, 2/10, 7/10\}$：$H \approx 1.157$
- 占比 $\{1, 0, 0\}$：$H = 0$

## ━━━━━━━━ 讲解 ━━━━━━━━

### 决策树为什么需要"纯度"度量

决策树的根本任务（[`../01-intro/`](../01-intro/)）：每个内部节点要挑一个特征切样本，目标是**让切完的子集尽量纯**。

```
切前（最杂）              切完（最纯）
─────────              ─────────
A 见  ████ 33%          子集左：A 见 ████████████ 100%
B 不见 ████ 33%   ──切──▶ ─────────
C 不见 ████ 33%          子集右：B/C 不见 ████████ 100%
                       
最难判（"猜"哪类）        一眼断（直接打标签）
```

但"纯"这个词在算法里没法用——你得给它一个**数字**。这个数字就是**信息熵 information entropy**：

> 一个集合越混乱越难猜里面的样本是哪类 → 熵高
> 越纯越好猜 → 熵低

熵 = 0 表示"全是一类，闭眼猜也对"；熵最大表示"各类均分，瞎猜准确率最低"。**整棵决策树的训练过程就是反复让熵下降。**

### 单事件信息量：罕见 = 信息量大

香农 1948 年从两件常识开始反推公式：

1. **罕见的事件信息量大**："太阳西升" 比 "旭日东升" 信息量大
2. **独立事件信息量可加**：知道"今天下雨"+"今天周三" 的总信息 = 知道下雨 + 知道周三

唯一同时满足这两条的函数是**对数函数 logarithm**（记作 $\log$，乘法转加法的工具）：罕见 → 概率小 → $-\log p$ 大；独立事件概率相乘 → 取对数变相加。所以：

$$h(x) = -\log_2 p(x)$$

| 事件 | 概率 $p$ | $h = -\log_2 p$ | 直觉 |
|---|---|---|---|
| 旭日东升 | 0.99 | $\approx 0.014$ bit | 几乎没信息 |
| 抛硬币正面 | 0.5 | 1 bit | 恰好 1 比特 |
| 太阳西升 | 0.01 | $\approx 6.64$ bit | 爆炸新闻 |

**为什么底数取 2**：单位是 **bit（比特，= 回答一个 yes/no 问题所需的信息量）**。也可以取 $e$（单位 nat）或 10（单位 hart），决策树里用哪个都行——**只比大小，不比绝对值**。

### 信息熵 = 信息量的期望

单事件有信息量，**整个数据集**的信息量怎么算？取**加权平均**（数学期望）：

$$H(D) = \mathbb{E}[h(x)] = \sum_{i=1}^{k} p_i \cdot h(x_i) = -\sum_{i=1}^{k} p_i \log_2 p_i$$

**符号扫盲**：

| 符号 | 读法 | 含义 |
|---|---|---|
| $H$ | Eta（音"伊塔"）| 熵的标准记号，跟物理热力学熵借的 |
| $\sum_{i=1}^{k}$ | sigma 求和 i 从 1 到 k | 所有 $k$ 个类别加起来 |
| $p_i$ | p sub i | 第 $i$ 类样本占总样本的比例 |
| $\log_2$ | log 以 2 为底 | $\log_2 8 = 3$（即 $2^3 = 8$） |

**为什么前面是负号**：$p_i \in (0, 1]$ → $\log p_i \le 0$ → 加负号让熵 $\ge 0$，符合"熵不能为负"的直觉。

### 三个对照实算（巩固公式）

直接套公式 $H = -\sum p_i \log_2 p_i$。

#### 例 1：均匀（最难猜）

3 类各占 1/3：

$$H = -3 \times \frac{1}{3} \log_2 \frac{1}{3} = \log_2 3 \approx 1.585 \text{ bit}$$

#### 例 2：偏斜（较易猜）

占比 $\{0.1, 0.2, 0.7\}$：

$$H = -(0.1 \log_2 0.1 + 0.2 \log_2 0.2 + 0.7 \log_2 0.7) \approx 0.332 + 0.464 + 0.360 = 1.157 \text{ bit}$$

#### 例 3：纯净（免猜）

占比 $\{1, 0, 0\}$：

$$H = -1 \log_2 1 - 0 \log_2 0 - 0 \log_2 0 = 0 \text{ bit}$$

> ⚠️ $0 \log_2 0$ 严格来说未定义（$\log 0 = -\infty$），约定取极限 $\lim_{p \to 0^+} p \log p = 0$。所以"空类别"项直接当 0 算，不报错。

#### 横向对比

| 占比 | 形状 | $H$ | 结论 |
|---|---|---|---|
| 1/3, 1/3, 1/3 | 完全均匀 | **1.585**（k 类的最大值 $\log_2 k$） | 最杂、最难猜 |
| 0.1, 0.2, 0.7 | 偏斜 | 1.157 | 中等 |
| 1, 0, 0 | 全一类 | **0** | 纯净、免猜 |

**关键性质**：$k$ 类时熵的最大值 = $\log_2 k$（均匀分布时取到）。二分类最大值 = 1，三分类最大值 ≈ 1.585。

### 代码落地：手算信息熵

```python
import numpy as np

def entropy(probs):
    """计算信息熵（bit）。probs 是各类占比的 list/array。"""
    probs = np.asarray(probs, dtype=float)
    # 过滤 0 概率项（避免 log(0) 警告，按约定取 0）
    probs = probs[probs > 0]
    return -np.sum(probs * np.log2(probs))

# Slide 21 三例验证
print(entropy([1/3, 1/3, 1/3]))   # → 1.5849...
print(entropy([0.1, 0.2, 0.7]))   # → 1.1567...
print(entropy([1.0, 0, 0]))       # → 0.0
```

### 一句话钉板

**这一节的关键启示**：**信息熵 = 不确定度的标尺，越混杂越大**。

展开：信息熵是一个数据集"猜中里面样本属于哪类"需要的平均比特数。0 = 免猜（纯），$\log_2 k$ = 最难猜（均匀）。

下一节：算了熵之后，怎么挑特征？答案是 **信息增益**——切之前的熵 减 切之后各子集的加权熵 → 熵下降越多的特征越值得选。
