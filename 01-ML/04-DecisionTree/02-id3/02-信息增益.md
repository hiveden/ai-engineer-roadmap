### PPT

> Slide 22 · 信息增益

**概念**：特征 $a$ 对训练数据集 $D$ 的信息增益 $g(D, a)$ 定义为：集合 $D$ 的**原始熵** $H(D)$ 与**以特征 $a$ 进行分割后的条件熵** $H(D|a)$ 之差。

**数学公式**：$g(D, A) = H(D) - H(D|A)$

信息增益 = 原总熵 - 条件熵（指以某特征进行分割后的总熵）

**条件熵**降低分类错误。

**权重**：每个叶节点的数目占比；**每个子类的信息熵**。

```
不确定性条对比：

切前  |████████████████████|   H(D)
                ↓ 用特征 a 切
切后  |██████████          |   H(D|a) = Σ |Dᵢ|/|D| × H(Dᵢ)
       └─ 已剩 ─┘└─ 增益 ─┘
                  g(D, a)
```

**Notes**：条件熵计算 公式说明 每个类别的分类后样本占比 / 每个分类后样本的信息熵

> Slide 23 · 信息增益计算

**特征 a：书本颜色**（蓝 / 红）

- 3 本科技书 (A) + 3 本言情小说 (B)
- α 部分（蓝）对应目标值：AAAB
- β 部分（红）对应目标值：BB

```
条件为 α 熵： -3/4 * math.log(3/4, 2) - 1/4 * math.log(1/4, 2) = 0.81
条件为 β 熵： -(2/2) * math.log(2/2, 2) = 0

条件熵：α 部分占 4/6，β 部分占 2/6
       (4/6) * 0.81 + (2/6) * 0 = 0.54

(原初) 熵： -3/6 * math.log(3/6, 2) - 3/6 * math.log(3/6, 2) = 1

信息增益： (原初) 熵 - 条件熵 = 1.0 - 0.54 = 0.46
```

**特征 2：书本厚度**（厚 / 薄）

- α 部分：1A 2B
- β 部分：2A 1B

```
特征 a 颜色（蓝 / 红）       特征 2 厚度（厚 / 薄）
6 样本: 3A + 3B              6 样本: 3A + 3B
        D                            D
        │                            │
   ┌────┴────┐                  ┌────┴────┐
   蓝(α 4)  红(β 2)             厚(α 3)  薄(β 3)
   AAAB     BB                  1A 2B    2A 1B
   3A 1B    0A 2B               H 较高    H 较高
   H≈0.81   H=0                 ↓

```

### 笔记

> 【理解】信息增益

**信息增益**（information gain）：知道特征 $A$ 后，数据集 $D$ 不确定性减少的量。

$$g(D, A) = H(D) - H(D \mid A)$$

**经验熵**（数据集 $D$，$K$ 个类）：

$$H(D) = -\sum_{k=1}^{K} \frac{|C_k|}{|D|} \log_2 \frac{|C_k|}{|D|}$$

**经验条件熵**（特征 $A$ 取 $n$ 个值，把 $D$ 切成 $D_1 \dots D_n$）：

$$H(D \mid A) = \sum_{i=1}^{n} \frac{|D_i|}{|D|} H(D_i)$$

**特征选择规则**：每个特征算一次 $g(D, A)$，选**最大**的那个作分裂点。

**最小例子**（6 样本，特征 $a$ 取 $\alpha/\beta$）：$\alpha$ 对应 AAAB（4 样本，3A1B），$\beta$ 对应 BB（2 样本纯净，熵=0）。算 $H(D)$、$H(D \mid a)$ 后得 $g(D, a)$。其中 $H(D) = -\frac{3}{6}\log_2\frac{3}{6} - \frac{3}{6}\log_2\frac{3}{6} = 1$（3A 3B 对半）。具体数值（Slide 23）：$H(D_\alpha)=-\frac{3}{4}\log_2\frac{3}{4}-\frac{1}{4}\log_2\frac{1}{4}\approx 0.81$，$H(D_\beta)=0$，$H(D\mid a)=\frac{4}{6}\times 0.81+\frac{2}{6}\times 0=0.54$，$g(D,a)=1-0.54=0.46$。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 算了熵之后，怎么挑特征

[`01-信息熵.md`](./01-信息熵.md) 给了"纯度"的数字定义。但决策树要决定的是**用哪个特征切**——我们需要一个**比较**指标：

> 切之前的熵 vs 切之后的熵，哪个特征让熵下降最多？

这个"下降量"就是 **信息增益 information gain**。简单一句话：

```
切之前         切之后（用特征 a 把数据分成几堆）
─────         ──────────────────────────
H(D)    →     堆 1 的熵 + 堆 2 的熵 + ...（按样本数加权）
              ──────────────────────────
                       H(D | a)

信息增益 g(D, a) = H(D) − H(D | a)   ← 越大越好
```

直觉：**熵下降得越多 = 这个特征越能让样本变纯 = 越值得当分裂特征**。ID3 的全部精髓就是这一句。

### 条件熵 H(D | a)：切完之后的"加权平均熵"

切完之后不是一个集合，是**多个子集**。每个子集自己算熵，再按子集大小**加权**：

$$H(D \mid a) = \sum_{i=1}^{n} \frac{|D_i|}{|D|} \cdot H(D_i)$$

**符号**：

| 符号 | 含义 |
|---|---|
| $a$ | 一个特征（如"颜色"） |
| $a$ 的取值数 $n$ | 这个特征有几种值（颜色：蓝/红 = 2） |
| $D_i$ | 第 $i$ 个取值对应的子集（如蓝色书的子集） |
| $\lvert D_i \rvert / \lvert D \rvert$ | 该子集占总样本的比例（权重） |
| $H(D_i)$ | 该子集自己的熵（用 [`01-信息熵.md`](./01-信息熵.md) 公式算） |

**为什么要加权**：100 个样本切成"99 个一堆 + 1 个一堆"，1 个的小堆熵再低也不重要。加权 = "大头说了算"。

### 颜色案例完整走一遍（PPT Slide 23）

**数据**：6 本书，3 科技 (A) + 3 言情 (B)，按封面颜色分蓝 (α) / 红 (β)：

```
颜色 α（蓝，4 本）         颜色 β（红，2 本）
─────────                  ─────────
A A A B                    B B
3 科技 / 1 言情             0 科技 / 2 言情
```

#### Step 1 · 算原始熵 H(D)

总集 6 本书，3A 3B 对半：

$$H(D) = -\frac{3}{6}\log_2 \frac{3}{6} - \frac{3}{6}\log_2 \frac{3}{6} = -\log_2 \frac{1}{2} = 1 \text{ bit}$$

> 二分类对半时熵刚好 = 1 bit（每问一个 yes/no 才能猜中）。

#### Step 2 · 算每个子集的熵

**蓝子集** $D_\alpha$（3A 1B）：

$$H(D_\alpha) = -\frac{3}{4}\log_2 \frac{3}{4} - \frac{1}{4}\log_2 \frac{1}{4} \approx 0.811 \text{ bit}$$

**红子集** $D_\beta$（0A 2B，纯净）：

$$H(D_\beta) = -1 \log_2 1 - 0 \log_2 0 = 0 \text{ bit}$$

#### Step 3 · 算条件熵 H(D | 颜色)

按子集大小加权（蓝占 4/6，红占 2/6）：

$$H(D \mid \text{颜色}) = \frac{4}{6} \times 0.811 + \frac{2}{6} \times 0 \approx 0.541 \text{ bit}$$

#### Step 4 · 算信息增益

$$g(D, \text{颜色}) = H(D) - H(D \mid \text{颜色}) = 1 - 0.541 = 0.459 \text{ bit}$$

> PPT 算到 0.46，是四舍五入的差别。

### 横向对比：颜色 vs 厚度

PPT Slide 23 还提了"厚度"特征作对比（蓝 = 1A 2B，红 = 2A 1B）：

```
特征 a：颜色                        特征 b：厚度
─────────                           ─────────
蓝 4 本 (3A 1B)  H≈0.811             厚 3 本 (1A 2B)  H≈0.918
红 2 本 (0A 2B)  H=0                 薄 3 本 (2A 1B)  H≈0.918
─────────                           ─────────
H(D|颜色) = 4/6×0.811 + 2/6×0       H(D|厚度) = 3/6×0.918 + 3/6×0.918
         ≈ 0.541                            ≈ 0.918
─────────                           ─────────
g(颜色) = 1 - 0.541 ≈ 0.459         g(厚度) = 1 - 0.918 ≈ 0.082
                  ↑                                  ↑
              熵下降很多                         几乎没下降
```

**结论**：颜色把数据切得更纯（红子集已是纯 B），所以 ID3 选**颜色**作分裂特征。厚度切完两边各 1A 2B / 2A 1B，"还是混乱"，信息增益小。

### 代码落地：手撸 information_gain

```python
import numpy as np
from collections import Counter

def entropy(labels):
    """直接从标签数组算熵。"""
    if len(labels) == 0:
        return 0.0
    counts = np.array(list(Counter(labels).values()), dtype=float)
    probs = counts / counts.sum()
    return -np.sum(probs * np.log2(probs))

def info_gain(D, y, feature_idx):
    """算特征 feature_idx 在数据集 D 上的信息增益。
    D: shape (n_samples, n_features)
    y: shape (n_samples,)
    """
    H_D = entropy(y)
    H_D_given_a = 0.0
    feature_col = D[:, feature_idx]
    for value in np.unique(feature_col):
        mask = feature_col == value
        weight = mask.sum() / len(y)
        H_D_given_a += weight * entropy(y[mask])
    return H_D - H_D_given_a

# 颜色案例验证
D = np.array([
    ['蓝'], ['蓝'], ['蓝'], ['蓝'],   # 4 本蓝
    ['红'], ['红'],                    # 2 本红
])
y = np.array(['A', 'A', 'A', 'B', 'B', 'B'])
print(info_gain(D, y, 0))   # → 0.459...
```

### ID3 的隐藏 bug：偏好"取值多"的特征

信息增益有个致命缺陷：**取值越多的特征，增益天然越大**。

极端反例——把"序号"（每个样本唯一）当特征：

```
6 样本（3A 3B），按"序号"切
                  D
                  │
       ┌──┬──┬──┴──┬──┬──┐
       1   2   3   4   5   6
       A   A   A   B   B   B
       │   │   │   │   │   │
      H=0 H=0 H=0 H=0 H=0 H=0   ← 每个子集只有 1 个样本，必然纯！

H(D|序号) = 6 × (1/6 × 0) = 0
g(序号) = H(D) - 0 = 1.0   ← 最大！
```

但"序号"作分裂毫无意义——新样本进来序号都没见过，不能泛化。

**根因**：ID3 没惩罚"分支太多"的特征。下一代算法 **C4.5** 用 *信息增益率*（除以"特征自己的熵"）修这个 bug，详见 [`../03-c45/`](../03-c45/)。

### 一句话钉板

**信息增益 = 切之前的熵 − 切之后各子集的加权熵。ID3 选增益最大的特征当分裂点。但它偏爱多值特征——这个偏爱在 C4.5 里被修。**
