### PPT

> Slide 24 · ID3 决策树构建流程

**Iterative Dichotomiser 3**（迭代二分器 3）

1. 计算每个特征的信息增益
2. 使用信息增益最大的特征将数据集拆分为子集
3. 使用该特征（信息增益最大的特征）作为决策树的一个节点
4. 使用剩余特征对子集重复上述（1, 2, 3）过程

**目的**：为了让子集更为纯净，不杂乱，从而方便决策。

```
              D（数据集）
                   │
       ① 算每个特征的信息增益 g
                   │
       ② 选 g 最大的特征 A*
                   │
       ③ A* 作分裂节点，按取值切子集
                   │
       ┌──────────┼──────────┐
       D₁         D₂   ...   Dₙ
       │          │          │
       ④ 用剩余特征对每个 Dᵢ 重复 ①~③
       │          │          │
       ▼          ▼          ▼
        （直到子集纯净 → 叶子）
```

> Slide 31 · ID3 决策树 - 信息增益案例（性别）

特征 1 / 特征 2 / 标签

- 计算熵
- 计算性别条件熵（$a$ = "性别"）
- 计算性别信息增益（$a$ = "性别"）

```
切前 H(D) ≈ 0.918
        D（5 流失 / 10 留存）
              │
          按"性别"切
       ┌──────┴──────┐
       男            女
       (8 人)        (7 人)
       3 流 5 留     2 流 5 留
       H≈0.954      H≈0.863
              ↓
       H(D|性别) ≈ 0.911
       g(性别) ≈ 0.007    ← 几乎没用
```

> Slide 32 · ID3 决策树 - 信息增益案例（活跃度）

- 计算活跃度条件熵（$a$ = "活跃度"）
- 计算活跃度信息增益（$a$ = "活跃度"）

**结论**：活跃度的信息增益比性别的信息增益大，对用户流失的影响比性别大。因此，决策树选择**活跃度**进行分裂。

```
                D（5 流失 / 10 留存，H≈0.918）
                       │
                  按"活跃度"切
       ┌───────────────┼───────────────┐
       高 (6)          中 (5)          低 (4)
       0 流 6 留       1 流 4 留       4 流 0 留
       H = 0           H ≈ 0.722       H = 0  ← 两端纯！
                       ↓
              H(D|活跃度) ≈ 0.241
              g(活跃度) ≈ 0.677    ← 远大于性别！

→ 选活跃度作根节点
```

> Slide 33 · 总结

1. **决策树是什么？**
   决策树是一种树形结构，树中每个内部节点表示一个特征上的判断，每个分支代表一个判断结果的输出，每个叶子节点代表一种分类结果。

2. **信息熵？**
   在信息论中代表随机变量不确定度的度量。

3. **信息增益？**
   由于特征 $A$ 而使得对数据 $D$ 的分类不确定性减少的程度。

4. **ID3 树的构建流程**
   - 计算每个特征的信息增益
   - 使用信息增益最大的特征将数据集拆分为子集
   - 使用信息增益最大的特征作为决策树的一个节点
   - 重复上述步骤

> Slide 34-35 · 选择题

**1. 下列关于决策树的概念描述错误的是？**

- A）决策树算法需要构建树结构
- B）决策树上的每一个节点代表一个判断条件
- C）决策树上的每一个叶节点代表一种分类结果
- D）通过决策树不能明确特征的重要性程度

**答案**：D （解析：决策树要根据特征重要性进行分裂）

**2. 下列关于熵和信息熵的描述错误的是？**

- A）熵越大，系统的混乱程度越小
- B）信息熵是用来描述信息的完整性和有序性的
- C）信息的有序状态越一致、数据越集中，信息熵越小，反之越大

**答案**：A （解析：信息越混乱熵越大）

**3. 下列关于信息增益的描述正确的是？（多选）**

- A）表达的是在用某个特征对数据集进行分裂
- B）是 ID3 算法中的核心
- C）消除的不确定性越大，信息增益越小，表示这个特征越不重要
- D）对类别数较多的特征比较青睐

**答案**：ABD

### 笔记

> 【知道】ID3 树构建流程

**递归四步**：

1. 计算每个特征的信息增益
2. 选信息增益**最大**的特征切分数据集 $S$
3. 该特征作为决策树当前节点
4. 对每个子集，用**剩余特征**重复 1-3，直到纯净 / 无特征可用 / 达到停止条件
5. 给叶节点打标签：按多数类别归类；若各类数目相同，则按**小索引的类别**打标签（Slide 29）

**经典案例**：论坛流失率（15 样本：5 流失 / 10 留存），比较"性别"和"活跃度"哪个特征更重要。

- $H(D) = -\frac{5}{15}\log_2\frac{5}{15} - \frac{10}{15}\log_2\frac{10}{15} \approx 0.918$
- 分别算 $H(D \mid \text{性别})$、$H(D \mid \text{活跃度})$，再求增益
- **结论**：活跃度增益更大，对流失影响更显著，作为根节点

**ID3 的坑**：

- 只能处理**离散**特征（连续特征要预先分箱）
- 偏向**取值多**的特征（极端：ID 列每个值唯一 → 条件熵=0 → 增益最大，但毫无泛化意义）→ C4.5 用增益率修正
- 不支持缺失值，无剪枝机制

## ━━━━━━━━ 讲解 ━━━━━━━━

### 单步算了，怎么递归到一棵完整的树

[`02-信息增益.md`](./02-信息增益.md) 解决了"挑哪个特征分根节点"。但根分完后子集还在 → 继续递归挑特征 → 直到子集纯了变成叶子。这就是 ID3 的全部。

```
ID3 = 在每个节点上贪心选信息增益最大的特征 → 切 → 递归
─────────────────────────────────────────────────
"贪心" = 只看当前局部最优，不回头不全局规划
"递归" = 同样的事情套娃做下去，停止条件简单
```

### 算法伪代码（5 行）

```
def ID3(D, features):
    if D 已纯净（所有样本同类）         → return 叶子(标签 = 该类)
    if features 为空                    → return 叶子(标签 = D 的多数类)
    A* = argmax over a in features: g(D, a)   # 信息增益最大者
    tree = Node(分裂特征 = A*)
    for value in A* 的所有取值:
        D_v = D 中 A* == value 的子集
        if D_v 为空                     → 该分支变叶子(标签 = D 的多数类)
        else                            → 该分支 = ID3(D_v, features - {A*})
    return tree
```

**3 个停止条件**（缺一会无限递归）：

| 条件 | 触发场景 | 叶子标签策略 |
|---|---|---|
| 子集已纯 | 全是同类 | 用该唯一类 |
| 特征用完 | 父节点已把所有特征用光 | 该子集的多数类 |
| 子集为空 | 某个特征值在 D 里没出现过 | 父节点的多数类 |

> ID3 没有"max_depth / min_samples_leaf" 这类工程刹车，所以**几乎必然过拟合**——这是它工业上不流行的根本原因之一。后剪枝 / 预剪枝主场在第 7 章。

### "特征用完不能再用" 是 ID3 的特性

**关键设计选择**：ID3 / C4.5 的多叉切法是把一个特征的**所有取值**一次性全切掉（颜色：蓝 → 一支，红 → 一支）——切完这个特征就**没信息可榨**了，子树里不能再用。

对比 CART（详见 [`../04-cart/`](../04-cart/)）：每次只切**两份**（蓝 vs 非蓝），同一个特征可以在不同深度反复用。这是 CART 二叉的天然优势，尤其对连续特征。

```
ID3 多叉                  CART 二叉
─────────                ─────────
颜色：蓝/红/绿/黄          颜色：是 蓝 ?
  │                        ┌──┴──┐
  ┌────┬────┬────┐         是    否
  蓝   红   绿   黄         ↓     ↓
  ─────────                  叶   颜色：是 红 ?
  特征"颜色"已用完                 ┌──┴──┐
  子树不能再用                     是    否
                                   ...   ...
                                ─────────
                                同一特征可重复
```

### 完整案例：论坛流失率（PPT Slide 30-32）

**业务**：15 个用户（5 流失 / 10 留存），考察"性别"和"活跃度"哪个对流失影响更大。

#### Step 1 · 算原始熵 H(D)

15 个样本，5 流失 / 10 留存：

$$H(D) = -\frac{5}{15} \log_2 \frac{5}{15} - \frac{10}{15} \log_2 \frac{10}{15} \approx 0.918 \text{ bit}$$

#### Step 2 · 算"性别"的信息增益

按性别切：男 8 人（3 流 5 留），女 7 人（2 流 5 留）。

$$H(D_\text{男}) = -\frac{3}{8}\log_2 \frac{3}{8} - \frac{5}{8}\log_2 \frac{5}{8} \approx 0.954$$

$$H(D_\text{女}) = -\frac{2}{7}\log_2 \frac{2}{7} - \frac{5}{7}\log_2 \frac{5}{7} \approx 0.863$$

$$H(D \mid \text{性别}) = \frac{8}{15} \times 0.954 + \frac{7}{15} \times 0.863 \approx 0.911$$

$$g(D, \text{性别}) = 0.918 - 0.911 \approx \textbf{0.007}$$

#### Step 3 · 算"活跃度"的信息增益

按活跃度切：高 6 人（0 流 6 留 → 全留），中 5 人（1 流 4 留），低 4 人（4 流 0 留 → 全流失）。

$$H(D_\text{高}) = 0 \quad H(D_\text{中}) = -\frac{1}{5}\log_2 \frac{1}{5} - \frac{4}{5}\log_2 \frac{4}{5} \approx 0.722 \quad H(D_\text{低}) = 0$$

$$H(D \mid \text{活跃度}) = \frac{6}{15} \times 0 + \frac{5}{15} \times 0.722 + \frac{4}{15} \times 0 \approx 0.241$$

$$g(D, \text{活跃度}) = 0.918 - 0.241 \approx \textbf{0.677}$$

#### Step 4 · 比较 → 选活跃度作根节点

```
信息增益对比：
  性别    |▏              | 0.007    ← 几乎没用
  活跃度  |██████████████ | 0.677    ← 强信号

→ 根节点选 "活跃度"
```

**业务解释**：高活跃 / 低活跃用户的流失行为完全两极（高活跃几乎不流失，低活跃几乎全流失），活跃度切完两端都纯，熵骤降。性别在留存 / 流失里分布差不多，切完没什么变化。

### 推理（Slide 29）：从根走到叶

训练完一棵树之后预测新样本：

```
新样本 → 根节点（看活跃度）
        ├─ 高 → 叶："留存"（多数类 / 历史数据中高活跃 100% 留存）
        ├─ 中 → 进入子树 → 看下一个特征 ...
        └─ 低 → 叶："流失"
```

**叶子打标签的规则**：

1. **多数类**：该叶子里哪类多就打哪类
2. **平局取小索引**：如果各类数目相同，按类别索引最小的（PPT Slide 29 原话）

> 实战里平局极少见。重要的是知道这是 ID3 的一个**确定性 tie-breaker**，不是随机。

### 工程视角：sklearn 为什么没实现 ID3

| 限制 | 影响 |
|---|---|
| **只能处理离散特征** | 连续特征（年龄 / 收入）必须预先分箱，丢失信息 |
| **偏好多值特征** | "用户 ID"作分裂点是经典反例，毫无泛化 |
| **不支持缺失值** | 真实数据普遍有缺失，预处理成本高 |
| **没剪枝机制** | 必然过拟合，叶子小到只剩 1 样本 |
| **多叉树** | 二叉树结构更好序列化、推理更快 |

**结论**：sklearn 默认实现的是 CART（[`../04-cart/`](../04-cart/)），ID3 仅作教学。但 ID3 的两个核心概念——**信息熵 + 信息增益**——是后续所有信息论方法（C4.5 增益率、互信息特征选择、决策树森林等）的基础，必须吃透。

### 一句话钉板

**ID3 = 贪心 + 递归 + 信息增益。每节点挑增益最大的特征切，子集纯了打标签。简单透明，但偏好多值 / 不支持连续 / 必过拟合——所以工业不用，教学必学。**
