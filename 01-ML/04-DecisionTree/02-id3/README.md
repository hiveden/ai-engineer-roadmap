> 02 · ID3 决策树

> 【理解】信息熵

**ID3 = Iterative Dichotomiser 3**：基于**信息增益**做特征选择。

**信息熵**（entropy）：随机变量不确定度的度量。

$$H = -\sum_{i=1}^{k} p_i \log(p_i)$$

**直觉**：

- 熵越大 → 越混乱、越不确定
- 熵越小 → 越纯、越确定
- 极端：单类纯净时 $H=0$；类别均匀时 $H$ 最大

**例子**：

- 占比 $\{1/3, 1/3, 1/3\}$：$H = \ln 3 \approx 1.0986$（最大）
- 占比 $\{1/10, 2/10, 7/10\}$：$H \approx 0.8018$
- 占比 $\{1, 0, 0\}$：$H = 0$

> 【理解】信息增益

**信息增益**（information gain）：知道特征 $A$ 后，数据集 $D$ 不确定性减少的量。

$$g(D, A) = H(D) - H(D \mid A)$$

**经验熵**（数据集 $D$，$K$ 个类）：

$$H(D) = -\sum_{k=1}^{K} \frac{|C_k|}{|D|} \log_2 \frac{|C_k|}{|D|}$$

**经验条件熵**（特征 $A$ 取 $n$ 个值，把 $D$ 切成 $D_1 \dots D_n$）：

$$H(D \mid A) = \sum_{i=1}^{n} \frac{|D_i|}{|D|} H(D_i)$$

**特征选择规则**：每个特征算一次 $g(D, A)$，选**最大**的那个作分裂点。

**最小例子**（6 样本，特征 $a$ 取 $\alpha/\beta$）：$\alpha$ 对应 AAAB（4 样本，3A1B），$\beta$ 对应 BB（2 样本纯净，熵=0）。算 $H(D)$、$H(D \mid a)$ 后得 $g(D, a)$。

> 【知道】ID3 树构建流程

**递归四步**：

1. 计算每个特征的信息增益
2. 选信息增益**最大**的特征切分数据集 $S$
3. 该特征作为决策树当前节点
4. 对每个子集，用**剩余特征**重复 1-3，直到纯净 / 无特征可用 / 达到停止条件

**经典案例**：论坛流失率（15 样本：5 流失 / 10 留存），比较"性别"和"活跃度"哪个特征更重要。

- $H(D) = -\frac{5}{15}\log_2\frac{5}{15} - \frac{10}{15}\log_2\frac{10}{15} \approx 0.918$
- 分别算 $H(D \mid \text{性别})$、$H(D \mid \text{活跃度})$，再求增益
- **结论**：活跃度增益更大，对流失影响更显著，作为根节点

**ID3 的坑**：

- 只能处理**离散**特征（连续特征要预先分箱）
- 偏向**取值多**的特征（极端：ID 列每个值唯一 → 条件熵=0 → 增益最大，但毫无泛化意义）→ C4.5 用增益率修正
- 不支持缺失值，无剪枝机制
