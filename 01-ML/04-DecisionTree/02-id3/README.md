### Slide 14 · 概念 2：信息熵

**信息熵**：代表随机变量的不确定度 / 预测难度的度量

- 熵越大，事件或变量越难以预测，不确定性越高 —— 样本较杂
- 熵越小，事件或变量越容易预测，不确定性越低 —— 样本较纯

数学上：信息熵 = 事件所有信息量的"数学期望"（加权平均）

$$H = -\sum_{i=1}^{k} p_i \log_2 p_i$$

### Slide 18 · 香农信息论 - 初步

**高信息熵 ≠ 有用信息多**，而是"样本太杂，预测难度、不确定度"高！

**低信息熵**：样本相对单一而纯粹，预测难度低

"熵"这个词最早来自物理学，描述宇宙越来越"乱"的趋势（比如香气四溢、墨水散开、再比如整齐码放的台球因为碰撞而四下滚落）。

"不确定性"和"混乱度"本质是一回事，所以借用了"熵"这个名字。

热力学熵 → 信息熵

### Slide 22 · 信息增益

**概念**：特征 $a$ 对训练数据集 $D$ 的信息增益 $g(D, a)$ 定义为：集合 $D$ 的**原始熵** $H(D)$ 与**以特征 $a$ 进行分割后的条件熵** $H(D|a)$ 之差。

**数学公式**：$g(D, A) = H(D) - H(D|A)$

信息增益 = 原总熵 - 条件熵（指以某特征进行分割后的总熵）

**条件熵**降低分类错误。

**权重**：每个叶节点的数目占比；**每个子类的信息熵**。

```
不确定性条对比：

切前  |████████████████████|   H(D)
                ↓ 用特征 a 切
切后  |██████████          |   H(D|a) = Σ |Dᵢ|/|D| × H(Dᵢ)
       └─ 已剩 ─┘└─ 增益 ─┘
                  g(D, a)
```

> **Notes**：条件熵计算 公式说明 每个类别的分类后样本占比 / 每个分类后样本的信息熵

### Slide 23 · 信息增益计算

**特征 a：书本颜色**（蓝 / 红）

- 3 本科技书 (A) + 3 本言情小说 (B)
- α 部分（蓝）对应目标值：AAAB
- β 部分（红）对应目标值：BB

```
条件为 α 熵： -3/4 * math.log(3/4, 2) - 1/4 * math.log(1/4, 2) = 0.81
条件为 β 熵： -(2/2) * math.log(2/2, 2) = 0

条件熵：α 部分占 4/6，β 部分占 2/6
       (4/6) * 0.81 + (2/6) * 0 = 0.54

(原初) 熵： -3/6 * math.log(3/6, 2) - 3/6 * math.log(3/6, 2) = 1

信息增益： (原初) 熵 - 条件熵 = 1.0 - 0.54 = 0.46
```

### Slide 24 · ID3 决策树构建流程

**Iterative Dichotomiser 3**（迭代二分器 3）

1. 计算每个特征的信息增益
2. 使用信息增益最大的特征将数据集拆分为子集
3. 使用该特征（信息增益最大的特征）作为决策树的一个节点
4. 使用剩余特征对子集重复上述（1, 2, 3）过程

**目的**：为了让子集更为纯净，不杂乱，从而方便决策。

```
              D（数据集）
                   │
       ① 算每个特征的信息增益 g
                   │
       ② 选 g 最大的特征 A*
                   │
       ③ A* 作分裂节点，按取值切子集
                   │
       ┌──────────┼──────────┐
       D₁         D₂   ...   Dₙ
       │          │          │
       ④ 用剩余特征对每个 Dᵢ 重复 ①~③
       │          │          │
       ▼          ▼          ▼
        （直到子集纯净 → 叶子）
```

### Slide 33 · 总结

1. **决策树是什么？**
   决策树是一种树形结构，树中每个内部节点表示一个特征上的判断，每个分支代表一个判断结果的输出，每个叶子节点代表一种分类结果。

2. **信息熵？**
   在信息论中代表随机变量不确定度的度量。

3. **信息增益？**
   由于特征 $A$ 而使得对数据 $D$ 的分类不确定性减少的程度。

4. **ID3 树的构建流程**
   - 计算每个特征的信息增益
   - 使用信息增益最大的特征将数据集拆分为子集
   - 使用信息增益最大的特征作为决策树的一个节点
   - 重复上述步骤

### 笔记

> 02 · ID3 决策树

**学习目标**：

1. 理解信息熵的意义
2. 理解信息增益的作用
3. 知道 ID3 树的构建流程

> 【理解】信息熵

**ID3 = Iterative Dichotomiser 3**：基于**信息增益**做特征选择。

**信息熵**（entropy）：随机变量不确定度的度量。

**单事件信息量**（Slide 12-14）：$h(x) = -\log_2 p(x)$。概率越小、信息量越大（"太阳西升"）；信息熵 = 所有事件信息量的数学期望（加权平均）。

$$H = -\sum_{i=1}^{k} p_i \log_2(p_i)$$

**直觉**：

- 熵越大 → 越混乱、越不确定
- 熵越小 → 越纯、越确定
- 极端：单类纯净时 $H=0$；类别均匀时 $H$ 最大

**例子**：

- 占比 $\{1/3, 1/3, 1/3\}$：$H = \log_2 3 \approx 1.585$（最大）
- 占比 $\{1/10, 2/10, 7/10\}$：$H \approx 1.157$
- 占比 $\{1, 0, 0\}$：$H = 0$

> 【理解】信息增益

**信息增益**（information gain）：知道特征 $A$ 后，数据集 $D$ 不确定性减少的量。

$$g(D, A) = H(D) - H(D \mid A)$$

**经验熵**（数据集 $D$，$K$ 个类）：

$$H(D) = -\sum_{k=1}^{K} \frac{|C_k|}{|D|} \log_2 \frac{|C_k|}{|D|}$$

**经验条件熵**（特征 $A$ 取 $n$ 个值，把 $D$ 切成 $D_1 \dots D_n$）：

$$H(D \mid A) = \sum_{i=1}^{n} \frac{|D_i|}{|D|} H(D_i)$$

**特征选择规则**：每个特征算一次 $g(D, A)$，选**最大**的那个作分裂点。

**最小例子**（6 样本，特征 $a$ 取 $\alpha/\beta$）：$\alpha$ 对应 AAAB（4 样本，3A1B），$\beta$ 对应 BB（2 样本纯净，熵=0）。算 $H(D)$、$H(D \mid a)$ 后得 $g(D, a)$。其中 $H(D) = -\frac{3}{6}\log_2\frac{3}{6} - \frac{3}{6}\log_2\frac{3}{6} = 1$（3A 3B 对半）。具体数值（Slide 23）：$H(D_\alpha)=-\frac{3}{4}\log_2\frac{3}{4}-\frac{1}{4}\log_2\frac{1}{4}\approx 0.81$，$H(D_\beta)=0$，$H(D\mid a)=\frac{4}{6}\times 0.81+\frac{2}{6}\times 0=0.54$，$g(D,a)=1-0.54=0.46$。

> 【知道】ID3 树构建流程

**递归四步**：

1. 计算每个特征的信息增益
2. 选信息增益**最大**的特征切分数据集 $S$
3. 该特征作为决策树当前节点
4. 对每个子集，用**剩余特征**重复 1-3，直到纯净 / 无特征可用 / 达到停止条件
5. 给叶节点打标签：按多数类别归类；若各类数目相同，则按**小索引的类别**打标签（Slide 29）

**经典案例**：论坛流失率（15 样本：5 流失 / 10 留存），比较"性别"和"活跃度"哪个特征更重要。

- $H(D) = -\frac{5}{15}\log_2\frac{5}{15} - \frac{10}{15}\log_2\frac{10}{15} \approx 0.918$
- 分别算 $H(D \mid \text{性别})$、$H(D \mid \text{活跃度})$，再求增益
- **结论**：活跃度增益更大，对流失影响更显著，作为根节点

**ID3 的坑**：

- 只能处理**离散**特征（连续特征要预先分箱）
- 偏向**取值多**的特征（极端：ID 列每个值唯一 → 条件熵=0 → 增益最大，但毫无泛化意义）→ C4.5 用增益率修正
- 不支持缺失值，无剪枝机制
