#### Slide 38 · C4.5 决策树（Classifier 4.5 / 分类器 4.5）

**ID3 树的不足**：偏向于选择种类多的特征作为分裂依据

```
反例：把"序号"当分裂特征（6 取值，每值唯一）

                    D（6 样本：3 科技 / 3 文学）
                              │
                          按"序号"切
       ┌────┬────┬────┴────┬────┬────┐
       1    2    3         4    5    6
       │    │    │         │    │    │
      科技 科技 文学      文学 文学 科技
      H=0  H=0  H=0       H=0  H=0  H=0   ← 每叶都纯！

  H(D|序号) = 0   →   g(序号) = H(D) - 0 = 1.0   ← 增益最大
                                                  但毫无泛化意义
                                                  （新样本序号都没见过）

  对比"封面颜色"（仅 2 取值）：g(颜色) 较小，却是真正有用的特征
```

**举个例子**：

| 序号 | 封面颜色 | 目标值 |
|---|---|---|
| 1 | 红 | 科技书 |
| 2 | 红 | 科技书 |
| 3 | 红 | 文学书 |
| 4 | 蓝 | 文学书 |
| 5 | 蓝 | 文学书 |
| 6 | 蓝 | 科技书 |

每个叶节点的样本都非常纯净，条件熵 = 0，信息增益大。

1. 将序号作为分裂特征，会构造一棵深度为 2 的决策树，该树的预测准确率可能非常高
2. 但分裂的特征没有意义，导致过拟合（换一波数据则无法准确预测）

封面颜色有 2 个取值，序号有 6 个取值。

> **Notes**：深度 2 的含义；图要换。
> 用特征 b（序号）作为分类（种类多），导致每个类别就一个样本，当然很纯（需要更换展示图）。
> 且依赖少数且分类多的特征，模型复杂、过拟合。

#### Slide 39 · 新的分裂依据：特征的信息增益率

**信息增益率 = 信息增益 / 特征熵**

```
                       g(D, A)             ← 分子：原本的信息增益
   Gain_Ratio(D, A) = ─────────
                       IV(A)               ← 分母：把"特征 A"自己当随机变量算的熵
                                              （即"分裂信息"，也叫特征熵）

  特征取值个数         IV(A)            惩罚系数 1/IV(A)        增益率
  ─────────────       ─────             ──────────────         ───────
  少（如 2 值）        小                大                     被放大 ↑
  多（如 6 值唯一）    大                小                     被压制 ↓


  直觉：分裂出越多分支 → IV 越大 → 把虚高的增益按比例"罚回去"

  切前  |████████████████████|   H(D)
                ↓ 用特征 A 切
  切后  |██████          |   H(D|A)
         └─已剩─┘└──g(D,A)──┘
                       ÷ IV(A)（A 自身的熵，分支越多越大）
                    ──────────
                     Gain_Ratio
```

**信息增益率的本质**：（特征 a 分类后有几个类别？对类别求信息熵作为惩罚）即不希望这个特征分裂后，有很多个分支。

**特征的信息增益 ÷ 特征的信息熵** —— 相当于对信息增益进行修正，增加一个惩罚系数。

特征取值个数**较多**时，惩罚系数较小；特征取值个数**较少**时，惩罚系数较大。

**惩罚系数**：数据集 D 以特征 a 作为随机变量的熵的**倒数**。

#### Slide 40 · 信息增益率（计算例子）

```
6 样本数据集
──────────────────────────────
特征 b：6 个取值（1/2/3/4/5/6） ← 多值
特征 a：2 个取值（α / β）        ← 少值
目标值：3 个 A，3 个 B
──────────────────────────────
任务：哪个特征更值得分裂？
      用信息增益率裁判 ID3 偏向多值的偏差
```

**已知数据集**：

| 特征 b | 特征 a | 目标值 |
|---|---|---|
| 1 | α | A |
| 2 | α | A |
| 3 | β | B |
| 4 | α | A |
| 5 | β | B |
| 6 | α | B |

**需求**：求特征 a、b 的信息增益率

```
原总熵 H(D) = 1.0（3A 3B 对半，最难猜）

特征 a（2 值）                       特征 b（6 值，每值唯一）
        D                                    D
        │                                    │
   ┌────┴────┐              ┌────┬────┬────┼────┬────┬────┐
   α(4)     β(2)            1    2    3    4    5    6
   3A 1B    0A 2B           A    A    B    A    B    B
   H≈0.81   H=0             H=0  H=0  H=0  H=0  H=0  H=0
   ↓                        ↓
   H(D|a) ≈ 0.54            H(D|b) = 0      ← 条件熵 ×6 全部为 0
   g(a)   ≈ 0.46            g(b)   = 1.0    ← 增益虚高
   IV(a)  ≈ 0.918           IV(b)  ≈ 2.585  ← 分裂信息（惩罚）
   ─────────────            ─────────────
   Gain_Ratio(a) ≈ 0.50     Gain_Ratio(b) ≈ 0.387


增益率 PK：
  特征 a |██████████ | 0.50    ← 胜
  特征 b |███████▌   | 0.387
                 ↑
        虚高的增益被 IV 罚下去了
```

原总熵 → 条件熵 ×6

**结论**：特征 a 的信息增益率大于特征 b 的信息增益率，根据信息增益率，应该选择**特征 a** 作为分裂特征。

#### Slide 41 · 总结

1. **信息增益率的作用**
   信息增益偏向于选择种类多的特征作为分裂依据 → 缓解 ID3 树中存在的不足
2. **信息增益率**
   信息增益率 = 信息增益 / 特征熵
   相当于对信息增益进行修正，增加一个惩罚系数

### 笔记

> 03 · C4.5 决策树

**学习目标**：

1. 理解信息增益率的意义
2. 知道 C4.5 树的构建方法

> 【理解】信息增益率

**动机**：ID3 倾向于挑取值多的特征（"用户 ID"作根节点的悖论）。C4.5 通过**惩罚项**修正。

**公式**：

$$\text{Gain\_Ratio}(D, A) = \frac{g(D, A)}{IV(A)}$$

其中**分裂信息**（intrinsic value，本质上是把特征 $A$ 当随机变量算的熵）：

$$IV(A) = -\sum_{i=1}^{n} \frac{|D_i|}{|D|} \log_2 \frac{|D_i|}{|D|}$$

**机制**：

- 特征取值种类**多** → $IV(A)$ 大 → 惩罚强 → 增益率被压小
- 特征取值种类**少** → $IV(A)$ 小 → 惩罚弱 → 增益率相对放大

**本质**：在信息增益的基础上乘惩罚参数 $1/IV(A)$。

> 【知道】C4.5 树构建说明

**对比计算**（6 样本 3A 3B，$H(D)=1$；特征 $a$ 取 2 值，特征 $b$ 取 6 值）：

| 指标 | 特征 $a$（2 值） | 特征 $b$（6 值，每值唯一） |
|---|---|---|
| 信息增益 | 0.46 | 1.0（看似完美） |
| 分裂信息 $IV$ | $-\frac{4}{6}\log_2\frac{4}{6} - \frac{2}{6}\log_2\frac{2}{6} \approx 0.918$ | $-\frac{1}{6}\log_2\frac{1}{6} \times 6 \approx 2.585$ |
| 信息增益率 | $0.46 / 0.918 \approx 0.50$ | $1.0 / 2.585 \approx 0.387$ |

**结论**：按增益率，特征 $a$ 反超 → 选 $a$ 作分裂特征。

**C4.5 相对 ID3 的工程改进**：

1. **信息增益率**：缓解偏向多值特征
2. **支持连续特征**：对连续值排序后用相邻中点作候选分裂点（与 CART 类似）
3. **支持缺失值**：用未缺失样本算增益、再按比例分配缺失样本
4. **后剪枝**：基于悲观误差估计 PEP（Pessimistic Error Pruning）

**局限**：

- 仍是**多叉树**（每个离散特征值一个分支）
- 全量数据驻留内存，不适合超大数据集
- sklearn 未实现 ID3/C4.5，工业界主用 CART
