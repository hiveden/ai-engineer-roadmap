## ━━━━━━━━ 讲解 ━━━━━━━━

### 分裂目的：让样本"切完更纯"

进 CART 之前先把决策树的根本动机说清——所有决策树（ID3 / C4.5 / CART）都在干同一件事：

> **找一个特征，把杂乱的样本切成尽量纯净的子集。**

```
切前（杂乱，难判定）           切后（更纯，好判定）
─────────────                  ─────────────
A ████ 33%                     左子集：A ████████ 100%（一眼断 A）
B ████ 33%      ── 切 ──▶      ─────────────
C ████ 33%                     右子集：B ████ 50%
                                       C ████ 50%
最难猜（H 高 / Gini 高）        左已纯（H=0 / Gini=0）
```

**纯**就好判——一个子集里全是 A，新样本掉进来直接判 A，错不了。**杂乱**就难判——三类各占 1/3，瞎猜准确率只有 33%。所以决策树整个训练过程就是**反复在每个节点上挑能让纯度提升最多的特征**。

三代算法挑特征的"提升尺子"不一样：

| 算法 | 挑特征用什么尺子 |
|---|---|
| ID3（1975） | **信息增益 information gain**（熵降得越多越好） |
| C4.5（1993） | **信息增益率 gain ratio**（增益除以特征自身熵） |
| **CART（1984）** | **基尼指数 Gini index**（切完不纯度越小越好） |

本章主角就是第三把尺子——CART 的基尼指数。

### 直觉锚：从"问问题"换成"抽两次"

ID3 / C4.5 的分裂依据是**信息熵 entropy**——猜中类别需要问几次问题。CART 1984 年换了个口子：

```
ID3 / C4.5 视角                  CART 视角
─────────────                    ─────────────
猜中要几次问题                   闭眼抓两次抓到不同色的概率
H 越小 → 越好猜                  Gini 越小 → 越容易抓到同色
```

举个直觉例子。一袋彩珠 100% 红色——闭眼抓两次必同色，"抓到不同"概率 = 0；如果红蓝各半——抓两次约一半概率不同，"抓到不同"概率 = 0.5。**纯净 → 0；混乱 → 大**，方向和熵完全一致，但不用算 $\log$。

```
全红 100%                  红蓝各半 50/50
████████████ 100%           █████ 50%
                            █████ 50%
抓 2 次必同色               抓 2 次约半数不同色
Gini = 0                    Gini = 0.5
```

这个"抓到不同色的概率"就是**基尼值 Gini value**——精确公式下一节展开，本节先把 CART 的全貌看清。

### CART 的两个硬规矩

CART 跟 ID3 / C4.5 的差别不止换了个准则，还多了两条工程取舍：

| 维度 | ID3 / C4.5 | CART |
|---|---|---|
| 分裂准则 | 熵相关（增益 / 增益率） | 基尼指数（分类） / 平方误差（回归） |
| 树形 | **多叉**（每个特征值开一支） | **严格二叉**（每次只切两份） |
| 任务 | 仅分类 | **分类 + 回归通吃** |
| sklearn | 未实现 | 默认实现（`DecisionTreeClassifier`） |

**严格二叉**怎么处理多取值特征？比如「婚姻状况：single / married / divorced」3 个值——CART 不会一次开 3 个分支，而是枚举所有"一对其余"的二分组合（{married} | {single, divorced} 等），挑基尼指数最小那种切（[`03-计算举例.md`](./03-计算举例.md)）。

### 双形态：一棵树两副面孔

CART 同一套分裂逻辑，换准则就能做两类活：

**① 分类 classification**——目标是离散类别（拖欠 / 不拖欠、垃圾邮件 / 正常邮件）：

- 切完看子集"够不够纯"
- 准则：**基尼指数 Gini index** 最小化

**② 回归 regression**——目标是连续数值（房价、年收入预测）：

- 切完看子集内部"够不够齐"
- 准则：**平方误差 squared error** 最小化（每一支的方差加权和）

```
CART 分类                        CART 回归
─────────────                   ─────────────
目标：类别                       目标：连续数值
切完看：纯不纯                   切完看：齐不齐
准则：基尼指数最小                准则：平方误差最小
叶子打：占多数的类别              叶子打：本组数值的均值
```

回归树主场不在本章，详见后续 `06-regression-tree/`。本章 03 / 04 只展开分类。

### 术语版

| 故事里的元素 | 术语 |
|---|---|
| 抓两次抓到不同色的概率 | 基尼值 Gini value（也叫基尼不纯度 Gini impurity） |
| 用特征切完后基尼值的加权和 | 基尼指数 Gini index |
| 1984 年提出的二叉决策树 | CART = Classification And Regression Tree |
| 严格只切两份的树形 | 二叉树 binary tree |
| 回归树的分裂准则 | 平方误差 squared error（也叫均方误差 MSE） |

### 关键启示

**CART 用「抓两次抓到不同色」的概率代替熵，强制二叉、分类回归通吃——sklearn 默认就是它。**
