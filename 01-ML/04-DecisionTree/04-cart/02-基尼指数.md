### PPT

> Slide 47 · CART 分类树 · 基尼值

```
随机抽 2 个样本，类别"不一致"的概率
─────────────────────────────────────
样本袋 D：● ● ● ● ○ ○ ○ ▲

  ┌── 抽 1 个 ──┐    ┌── 再抽 1 个 ──┐
  │             │    │               │
  ▼             ▼    ▼               ▼
 颜色 X       记下     颜色 Y      对比
                                    │
                  X ≠ Y 的概率 = Gini(D)
─────────────────────────────────────
Gini 越小 → 越容易抽到同色 → 越纯
```

**基尼值 Gini(D)**：从数据集 D 中随机抽取两个样本，其类别标记不一致的概率。故，**Gini(D) 值越小，数据集 D 的纯度越高**。

可以理解为：总概率 100% - 2 次抽取到相同类别的"联合概率"。

即：100% - （类别占比的平方之和）

```
3 类示例：纯净 vs 偏混
─────────────────              ─────────────────
A ████████████ 100%            A ██████ 50%
B ·             0%             B ███ 25%
C ·             0%             C ███ 25%
─────────────────              ─────────────────
Gini = 1 - 1² = 0              Gini = 1 - (.5²+.25²+.25²)
                                    = 1 - 0.375 = 0.625
（最纯）                        （较杂）
```

**例**：样本包含 3 种颜色：

- $1 - 100\%^2 = 0$ （Gini 越小，数据纯度越高）
- $1 - (50\%^2 + 25\%^2 + 25\%^2) = 0.625$ （数据纯度低！）

**基尼指数 Gini_index(D)**：选择使划分后基尼系数最小的属性作为最优化分属性。

```
特征 A 把 D 切成 D₁、D₂
─────────────────────────────
        D
        │
    用特征 A 切
        │
   ┌────┴────┐
   D₁        D₂
   Gini(D₁)  Gini(D₂)
        ↓
Gini_index(D, A) = |D₁|/|D| × Gini(D₁)
                 + |D₂|/|D| × Gini(D₂)
                       ↓
              枚举所有特征 / 分裂点
              选 Gini_index 最小者
```

**注意**：

- 信息增益（ID3）、信息增益率（C4.5）值越**大**，则说明优先选择该特征
- 基尼指数值越**小**（cart），则说明优先选择该特征

> **Notes**：基尼值用减法理解：100% - 两次抽到相同值的概率。基尼系数：社会财富平均程度。

### 笔记

> 【理解】基尼指数计算方法

**基尼值**（衡量集合纯度）：

$$\text{Gini}(D) = 1 - \sum_{k=1}^{K} \left(\frac{|C_k|}{|D|}\right)^2$$

**基尼指数**（特征 $A$ 把 $D$ 切成 $D_1, D_2$ 的加权基尼值）：

$$\text{Gini\_index}(D, A) = \frac{|D_1|}{|D|} \text{Gini}(D_1) + \frac{|D_2|}{|D|} \text{Gini}(D_2)$$

**为什么 Gini 比熵快**：无 $\log$ 计算，工程上更高效；两者效果接近。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉锚：闭眼摸两次摸到不同色的概率

把数据集想成一袋彩珠，每颗珠子的颜色就是它的类别标签。**基尼值 Gini value** 在问一个具体得不能再具体的问题：

> **闭眼抓一颗、记下颜色、放回去；再闭眼抓一颗——两次颜色不一样的概率是多少？**

```
全红（10/10）              红蓝各半（5/5）
─────────────              ─────────────
🔴🔴🔴🔴🔴                  🔴🔴🔴🔴🔴
🔴🔴🔴🔴🔴                  🔵🔵🔵🔵🔵

两次必同色                  两次约一半同一半不同
Gini = 0                    Gini = 0.5
（最纯）                    （最乱，二分类极限）
```

**Gini 越小 → 越容易抓到同色 → 这袋珠子越"纯"。**

为什么不直接抽一次看是什么色？因为抽一次只能看到点，抽两次问"是否同色"才能反映**整袋的混乱程度**——这是同一族（与熵共享思想）的不纯度衡量法。

### 怎么算（精确公式）

抓到同色的概率 = 各类别"自己抓两次"的概率加起来。设类别 $k$ 的占比是 $p_k$，"两次都抓到 $k$"概率是 $p_k^2$。**抓到不同色（基尼值）= 1 减去抓到同色**：

$$
\text{Gini}(D) = 1 - \sum_{k=1}^{K} p_k^2
$$

> **符号扫盲**：$K$ = 类别总数；$p_k = |C_k| / |D|$ = 第 $k$ 类样本数除以总样本数（占比）。读"基尼 Gini"就是公式作者意大利统计学家 Corrado Gini 的姓。

实算 3 个对照（教材原例）：

```
分布形态                         占比                Gini 计算                值
─────────────                    ─────               ─────────────             ─────
① 一类占满（100/0/0）           1, 0, 0             1 - 1²                   0      ← 最纯
② 偏斜（50/25/25）              0.5, 0.25, 0.25     1 - 0.25 - 0.0625×2     0.625
③ 三类均匀（1/3, 1/3, 1/3）     ⅓, ⅓, ⅓             1 - 3 × (1/3)²           0.667  ← 三类极限
```

**二分类时**，Gini 值最大就是 0.5（50/50）；类别越多，均匀时极限越大（3 类 0.667，4 类 0.75，$K$ 类 $1 - 1/K$）。

### 加权一步：从基尼值到基尼指数

基尼值衡量**单个集合**的纯度。可决策树要选**特征**——切完后看左右两支总体纯不纯。把切出来的两支基尼值**按样本占比加权**就行：

$$
\text{Gini\_index}(D, A) = \frac{|D_1|}{|D|}\text{Gini}(D_1) + \frac{|D_2|}{|D|}\text{Gini}(D_2)
$$

```
              D（10 样本）
                  │
              用特征 A 切
              │
        ┌─────┴─────┐
        D₁（3 样本）  D₂（7 样本）
        Gini(D₁)     Gini(D₂)
              │
        Gini_index = 3/10 × Gini(D₁) + 7/10 × Gini(D₂)
                       ↑
                  样本多的子集占主导，权重大
```

**为什么要加权**？大子集没切干净比小子集没切干净糟糕得多——3 样本的子集再脏总污染面有限，70 样本的子集脏意味着大部分数据还没切清。

### CART 用基尼指数切分的完整流程

公式有了——但**怎么把 Gini 公式变成一棵树**？把零散动作串成 5 步算法：

```
节点 N（待切的样本子集）
        │
   ┌────▼─────────────────────────────────────┐
   │ ① 收集候选分裂点（每个特征都生成一批）   │
   │   ─ 二值特征：天然 1 个二分              │
   │   ─ 多值特征：枚举 2^(n-1)-1 种"一对其余"│
   │   ─ 连续特征：排序后取相邻中点           │
   └────┬─────────────────────────────────────┘
        │
   ┌────▼─────────────────────────────────────┐
   │ ② 对每个候选分裂点算 Gini_index          │
   │   = |D₁|/|D|·Gini(D₁) + |D₂|/|D|·Gini(D₂)│
   └────┬─────────────────────────────────────┘
        │
   ┌────▼─────────────────────────────────────┐
   │ ③ 全局取 Gini_index 最小的(特征, 分裂点) │
   │   作为本节点的分裂方案                    │
   └────┬─────────────────────────────────────┘
        │
   ┌────▼──────────────────────────────────────┐
   │ ④ 按该方案把 N 切成 D₁ 和 D₂（严格二叉）  │
   └────┬──────────────────────────────────────┘
        │
   ┌────▼──────────────────────────────────────┐
   │ ⑤ 对 D₁、D₂ 各自递归回 ①                  │
   │   停止条件任一触达即作叶子（多数类标签）：│
   │     ─ 子集纯净（Gini = 0）                │
   │     ─ 没特征可切                          │
   │     ─ 触达 max_depth / min_samples_leaf   │
   └───────────────────────────────────────────┘
```

**关键三点**：

1. **候选分裂点不是随便选**——每种特征类型有自己的生成规则（见 ① 三条）
2. **"特征 × 分裂点"是同一个搜索空间**——不是先挑特征再挑分裂点，而是**所有 (特征, 分裂点) 组合一起 PK**，谁 Gini_index 最小谁赢
3. **递归时同特征可重复使用**——尤其连续特征（年收入），每层在不同区间找最优阈值

> 三种特征类型的候选生成 + 三特征 PK 的完整算例在 [`03-计算举例.md`](./03-计算举例.md) 跑了一遍贷款数据。

**一句话装进脑子**：**枚举所有 (特征, 分裂点) → 算 Gini_index → 取最小者切 → 子集递归。**

### 越小越优：和 ID3 / C4.5 反方向

这是新手最容易踩的坑——**三个算法的方向不一样**：

| 算法 | 准则 | 方向 |
|---|---|---|
| ID3 | 信息增益 information gain | 越**大**越好（熵降得多） |
| C4.5 | 信息增益率 gain ratio | 越**大**越好（修偏后熵降得多） |
| **CART** | 基尼指数 Gini index | 越**小**越好（切完不纯度低） |

直觉对得上：增益是"切前减切后"，**降得多就是好**；基尼指数是"切完后剩多少不纯度"，**剩得少就是好**。算什么、看什么、方向就反过来。

### 为什么 Gini 比熵快

工程上 CART 选基尼不光是数学口味，还有个硬理由——**没有 $\log$**：

```
熵                          基尼值
H = -∑ p log p              Gini = 1 - ∑ p²

 一个 log 调用              一个乘法
 通常用查表/泰勒展开         CPU 一个指令
```

每次特征筛选都要在所有分裂点上把这个公式算一遍。大数据集 / 多特征下，砍掉 $\log$ 累计能省一大块时间。两者效果接近——基尼值可以视作熵的一阶近似（$-\log p \approx 1 - p$ 当 $p$ 接近 1 时）——所以工业界换便宜的那个完全划算。

### 同名澄清：经济学的"基尼系数"

底稿 Notes 提了一句"**基尼系数：社会财富平均程度**"——这是个善意的同名提醒，但**两个 Gini 不是同一个数**，搞混很尴尬：

| 维度 | ML 的基尼指数 Gini index | 经济学的基尼系数 Gini coefficient |
|---|---|---|
| 衡量 | 数据集类别**不纯度** | 收入 / 财富**不平等度** |
| 公式 | $1 - \sum p_k^2$ | 洛伦兹曲线下面积比（积分） |
| 取值 | $[0, 1-1/K]$ | $[0, 1]$ |
| 越大 | 越乱 | 越不平等 |
| 提出人 | 共用名字（Corrado Gini 1912） | 同一人 1912 论文 |

**共同思想**：两者都问"随机抽两个，差多少？"——ML 抽两个样本看类别是否同；经济学抽两个人看收入是否同。**思路同源，量纲不同**。教材作者拿"社会财富平均"做类比是想让你抓"不均匀"这个直觉，**别记成同一个公式**就行。

### 边界对照表（不用算，记结论）

| 分布形态 | Gini |
|---|---|
| 一类占满（任意 $K$） | **0** |
| 2 类均匀（公平硬币） | 0.5 |
| 3 类均匀 | 0.667 |
| 4 类均匀 | 0.75 |
| $K$ 类均匀（一般式） | $1 - 1/K$ |

记忆：**$K$ 类均匀 → Gini = $1 - 1/K$**，永远小于 1，但永远逼近不到 1。

### 反例：只看 Gini(D₁)、Gini(D₂) 不看权重

新手常见错误——切完直接挑"哪一支基尼最小"当依据：

```
错误判断                        正确判断
─────────────                   ─────────────
"D₁ Gini=0 真纯！选 A"         "Gini_index = |D₁|/|D|·0 + |D₂|/|D|·0.49"
                                 = 0.343
                                 还要和其他特征的 Gini_index 比

只看局部 → 漏算大子集污染        加权后比 → 公平裁判
```

**永远算 Gini_index 再比**，单看一支没意义。

### 术语版

| 故事里的元素 | 术语 |
|---|---|
| 抓两次抓到不同色的概率 | 基尼值 Gini value（也叫基尼不纯度 Gini impurity） |
| 切完后两支基尼值的加权和 | 基尼指数 Gini index |
| 类别占比平方之和 | 同色概率 $\sum p_k^2$ |
| 切出的左右子集 | $D_1, D_2$ |
| 子集样本数 | 基数 cardinality $\lvert D_i \rvert$ |
| CART 的分裂准则 | 选 Gini_index 最小的特征 / 分裂点 |

### 关键启示

**基尼指数 = 闭眼抓两次抓到不同色的概率，按子集大小加权——越小越优，没 log 跑得快。**
