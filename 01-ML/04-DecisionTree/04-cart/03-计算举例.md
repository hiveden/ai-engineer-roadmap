### PPT

> Slide 48 · CART 分类树 · 案例

**已知**：是否拖欠贷款数据。
**需求**：计算各特征的基尼指数，选择最优分裂点

```
ID  是否有房  婚姻状况   年收入   拖欠贷款
────────────────────────────────────────
 1     有     single     125k       no
 2     无     married    100k       no
 3     无     single      70k       no
 4     有     married    120k       no
 5     无     divorced    95k      yes
 6     无     married     60k       no
 7     有     divorced   220k       no
 8     无     single      85k      yes
 9     无     married     75k       no
10     无     single      90k      yes
────────────────────────────────────────
分布：no ███████ 7   yes ███ 3
```

共 10 人：**7 no : 3 yes**

> Slide 49 · CART 决策树 · 第一个特征：是否有房

Gini 值：100% - （类别占比的平方之和）

**第一个特征：是否有房**（有房 / 无房）

- **有房子的基尼值**：有房子有 1、4、7 共计 3 个样本，对应 3 个 no、0 个 yes（共 3 个）
- **无房子的基尼值**：无房子有 2、3、5、6、8、9、10 共 7 个样本，对应 4 个 no、3 个 yes（共 7 个）

```
              D（10 人，7 no / 3 yes）
                       │
                  按"是否有房"切
              ┌────────┴────────┐
              有房              无房
            (1, 4, 7)      (2,3,5,6,8,9,10)
            3 样本           7 样本
            3 no / 0 yes     4 no / 3 yes
            ████████ 100%    ████ 57% no
            ·          0%    ███  43% yes
              │                │
        Gini = 1 - 1² = 0      Gini = 1 - (4/7)² - (3/7)²
                                    ≈ 0.4898
                       ↓
        Gini_index = 3/10 × 0  +  7/10 × 0.4898
                   = 0.343
```

**基尼系数**：第一部分样本占了总样本的 3/10，第二部分样本占了总样本的 7/10。

> Slide 50-51 · 第二个特征：婚姻状况

```
婚姻状况：3 个取值
─────────────────────────
single    1, 3, 8, 10  → 4 人 (1 no / 3 yes)
married   2, 4, 6, 9   → 4 人 (4 no / 0 yes)
divorced  5, 7         → 2 人 (1 no / 1 yes)
─────────────────────────
3 类不能直接二分 → 枚举 3 种"一对其他"组合
```

**第二个特征：婚姻状况**（多分类：三个类别，但 CART 树每次分裂只有 2 个分支）

```
3 种二分组合
─────────────────────────────────────
① {married}   |  {single, divorced}
② {single}    |  {married, divorced}
③ {divorced}  |  {single, married}
─────────────────────────────────────
分别算 Gini_index，挑最小者作该特征的分裂点
```

1. 计算 {married} 和 {single, divorced} 情况下的基尼指数
   - 结婚的基尼值：有 2、4、6、9 共 4 个样本，并且对应目标值全部为 no
   - 不结婚的基尼值：有 1、3、5、7、8、10 共 6 个样本，并且对应 3 个 no，3 个 yes

   ```
                D（10 人，7 no / 3 yes）
                       │
              按"婚姻 = married?"切
              ┌────────┴────────┐
              married        其他
            (2, 4, 6, 9)   (1,3,5,7,8,10)
            4 no / 0 yes   3 no / 3 yes
            Gini = 0       Gini = 1 - .5² - .5² = 0.5
                       ↓
        Gini_index = 4/10 × 0 + 6/10 × 0.5 = 0.3
   ```

> **Notes**：各个子分类之间相互计算，最终确定可能的分裂点，而不是以 single 作为分裂点，也不是以 divorced 作为分裂点。

2. 计算 {single} | {married, divorced} 情况下基尼指数

   ```
                D（10 人，7 no / 3 yes）
                       │
              按"婚姻 = single?"切
              ┌────────┴────────┐
              single         其他
            (1, 3, 8, 10)  (2,4,5,6,7,9)
            1 no / 3 yes   5 no / 1 yes (married+divorced)
            Gini ≈ 0.375   Gini ≈ 0.278
                       ↓
        Gini_index = 4/10 × 0.375 + 6/10 × 0.278 ≈ 0.367
   ```

3. 计算 {divorced} | {single, married} 情况下基尼指数

   ```
                D（10 人，7 no / 3 yes）
                       │
              按"婚姻 = divorced?"切
              ┌────────┴────────┐
              divorced       其他
            (5, 7)         (1,2,3,4,6,8,9,10)
            1 no / 1 yes   6 no / 2 yes
            Gini = 0.5     Gini ≈ 0.375
                       ↓
        Gini_index = 2/10 × 0.5 + 8/10 × 0.375 ≈ 0.4
   ```

```
3 种二分组合对比（婚姻状况特征）
────────────────────────────────────
① {married}   | 其他   |█████      | 0.3   ← 最小，选！
② {single}    | 其他   |██████     | 0.367
③ {divorced}  | 其他   |███████    | 0.4
────────────────────────────────────
```

**最终**：该特征的基尼值为 **0.3**，并且预选分裂点为：{married} 和 {single, divorced}

> Slide 52-54 · 第三个特征：年收入连续值切分

```
年收入升序排列（10 个样本）
────────────────────────────────────────
60   70   75   85   90   95  100  120  125  220
 │    │    │    │    │    │    │    │    │    │
 ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼
no   no   no  yes  yes  yes  no   no   no   no
────────────────────────────────────────
相邻中点作候选分裂点：
   65  72.5  80  87.5  92.5  97.5  110  122.5  172.5
```

**年收入**（连续值，如何切分？CART 树每次分裂只有 2 个分支）

1. 先将数值型属性**升序排列**，以**相邻中间值**作为待确定分裂点
2. 以年收入 65 将样本分为两部分，计算基尼指数

   ```
                D（10 人，7 no / 3 yes）
                       │
              按"年收入 < 65?"切
              ┌────────┴────────┐
              是                否
            (1 样本)         (9 样本)
            1 no / 0 yes     6 no / 3 yes
            Gini = 0         Gini = 1 - (6/9)² - (3/9)² ≈ 0.444
                       ↓
        Gini_index = 1/10 × 0 + 9/10 × 0.444 ≈ 0.4
   ```

3. 以此类推计算所有分割点的基尼指数，最小的基尼指数为 **0.3**，此时分割点为 **97.5**

> Slide 55 · 第 1 轮 + 第 2 轮

```
第 1 轮：D（10 人，7 no / 3 yes）
        ┌────────────┬────────────┐
   是否有房      婚姻=married?    年收入<97.5?
   |███████|     |██████ |        |██████ |
```

**第 1 轮结果**：

- 以是否有房作为分裂点的基尼指数为：**0.343**（不恰当的切分）
- 以婚姻状况为分裂特征、以 married 作为分裂点的基尼指数为：**0.3**（选中）
- 以年收入作为分裂特征、以 97.5 作为分裂点的基尼指数为：**0.3**

**第 2 轮**：

1. 样本 2、4、6、9 样本的类别都是 no，已经达到最大纯度 → 该节点不需要再继续分裂
2. 样本 1、3、5、7、8、10 样本中仍然包含 4 个 no，2 个 yes → 该节点并未达到要求的纯度，需要继续划分
3. 右子树的数据集变为：1、3、5、7、8、10，在该数据集中计算不同特征的基尼指数，选择基尼指数最小的特征继续分裂

**训练过程**：

1. 重复上述过程，直到构建完成整个决策树
2. 为每个叶节点打上标签（占多数的类别）

注意：**特征可以在多层重复使用**

### 笔记

> 【理解】基尼指数计算举例

**贷款数据集**（10 样本，3 特征：是否有房 / 婚姻状况 / 年收入）。

**1. 是否有房**（二值，天然二分）：

- 有房（3 样本，全 no）：$\text{Gini} = 1 - 0^2 - 1^2 = 0$
- 无房（7 样本，4 no / 3 yes）：$\text{Gini} = 1 - (3/7)^2 - (4/7)^2 = 0.4898$
- 加权：$\frac{3}{10} \times 0 + \frac{7}{10} \times 0.4898 = 0.343$

**2. 婚姻状况**（三值，需枚举所有二分组合）：

- $\{married\} \mid \{single, divorced\}$：Gini = **0.3**
- $\{single\} \mid \{married, divorced\}$：Gini = 0.367
- $\{divorced\} \mid \{single, married\}$：Gini = 0.4
- 选最小的 0.3，分裂点 = $\{married\}$ vs 其他

**3. 年收入**（连续，排序后取相邻中点作候选）：

- 候选点：65、72.5、80、87.5、92.5、97.5、110、122.5、172.5
- 全扫一遍找最小：97.5 处取得 **0.3**

**最终选择**：三特征最小 Gini 分别 0.343 / 0.3 / 0.3 → 0.3 平局，随机选（如婚姻状况）作根节点。注意：CART 中**同一特征可在多层重复使用**（例如年收入这种连续特征可在不同深度多次切分）。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉锚：CART 处理三种特征的三招

CART 强制二叉，遇到三种特征类型有三种切法：

```
特征类型 feature type              处理招式                          示例
─────────────────────              ─────────────────                 ─────────
二值 binary（是否有房）            天然二分，直接切                   有房 | 无房
多值 multiclass（婚姻状况）        枚举所有"一对其余"组合             {married} | {single, divorced}
连续 continuous（年收入）          排序取相邻中点作候选阈值           收入 < 97.5 | 收入 ≥ 97.5
```

本节跑一遍贷款数据 10 样本 7 no / 3 yes，把三招都过一遍。

### 招式一：二值特征——是否有房（天然二分）

3 有房全 no，7 无房 4 no / 3 yes：

```
有房 (3 样本)              无房 (7 样本)
─────────                  ─────────
no  ████████ 100%          no  ████ 57%
yes ·         0%           yes ███ 43%
Gini = 1 - 1² = 0          Gini = 1 - (4/7)² - (3/7)²
                                ≈ 1 - 0.327 - 0.184
                                ≈ 0.4898
```

**Gini_index** = $\frac{3}{10} \times 0 + \frac{7}{10} \times 0.4898 \approx \mathbf{0.343}$

注意：左支 Gini = 0（"全 no"看起来完美），但右支 7 样本 Gini 接近极限 0.5，加权拉高了总分。**单看一支没用，永远算加权**。

### 招式二：多值特征——婚姻状况（枚举二分）

`single / married / divorced` 3 个值。CART 不能一次开 3 支，得**枚举所有"一对其余"二分组合**：

| 组合 | 左支 | 右支 | Gini_index |
|---|---|---|---|
| ① **{married}** \| {single, divorced} | 4 样本 4 no / 0 yes，Gini = 0 | 6 样本 3 no / 3 yes，Gini = 0.5 | **0.3** ✓ |
| ② {single} \| {married, divorced} | 4 样本 1 no / 3 yes，Gini ≈ 0.375 | 6 样本 5 no / 1 yes，Gini ≈ 0.278 | 0.367 |
| ③ {divorced} \| {single, married} | 2 样本 1 no / 1 yes，Gini = 0.5 | 8 样本 6 no / 2 yes，Gini = 0.375 | 0.4 |

**该特征最优分裂点 = {married} vs 其他，Gini_index = 0.3**。

为什么不能"按原值开 3 支"？CART 强制二叉的工程理由：**多叉树深度不可控、子集越切越小、不便剪枝**。代价是多值特征要枚举 $2^{n-1} - 1$ 种二分（这里 3 值刚好 3 种）——值多就贵。

### 招式三：连续特征——年收入（排序找中点）

10 个收入排序：

```
60  70  75  85  90  95  100  120  125  220
no  no  no  yes yes yes no   no   no   no
```

候选分裂点 = **相邻中点**（每两个相邻值取均值）：

```
65  72.5  80  87.5  92.5  97.5  110  122.5  172.5
                          ↑
                    最优阈值在这
```

**为什么取相邻中点**？因为分裂条件 `收入 < 阈值`：阈值在两个相邻值之间任何位置，切出来的左右子集都一样——取中点只是个稳定的代表。

逐个候选都算 Gini_index。先把教材点名要算的两个跑一遍：

**阈值 65**（Slide 52）：

```
按"收入 < 65?"切
  是（1 样本：60）          否（9 样本：70-220）
  no=1, yes=0               no=6, yes=3
  Gini = 1 - 1² = 0         Gini = 1 - (6/9)² - (3/9)² ≈ 0.444
                  ↓
  Gini_index = 1/10 × 0 + 9/10 × 0.444 ≈ 0.4
```

**阈值 72.5**（Slide 53 留的"请同学们算一算"）：

```
按"收入 < 72.5?"切
  是（2 样本：60, 70）       否（8 样本：75-220）
  no=2, yes=0                no=5, yes=3
  Gini = 1 - 1² = 0          Gini = 1 - (5/8)² - (3/8)² ≈ 0.469
                  ↓
  Gini_index = 2/10 × 0 + 8/10 × 0.469 ≈ 0.375
```

**节奏感**：从 65（0.4）到 72.5（0.375）——把第二个 no 切到左支后，左支仍纯净，右支 yes 占比上升一点点，整体降了。这就是"扫候选阈值"的过程——一路扫下去看哪个最低。

**全部候选汇总**（教材 Slide 54）：

| 阈值 | 左支样本 | 右支样本 | Gini_index |
|---|---|---|---|
| 65 | 1（1 no / 0 yes） | 9（6 no / 3 yes） | 0.4 |
| 72.5 | 2（2 no / 0 yes） | 8（5 no / 3 yes） | 0.375 |
| 80 | 3（3 no / 0 yes） | 7（4 no / 3 yes） | 0.343 |
| 87.5 | 4（3 no / 1 yes） | 6（4 no / 2 yes） | 约 0.417 |
| 92.5 | 5（3 no / 2 yes） | 5（4 no / 1 yes） | 0.4 |
| **97.5** | 6（3 no / 3 yes，前 6 个） | 4（4 no / 0 yes，后 4 个） | **0.3** ✓ |
| 110 | 7（4 no / 3 yes） | 3（3 no / 0 yes） | 0.343 |
| 122.5 | 8（5 no / 3 yes） | 2（2 no / 0 yes） | 0.375 |
| 172.5 | 9（6 no / 3 yes） | 1（1 no / 0 yes） | 0.4 |

**该特征最优分裂点 = 收入 < 97.5，Gini_index = 0.3**。

> 注意 97.5 这个点的妙处：左支 60-95 包含全部 3 个 yes + 3 个 no，右支 100-220 全 no（4 个）——右支被切纯了，左支虽然 50/50 但只占 6/10 权重。两端阈值（65 / 172.5）只能切出 1 样本子集，剩下 9 个还是混的，所以两端高、中间低，最低点正好压在 yes 簇的右边界。

### 第 1 轮：三特征 PK

```
特征             分裂点                Gini_index
─────────────    ────────────────      ─────────
是否有房          有房 | 无房            0.343
婚姻状况          {married} | 其他      0.3   ← 平局之一
年收入            < 97.5 | ≥ 97.5      0.3   ← 平局之一
```

0.3 平局——sklearn 默认按特征顺序选第一个，或随机选。教材选**婚姻状况**作根节点：

```
                 根：婚姻 = married?
                 ┌──────┴──────┐
              是（4 人）        否（6 人）
              4 no / 0 yes      4 no / 2 yes
              Gini = 0          Gini ≈ 0.444
              ●叶子（标 no）    ○继续切
```

### 第 2 轮：右支递归

左支 4 人全 no——已纯净，**打叶子标签 no** 直接收工。

右支 6 人（样本 1, 3, 5, 7, 8, 10）4 no / 2 yes，没纯——**在这 6 样本上重新算**所有特征的 Gini_index：

- 是否有房：在子集上重算
- 婚姻状况：剩下两值（single / divorced），二分组合也重新枚举
- **年收入：可以再用一次**——在子集 60、70、85、90、95、220 上重新排序取中点

> **关键**：CART 中**同一特征可以在多层重复使用**——尤其连续特征，每层在不同区间找最优阈值。这跟 ID3 / C4.5 离散特征"用过就废"的多叉模式不同。

递归直到停止条件：
1. 子集纯净（Gini = 0）
2. 没特征可切了
3. 触达预设的 `max_depth` / `min_samples_leaf` 等剪枝阈值

每个叶子最后**打上多数类的标签**。

### 反例：不加权直接挑"哪一支最纯"

```
错误判断                              正确判断
─────────────                         ─────────────
"有房支 Gini = 0 完美！选这个"        Gini_index 加权 = 0.343
                                       （右支 0.4898 拖累）

只看局部 → 漏算大子集污染              三特征加权后 PK → 婚姻 / 收入并列 0.3
```

本节大半篇幅都在算加权——原因在此。

### 实战 hint：sklearn 怎么对应

适用 sklearn ≥ 1.0：

| 教材步骤 | sklearn `DecisionTreeClassifier` |
|---|---|
| 用 Gini 算分裂 | `criterion='gini'`（默认） |
| 多值二分枚举 | 内部自动枚举 |
| 连续中点 | 内部自动处理 |
| 第 2 轮递归 | 自动递归到 `max_depth` / `min_samples_split` |
| 打叶子标签 | `predict()` 取叶子多数类 |

### 术语版

| 故事里的元素 | 术语 |
|---|---|
| 二值特征直接二分 | 二元分裂 binary split |
| 多值"一对其余"枚举 | 子集枚举 subset enumeration（CART 二叉化必经） |
| 连续值取相邻中点 | 阈值二分 threshold split |
| 切完后两支基尼值的加权和 | 基尼指数 Gini index |
| 子集已全同类 | 纯净 / 叶节点 leaf node |
| 同特征多层重复用 | 特征可重复使用 feature reuse |

### 关键启示

**CART 三招——二值直分、多值枚举二分、连续找最优中点；三特征 Gini_index 加权 PK，最小者胜，递归直到纯净。**
