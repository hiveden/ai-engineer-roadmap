### PPT

> Slide 56 · 三种分类树的对比

| 名称 | 提出时间 | 分支方式 | 特点 |
|---|---|---|---|
| ID3 | 1975 | 信息增益 | 1. ID3 只能对离散属性的数据集构成决策树；2. 倾向于选择取值较多的属性 |
| C4.5 | 1993 | 信息增益率 | 1. 缓解了 ID3 分支过程中总喜欢偏向选择值较多的属性；2. 可处理连续数值型属性，也增加了对缺失值的处理方法；3. 只适合于能够驻留于内存的数据集，大数据集无能为力 |
| CART | 1984 | 基尼指数 | 1. 可以进行分类和回归，可处理离散属性，也可以处理连续属性；2. 采用基尼指数，计算量减小；3. 一定是二叉树 |

> Slide 57 · 总结

1. **CART 决策树的作用**？分类和回归
2. **基尼指数的作用**？特征筛选，基尼指数值越**小**，则说明优先选择该特征

- C）基尼指数的计算使用到了自然对数
- D）CART 算法不能用于回归场景

### 笔记

> 【知道】比较

| 算法 | 年份 | 准则 | 树形 | 特点 |
|---|---|---|---|---|
| ID3 | 1975 | 信息增益 | 多叉 | 仅离散；偏向多值特征 |
| C4.5 | 1993 | 信息增益率 | 多叉 | 修正多值偏好；支持连续 + 缺失；不适合大数据 |

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉锚：三代决策树的演化故事

三个算法都是 **Quinlan / Breiman 一代代修补出来的**，每一代解决前一代的痛点。把它当一条时间线来记最简单：

```
时间线
─────────────────────────────────────────────────────────────
1975        1984              1993
ID3        CART              C4.5
─────       ─────             ─────
信息增益    基尼指数          信息增益率
多叉        严格二叉          多叉
仅离散      离散+连续         离散+连续
仅分类      分类+回归         仅分类
↓痛点       ↓改进             ↓改进
偏向多值    强制二叉          修偏 + 缺失值
仅分类      工程级            后剪枝 PEP
```

**注意**：CART 1984 比 C4.5 1993 还早出来——它不是 ID3 的"直接继承者"，而是 Breiman 团队走的另一条路；C4.5 是 Quinlan 在 ID3 基础上修补的"亲儿子"。**最后赢的是另一条路上的 CART**——sklearn / XGBoost / LightGBM 全部基于 CART。

### 三视角横向对比

```
                  ID3              C4.5             CART
                  (1975)           (1993)           (1984)
─────────────     ───────────      ───────────      ───────────
分裂准则           信息增益          信息增益率        基尼指数 / 平方误差
方向               越大越好          越大越好          越小越好
树形               多叉              多叉              严格二叉
连续特征           ✗ 不支持          ✓ 排序找中点      ✓ 排序找中点
缺失值             ✗                ✓ 概率分配        ✓ 替代分裂
任务               仅分类            仅分类            分类 + 回归
计算开销           中（要算 log）    高（log + IV）    低（无 log）
内存占用           全量驻留          全量驻留          全量驻留
后剪枝             ✗                ✓ PEP             ✓ CCP（代价复杂度）
sklearn 实现       ✗                ✗                 ✓ 默认
工业应用           教学              历史              主流（含集成模型基学习器）
```

### 各自的痛点 → 下代怎么修

#### ID3：偏向多值特征

```
反例：把"用户 ID"列丢进去
D（6 样本）：A A A B B B
按 ID 切    →  6 个叶子，每叶 1 样本
H(D|ID) = 0   →  g(ID) = 1.0（满分）
但新数据 ID 树里压根没见过 → 完全不能泛化
```

**根因**：取值多 → 切完每堆样本少 → 天然纯 → 增益虚高。
**C4.5 的解**：除以**分裂信息 IV(A)**——特征 $A$ 自身的熵，分支多时大 → 把虚高罚回去。

#### C4.5：仍是多叉 + 内存吃紧

C4.5 修了 ID3 的偏差，但留下两个工程坑：

1. **多叉树**：每个离散特征值开一支——树形不规整、深度不可控、剪枝复杂
2. **全量驻留**：每次算增益率都要扫整张表——大数据集吃不消

**CART 的解**：

1. **强制二叉**：多值特征枚举"一对其余"二分，深度可控、剪枝好做
2. **基尼指数**：没 $\log$，每次分裂判定更便宜

#### CART 自己的取舍

CART 也不是完美的：

- **多值特征枚举贵**：值有 $n$ 个 → 要枚举 $2^{n-1} - 1$ 种二分组合（10 值时 = 511 种）
- **基尼相比熵略不敏感**：极端分布下数值差距小，但实践效果接近
- **二叉树更深**：多值特征分裂一次只能把一个值分出去，深度比多叉大

**取舍合理**：工业场景多值特征通常会预编码（label encoding / target encoding），且二叉树深度的代价能用剪枝 / `max_depth` 控住——换来的是工程级的可调控性。

### 为什么 sklearn 选 CART

sklearn `DecisionTreeClassifier` / `DecisionTreeRegressor` 内部**只实现 CART**——不提供 ID3 / C4.5。原因：

| 维度 | CART 优势 |
|---|---|
| **统一接口**（分类+回归） | `criterion='gini'` 切分类，`criterion='squared_error'` 切回归，同一棵树代码 |
| **二叉树结构** | 实现简单、深度可控、剪枝（CCP）有现成理论 |
| **集成模型基础** | RandomForest / GradientBoosting / XGBoost / LightGBM 全部基于二叉 CART |
| **工程效率** | 无 $\log$、剪枝接口稳定 |

**实战提示**：

- 想用 ID3 / C4.5：R 语言有 `C50` 包；Python 没主流实现，自己写或用 `chefboost` 之类小包
- 99% 场景下默认 CART 即可
- `criterion='entropy'` 是 sklearn 在 CART 框架下用熵做分裂——**树形仍是二叉**，不是真的 ID3 / C4.5

### 选择题复盘（Slide 58）

**Q：下列关于 CART 树的说法正确的是？**

- A）基尼指数值越大，则说明优先选择该特征 → ❌ **越小越优**（与 ID3/C4.5 反方向）
- B）基尼指数是 CART 算法中用于划分属性的重要依据 → ✓ **正确答案**
- C）基尼指数的计算使用到了自然对数 → ❌ Gini 用平方和，**不用 log**（这正是它比熵快的原因）
- D）CART 算法不能用于回归场景 → ❌ CART 名字里就有 R = Regression，**分类回归通吃**

四个选项正好把 CART 四个核心特点的常见误解全踩一遍——**越小越优 / 不用 log / 二叉 / 通吃**。

### 选型决策表

实际项目挑算法的一句话指引：

| 场景 | 选什么 |
|---|---|
| Python 单棵决策树 | sklearn CART（默认） |
| Python 集成模型 | XGBoost / LightGBM / CatBoost（基学习器都是 CART） |
| R 项目纯分类 | `rpart`（CART） / `C50`（C4.5） |
| 教学 / 解释 ID3 思想 | 自己写或 `chefboost` |
| 大数据流式训练 | LightGBM / Spark MLlib（都是 CART 变体） |

**99% 场景的默认答案：CART**——所以本章把它讲透。

### 术语版

| 故事里的元素 | 术语 |
|---|---|
| Quinlan 1975 | ID3 = Iterative Dichotomiser 3 |
| Quinlan 1993 | C4.5 = Classifier 4.5 |
| Breiman 1984 | CART = Classification And Regression Tree |
| ID3 准则 | 信息增益 information gain |
| C4.5 准则 | 信息增益率 gain ratio |
| CART 分类准则 | 基尼指数 Gini index |
| CART 回归准则 | 平方误差 squared error |
| sklearn 默认实现 | `DecisionTreeClassifier` / `DecisionTreeRegressor` |
| CART 后剪枝方法 | 代价复杂度剪枝 CCP（Cost Complexity Pruning） |
| C4.5 后剪枝方法 | 悲观误差估计 PEP（Pessimistic Error Pruning） |

### 关键启示

**ID3 偏多值 → C4.5 修偏但仍多叉 → CART 二叉 + 基尼通吃；工业首选 CART，sklearn 默认就是它。**
