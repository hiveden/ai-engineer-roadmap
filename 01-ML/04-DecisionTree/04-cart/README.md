#### Slide 46 · CART 决策树（Classification and Regression Tree）

CART 模型是一种决策树模型，它即可以用于分类，也可以用于回归。

- CART 分类生成树采用的**基尼指数最小化**策略
- CART 回归树使用**平方误差最小化**策略

#### Slide 47 · CART 分类树 · 基尼值

```
随机抽 2 个样本，类别"不一致"的概率
─────────────────────────────────────
样本袋 D：● ● ● ● ○ ○ ○ ▲

  ┌── 抽 1 个 ──┐    ┌── 再抽 1 个 ──┐
  │             │    │               │
  ▼             ▼    ▼               ▼
 颜色 X       记下     颜色 Y      对比
                                    │
                  X ≠ Y 的概率 = Gini(D)
─────────────────────────────────────
Gini 越小 → 越容易抽到同色 → 越纯
```

**基尼值 Gini(D)**：从数据集 D 中随机抽取两个样本，其类别标记不一致的概率。故，**Gini(D) 值越小，数据集 D 的纯度越高**。

可以理解为：总概率 100% - 2 次抽取到相同类别的"联合概率"。

即：100% - （类别占比的平方之和）

```
3 类示例：纯净 vs 偏混
─────────────────              ─────────────────
A ████████████ 100%            A ██████ 50%
B ·             0%             B ███ 25%
C ·             0%             C ███ 25%
─────────────────              ─────────────────
Gini = 1 - 1² = 0              Gini = 1 - (.5²+.25²+.25²)
                                    = 1 - 0.375 = 0.625
（最纯）                        （较杂）
```

**例**：样本包含 3 种颜色：

- $1 - 100\%^2 = 0$ （Gini 越小，数据纯度越高）
- $1 - (50\%^2 + 25\%^2 + 25\%^2) = 0.625$ （数据纯度低！）

**基尼指数 Gini_index(D)**：选择使划分后基尼系数最小的属性作为最优化分属性。

```
特征 A 把 D 切成 D₁、D₂
─────────────────────────────
        D
        │
    用特征 A 切
        │
   ┌────┴────┐
   D₁        D₂
   Gini(D₁)  Gini(D₂)
        ↓
Gini_index(D, A) = |D₁|/|D| × Gini(D₁)
                 + |D₂|/|D| × Gini(D₂)
                       ↓
              枚举所有特征 / 分裂点
              选 Gini_index 最小者
```

**注意**：

- 信息增益（ID3）、信息增益率（C4.5）值越**大**，则说明优先选择该特征
- 基尼指数值越**小**（cart），则说明优先选择该特征

> **Notes**：基尼值用减法理解：100% - 两次抽到相同值的概率。基尼系数：社会财富平均程度。

#### Slide 49 · CART 决策树 · 第一个特征：是否有房

Gini 值：100% - （类别占比的平方之和）

**第一个特征：是否有房**（有房 / 无房）

- **有房子的基尼值**：有房子有 1、4、7 共计 3 个样本，对应 3 个 no、0 个 yes（共 3 个）
- **无房子的基尼值**：无房子有 2、3、5、6、8、9、10 共 7 个样本，对应 4 个 no、3 个 yes（共 7 个）

```
              D（10 人，7 no / 3 yes）
                       │
                  按"是否有房"切
              ┌────────┴────────┐
              有房              无房
            (1, 4, 7)      (2,3,5,6,8,9,10)
            3 样本           7 样本
            3 no / 0 yes     4 no / 3 yes
            ████████ 100%    ████ 57% no
            ·          0%    ███  43% yes
              │                │
        Gini = 1 - 1² = 0      Gini = 1 - (4/7)² - (3/7)²
                                    ≈ 0.4898
                       ↓
        Gini_index = 3/10 × 0  +  7/10 × 0.4898
                   = 0.343
```

**基尼系数**：第一部分样本占了总样本的 3/10，第二部分样本占了总样本的 7/10。

#### Slide 50 · CART 决策树原理举例 · 第二个特征：婚姻状况

是否单身 / 结婚 / 非结婚

```

─────────────────────────────────────

② {single}    |  {married, divorced}
③ {divorced}  |  {single, married}
─────────────────────────────────────
分别算 Gini_index，挑最小者作该特征的分裂点
```

1. 计算 {married} 和 {single, divorced} 情况下的基尼指数
   - 结婚的基尼值：有 2、4、6、9 共 4 个样本，并且对应目标值全部为 no
   - 不结婚的基尼值：有 1、3、5、7、8、10 共 6 个样本，并且对应 3 个 no，3 个 yes
   -

   ```
                D（10 人，7 no / 3 yes）
                       │
              按"婚姻 = married?"切
              ┌────────┴────────┐
              married        其他
            (2, 4, 6, 9)   (1,3,5,7,8,10)
            4 no / 0 yes   3 no / 3 yes

   ```

### 笔记

> 04 · CART 决策树

**学习目标**：

1. 理解基尼指数的作用
2. 知道 CART 构建的特征选择方法

> 【知道】CART 树简介

**CART = Classification And Regression Tree**（1984）。**始终是二叉树**，分类回归通吃：

- **分类树**：基尼指数最小化作准则
- **回归树**：平方误差最小化（详见第 6 章）

> 【理解】基尼指数计算方法

**基尼值**（衡量集合纯度）：

$$\text{Gini}(D) = 1 - \sum_{k=1}^{K} \left(\frac{|C_k|}{|D|}\right)^2$$

**基尼指数**（特征 $A$ 把 $D$ 切成 $D_1, D_2$ 的加权基尼值）：

$$\text{Gini\_index}(D, A) = \frac{|D_1|}{|D|} \text{Gini}(D_1) + \frac{|D_2|}{|D|} \text{Gini}(D_2)$$

**选择准则对比**：

| 算法 | 准则 | 方向 |
|---|---|---|
| ID3 | 信息增益 | 越**大**越好 |
| C4.5 | 信息增益率 | 越**大**越好 |
| CART | 基尼指数 | 越**小**越好 |

**为什么 Gini 比熵快**：无 $\log$ 计算，工程上更高效；两者效果接近。

> 【理解】基尼指数计算举例

**贷款数据集**（10 样本，3 特征：是否有房 / 婚姻状况 / 年收入）。

**1. 是否有房**（二值，天然二分）：

- 有房（3 样本，全 no）：$\text{Gini} = 1 - 0^2 - 1^2 = 0$
- 无房（7 样本，4 no / 3 yes）：$\text{Gini} = 1 - (3/7)^2 - (4/7)^2 = 0.4898$
- 加权：$\frac{3}{10} \times 0 + \frac{7}{10} \times 0.4898 = 0.343$

**2. 婚姻状况**（三值，需枚举所有二分组合）：

- $\{married\} \mid \{single, divorced\}$：Gini = **0.3**
- $\{single\} \mid \{married, divorced\}$：Gini = 0.367
- $\{divorced\} \mid \{single, married\}$：Gini = 0.4
- 选最小的 0.3，分裂点 = $\{married\}$ vs 其他

**3. 年收入**（连续，排序后取相邻中点作候选）：

- 候选点：65、72.5、80、87.5、92.5、97.5、110、122.5、172.5
- 例：以 65 切，左 1 样本（no）Gini=0；右 9 样本（6 no / 3 yes），Gini = $\frac{1}{10} \times 0 + \frac{9}{10} \times [1 - (6/9)^2 - (3/9)^2] = 0.4$
- 全扫一遍找最小：97.5 处取得 **0.3**

**最终选择**：三特征最小 Gini 分别 0.343 / 0.3 / 0.3 → 0.3 平局，随机选（如婚姻状况）作根节点。递归切剩余子集，直到叶子纯净。注意：CART 中**同一特征可在多层重复使用**（Slide 55，例如年收入这种连续特征可在不同深度多次切分）。

> 【知道】比较

| 算法 | 年份 | 准则 | 树形 | 特点 |
|---|---|---|---|---|
| ID3 | 1975 | 信息增益 | 多叉 | 仅离散；偏向多值特征 |
| C4.5 | 1993 | 信息增益率 | 多叉 | 修正多值偏好；支持连续 + 缺失；不适合大数据 |
| CART | 1984 | 基尼指数 / 平方误差 | **二叉** | 分类 + 回归；计算量小；工业首选 |

**sklearn 默认实现的就是 CART**（`DecisionTreeClassifier` / `DecisionTreeRegressor`）。
