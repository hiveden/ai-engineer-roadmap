### PPT

> Slide 63 · 案例背景

**1912 年 4 月 15 日**，在她的处女航中，泰坦尼克号在与冰山相撞后沉没，在 2224 名乘客和机组人员中造成 1502 人死亡。

造成海难失事的原因之一是乘客和机组人员没有足够的救生艇。尽管幸存生存有一些运气因素，但有些人比其他人更容易生存，例如妇女，儿童和上流社会。

有了遇难和幸运数据，运用机器学习工具来预测哪些乘客可幸免于悲剧。

**数据情况**：

- 数据集中的特征包括票的类别、是否存活、乘坐班次、年龄、登陆、home.dest、房间、船和性别等
- 乘坐班是指乘客班（1，2，3），是社会经济阶层的代表
- age 数据存在缺失

```
1912.04.15  RMS Titanic  ❄ 撞冰山沉没
─────────────────────────────────────────
  乘客 + 机组：2224       遇难：1502
  存活率：约 32%          女/童/上流：偏高

  数据片段（train.csv）：
  ┌─────────┬────────┬──────┬──────┬───────────┐
  │ Pclass  │ Sex    │ Age  │ ...  │ Survived  │
  ├─────────┼────────┼──────┼──────┼───────────┤
  │   3     │ male   │ 22.0 │ ...  │    0      │
  │   1     │ female │ 38.0 │ ...  │    1      │
  │   3     │ female │ 26.0 │ ...  │    1      │
  │   1     │ female │ 35.0 │ ...  │    1      │
  │   3     │ male   │  NaN │ ...  │    0      │  ← Age 有缺失
  │  ...    │  ...   │ ...  │ ...  │   ...     │
```

### 笔记

> 【实践】案例背景

1912 年泰坦尼克号沉没，2224 人中 1502 人遇难。任务：根据**票类、班次、姓名、年龄、上船港口、房间、性别**等特征，预测乘客是否生还。

**业务侧观察**：妇女、儿童、社会地位高者存活率显著更高 → 决策树天然适合表达这类"分群规则"。

**特征工程要点**：

- 类别特征（Sex、Embarked）用 `pd.get_dummies` 或 `OrdinalEncoder`
- 缺失值（Age、Cabin）填充或丢弃
- 决策树**不需要**标准化

## ━━━━━━━━ 讲解 ━━━━━━━━

### 为什么是泰坦尼克

ML 教学界把它当"Hello World 第二步"——比鸢尾花复杂，比真实业务干净。它一次性给齐了**4 个工程要素**：

| 要素 | 体现 | 你能学到 |
|---|---|---|
| **混合特征类型** | 数值（Age）+ 类别（Sex / Embarked）+ 序数（Pclass） | `pd.get_dummies` / 编码选型 |
| **缺失值** | Age 缺 ~20%、Cabin 缺 ~77% | 填充 vs 丢弃的取舍 |
| **二分类** | Survived ∈ {0, 1} | 准确率 / precision / recall |
| **业务直觉** | "妇女儿童优先" + "头等舱靠近甲板" | 特征工程能讲故事 |

### 业务规则天然像决策树

事故现场的**逃生协议**本身就是一棵决策树：

```
遇难者画像（业务直觉）

         你是女性吗？
          │
     ┌────┴────┐
    是          否
    │          │
  存活率↑    你是儿童吗？
            │
       ┌────┴────┐
      是          否
      │          │
   存活率↑    你是头等舱吗？
              │
          ┌───┴───┐
         是        否
         │        │
       存活率↑   存活率↓↓
```

模型要做的不是发明规则——而是**把这棵树从数据里"长"出来**，并且告诉你每个分裂的纯度（gini）和样本数（samples）。这正是决策树相对其他模型的看家本领：**业务方能直接审。**

**这一节的关键启示**：业务规则像树，决策树就把树从数据里长出来。

### 数据集详解

#### 来源与版本

教学界主要流通**两个版本**，字段不同，别混：

| 版本 | 提供方 | 行数 | 字段数 | 何时用 |
|---|---|---|---|---|
| **Kaggle** `titanic/train.csv` | Kaggle Getting Started Competition | 891 | 12 | **本章用** + 几乎所有教程 |
| **Vanderbilt** `titanic3.csv` | Vanderbilt 大学 Frank Harrell | 1309 | 14 | 老教材 / R 包 |

差别：

- Kaggle 是 Vanderbilt 的子集（少 ~400 行），把 Survived 拆成 train/test（test 不公开 y，逼你提交评估）
- Vanderbilt 多了 `home.dest`（家乡-目的地）/ `boat`（救生艇号）/ `body`（遗体识别号）—— PPT 提到的 "home.dest / 船" 就是这版
- ⚠️ `boat` 字段是**数据泄漏陷阱**：知道上了几号救生艇当然能预测生还。教学里千万别当特征用

> 本章统一用 Kaggle 版，2224 / 1502 这组数字是**整艘船的真实数据**，不是数据集行数（数据集只有 891 + 418 = 1309 行，是部分乘客的记录）。

#### 数据规模

```
Kaggle Titanic
─────────────────────────────────────────
  train.csv   891 行 × 12 列   含 Survived
  test.csv    418 行 × 11 列   不含 Survived（提交评估）
  ─────────
  合计        1309 行          ≈ 真实 2224 人的 59%

  目标分布（train）：
    Survived = 0  ████████████████ 549 (61.6%)  遇难
    Survived = 1  ██████████       342 (38.4%)  生还
    ↑ 不是 1:1，但还算平衡，不需要 SMOTE / class_weight
```

#### 字段全表

| # | 字段 | 类型 | 取值范围 / 示例 | 缺失率 | 业务含义 | 本章用？ |
|---|---|---|---|---|---|---|
| 1 | `PassengerId` | int | 1 - 891 | 0% | 行号，无意义 | ✗ |
| 2 | **`Survived`** | bin 0/1 | 0=died, 1=survived | 0% | **目标 y** | ✓ |
| 3 | **`Pclass`** | 序数 | 1 / 2 / 3 | 0% | 舱位 = 社会经济地位 | ✓ |
| 4 | `Name` | str | "Braund, Mr. Owen Harris" | 0% | 含称谓（Mr/Mrs/Miss/Master） | ✗ |
| 5 | **`Sex`** | cat | male / female | 0% | 性别，最强单特征 | ✓ |
| 6 | **`Age`** | float | 0.42 - 80.0 | **19.9%** | 年龄，需填充 | ✓ |
| 7 | `SibSp` | int | 0 - 8 | 0% | 同船兄弟姐妹+配偶数 | ✗ |
| 8 | `Parch` | int | 0 - 6 | 0% | 同船父母+子女数 | ✗ |
| 9 | `Ticket` | str | "A/5 21171" | 0% | 票号，格式杂乱 | ✗ |
| 10 | `Fare` | float | 0 - 512.33 | 0%（test 缺 1） | 票价，与 Pclass 强相关 | ✗ |
| 11 | `Cabin` | str | "C85" / "B96 B98" | **77.1%** | 房间号，缺太多 | ✗ |
| 12 | `Embarked` | cat | C / Q / S | 0.2% | 登船港口 | ✗ |

> Embarked 三个值：C=Cherbourg（法国瑟堡）/ Q=Queenstown（爱尔兰昆士敦，今 Cobh）/ S=Southampton（英国南安普顿，起航港）。

**这一节的关键启示**：数据看清楚再上模型，缺失和泄漏比算法更致命。

#### 业务侧 EDA 速看

不用跑代码也能预判的几个分群差异——决策树会"自己发现"它们：

```
存活率拆分（PPT "妇女 / 儿童 / 上流" 三结论的数据印证）

         整体：38%
    ┌────────┴─────────┐
   女：74%             男：19%
    │                   │
  ┌─┴─┐               ┌─┴─┐
  1 等：97%            ≤16 岁：59%
  2 等：92%            >16 岁：17%
  3 等：50%

→ 性别 > 年龄 > 舱位（按区分度）
→ 决策树第一刀大概率切 Sex
```

### 选 3 个特征的理由

PPT 的 demo 只挑了 `Pclass / Age / Sex` 三个：

- **够强**：单凭这 3 列已经能跑到 ~78% 准确率
- **够杂**：覆盖序数 / 数值 / 类别三种类型，把特征工程的坑全踩一遍
- **够短**：决策树可视化能塞进一张图，业务方一眼看懂

剩下的字段（Fare / Embarked / SibSp 等）留给后续做特征工程练习。
