> Slide 77 · 过拟合：树的切分太多次

```
深树：每个叶子只剩 1 个样本（10 叶子）

         x ≤ 5.5
         ┌─┴─┐
        ...   ...
       /   \  /   \
      ...  ...    ...
     /\  /\  /\  /\  /\
    ●● ●● ●● ●● ●● ●● ●● ●● ●● ●●
    每片叶子 = 1 样本 → 预测值 = 该样本 y
```

训练时，树每个分支只有 1 个样本，回归的预测值（样本均值），只能是它的标签值。
测试时，表现拉胯。

**解决方案**：设置参数

```
预测线穿过每个数据点（过拟合典型）

y
9.5 │                            ●─●─●
9.0 │                       ●────┘
8.5 │                   ●───┘
8.0 │              ┌────┘
7.5 │              │
7.0 │           ●──┘
6.5 │       ●───┘
6.0 │    ●──┘
5.5 │ ●─●─●
    └───────────────────────────────── x

训练误差 = 0   |   测试误差 ↑↑
```

- 决策树模型过于复杂
- 精准穿过了每个数据点
- 过拟合 ！！

参数：

- `min_samples_split`
- `min_samples_leaf`
- `max_depth`

> Slide 78 · 案例代码

```python
import numpy as np
import pandas as pd
from sklearn.tree import DecisionTreeRegressor  # 回归决策树
from sklearn.linear_model import LinearRegression
import matplotlib.pyplot as plt

def dm01_回归分类():
    # 1 准备数据
    x = np.array(list(range(1,11))).reshape(-1, 1)
    y = np.array([5.56, 5.70, 5.91, 6.40, 6.80, 7.05, 8.90, 8.70, 9.00, 9.05])
    print('x-->\n', x)
    print('y-->\n', y)

    # 2 实例化模型 模型训练
    model1 = DecisionTreeRegressor(max_depth=1)
    model2 = DecisionTreeRegressor(max_depth=3)
    model3 = LinearRegression()
    model1.fit(x, y)
    model2.fit(x, y)
    model3.fit(x, y)

    # 3 模型预测 # 等差数组-按照间隔
    x_test = np.arange(0.0, 10.0, 0.01).reshape(-1, 1)
    y_pre1 = model1.predict(x_test)
    y_pre2 = model2.predict(x_test)
    y_pre3 = model3.predict(x_test)
    print(y_pre1.shape, y_pre2.shape, y_pre3.shape)

    # 4 结果可视化
    plt.figure(figsize=(10, 6), dpi=100)
    plt.scatter(x, y, label='data')
    plt.plot(x_test, y_pre1, label='max_depth=1')   # 深度 1 层
    plt.plot(x_test, y_pre2, label='max_depth=3')   # 深度 3 层
    plt.plot(x_test, y_pre3, label='linear')
    plt.xlabel('data')
    plt.ylabel('target')
    plt.title('DecisionTreeRegressor')
    plt.legend()
    plt.show()
```

**vs 线性回归对比实验**（典型可视化）：

```python
import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.tree import DecisionTreeRegressor

X = np.arange(1, 11).reshape(-1, 1)
y = np.array([5.56, 5.7, 5.91, 6.4, 6.8, 7.05, 8.9, 8.7, 9, 9.05])

lr = LinearRegression().fit(X, y)
dt = DecisionTreeRegressor(max_depth=3).fit(X, y)

X_test = np.linspace(0, 10, 1000).reshape(-1, 1)
# lr.predict 是直线；dt.predict 是阶梯曲线
```

**观察结论**：

1. 线性回归 → **直线**
2. 决策树 → **阶梯曲线**（每个叶子一段常数）
3. 树**拟合能力强**，但极易过拟合 → 必须配合 `max_depth` / 剪枝 / 集成（随机森林、GBDT）

### 什么时候用回归树，什么时候用线性回归

拿到一个连续值预测任务，首先要问：**数据是线性的，还是分段的？**

```
线性关系（适合线性回归）        非线性 / 分段关系（适合回归树）
─────────────────────        ────────────────────────────
y 随 x 单调线性增长            y 在某些区间跳变（阶梯状）
无明显分段或折点               数据明显存在若干"簇"
需要斜率有业务意义             只需知道"这段预测什么值"
```

三个选回归树的实际理由：

1. **非线性**：关系是折线 / 分段 / 跳变的，线性回归假设成立不了
2. **阶梯特性**：业务天然分段（按地段定价 / 按用量梯度计费 / 生物生长阶段）
3. **易解释**：每条预测路径是几个 if-else，产品 / 运营可以读懂规则

**注意**：这 10 样本的案例数据本身确实有轻微线性趋势，用线性回归也不是不能——这里的目的是演示两者的**形态差异**，不是说线性回归错了。

### sklearn API 速查

```python
from sklearn.tree import DecisionTreeRegressor

reg = DecisionTreeRegressor(
    criterion='squared_error',   # 分裂准则：平方损失（sklearn ≥1.0 改名，旧版叫 'mse'）
    max_depth=None,              # 树最大深度，None = 不限
    min_samples_split=2,         # 节点分裂需要的最少样本数
    min_samples_leaf=1,          # 叶子节点最少样本数
    random_state=None,           # 随机种子（tie-breaking 用）
)

reg.fit(X_train, y_train)        # X_train shape: (n_samples, n_features)
reg.predict(X_test)              # 返回 shape (n_samples,) 的连续值
```

**关键参数一览**：

| 参数 | 默认值 | 作用 | 调参起点 |
|---|---|---|---|
| `criterion` | `'squared_error'` | 分裂准则，回归树固定用平方损失 | 不动 |
| `max_depth` | `None` | 树最大深度 | 先试 3-5，看欠/过拟合再调 |
| `min_samples_split` | `2` | 父节点最少几个样本才分裂 | 小数据集 2-5，大数据集 10-50 |
| `min_samples_leaf` | `1` | 叶子最少几个样本 | 设 `min_samples_split` 的一半 |
| `random_state` | `None` | 随机种子 | 复现实验时填 42 |

> `criterion` 在 sklearn 1.0 之前叫 `'mse'`（均方误差），1.0 后改为 `'squared_error'`，功能完全相同。旧代码里见到 `'mse'` 不用慌。

### 代码走读：三模型对比（PPT Slide 78）

逐段看 PPT 代码的设计思路：

**Step 1 · 准备数据**

```python
x = np.array(list(range(1, 11))).reshape(-1, 1)   # shape (10, 1)
y = np.array([5.56, 5.70, 5.91, 6.40, 6.80, 7.05, 8.90, 8.70, 9.00, 9.05])
```

- `reshape(-1, 1)`：sklearn 要求 X 是 2D，-1 让 numpy 自动算行数（这里就是 10）
- y 是 1D，不需要 reshape

**Step 2 · 三个模型**

```python
model1 = DecisionTreeRegressor(max_depth=1)   # 1 刀 → 2 段阶梯
model2 = DecisionTreeRegressor(max_depth=3)   # 最多 3 刀 → 最多 8 段阶梯
model3 = LinearRegression()                   # 一条直线
```

**为什么选 max_depth=1 和 3？** 对比极端欠拟合（1 刀）和适度拟合（3 刀），直观展示深度对形状的影响。

**Step 3 · 密集测试点**

```python
x_test = np.arange(0.0, 10.0, 0.01).reshape(-1, 1)   # 1000 个点
```

关键细节：**不是用训练数据的 x 来预测**，而是用 1000 个细密的点预测，这样画出来的曲线才平滑。如果只用 x=[1..10] 预测，阶梯看起来只有 10 个台阶断点，锯齿感强。

**Step 4 · 可视化**

```python
plt.scatter(x, y, label='data')            # 原始散点
plt.plot(x_test, y_pre1, label='max_depth=1')
plt.plot(x_test, y_pre2, label='max_depth=3')
plt.plot(x_test, y_pre3, label='linear')
```

- `scatter`：原始数据点
- `plot`：三条预测线

### 阶梯曲线 vs 直线：视觉解读

运行上述代码，图里会看到三条线：

```
预测形态对比

max_depth=1（橙色，2 段阶梯）：
  x ≤ 6.5 → 全预测 6.24
  x > 6.5 → 全预测 8.91
  只切一刀，左边 6 个样本被迫共享一个预测值，误差大

max_depth=3（绿色，多段细阶梯）：
  以 6.5 / 3.5 / ... 切多刀
  每段更短，样本更集中，更贴近真实数据
  但过深会过拟合（见 Slide 77）

linear（蓝色，直线）：
  y = wx + b，斜率由训练集拟合
  在 x 两端可能偏高或偏低（受"群体趋势"牵引）
  无法反映 6.5 处的跳变
```

**关键观察**：这个数据集在 x=6-7 之间 y 从 7.05 跳到 8.90，跨度约 1.85。线性回归的直线只能在这里斜着穿过，必然在左边高估、右边低估。回归树则用一刀切在 6.5，左右各用均值，直接对齐了这个跳变。

### 过拟合的 3 把刹车

PPT Slide 77 演示了**不设参数的极端情况**：树一直分到每片叶子只剩 1 个样本，训练误差归零，测试误差爆炸。

三个参数都是刹车，和分类树的同名参数语义完全相同（详见 [`../05-titanic/02-API.md`](../05-titanic/02-API.md)），只是作用对象从分类换到了回归：

```
样本流到某节点（n 个样本）
        │
        ▼
   n < min_samples_split？  ──是──> 停（叶子输出均值）
        │否
        ▼
   尝试切分 → 左(n_L) / 右(n_R)
        │
        ▼
   min(n_L, n_R) < min_samples_leaf？ ──是──> 这刀不要，停
        │否
        ▼
   当前深度 == max_depth？  ──是──> 停
        │否
        ▼
       继续递归切分
```

**三把刹车的直觉**：

| 参数 | 类比 | 控的是哪里 |
|---|---|---|
| `max_depth` | "最多问 3 个问题就收手" | 树的总层数上限 |
| `min_samples_split` | "至少 10 个样本才值得分家" | 内部节点的分裂门槛 |
| `min_samples_leaf` | "每个判决需要 5 个以上证人" | 叶子的样本下限 |

**调参起点**（回归树经验值）：

```python
DecisionTreeRegressor(
    max_depth=5,
    min_samples_split=10,
    min_samples_leaf=5,
    random_state=42,
)
```

> 这只是起点，实际用 CV + GridSearch 调。

### 选模型决策表

| 场景 | 推荐 | 理由 |
|---|---|---|
| 数据关系线性，需要系数解释 | **线性回归** | 斜率直接对应业务含义（每增 1 平米涨多少） |
| 数据非线性 / 有阶梯跳变 | **回归树** | 天然分段，不假设线性 |
| 特征间有复杂交互（A 高且 B 低才影响 y） | **回归树** | 树结构自然捕捉交互 |
| 准确率优先、解释性其次 | **GBDT / XGBoost** | 集成方法，多棵树叠加，准确率显著高于单树 |
| 数据量小（< 1000 样本），特征少 | **线性回归 / 单棵决策树** | 集成方法在小数据上容易过拟合 |
| 生产环境需要规则部署（if-else 白盒） | **回归树**（浅树） | 可直接把树导出成 if-else 代码 |

**实战中单棵回归树的定位**：多数情况下是 **GBDT / 随机森林的积木**，而不是最终交付的模型。单棵树拿来学原理，集成方法才是打实际比赛 / 上线的主力。

### 选择题解析（PPT Slide 80）

**题 1**：回归决策树说法正确的是（多选）？

- A）可以处理非线性关系 ✓ — 阶梯形状不假设线性
- B）可以处理多个输入变量 ✓ — 多特征时枚举每个特征的最优切分点
- C）可以处理分类和回归问题 ✗ — `DecisionTreeRegressor` 只做回归；分类用 `DecisionTreeClassifier`；CART 整体框架可以做两种，但一个实例对象只做一种
- D）无需干预，天然不会过拟合 ✗ — 恰恰相反，默认参数下极容易过拟合

**答案：AB**

**题 2**：回归决策树如何选择最佳分割点？

- A）信息增益 — ID3 / C4.5 用，适合分类，不适合回归
- B）基尼指数 — CART 分类树用
- C）均方误差（平方损失）✓
- D）叶节点样本数量 — 不是准则，是刹车参数

**答案：C**
