# 什么是剪枝

```
     ●   ●     C   ●                ●       C   ●

            ●   ●                         ●   ●
       太细 → 把训练噪声学进去      规则更"泛化"
```

**为什么要剪枝？**
决策树剪枝是一种防止决策树过拟合的一种正则化方法；提高其泛化能力。

```
剪枝操作：把整棵子树折成一个叶子

   ◇ 内部节点                    ◇ 内部节点
   ┌─┴─┐                         │
   ◇   ◇        ──剪枝──▶       [叶子]
  ┌┴┐ ┌┴┐                        多数类标签
  ● ● ● ●
  （子树）                      （单个叶节点）
```

**剪枝**：把子树的节点全部删掉，使用叶子节点来替换。

**剪枝方法**：

- **预剪枝**：指在决策树生成过程中，对每个节点在划分前先进行估计，若当前节点的划分不能带来决策树泛化性能提升，则停止划分并将当前节点标记为叶节点
- **后剪枝**：是先从训练集生成一棵完整的决策树，然后自底向上地对非叶节点进行考察，若将该节点对应的子树替换为叶节点能带来决策树泛化性能提升，则将该子树替换为叶节点

### 笔记

> 【知道】什么是剪枝？

**剪枝（pruning）是决策树对付过拟合的主要手段**。

**为什么会过拟合**：决策树为了正确分类训练样本会不断划分，分支太多时会把训练集的噪声当成"普遍规律"学进去。

**剪枝操作**：把一棵子树的子节点全删掉，用一个叶子节点替换该子树（实质是后剪枝）。也可以只保留根节点本身。

**工程类比**：相当于代码 review 砍掉过度细化的 if-else 分支，让规则更"泛化"。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 为什么决策树天生过拟合

回忆 06 章回归树里"过拟合那张穿过每个数据点的图"——模型不放过训练集里任何一个点，曲线蛇形穿越所有样本。分类树也一样，只要不加限制就会无限分裂：

```
信息增益 / 基尼指数只要还能降 → 算法继续切 → 没有"停"的天然信号

结果：
  训练集准确率 = 100%（每个样本单独一个叶子，必然正确）
  测试集准确率 = 崩盘（叶子规则全是训练噪声）
```

**具体机制**：

```
训练样本到某个叶子：1 科技书 + 1 文学书（不纯）
        ↓
决策树还要分裂：按"插图"切
        ↓
有插图叶：1 文学书（纯了 ✓）
无插图叶：1 科技书（纯了 ✓）
        ↓
训练集完美，但这条规则可能只是 2 个样本的噪声
```

这和线性模型不同——线性模型参数有解析解有界；决策树的"深度"在数学上没有上限，不踩刹车就一定过拟合。

### 剪枝的核心操作：子树 → 叶子

**剪枝 = 把一棵子树整体替换成单个叶节点**，叶子的标签按原子树里的**多数类**打。

```
剪枝前                          剪枝后
──────                          ──────
   ◇ 父节点                       ◇ 父节点（变成叶子）
   ┌──┴──┐                        │
   ◇     ◇      ──剪掉──▶        [叶子]
  ┌┴┐   ┌┴┐                       标签 = 原子树多数类
  ● ● ● ●                         规则更粗，但更稳定
（4 叶子，细碎）                  （1 叶子，保守）
```

操作前后的变化：
- **规则变粗**：原来"颜色=红 AND 插图=有 → 文学书"，剪完变"颜色=红 → 文学书（统一判）"
- **深度减少**：树变浅，复杂度降低
- **训练精度微降**：可能从 100% 降到 95%；但测试精度往往上升

### 剪枝 = 决策树版的正则化

正则化（regularization）是 ML 里控制模型复杂度、防止过拟合的通用手段：

| 模型 | 正则化形式 | 机制 |
|---|---|---|
| 线性回归 | L2（Ridge）/ L1（Lasso） | 参数惩罚，让权重趋小 / 稀疏 |
| 神经网络 | Dropout / Weight Decay | 随机丢节点 / 权重衰减 |
| **决策树** | **剪枝** | **删掉"收益不够"的子树** |
| 决策树（参数形式） | `max_depth` / `min_samples_*` | 限制复杂度上限（预剪枝参数化） |

剪枝让决策树"学得没那么细"——用一点训练精度换更好的泛化。这和 L2 惩罚让权重缩小是同一个哲学：**限制模型自由度**。

### 剪枝 vs 集成：两条对付过拟合的路

```
过拟合对策
─────────────────────────────────────────────
路线 A · 剪枝（控制单棵树复杂度）
  预剪枝：边长边停（max_depth / min_samples_*）
  后剪枝：长完再砍（ccp_alpha）
  优点：树小、可解释、推理快
  缺点：一棵树再怎么调，泛化天花板不高

路线 B · 集成（多棵树投票 / 加权）
  Random Forest：并行 bagging，每棵浅树
  GBDT / XGBoost / LightGBM：串行 boosting
  优点：泛化强、精度高
  缺点：可解释性差、调参复杂
─────────────────────────────────────────────
工业实践：
  可解释性要求高 → 单棵 + 剪枝
  追求预测精度   → 直接集成
```

**不懂剪枝，就不懂集成**：集成模型的基学习器都是"浅树"（通常 max_depth=3-8）。这不是随意的设定——浅树本身就是一种预剪枝，让每棵基学习器"有点偏差但方差低"，集成后互补。理解剪枝才能理解这个设计背后的逻辑。

集成学习是后续章节的主场，此处只点名。

### 一句话钉板

**剪枝 = 把"收益不够"的子树砍成叶子，是决策树对付过拟合的正则化手段。分两类：边长边停（预剪枝）和长完再砍（后剪枝）。工业追求精度直接上集成；但不懂剪枝就不懂集成为什么要用浅树。**
