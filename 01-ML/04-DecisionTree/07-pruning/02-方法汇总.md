**已知**：训练集和验证集

```
数据划分                       剪枝两条路线
─────────────                  ─────────────
   全部数据                    ① 预剪枝：边长边停
   ────────                       生成中 → 验证 → 不升则停
   ┌──────────┐
   │  训练集   │ → 决定结构    ② 后剪枝：先长全再砍

   ├──────────┤
   │  验证集   │ → 决定剪不剪
   │（少量）   │   （用准确率）
   └──────────┘
```

> Slide 85 · 剪枝原理

```
分裂前                  分裂后
─────                  ─────
   父节点                 父节点
   acc_v = p%             │
                       ┌──┴──┐
                      左叶   右叶
                     acc_L  acc_R
                     ─────────────
       判定：左、右验证准确率均值 vs 父节点
              ↑ 升  → 允许分裂
              ↓ 不升 → 禁止分裂（预剪枝）
              或 折回叶子（后剪枝）
```

### 笔记

> 【知道】常见剪枝方法汇总

两大策略：

| 策略 | 时机 | 做法 |
|---|---|---|
| **预剪枝** pre-pruning | 树**生成中** | 划分前先估计：若划分不能提升验证集泛化性能，就停止划分，标记为叶节点 |
| **后剪枝** post-pruning | 树**完整生成后** | 自底向上考察非叶节点：若把子树替换为叶节点能提升验证集精度，就替换 |

**判停依据**：通常预留**验证集**算精度。分工原则（Slide 88）：**训练集**决定树的结构和叶子节点的标签；**验证集**决定是否剪枝。

**sklearn 实现**：

- 预剪枝：`max_depth` / `min_samples_split` / `min_samples_leaf` / `min_impurity_decrease`
- 后剪枝：`ccp_alpha`（Cost-Complexity Pruning，CART 原版剪枝）

## ━━━━━━━━ 讲解 ━━━━━━━━

### 数据分工：训练集管结构，验证集管剪枝

这是剪枝的根本原则，必须先搞清楚：

```
全部数据
   │
   ├── 训练集（多数，通常 70-80%）
   │     → 决定：树结构（哪个特征分裂）
   │     → 决定：叶子标签（多数类）
   │
   └── 验证集（少量，通常 20-30%）
         → 决定：这个分裂值不值得做（剪枝判据）
```

**为什么要分开？**：用训练集既长树又决定剪不剪，相当于"自己出题自己答"——必然过拟合。用独立验证集评判，才能检验规则的**泛化**价值，而不只是记忆能力。

### 预剪枝 vs 后剪枝：时机对比

```
预剪枝（pre-pruning）           后剪枝（post-pruning）
────────────────────           ──────────────────────
        根                              根（完整树，先长全）
        │                              │
    按特征A试分裂                  ┌───┴───┐
        │                          ②       ③
    验证集精度升了吗？              │       │
        │                         ┌┴┐   ┌─┴─┐
    ↓ 没升 → 停，当前节点变叶子    ④ ⑤  ●  ⑥
    ↓ 升了 → 允许分裂继续                │
        │                            ┌─┴─┐
       [叶子]                        ● ●
    （边长边停）
                                 ↓ 长完后自底向上考察
                                 ⑥ → ⑤ → ④ → ③ → ② → ①
                                 每个节点：折成叶子，验证集精度升了吗？
                                 ↑ 升 → 剪掉 ✓
                                 ↓ 不升 → 保留 ✗
                                （先长全，再砍）
```

### 准确率均值判定的算法

**判定规则**（Slide 85 核心）：

```
当前节点（父）验证集准确率 = acc_parent

尝试按特征 A 分裂 → 得到左叶、右叶

左叶验证集准确率 = acc_L
右叶验证集准确率 = acc_R
均值 = (acc_L + acc_R) / 2

判定：
  均值 > acc_parent  → 分裂后更好 → 允许分裂（或保留分裂）
  均值 ≤ acc_parent  → 分裂后没好 → 停止分裂（预剪枝）/ 折回叶子（后剪枝）
```

**注意**：这里用的是左右叶子各自准确率的**均值**，不是把验证样本混在一起算总体准确率。为什么用均值？——防止一个大叶子拖着小叶子"平摊"稀释精度变化的判断。

**伪代码**（预剪枝判定一个节点）：

```
def should_split(node, feature, train_data, val_data):
    # 不分裂时，父节点按多数类预测
    acc_parent = accuracy(node.majority_label, val_data)

    # 试着分裂
    left_data, right_data = split(train_data, feature)
    left_label = majority_class(left_data)
    right_label = majority_class(right_data)

    left_val, right_val = split(val_data, feature)
    acc_L = accuracy(left_label, left_val)
    acc_R = accuracy(right_label, right_val)

    avg = (acc_L + acc_R) / 2

    return avg > acc_parent  # True → 允许分裂
```

### sklearn 实现速查

**预剪枝参数**（主场在 [`../05-titanic/02-API.md`](../05-titanic/02-API.md)）：

| 参数 | 作用 | 调参起点 |
|---|---|---|
| `max_depth` | 树最大深度 | 5（大数据集可更深） |
| `min_samples_split` | 节点继续分裂的最小样本数 | 10-50 |
| `min_samples_leaf` | 叶子最少样本数 | 5-20 |
| `min_impurity_decrease` | 分裂至少降低多少不纯度才做 | 默认 0，可试 0.01 |

**后剪枝参数**（本章主场）：

```python
from sklearn.tree import DecisionTreeClassifier

# Step 1：获取不同 ccp_alpha 对应的剪枝路径
clf = DecisionTreeClassifier(random_state=42)
path = clf.cost_complexity_pruning_path(X_train, y_train)
alphas = path.ccp_alphas  # 从 0 到最大 alpha 的序列

# Step 2：GridSearch 找最优 alpha
from sklearn.model_selection import cross_val_score
best_alpha, best_score = 0, 0
for alpha in alphas:
    clf = DecisionTreeClassifier(ccp_alpha=alpha, random_state=42)
    score = cross_val_score(clf, X_train, y_train, cv=5).mean()
    if score > best_score:
        best_score, best_alpha = score, alpha

# Step 3：用最优 alpha 训练最终模型
final_clf = DecisionTreeClassifier(ccp_alpha=best_alpha, random_state=42)
final_clf.fit(X_train, y_train)
```

**`ccp_alpha` 的含义**：cost-complexity pruning 的惩罚系数。越大 → 剪得越猛 → 树越浅。alpha=0 等于不剪枝；alpha 过大会把树剪到只剩根节点。

### 一句话钉板

**训练集长树，验证集判断值不值得分裂。预剪枝边长边停（参数刹车），后剪枝先长全再从底往上砍。判停依据：分裂后左右叶准确率均值 vs 父节点——不升就不分。**
