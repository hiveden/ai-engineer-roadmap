### 预剪枝：Step by Step

#### Step 1 · 训练阶段建树（按颜色分根节点）

训练集按 ID3/CART 逻辑选特征：颜色信息增益最大 → 作为根节点分裂。

```
训练集分析：
  蓝色书：3 科技（#1,2,3）+ 0 文学 → 纯 ✓ → 标签 [科技书]
  红色书：1 科技（#4）+ 4 文学（#5,6,7,8）→ 不纯 → 标签 [文学书]（多数类）

              [?]                 ← 父节点：4科+4文，50%/50%，无多数类
              4科+4文              → 随机选，打 [科技书] 标签
                 │
            按"颜色"
          ┌────┴────┐
          蓝         红
       3科+0文     1科+4文
       [科技书]    [文学书]
       （纯）      （不纯，还可以再分）
```

此时红子树不纯，如果不剪枝会继续分裂。预剪枝在这里要先**暂停，去问验证集**。

#### Step 2 · 验证集第一步：颜色分裂值不值得做

**不分裂时**（父节点统一预测 [科技书]）：

```
验证集 4 条：
  #1 蓝/科技 → 预测 科技 ✓
  #2 蓝/科技 → 预测 科技 ✓
  #3 红/文学 → 预测 科技 ✗
  #4 红/文学 → 预测 科技 ✗
父节点准确率 = 2/4 = 50%
```

**分裂后**（按颜色分两叶）：

```
蓝叶 [科技书]：
  #1 蓝/科技 → 科技 ✓
  #2 蓝/科技 → 科技 ✓
  → 蓝叶准确率 = 2/2 = 100%

红叶 [文学书]：
  #3 红/文学 → 文学 ✓
  #4 红/文学 → 文学 ✓
  → 红叶准确率 = 2/2 = 100%

均值 = (100% + 100%) / 2 = 100%
```

**判定**：100% > 50% → **允许分裂** ✓

#### Step 3 · 红子树是否继续分裂（插图特征）

红子树目前是叶节点 [文学书]，训练集里有 1 科技（#4 红+有插图）+ 4 文学。算法考虑按"插图"再切：

```
训练集红子树：
  有插图：1 科技(#4) + 1 文学(#8) → 1v1（一对一打平）打 [文学书]（选"文学"因为这样才有意义，否则与父同标签）
  无插图：0 科技 + 3 文学 → 全文学 → [文学书]

按"插图"分裂后，红子树变为：
         [文学书]（红父）
         ┌──┴──┐
         有     无
     [文学书] [文学书]
      1科1文   0科3文
```

**验证集中红色书样本**：
- #3 红/无插图/文学 → 走无插图叶，预测 [文学书] ✓
- #4 红/有插图/文学 → 走有插图叶，预测 [文学书] ✓

**父节点（不分裂，红叶仍是 [文学书]）**：

```
红叶父节点验证准确率（已知 = 100%，来自 Step 2 的红叶）

按插图分裂后：
  有插图叶[文学书]: #4红/有/文学 → 文学 ✓ → 1/1 = 100%
  无插图叶[文学书]: #3红/无/文学 → 文学 ✓ → 1/1 = 100%
  均值 = (100% + 100%) / 2 = 100%
```

**判定**：均值 100% = 父节点 100%，没有**提升**（≤，不升） → **预剪枝：不允许分裂** ✗

> 注意：PPT Slide 92 实际上描述的是插图分裂后有一个叶子准确率为 0%。这是因为 PPT 中"科技书"的标签分配不同——PPT 里"有插图"叶打了 [文学书]，"无插图"叶打了 [科技书]。验证集 #3（红/无/文学）走无插图叶预测 [科技书] → 错误 → 0%；验证集 #4（红/有/文学）走有插图叶预测 [文学书] → 正确 → 100%。均值 = (100%+0%)/2 = 50% < 父节点 100%，不允许分裂。结论一致，均为预剪枝第 2 步 ✗。

### 后剪枝：先长全再砍

**核心逻辑**（Slide 94）：

```
Step 1：用全部训练集长出完整树（不剪枝，长到叶子纯为止）
Step 2：编号所有内部节点（例：6 个）
Step 3：自底向上逐一考察
        对每个内部节点 k：
          试验：把 k 整棵子树替换成 1 个叶子（按子树里多数类打标签）
          计算：替换后验证集准确率
          判定：精度上升 → 确认剪掉 ✓
                精度不升 → 恢复，保留原子树 ✗
Step 4：所有节点过完 → 得到最终后剪枝树
```

**伪代码**：

```
def post_prune(tree, val_data):
    nodes = get_internal_nodes_bottom_up(tree)  # 自底向上排序
    for node in nodes:
        # 折叠试验
        original_subtree = node.subtree
        node.make_leaf(majority_class(node.train_samples))

        acc_after = accuracy(tree, val_data)
        acc_before = accuracy_with_original(original_subtree, val_data)

        if acc_after >= acc_before:
            pass  # 保留剪枝（叶子）
        else:
            node.restore(original_subtree)  # 恢复

    return tree
```

### 预剪枝 vs 后剪枝：本案例对比小结

| 维度 | 预剪枝 | 后剪枝 |
|---|---|---|
| **书本案例最终树深度** | 1 层（只有颜色节点） | 通常更深 |
| **西瓜书最终精度** | 71.4% | 71.4% |
| **决策过程** | 每步分裂前问验证集 | 长完后自底向上试验 |
| **计算量** | 小（早停） | 大（先长全，再逐节点试） |

两种方法在这个小案例里精度一样，但后剪枝保留了更多有价值的分支（见下章 [`04-对比.md`](./04-对比.md)）。

### 一句话钉板

**预剪枝：每步问验证集，均值不升就停。**

**后剪枝：长完整树再自底向上逐节点折叠试验。**

**书本案例 50% → 100%；西瓜书两法都到 71.4%。**
