### PPT

> Slide 95 · 决策树正则化 – 剪枝技术对比

| 维度 | 预剪枝 | 后剪枝 |
|---|---|---|
| **优点** | 预剪枝使决策树的很多分支没有展开，不单降低了过拟合风险，还显著减少了决策树的训练、测试时间开销 | 比预剪枝保留了更多的分支。一般情况下，后剪枝决策树的欠拟合风险很小，泛化性能往往优于预剪枝 |
| **缺点** | 有些分支的当前划分虽不能提升泛化性能，但后续划分却有可能导致性能的显著提高；预剪枝决策树也带来了欠拟合的风险 | 后剪枝先生成，后剪枝。自底向上地对树中所有非叶子节点进行逐一考察，训练时间开销比未剪枝的决策树和预剪枝的决策树都要大得多 |

> Slide 96 · 总结

1. **剪枝？**
   把叶子节点、子节点删掉，用更大的叶子节点（子树）替换叶子节点
2. **预剪枝和后剪枝？**
   - 预剪枝：指在决策树生成过程中，对每个节点在划分前先进行估计，若当前节点的划分不能带来决策树泛化性能提升，则停止划分并将当前节点标记为叶节点
   - 后剪枝：是先从训练集生成一棵完整的决策树，然后自底向上地对非叶节点进行考察，若将该节点对应的子树替换为叶节点能带来决策树泛化性能提升，则将该子树替换为叶节点

### 笔记

> 【了解】剪枝方法对比

| 维度 | 预剪枝 | 后剪枝 |
|---|---|---|
| **训练开销** | 小（提前停） | 大（先长全再砍） |
| **泛化性能** | 一般 | 通常更优 |
| **欠拟合风险** | **较高**（贪心可能错过"先降后升"的好分支） | 很低 |
| **过拟合风险** | 低 | 低 |

**经验法则**：

- 数据量小、追求训练速度 → 预剪枝（设 `max_depth` 等）
- 数据量充足、追求泛化 → 后剪枝（`ccp_alpha`）或直接上**集成学习**（RF / GBDT / XGBoost）规避单棵树过拟合
- 工业实践中：单棵决策树几乎只用于可解释性场景；预测任务直接上集成

### 4 维度对比表

| 维度 | 预剪枝 | 后剪枝 |
|---|---|---|
| **训练开销** | **小**（早停，不需要长满） | **大**（先长满，再逐节点试验） |
| **泛化性能** | 一般（保守） | 通常更优（保留更多有用分支） |
| **欠拟合风险** | **较高** | 很低 |
| **过拟合风险** | 低 | 低（均靠验证集控制） |

**欠拟合风险的根源**：预剪枝是贪心判断——当前分裂不提升精度就立刻停。但有些分支"先降后升"，即当前分裂看起来没用，但它是后续某个关键分裂的前提。预剪枝把这类分支也砍掉了，可能错过更深处的有效规则。

```
"先降后升"示意：
                         根
                         │
                    按 A 分裂（精度不升 ← 预剪枝会在这里停）
                 ┌──────┴──────┐
                 ◇             ◇
                 │             │
            按 B 分裂      按 C 分裂（如果允许进来，精度大升）
            ┌──┴──┐          ┌──┴──┐
            ●     ●          ●     ●（关键规则在这一层）

预剪枝：A 那层就停了 → 错过下面的 C
后剪枝：先长到 C，然后从底向上考察，保留 C 砍掉没用的
```

### 经验法则决策表

```
场景                           推荐方案
──────────────────────         ─────────────────────────
数据量小（< 1k）               预剪枝（max_depth=3-5）
  理由：数据少，过长容易过拟合；验证集也小，后剪枝试验噪声大

数据量中等（1k-100k）          后剪枝（ccp_alpha 调参）
  理由：数据够多，验证集信号可靠；泛化更重要

追求最高泛化性能               直接上集成（RF / GBDT）
  理由：单棵树天花板有限；集成模型泛化几乎必胜

可解释性优先（金融 / 医疗）     单棵 + 预剪枝（max_depth 限死）
  理由：规则要能给业务解释；集成太黑盒

快速 baseline                  max_depth=5 跑个预剪枝版
  理由：1 行参数，5 秒出结果，先有个锚点再说
```

### 集成学习：跳板

剪枝是对单棵树的"补丁"——治标。集成学习是从架构层面解决过拟合——治本：

```
为什么集成比单棵强？
─────────────────────────────────────────
单棵树：一个模型，偏差 + 方差都集中在一棵树上
        剪枝只是限制方差，但精度天花板有限

集成（bagging）：
  Random Forest = N 棵浅决策树，各自在随机子集上训练
  每棵树：高偏差但低方差（浅树）
  投票叠加：偏差均出 + 方差消除 → 精度大幅提升

集成（boosting）：
  GBDT / XGBoost / LightGBM = 逐棵树纠错
  每棵树：专门拟合前一轮的残差
  叠加：偏差逐步消除 → 精度更高

─────────────────────────────────────────
下一阶段主场：集成学习（Random Forest / GBDT / XGBoost）
```

### 一句话钉板

**预剪枝快但欠拟合风险高，后剪枝慢但泛化更好。**

**小数据用预剪枝刹车，中大数据上 ccp_alpha 后剪枝。**

**追求精度直接上集成，单棵树留给可解释性。**
