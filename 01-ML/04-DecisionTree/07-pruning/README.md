#### Slide 83 · 决策树正则化 – 剪枝

```
       完整树（过拟合）              剪枝后（泛化好）
       ───────────────              ───────────────
            根                            根
       ┌────┴────┐                  ┌────┴────┐
       A         B                  A         B
     ┌─┴─┐     ┌─┴─┐                │       ┌─┴─┐
     ●   ●     C   ●                ●       C   ●
            ┌─┴─┐                         ┌─┴─┐
            ●   ●                         ●   ●
       太细 → 把训练噪声学进去      规则更"泛化"
```

**为什么要剪枝？**
决策树剪枝是一种防止决策树过拟合的一种正则化方法；提高其泛化能力。

```
剪枝操作：把整棵子树折成一个叶子

   ◇ 内部节点                    ◇ 内部节点
   ┌─┴─┐                         │
   ◇   ◇        ──剪枝──▶       [叶子]
  ┌┴┐ ┌┴┐                        多数类标签
  ● ● ● ●
  （子树）                      （单个叶节点）
```

**剪枝**：把子树的节点全部删掉，使用叶子节点来替换。

**剪枝方法**：

- **预剪枝**：指在决策树生成过程中，对每个节点在划分前先进行估计，若当前节点的划分不能带来决策树泛化性能提升，则停止划分并将当前节点标记为叶节点
- **后剪枝**：是先从训练集生成一棵完整的决策树，然后自底向上地对非叶节点进行考察，若将该节点对应的子树替换为叶节点能带来决策树泛化性能提升，则将该子树替换为叶节点

```

    ┌───┴───┐
    ②       ③                       每次试验：
    │       │                       ─────────────
  ┌─┴─┐   ┌─┴─┐                     考察节点 k：
  ④   ⑤   ●   ⑥                       把它整棵子树折成一个叶子
  │   │      │                          → 算验证集准确率
┌─┴─┐ ●   ┌─┴─┐                        ↑ 升 → 剪 ✓
● ●     ● ●                             ↓ 不升 → 保留 ✗
（圆点 = 叶节点）
```

先利用训练集完整的生成一颗树，有 6 个内部节点。分别考察这些节点作为叶子节点模型的准确率，若准确率上升，则剪掉，否则保留。

#### Slide 95 · 决策树正则化 – 剪枝技术对比

| 维度 | 预剪枝 | 后剪枝 |
|---|---|---|
| **优点** | 预剪枝使决策树的很多分支没有展开，不单降低了过拟合风险，还显著减少了决策树的训练、测试时间开销 | 比预剪枝保留了更多的分支。一般情况下，后剪枝决策树的欠拟合风险很小，泛化性能往往优于预剪枝 |
| **缺点** | 有些分支的当前划分虽不能提升泛化性能，但后续划分却有可能导致性能的显著提高；预剪枝决策树也带来了欠拟合的风险 | 后剪枝先生成，后剪枝。自底向上地对树中所有非叶子节点进行逐一考察，训练时间开销比未剪枝的决策树和预剪枝的决策树都要大得多 |

#### Slide 96 · 总结

1. **剪枝？**
   把叶子节点、子节点删掉，用更大的叶子节点（子树）替换叶子节点
2. **预剪枝和后剪枝？**
   - 预剪枝：指在决策树生成过程中，对每个节点在划分前先进行估计，若当前节点的划分不能带来决策树泛化性能提升，则停止划分并将当前节点标记为叶节点
   - 后剪枝：是先从训练集生成一棵完整的决策树，然后自底向上地对非叶节点进行考察，若将该节点对应的子树替换为叶节点能带来决策树泛化性能提升，则将该子树替换为叶节点

> 07 · 决策树剪枝

**学习目标**：

1. 知道什么是剪枝
2. 理解剪枝的作用
3. 知道常用剪枝方法
4. 了解不同剪枝方法的优缺点

> 【知道】什么是剪枝？

**剪枝（pruning）是决策树对付过拟合的主要手段**。

**为什么会过拟合**：决策树为了正确分类训练样本会不断划分，分支太多时会把训练集的噪声当成"普遍规律"学进去。

**剪枝操作**：把一棵子树的子节点全删掉，用一个叶子节点替换该子树（实质是后剪枝）。也可以只保留根节点本身。

**工程类比**：相当于代码 review 砍掉过度细化的 if-else 分支，让规则更"泛化"。

> 【知道】常见剪枝方法汇总

两大策略：

| 策略 | 时机 | 做法 |
|---|---|---|
| **预剪枝** pre-pruning | 树**生成中** | 划分前先估计：若划分不能提升验证集泛化性能，就停止划分，标记为叶节点 |
| **后剪枝** post-pruning | 树**完整生成后** | 自底向上考察非叶节点：若把子树替换为叶节点能提升验证集精度，就替换 |

**判停依据**：通常预留**验证集**算精度。分工原则（Slide 88）：**训练集**决定树的结构和叶子节点的标签；**验证集**决定是否剪枝。

**sklearn 实现**：

- 预剪枝：`max_depth` / `min_samples_split` / `min_samples_leaf` / `min_impurity_decrease`
- 后剪枝：`ccp_alpha`（Cost-Complexity Pruning，CART 原版剪枝）

> 【理解】例子

**西瓜书"脐部"案例**（训练集 + 验证集划分）：

**预剪枝流程**：

1. 不分裂，全部预测多数类"好瓜" → 验证集精度 3/7
2. 用"脐部"分裂成 3 个叶子（编号 2/3/4）→ 验证集精度 5/7 ✓ 值得分裂
3. 对结点 2 用"色泽"再分 → 验证集精度降到 4/7 ✗ **禁止划分**
4. 对结点 3 用"根蒂"再分 → 验证集精度仍 5/7（无提升）✗ 禁止划分
5. 最终：仅一层划分的浅树，精度 71.4%

**后剪枝流程**（先长完整树，验证集精度仅 42.9%）：

1. 自底向上考察结点 6 → 替换为叶子，精度升至 57.1% ✓ 剪
2. 考察结点 5 → 精度仍 57.1%（无提升）✗ 不剪
3. 考察结点 2 → 精度升至 71.4% ✓ 剪
4. 考察结点 3、1 → 无提升 ✗ 保留
5. 最终精度 71.4%

> 【了解】剪枝方法对比

| 维度 | 预剪枝 | 后剪枝 |
|---|---|---|
| **训练开销** | 小（提前停） | 大（先长全再砍） |
| **泛化性能** | 一般 | 通常更优 |
| **欠拟合风险** | **较高**（贪心可能错过"先降后升"的好分支） | 很低 |
| **过拟合风险** | 低 | 低 |

**经验法则**：

- 数据量小、追求训练速度 → 预剪枝（设 `max_depth` 等）
- 数据量充足、追求泛化 → 后剪枝（`ccp_alpha`）或直接上**集成学习**（RF / GBDT / XGBoost）规避单棵树过拟合
- 工业实践中：单棵决策树几乎只用于可解释性场景；预测任务直接上集成
