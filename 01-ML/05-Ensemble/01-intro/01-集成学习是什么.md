## ━━━━━━━━ 底稿 ━━━━━━━━

### PPT

> Slide 4 · 集成学习定义

集成学习是机器学习中的一种思想，它通过多个模型的组合形成一个精度更高的模型，参与组合的模型成为弱学习器（弱学习器）。训练时，使用训练集依次训练出这些弱学习器，对未知的样本进行预测时，使用这些弱学习器联合进行预测。

### 笔记

> 【知道】集成学习是什么

**Ensemble Learning**：组合多个**弱学习器（weak learner / base learner）** 形成精度更高的强学习器。每个弱学习器独立训练、独立预测，最后把预测结果合并。

工程类比：微服务集群 + 网关聚合。单个服务能力有限，多副本投票/加权得到更稳的输出。

俗语版：三个臭皮匠，赛过诸葛亮。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

**业务痛点**：训练单棵决策树预测"邮件是不是垃圾"，训练集准确率 **99%**，测试集只剩 **72%** —— 典型过拟合。换一棵树（不同特征顺序、不同采样）测试集 70%，半斤八两。单棵都不靠谱，怎么办？

**生活锚**：判一张 CT 片有没有肿瘤。单个医生可能漏诊；**5 位医生各自独立读片再投票**，把偶发误判抹平 —— 医院遇到疑难病例时的「多学科会诊」就是这思路，俗称「三个臭皮匠，赛过诸葛亮」。

一句话：**别赌一个模型，赌一群**。

### 主线案例：5 棵决策树判垃圾邮件

把每棵决策树看作一个**弱学习器（weak learner，也叫基学习器 base learner）**——单干能力一般、独立训练、独立投票。集成就是把 N 个弱学习器凑一起合议。

**正例（集成救场）**：

5 棵决策树独立训练，每棵看训练集的不同子集，对一封新邮件投票：

```
树1: 垃圾   ─┐
树2: 垃圾   ─┤
树3: 正常   ─┼─→ 投票 3:2 → 判"垃圾"
树4: 垃圾   ─┤
树5: 正常   ─┘
```

下表为**虚构示意数字**，方便对比效果（不是实测结果）：

| 模型 | 测试集准确率 |
|---|---|
| 单棵决策树（任选一棵）| 72% |
| 5 棵投票集成 | **87%** |

只要 5 棵树**不同时犯错**，多数票就能把单棵的偶发错误抹平。

**反例（集成失效）**：

5 棵**完全相同**的决策树（同样的数据、同样的参数、同样的随机种子）：

```
树1: 垃圾   ─┐
树2: 垃圾   ─┤
树3: 垃圾   ─┼─→ 投票 5:0 → 判"垃圾"（和单棵一模一样）
树4: 垃圾   ─┤
树5: 垃圾   ─┘
```

集成后准确率还是 **72%**，一点没涨。**5 个克隆体一起错也是错**。

→ 启示：**差异性（diversity）是集成有效的前提**。怎么制造差异是后两节（[`03-bagging.md`](./03-bagging.md) / [`04-boosting.md`](./04-boosting.md)）的事。

### 常见误区

- **不是"模型越多越好"**：弱学习器之间太像时（极端就是反例里的克隆体），集成增益迅速衰减到 0。
- **不是"非要用决策树"**：bagging / boosting 对基学习器（base learner）类型不挑，理论上 KNN / LR 都能当弱学习器。但实际生态里决策树是默认选择 —— 训练快，且**换个深度或随机种子，长出来的树就不一样**，天然能造差异。

### 不在本章范围

- **stacking（堆叠融合）**：再加一层模型学"怎么融合前面 N 个模型的输出"。本章只讲**固定聚合规则**的 bagging / boosting，stacking 暂不展开。
