- **Bagging**：随机森林（Random Forest）

```
自助聚合（装袋法）                    增强法
─────────────────────────────────────────────
学习器之间相互独立          学习器之间需要通信

```

> **Notes**：每个样本所属的类别 圈圈还是方块，由 3 个分类器平权投票决定。三个分类器的训练样本不同。

> Slide 11 · Bagging & Boosting 对比

| | bagging | boosting |
| --- | --- | --- |
| 数据采样 | 对数据进行有放回的采样训练 | 全部样本，根据前一轮学习结果调整数据的重要性 |
| 投票方式 | 所有学习器平权投票 | 对学习器进行加权投票 |
| 学习顺序 | 并行的，每个学习器没有依赖关系 | 串行，学习有先后顺序 |

> **Notes**：也能做回归，各弱分类器的平均预测值。CART 回归树。

> Slide 13 · 自检题

**1、Bagging 是什么意思，下面说法正确的是（）**

- A）一个精度比较高的分类器
- B）从数据集中随机采样出多个子集，每个子集训练一个弱分类器，然后将这些弱分类器组合成一个强分类器
- C）从特征集中随机选择多个特征，使用这些特征训练一个弱分类器，然后将多个弱分类器组合成一个强分类器
- D）以上都不是

**正确答案：B**（C 是随机森林的定义）

### 笔记

> 【了解】集成学习分类

主流两类：

| 流派 | 核心 | 代表 |
|---|---|---|
| **Bagging**（Bootstrap Aggregating）| 并行 + 有放回采样 + 平权投票 | Random Forest |
| **Boosting** | 串行 + 关注前一轮错样本 + 加权投票 | AdaBoost / GBDT / XGBoost / LightGBM |

> Bagging vs Boosting

| 维度 | Bagging | Boosting |
|---|---|---|
| **数据** | 有放回采样的子集 | 全量数据 + 样本权重(AdaBoost) / 残差或负梯度(GBDT) |
| **投票** | 平权 | 加权（按学习器误差） |
| **顺序** | 并行（无依赖） | 串行（后一个依赖前一个） |
| **侧重** | 降方差 | 降偏差 |

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

一句话：**Bagging 是并行集群，Boosting 是流水线**。前者各干各的最后投票，后者一个接一个地补前一个的锅。

### 工程师视角

| 角度 | Bagging | Boosting |
|---|---|---|
| 部署 | 多副本横向扩容（kubectl scale） | 串行 pipeline（A → B → C） |
| 失败模式 | 单副本错不影响多数票 | 前序错传染后序，需正则约束 |
| 训练成本 | 易并行（多核 / 多机） | 顺序依赖，难并行 |
| 调参痛点 | 树的数量、采样比 | 学习率（learning rate）、轮数、早停 |

### 怎么选

- **数据噪声大 / 想稳**：bagging（随机森林 Random Forest）—— 降方差（variance）是它的本职
- **想榨干精度 / Kaggle 打榜**：boosting（XGBoost / LightGBM）—— 降偏差（bias），但需要更细的调参
- **生产里默认**：先 RF 拿基线，再换 LightGBM 看是否值得复杂度
