### 笔记

> 【理解】bagging 集成

**思想**：有放回抽样（bootstrap）产生 N 个不同训练集 → 各自训练弱学习器 → 平权投票（分类）/ 平均（回归）。

**关键点**：

- 弱学习器之间**独立**，训练可并行
- 抽样制造**差异性**，差异越大集成增益越高
- 投票权重相等（所以叫 "平权"）

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

一句话：**给每个模型发一份"打了码的数据集"，逼它们看到不同的世界，最后开会投票**。

### 关键词解释

- **Bootstrap（有放回抽样）**：原数据集 1000 条，抽 1000 次，每次抽完放回。结果某些样本被抽多次、约 36.8% 样本一次没被抽到（OOB, out-of-bag），这部分天然可以做验证集
- **Aggregating（聚合）**：分类用多数票（majority vote），回归取平均
- **平权（equal weight）**：所有弱学习器一票一票算，没有"资深员工票更重"的事

### 工程师视角

像**多副本服务 + 负载均衡**，但每个副本训练时只能看部分日志：

```
原始数据 ──► [bootstrap 抽样 × N] ──► N 份子训练集
                                          │
                                          ▼
                                  并行训练 N 个弱学习器
                                          │
                                          ▼
                          预测时：N 个模型独立打分 → 投票/平均
```

并行性是免费的红利 —— 多核机器直接 `n_jobs=-1`，训练时间几乎线性下降。

### 为什么能涨点

单棵决策树方差（variance）大：换一批数据，长出来的树形状差很多。bagging 让 N 棵树看到不同子集，各自的"过拟合方向"不同，平均后**抵消随机噪声**，方差显著下降，偏差几乎不变。

### 常见误区

- **N 不是越大越好**：边际收益递减，N 到几百就饱和，再加只费 CPU
- **基学习器要"不稳定"才划算**：决策树这种对数据敏感的模型适合，KNN / 线性回归这种本来就稳的模型 bagging 增益很小
