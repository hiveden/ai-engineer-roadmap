### 笔记

> 【理解】bagging 集成

**思想**：有放回抽样（bootstrap）产生 N 个不同训练集 → 各自训练弱学习器 → 平权投票（分类）/ 平均（回归）。

**关键点**：

- 弱学习器之间**独立**，训练可并行
- 抽样制造**差异性**，差异越大集成增益越高
- 投票权重相等（所以叫 "平权"）

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

**业务痛点**：单棵决策树**方差大** —— 换一批训练数据，长出来的树形状差很多，预测跟着变。怎么把这种"看运气"的不稳定性压下去？

**生活锚（接 [`02-分类.md`](./02-分类.md) 的医生类比）**：5 位医生看同一题 CT，但每人手上的**病例库不一样** —— 病例库怎么来？把医院总库的 1000 份病例**有放回**地随机抽 1000 份，重复 5 次，得到 5 份**有重叠但不完全相同**的病例库。各看各的、最后投票。

一句话：**逼每棵树看不一样的数据，最后开会投票**。

### 主线案例：5 棵决策树 + bootstrap 抽样

```
原始训练集 1000 条
   │
   ├─► bootstrap 抽 1000 条 ─► 子集 1（约 632 条不重复 + 部分重复）─► 训练树 1
   ├─► bootstrap 抽 1000 条 ─► 子集 2 ─► 训练树 2
   ├─► bootstrap 抽 1000 条 ─► 子集 3 ─► 训练树 3
   ├─► bootstrap 抽 1000 条 ─► 子集 4 ─► 训练树 4
   └─► bootstrap 抽 1000 条 ─► 子集 5 ─► 训练树 5
                                          │
                                          ▼
预测：5 棵树各打分 ─► 平权投票（分类）/ 平均（回归）
```

下表为**虚构示意数字**，方便对比：

| 模型 | 测试集准确率 | 重训稳定性 |
|---|---|---|
| 单棵决策树 | 78% | 多次重训波动 ±5% |
| Bagging 5 棵 | 86% | 波动 ±1.5% |

→ 不仅准确率提升，**重训稳定性**大幅改善 —— 这是 bagging 的核心收益。

### 关键概念

- **Bootstrap（自助 / 有放回抽样）**：从 N 条数据里**有放回**随机抽 N 次。某些样本被抽多次，约 **36.8%** 样本一次都没被抽到（数学事实：N 大时极限是 1/e），这部分叫 **OOB（Out-of-Bag，袋外样本）**，可天然当验证集，不用另划数据
- **Aggregating（聚合）**：分类用多数投票（majority vote），回归取平均
- **平权（equal weight）**：所有树一棵一票，没有"资深员工票更重"

### 为什么能涨点 —— 方差视角

单棵决策树**方差大**：换一批数据，长出来的树形状差很多 → 预测看运气。
Bagging 让 N 棵树看到不同 bootstrap 子集，各自的"过拟合方向"互不相同 → 平均后**抵消随机噪声**，方差显著下降，**偏差几乎不变**（每棵树学的还是同一个原始问题）。

数字直觉：N 棵**完全独立**的树平均后，方差降为单树的 **1/N**。实际中 bootstrap 子集有重叠，树之间不完全独立，方差降幅打折，但仍显著。

### 常见误区

- **N 不是越大越好**：边际收益递减，N 到几百棵就饱和，再加只费 CPU
- **基学习器要"不稳定"才划算**：决策树对数据敏感、方差大，bagging 收益高；KNN / 线性回归本身方差小，bagging 增益很小
