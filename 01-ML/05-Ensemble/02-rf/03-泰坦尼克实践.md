> Slide 22 · 泰坦尼克号案例（代码 1）

```python
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV

def dm01_随机森林():
    # 1 获取数据集
    titan = pd.read_csv("./data/titanic/train.csv")
    # 2 确定特征值和目标值
    x = titan[["Pclass", "Age", "Sex"]].copy()
    y = titan["Survived"]
    # 3-1 处理数据-处理缺失值
    x['Age'].fillna(value=titan["Age"].mean(), inplace=True)
    print(x.head())
    # 3-2 one-hot 编码
    x = pd.get_dummies(x)
    # 4 数据集划分
    x_train, x_test, y_train, y_test = \
        train_test_split(x, y, random_state=22, test_size=0.2)
```

> **Notes**：模型权重，用于...；样本权重，用于...

```python
    # 5-1 使用决策树进行模型训练和评估

```

## ━━━━━━━━ 讲解 ━━━━━━━━

### 直觉

**业务痛点**：泰坦尼克生存预测在 [`../../04-DecisionTree/05-titanic/`](../../04-DecisionTree/05-titanic/) 用单棵决策树做过 —— 训练 acc ≈ 100%、测试 acc ≈ 75%，**过拟合明显**。换一批样本重训，预测变化大、模型不可靠。生产场景需要**稳定的预测**，单棵树达不到。

**生活锚（接 5 位医生的类比）**：把 891 名乘客的"生存还是遇难"看成一道分类题：

| 单棵树 | 随机森林 |
|---|---|
| 1 位医生看全部病例 + 全部指标 | 100 位医生各看不同病例库，每位每题只看部分指标 |
| 容易钻牛角尖（叶子只剩 1 人 = 死记）| 投票把单点偏见抹平 |
| 测试 acc ≈ 75% | 测试 acc ≈ 82% |

一句话：**同一份泰坦尼克数据，从单棵树 75% → RF 82%，靠的是两层随机 + 投票**。

### 业务问题

数据集 891 行，特征只用 `Pclass / Age / Sex` 三列，目标列 `Survived`（0/1）。要解决三件事：

1. `Age` 列约 20% 缺失 —— 怎么补
2. `Sex` 是字符串 male/female —— 怎么喂给模型
3. 模型怎么调参才能稳过 80% acc

下面用 sklearn 的 6 步 pipeline 把这三件事一次串完。

---

### 【代码】Pipeline 6 步骨架

```
① 读数据  →  ② 选特征/处理缺失  →  ③ One-hot  →
④ train/test split  →  ⑤ 训练+评估  →  ⑥ GridSearchCV 调参
```

完整代码走读：

#### Step 1 · 读数据 + 选特征

```python
titan = pd.read_csv("./data/titanic/train.csv")
x = titan[["Pclass", "Age", "Sex"]].copy()   # 3 个特征
y = titan["Survived"]
```

**为什么只用 3 个特征**：PPT 保持简单，聚焦算法而非特征工程。真实项目还会加 `Fare`、`Embarked`、`SibSp`、`Parch` 等，效果能进一步提升。

#### Step 2 · 缺失值处理

```python
x['Age'].fillna(value=titan["Age"].mean(), inplace=True)
```

`Age` 列约缺失 20%，用全集均值填。

> ⚠️ 注意数据泄漏：`titan["Age"].mean()` 用的是全数据集（含测试集）的均值。严格做法是 `x_train["Age"].mean()` 只用训练集均值，然后同样值填测试集。PPT 教学简化，实际项目用 `sklearn.impute.SimpleImputer` 拼进 Pipeline 避免此问题。

#### Step 3 · One-hot 编码

```python
x = pd.get_dummies(x)
```

`Sex` 列（male/female）展开为 `Sex_female` / `Sex_male` 两列 0/1。`Pclass`（1/2/3）作为数值保留（决策树对单调变换不敏感）。

```
原始                        get_dummies 后
────────────────            ──────────────────────────────────
Pclass  Age   Sex           Pclass  Age   Sex_female  Sex_male
3       22    male          3       22    0           1
1       38    female        1       38    1           0
```

#### Step 4 · 数据集划分

```python
x_train, x_test, y_train, y_test = \
    train_test_split(x, y, random_state=22, test_size=0.2)
```

891 条样本 → 712 训练 / 179 测试，固定种子保证可复现。

#### Step 5 · 训练 + 评估（对比实验）

```python
# 5-1 单棵决策树（基准）
dtc = DecisionTreeClassifier()          # ← 没有任何刹车，必然过拟合
dtc.fit(x_train, y_train)
print('单一决策树 accuracy:', dtc.score(x_test, y_test))
# 典型输出：~0.75

# 5-2 随机森林（固定参数）
rfc = RandomForestClassifier(max_depth=6, random_state=9)
rfc.fit(x_train, y_train)
print('随机森林 accuracy:', rfc.score(x_test, y_test))
# 典型输出：~0.82
```

**为什么 PPT 的 RF 设了 `max_depth=6`**：即使是随机森林，深度不限时每棵树仍会记住训练集噪声；加 `max_depth=6` 是限制单棵树深度，配合 N 棵投票效果最好。

#### Step 6 · GridSearchCV 调超参（5 折版）

PPT 用 `cv=2`（2 折）是为了速度。生产/作业建议改成 5 折：

```python
from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import RandomForestClassifier

estimator = RandomForestClassifier(random_state=9)
param_grid = {
    "n_estimators": [40, 50, 60, 70, 100],
    "max_depth": [2, 4, 6, 8, 10],
}
grid_search = GridSearchCV(
    estimator,
    param_grid=param_grid,
    cv=5,          # 5 折交叉验证
    scoring="accuracy",
    n_jobs=-1,     # 并行跑
)
grid_search.fit(x_train, y_train)

print("最优参数:", grid_search.best_params_)
print("CV 最优 acc:", grid_search.best_score_)
print("测试集 acc:", grid_search.score(x_test, y_test))
```

**网格大小**：5 × 5 = 25 个参数组合 × 5 折 = 125 次训练。这个规模用 `n_jobs=-1` 几秒内跑完。

### 【代码】混淆矩阵 + 特征重要性

PPT 代码只打印 accuracy，实战再加两块：

#### 混淆矩阵

```python
from sklearn.metrics import confusion_matrix, classification_report

best_rfc = grid_search.best_estimator_
y_pred = best_rfc.predict(x_test)

print(confusion_matrix(y_test, y_pred))
print(classification_report(y_test, y_pred, target_names=["died", "survived"]))
```

混淆矩阵读法：

```
                预测 died    预测 survived
实际 died   [   TN        FP   ]    ← 行 = 实际类别
实际 survived [  FN        TP   ]

典型结果（RF max_depth=6）：
              predicted
actual        died  survived
died           99     11        ← 11 个实际死亡被预测为生还（FP）
survived       22     47        ← 22 个实际生还被预测为死亡（FN）

acc = (99+47) / 179 ≈ 0.816
```

> 泰坦尼克数据类别不平衡（死亡 ~61% / 生还 ~39%），光看 accuracy 不够——还要看 recall（生还者里识别出几个）。

#### 特征重要性

```python
import matplotlib.pyplot as plt
import numpy as np

importances = best_rfc.feature_importances_
feature_names = x_train.columns.tolist()   # ['Pclass', 'Age', 'Sex_female', 'Sex_male']

# 排序后画横向柱状图
idx = np.argsort(importances)
plt.figure(figsize=(6, 3))
plt.barh(np.array(feature_names)[idx], importances[idx])
plt.xlabel("Feature Importance (Gini Impurity Decrease)")
plt.title("Random Forest Feature Importances - Titanic")
plt.tight_layout()
plt.show()
```

预期结果（典型）：

```
特征重要性（从高到低）：
Sex_female  ██████████████████  0.38    ← 性别是最强信号（"女士优先"）
Sex_male    ████████████        0.24
Age         ██████████          0.20
Pclass      ████████            0.18

→ 性别（Sex_female + Sex_male）合计约 62%，符合历史常识
```

### 单棵树 vs RF 对比总结（虚构示意）

| 维度 | 单棵 DecisionTree | 随机森林 |
|---|---|---|
| 测试 acc（典型） | ~75% | ~82% |
| 过拟合风险 | 高（不限深度则训练 100%） | 低（投票平均掉方差） |
| 可解释性 | 高（plot_tree 可视化） | 低（N 棵树无法直接画） |
| 训练速度 | 快 | 慢（N 倍） |
| 特征重要性 | 有（但单棵树不稳定） | 有（更稳定，N 棵平均） |
| 调参复杂度 | 低（3 个刹车） | 中（多 `n_estimators` + `max_features`） |

**工程决策**：泰坦尼克这类小数据集（< 1k）RF 往往比单棵树高 5-8% acc，且调参简单，是默认首选。可解释性要求极高时（如要给监管看每条规则）才退回单棵树。

### 常见误区

- **`titan["Age"].mean()` 用全集填缺失值**：数据泄漏 —— 测试集统计量混进训练流程。生产做法用训练集均值或 `SimpleImputer` 接 Pipeline
- **GridSearchCV 用 `cv=2`**：PPT 为速度示范，作业/生产至少 5 折，否则 CV 估计方差大
- **看 accuracy 拍板**：泰坦尼克类别不平衡（死亡 ~61% / 生还 ~39%），还得看 recall 和混淆矩阵
