> Slide 31-32 · Adaboost 算法（第 2、3 个弱学习器 + 整体过程）

AdaBoost 通过迭代训练一系列弱分类器，每一轮根据前一轮的表现，动态调整样本权重——分错的样本被"加重"，分对的被"减轻"，让后续分类器更关注难例。

**每个模型的发言权**（模型权重 α）：误差小的模型发言权大，误差大的模型发言权小。

### 笔记

> 【知道】AdaBoost 算法简介

**Adaptive Boosting**：基于 Boosting 思想，通过**逐步提高被前一轮分类错误样本的权重**来训练强分类器。

**两个动态权重**：

| 对象 | 调整方式 |
|---|---|
| **样本权重** $D_t$ | 错分样本权重 ↑，正确样本权重 ↓ |
| **学习器权重** $\alpha_t$ | 误差率小 → 权重大；误差率大 → 权重小 |

**核心步骤**：

- **权值调整**：错分样本权重提高，让后续学习器更关注难样本
- **加权多数表决**：好学习器权重大，差的小

**同质要求**：所有基学习器必须同类型（如都是决策桩 / 都是 LR）；混搭不同类型称为"异构集成（heterogeneous ensemble）"，不属于 AdaBoost。

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题：一个弱分类器为什么不够

设想一个人脸/非人脸检测器。用一个简单规则——"眼睛区域色差大 → 是人脸"——单独上线，假阳性率奇高。再加一个规则——"鼻梁处有竖向高光"——单独也很弱。两个加起来好一些，但还是不够。

这是 **弱分类器（weak classifier）** 的典型困境：

```
单个弱分类器          多个弱分类器组合
────────────         ─────────────────────────────
准确率略高于随机猜   → 串行纠错，组合后逼近强分类器
  （比如 60%）          （目标：>90%）
```

关键问题：多个弱分类器怎么"串行纠错"？AdaBoost 的答案是 **用样本权重传递信息**。

### 加法模型直觉

AdaBoost 的最终输出是：

$$H(x) = \text{sign}\left(\sum_{t=1}^{m} \alpha_t h_t(x)\right)$$

中文直读：把 $m$ 个弱分类器的输出 $h_t(x)$，按各自的"发言权" $\alpha_t$ 加权求和，取符号（正 → 正类，负 → 负类）。

这是**加法模型（additive model）**：一系列基础模型的加权叠加。

```
最终判断
   ↑
   α₁ × [弱分类器1 说 +1/-1]
 + α₂ × [弱分类器2 说 +1/-1]
 + α₃ × [弱分类器3 说 +1/-1]
 + ...
   ↓
  > 0 → 正类
  < 0 → 负类
```

**类比投票**：不是一人一票，而是能力强（误差率低）的专家票权重更大。

### 三个核心概念

#### 1. 样本权重 $D_t(i)$

第 $t$ 轮时，第 $i$ 个样本的"重要程度"，初始均匀 = $\frac{1}{N}$。

**物理意义**：训练下一个分类器时，权重大的样本相当于"被更多次采样"——分类器更努力拟合它们。

#### 2. 弱分类器权重 $\alpha_t$

第 $t$ 个弱分类器的"发言权"，由它的误差率决定（公式详见 `02-推导.md`）。

- 误差率越低 → $\alpha_t$ 越大 → 发言权越大
- 误差率接近 0.5（和随机猜差不多）→ $\alpha_t$ 接近 0 → 发言权极小

#### 3. 误差率 $\varepsilon_t$

第 $t$ 轮中，以**当前样本权重**加权后的错误率：

$$\varepsilon_t = \sum_{i=1}^{N} D_t(i) \cdot \mathbf{1}[h_t(x_i) \neq y_i]$$

符号扫盲：

| 符号 | 读法 | 直觉 |
|---|---|---|
| $\varepsilon_t$ | epsilon-t | 第 $t$ 轮弱分类器的加权错误率 |
| $D_t(i)$ | D-t of i | 第 $t$ 轮第 $i$ 个样本的权重 |
| $\mathbf{1}[\cdot]$ | indicator function（示性函数） | 括号内条件为真 = 1，为假 = 0 |
| $h_t(x_i)$ | h-t of x-i | 第 $t$ 个弱分类器对样本 $i$ 的预测 |
| $y_i$ | y-i | 样本 $i$ 的真实标签 |

**注意**：$\varepsilon_t$ 不是"错了几个"，而是"错的样本的权重之和"——权重大的样本分错，误差率升得更狠。

### 与 Bagging 的本质差异

AdaBoost 和 Random Forest（Bagging 类）都是集成多个弱分类器，但串并行机制完全不同：

```
Bagging（随机森林）             AdaBoost（串行纠错）
────────────────────          ────────────────────────────
并行：各树独立训练              串行：第 t+1 棵树在第 t 棵的"失误信息"上训练
重采样：Bootstrap 有放回抽样    重加权：改变样本权重，不重采样
目标：降方差（减少过拟合）       目标：降偏差（逼近强分类器）
树之间无依赖关系               树之间强依赖：每棵树依赖前一棵的结果
```

一句话：**Bagging 是并行民主投票，AdaBoost 是串行专家会诊——后一位专家重点看前一位的漏诊**。

### 同质要求

AdaBoost 要求所有弱分类器必须**同类型**（homogeneous）：

- 全是 `DecisionTreeClassifier(max_depth=1)`（决策桩 decision stump）
- 全是 `LogisticRegression`

混搭不同类型叫"**异构集成（heterogeneous ensemble）**"，AdaBoost 框架不支持。

最常见的基学习器：**决策桩**（max_depth=1 的决策树）——每个分类器只问一个问题（"身高 > 170？"），够弱但不太弱。

### 一句话钉板

**AdaBoost = 串行 + 重加权 + 加权投票：每轮在上一轮的"错题"上加重注意力训练新分类器，最终让误差小的分类器发言权大。**
