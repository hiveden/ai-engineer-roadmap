## ━━━━━━━━ 讲解 ━━━━━━━━

### GBDT 用在分类：损失函数换了，框架不变

前面两章（01、02）讲的是回归——预测连续值，损失函数用平方损失，负梯度就是残差。

泰坦尼克是**二分类**（Survived = 0/1）。GBDT 换一个损失函数（logloss / 对数损失），框架完全不变：

```
GBDT 回归                        GBDT 分类
────────────────                 ────────────────
损失函数：平方损失                损失函数：对数损失（logloss）
负梯度：y - f(x)（残差）          负梯度：y - p（真实标签 - 预测概率）
输出：连续值                      输出：概率（经 sigmoid 变换）
```

sklearn 的 `GradientBoostingClassifier` 默认用对数损失，用户不用手动处理，传入标签直接跑。

### 关键参数扫盲

| 参数 | 默认值 | 直觉 | 调参方向 |
|---|---|---|---|
| `n_estimators` | 100 | 弱学习器（树）的个数；越多越准但越慢 | 先粗调：50/100/200；再精调 |
| `learning_rate` | 0.1 | 每棵树贡献的缩放系数（shrinkage）；小则稳，需配合大 n_estimators | 0.05-0.3；通常与 n_estimators 联动 |
| `max_depth` | 3 | 每棵树的最大深度；GBDT 弱树不需要很深 | 3-5；太深会过拟合 |
| `subsample` | 1.0 | 每轮随机抽多少比例样本建树（随机梯度）；<1.0 引入随机性防过拟 | 0.6-0.9，配合小 learning_rate |
| `min_samples_split` | 2 | 节点继续分裂的最少样本数 | 增大可防过拟合 |
| `random_state` | None | 随机种子，保证可复现 | 固定一个值即可 |

**`learning_rate` 和 `n_estimators` 是跷跷板**：

```
learning_rate 小  ←→  n_estimators 大
learning_rate 大  ←→  n_estimators 小

经验：先固定 learning_rate=0.1，调 n_estimators；
      确定棵数后再调小 learning_rate、增大棵数做精调
```

### 完整 Pipeline（按 sklearn 标准模板补全）

PPT 代码片段拼合成可运行的完整版本。数据预处理与 PPT 保持一致（Pclass + Age + Sex，Age 均值填充，get_dummies 编码）。

```python
import pandas as pd
from sklearn.model_selection import train_test_split, GridSearchCV
from sklearn.ensemble import GradientBoostingClassifier, RandomForestClassifier, AdaBoostClassifier
from sklearn.metrics import classification_report

# ── 1. 读数据 ──────────────────────────────
df = pd.read_csv('./data/titanic/train.csv')

# ── 2. 特征工程 ────────────────────────────
x = df[['Pclass', 'Age', 'Sex']].copy()
y = df['Survived'].copy()

# 缺失值：Age 用均值填充（PPT 原方案）
x['Age'].fillna(x['Age'].mean(), inplace=True)

# 类别特征：Sex（male/female） + Pclass（1/2/3）用 get_dummies
# get_dummies 会把 Sex 拆成 Sex_female / Sex_male 两列
x = pd.get_dummies(x)

# ── 3. 划分训练/测试集 ─────────────────────
x_train, x_test, y_train, y_test = train_test_split(
    x, y, random_state=22, test_size=0.2
)

# ── 4. 默认参数跑一遍（基线） ───────────────
gbdt = GradientBoostingClassifier(random_state=42)
gbdt.fit(x_train, y_train)
print(f"GBDT 默认参数准确率: {gbdt.score(x_test, y_test):.4f}")

# ── 5. 网格搜索调参（PPT 方案） ─────────────
param_grid = {
    "n_estimators": [100, 110, 120, 130],
    "max_depth": [2, 3, 4],
    "random_state": [9],
}
gs = GridSearchCV(
    GradientBoostingClassifier(),
    param_grid=param_grid,
    cv=3,
    n_jobs=-1,   # 并行加速
    verbose=1,
)
gs.fit(x_train, y_train)
print(f"GBDT 调参后准确率:   {gs.score(x_test, y_test):.4f}")
print(f"最优参数:             {gs.best_params_}")

# ── 6. 与 RF / AdaBoost 对比 ───────────────
models = {
    "Random Forest": RandomForestClassifier(n_estimators=100, random_state=42),
    "AdaBoost":      AdaBoostClassifier(n_estimators=100, random_state=42),
    "GBDT":          GradientBoostingClassifier(n_estimators=100, random_state=42),
}
print("\n── 模型准确率对比 ──")
for name, model in models.items():
    model.fit(x_train, y_train)
    score = model.score(x_test, y_test)
    print(f"  {name:<16}: {score:.4f}")
```

**预期输出**（数值因随机种子略有浮动，趋势稳定）：

```
GBDT 默认参数准确率: ~0.816
GBDT 调参后准确率:   ~0.821

── 模型准确率对比 ──
  Random Forest   : ~0.804
  AdaBoost        : ~0.793
  GBDT            : ~0.816
```

三种 Boosting/Bagging 模型在同一数据集上，GBDT 通常略优，但差距不大——差别在 1-3 个百分点。

### GBDT vs RF vs AdaBoost：工程对比

| 维度 | Random Forest（RF） | AdaBoost | GBDT |
|---|---|---|---|
| 训练方式 | **并行**（各树独立） | **串行**（依赖上一轮权重） | **串行**（依赖上一轮残差） |
| 训练速度 | 快（可多核） | 中 | 慢（纯串行，无法并行） |
| 推理速度 | 快 | 快 | 快（推理时各树独立） |
| 防过拟合机制 | 随机特征 + Bagging | 样本权重 + 早停 | learning_rate + max_depth + subsample |
| 离群点鲁棒性 | 一般 | 差（离群点被反复加权） | 可选 Huber/MAE 损失 |
| 准确率上限 | 中高 | 中 | **高**（工业界基准之一） |
| 调参难度 | 低（n_estimators + max_features） | 低 | 中（多参数联动） |
| 可解释性 | 中（feature importance） | 中 | 中（feature importance） |

```
训练流程对比

RF：  tree₁ ──┐
      tree₂ ──┼── 投票/平均 → 输出
      tree₃ ──┘
      （并行，各树独立，快）

GBDT：f₀ → tree₁ → f₁ → tree₂ → f₂ → tree₃ → ...
      （串行，每棵树依赖前一轮结果，慢）
      
推理时 GBDT 和 RF 一样快：都是过 T 棵树然后累加/平均。
慢在训练，不在推理。
```

**工程结论**：数据量不大、追求精度 → GBDT；数据量大、追求训练速度 → RF；GBDT 精度想要但训练太慢 → 下一章 XGBoost（GBDT 工程化升级，支持并行）。

### 一句话钉板

**`GradientBoostingClassifier` 把 GBDT 的损失函数换成 logloss，其余框架不变；关键参数是 n_estimators × learning_rate 的跷跷板；GBDT 串行训练比 RF 慢但准确率更高，XGBoost 就是为了解决这个训练慢问题。**
