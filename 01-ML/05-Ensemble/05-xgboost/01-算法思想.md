### PPT

> Slide 69 · XGBoost 思想（引入）

**XGBoost（Extreme Gradient Boosting）**：极端梯度提升树，集成学习方法的王牌，在数据挖掘比赛中，大部分获胜者用了 XGBoost。

**Xgb 的构建思想**：

1. 构建模型的方法是最小化训练数据的损失函数：训练的模型复杂度较高，易过拟合。
2. 在损失函数中加入正则化项，提高对未知的测试数据的泛化性能。

> Slide 70 · XGBoost 正则化项

**XGBoost（Extreme Gradient Boosting）**：是对 GBDT 的改进，并且在损失函数中加入了正则化项

正则化项用来降低模型的复杂度

$$\Omega(f) = \gamma T + \frac{1}{2}\lambda \|w\|^2$$

- $\gamma T$：$T$ 表示一棵树的叶子结点数量
- $\lambda$：$w$ 表示叶子结点输出值组成的向量，向量的模；$\lambda$ 对该项的调节系数

> Slide 18（补充）· XGBoost Loss 的复杂性

但是问题是 XGBoost 的 loss 更为复杂：

$$\text{XGBoost loss} = \text{损失} + \text{正则项}$$

（不一定是平方损失）+（限制模型复杂度）

也希望将该 loss 降到最低。但是存在正则项，难以求负梯度，从而优化困难。

**对策：化繁为简**——利用泰勒展开公式来近似该 loss 表达式。

### 笔记

> 【知道】XGBoost 算法思想

XGBoost 是对 GBDT 的三点改进：

1. **泰勒二阶展开**求解损失（GBDT 只用一阶）
2. **加正则项**控制树复杂度（防过拟合）
3. **自创分裂指标**（从损失函数推导，分裂时考虑复杂度增益）

## ━━━━━━━━ 讲解 ━━━━━━━━

### 业务问题：GBDT 在工程上卡在哪里

GBDT（上游 [`../04-gbdt/`](../04-gbdt/)）本质是串行 Boosting：第 $t$ 棵树拟合前 $t-1$ 棵树的残差，树之间有严格的依赖关系。这在理论上优雅，但上到工业场景时暴露 3 个工程痛点：

```
GBDT 的工程痛点
──────────────────────────────────────────────────────
痛点 1：无内置正则 = 容易过拟合
  目标函数只有"损失最小化"，没有对树结构的约束
  叶子数 / 叶子值 无限自由 → 模型随意复杂 → 训练好测试差

痛点 2：分裂准则缺统一的理论依据
  残差拟合的分裂没有明确的"增益 > 阈值才分裂"机制
  何时停止？靠 max_depth / min_samples_leaf 人工限制

痛点 3：只用一阶信息（负梯度），近似精度有限
  一阶近似只知道"往哪方向走"
  不知道"走多远最优"——步长靠 learning_rate 人工控制
```

XGBoost 2014 年由陈天奇提出，核心思路是：**把正则项直接写进目标函数，从目标函数推导出所有东西——叶子值、分裂准则、停止条件，全部数学导出**。

### 4 大改进

#### 改进 1：二阶泰勒展开——拟合更精确

GBDT 优化损失时，用**负梯度**（一阶导 $g_i$）拟合残差，是一阶泰勒近似。XGBoost 同时使用一阶导 $g_i$ 和**二阶导** $h_i$：

```
一阶（GBDT）:  f(x + Δx) ≈ f(x) + f'(x)Δx
              知道"往哪走"，不知道"走多远合适"

二阶（XGB）:   f(x + Δx) ≈ f(x) + f'(x)Δx + ½f''(x)Δx²
              知道"往哪走"，也知道"曲率"
              → 闭式求出叶子最优值 w* = -G/(H+λ)
```

直觉：一阶用直线逼近损失曲面，二阶用抛物线——弯曲的曲面上，抛物线更贴合，最优点有解析式。

#### 改进 2：目标函数加正则项——树的"复杂度税"

$$\text{Obj} = \underbrace{\sum_{i=1}^n L(y_i, \hat{y}_i)}_{\text{拟合损失}} + \underbrace{\Omega(f)}_{\text{复杂度惩罚}}$$

$$\Omega(f) = \gamma T + \frac{1}{2}\lambda \|w\|^2$$

| 正则项 | 含义 | 效果 |
|---|---|---|
| $\gamma T$ | 叶子数 $T$ × 惩罚系数 $\gamma$ | 少长叶子（树更矮） |
| $\frac{1}{2}\lambda \lVert w \rVert^2$ | 叶子值 L2 范数 | 叶子输出值不能太极端 |

$\gamma$ 和 $\lambda$ 都是超参数。过拟合时直接调大，比 GBDT 更可控。

#### 改进 3：数学推导的分裂 Gain——增益有保证

XGBoost 从目标函数直接推导出分裂增益（推导见 [`02-目标函数.md`](./02-目标函数.md)）：

$$\text{Gain} = \frac{1}{2}\left[\frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L + G_R)^2}{H_L + H_R + \lambda}\right] - \gamma$$

只有 $\text{Gain} > 0$ 才分裂。增益里已经减掉 $\gamma$，所以**停止条件是内置的**，不需要额外人为设阈值。

#### 改进 4：工程优化——列采样 + 并行直方图

```
列采样（colsample_bytree）
  每棵树随机只选 k% 特征参与分裂
  → 类似随机森林，减少计算 + 增加多样性

直方图分桶（histogram approximation）
  连续值离散成 256 个桶，不枚举所有切分点
  → 枚举从 O(样本数) → O(256)，大幅提速
  → 各特征直方图可并行计算
```

### RF / GBDT / XGBoost 三足鼎立

```
              随机森林 RF          GBDT               XGBoost
─────────────────────────────────────────────────────────────────
策略          Bagging             Boosting            Boosting
树间关系      独立（并行训练）     串行（依赖残差）    串行（依赖 gᵢ/hᵢ）
正则化        无（靠随机性）       无内置正则          内置 γT + λ‖w‖²
分裂准则      基尼 / 信息增益      负梯度（一阶）      二阶泰勒推导的 Gain
叶子值        多数类 / 均值        残差均值            -Gⱼ/(Hⱼ+λ)
并行度        高（树间完全独立）   低（串行依赖）      中（特征级可并行）
调参难度      低                  中                  高（参数多）
过拟合风险    低-中（随机性兜底）  中（无正则刹车）    低（正则可调）
典型场景      特征多 / 噪声大      连续值回归          竞赛 / 高精度场景
```

**一句话区分**：RF 靠随机多数投票，GBDT 靠一步步修残差，XGBoost 靠把正则和二阶导一起解出的数学最优。

### 一句话钉板

**XGBoost = GBDT + 二阶泰勒近似 + 目标函数内置正则 + 数学推导的分裂 Gain。三个改进围绕同一个目标：把"目标函数最小化"做得更彻底、更可控。**
