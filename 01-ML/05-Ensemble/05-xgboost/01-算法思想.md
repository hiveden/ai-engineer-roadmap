### PPT

> Slide 69 · XGBoost 思想（引入）

**XGBoost（Extreme Gradient Boosting）**：极端梯度提升树，集成学习方法的王牌，在数据挖掘比赛中，大部分获胜者用了 XGBoost。

**Xgb 的构建思想**：

1. 构建模型的方法是最小化训练数据的损失函数：训练的模型复杂度较高，易过拟合。
2. 在损失函数中加入正则化项，提高对未知的测试数据的泛化性能。

> Slide 70 · XGBoost 正则化项

**XGBoost（Extreme Gradient Boosting）**：是对 GBDT 的改进，并且在损失函数中加入了正则化项

正则化项用来降低模型的复杂度

$$\Omega(f) = \gamma T + \frac{1}{2}\lambda \|w\|^2$$

- $\gamma T$：$T$ 表示一棵树的叶子结点数量
- $\lambda$：$w$ 表示叶子结点输出值组成的向量，向量的模；$\lambda$ 对该项的调节系数

> Slide 18（补充）· XGBoost Loss 的复杂性

但是问题是 XGBoost 的 loss 更为复杂：

$$\text{XGBoost loss} = \text{损失} + \text{正则项}$$

（不一定是平方损失）+（限制模型复杂度）

也希望将该 loss 降到最低。但是存在正则项，难以求负梯度，从而优化困难。

**对策：化繁为简**——利用泰勒展开公式来近似该 loss 表达式。

### 笔记

> 【知道】XGBoost 算法思想

XGBoost 是对 GBDT 的三点改进：

1. **泰勒二阶展开**求解损失（GBDT 只用一阶）
2. **加正则项**控制树复杂度（防过拟合）
3. **自创分裂指标**（从损失函数推导，分裂时考虑复杂度增益）

## ━━━━━━━━ 讲解 ━━━━━━━━

### 生活锚 · 升级版接力医生：看方向也看趋势，还设开方上限

延续 [`../01-intro/02-分类.md`](../01-intro/02-分类.md) 的医生会诊场景。GBDT（[`../04-gbdt/`](../04-gbdt/)）那一棒接力医生，每位**只看上一位漏掉的差距**（残差 residual），照差距往一个方向开方——这相当于看「方向」。

XGBoost 给同一群医生升级三件装备：

- **多看一层趋势**：除「差距方向」（一阶导 first-order gradient），再看「差距收敛得快不快」（二阶导 second-order Hessian）——方向加趋势，下手更准
- **开方上限**：每位医生写处方前，先按"用药种类数 × 剂量大小"扣一笔分；想开重药就得抗住扣分（正则项 regularization term）
- **流水线优化**：化验单提前预排序、桶装分发，几位医生同时跑各自的化验台，互不干扰（特征级并行 feature-level parallelism）

一句话：**XGBoost = GBDT 升级版——方向看一阶导，趋势看二阶导，开方设正则，流水线还并行**。本节把这套升级到底加了什么、为什么有效讲清楚。

### 业务问题：GBDT 跑到生产环境会卡哪几下

GBDT 在 Kaggle 早期很能打，但 2014 年前后工业数据量越拉越大、对泛化越拉越严，几个真痛点冒出来（数字均为**虚构示意**）：

- **过拟合刹不住车**：风控同事用 500 棵树拟合违约样本，训练 AUC 0.95，上线测试集掉到 0.72——树长太深、叶子值太大没人管
- **何时该停拍脑袋**：业务方问「为什么 max_depth=6 不是 7」，工程师只能说「调出来的」——没数学依据
- **走多远靠手调**：learning_rate 调小到 0.01 才稳，但 1000 棵树训练一晚才出结果，速度顶不住
- **大数据卡住**：千万样本枚举所有切分点，单棵树几小时——产品 PM 不答应

陈天奇 2014 年提出 XGBoost，定调一句话：**把正则项直接焊进目标函数，叶子值 / 分裂准则 / 停止条件全部数学导出，再配一套工程优化把速度榨干**。下面把四件升级展开。

### 【数学】4 大改进

#### 改进 1：二阶泰勒展开——既看方向也看曲率

GBDT 拟合残差用**一阶导 first-order gradient $g_i$**（损失曲面的"坡度"），是一阶泰勒近似 first-order Taylor approximation。XGBoost 同时用 $g_i$ 和**二阶导 Hessian $h_i$**（损失曲面的"曲率"）：

```
一阶（GBDT）：f(x + Δx) ≈ f(x) + f'(x)·Δx
            知道"往哪走"，不知道"走多远合适"

二阶（XGB）：f(x + Δx) ≈ f(x) + f'(x)·Δx + ½·f''(x)·Δx²
            既知"往哪走"，又知"曲率多大"
            → 叶子最优值闭式可解：w* = -G/(H+λ)
```

直觉：一阶用直线逼近损失曲面，二阶用抛物线——弯曲面上抛物线贴得更紧，最优点有解析式直接算出来，不像 GBDT 还要靠 learning_rate 一点点试。

#### 改进 2：目标函数加正则项——给"开重药"扣分

$$\text{Obj} = \underbrace{\sum_{i=1}^n L(y_i, \hat{y}_i)}_{\text{拟合损失}} + \underbrace{\Omega(f)}_{\text{复杂度惩罚}}$$

$$\Omega(f) = \gamma T + \frac{1}{2}\lambda \|w\|^2$$

| 正则项 | 含义 | 效果 |
|---|---|---|
| $\gamma T$ | 叶子数 $T$ × 惩罚系数 $\gamma$ | 少长叶子（树更矮） |
| $\frac{1}{2}\lambda \lVert w \rVert^2$ | 叶子值 L2 范数 | 叶子输出值不能太极端 |

$\gamma$ 和 $\lambda$ 都是超参数 hyperparameter。生活锚里"开方上限"对应的就是这两项——叶子越多扣 $\gamma$，剂量越猛扣 $\lambda$。过拟合时直接调大即可，比 GBDT 多了正则刹车。

#### 改进 3：数学推导的分裂 Gain——停车条件内置

XGBoost 从目标函数直接推导出分裂增益 split gain（推导见 [`./02-目标函数.md`](./02-目标函数.md)）：

$$\text{Gain} = \frac{1}{2}\left[\frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L + G_R)^2}{H_L + H_R + \lambda}\right] - \gamma$$

只有 $\text{Gain} > 0$ 才分裂。增益里已经减掉 $\gamma$，**停止条件是数学内置的**，不靠手设 max_depth 临时挡——这点对工业调参成本意义重大。

#### 改进 4：工程优化——列采样 + 直方图 + 并行

```
列采样 column subsampling（colsample_bytree）
  每棵树随机只选 k% 特征参与分裂
  → 类似随机森林，减少计算 + 增加多样性

直方图分桶 histogram approximation
  连续值离散成 256 个桶，不枚举所有切分点
  → 枚举从 O(样本数) → O(256)，大幅提速

特征级并行 feature parallelism
  各特征的直方图可并行计算
  注意：树之间仍是串行的 boosting，并非树间并行

缺失值自动处理 sparsity-aware split
  分裂时同时尝试"缺失走左 / 缺失走右"，选 Gain 高的方向，不用预先填充
```

XGBoost 的并行**不是树之间并行**（boosting 框架决定树必须串行），而是单棵树构建时各特征直方图并行——这点常被误解。

### RF / GBDT / XGBoost 三足鼎立

```
              随机森林 RF          GBDT               XGBoost
─────────────────────────────────────────────────────────────────
策略          Bagging             Boosting            Boosting
树间关系      独立（并行训练）     串行（依赖残差）    串行（依赖 gᵢ/hᵢ）
正则化        无（靠随机性）       无内置正则          内置 γT + λ‖w‖²
分裂准则      基尼 / 信息增益      负梯度（一阶）      二阶泰勒推导的 Gain
叶子值        多数类 / 均值        残差均值            -Gⱼ/(Hⱼ+λ)
并行度        高（树间完全独立）   低（串行依赖）      中（特征级可并行）
调参难度      低                  中                  高（参数多）
过拟合风险    低-中（随机性兜底）  中（无正则刹车）    低（正则可调）
典型场景      特征多 / 噪声大      连续值回归          竞赛 / 高精度场景
```

**一句话区分**：RF 靠随机多数投票，GBDT 靠一步步修残差，XGBoost 靠正则和二阶导一起解出的数学最优。
