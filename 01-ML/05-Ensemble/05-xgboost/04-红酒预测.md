```python
from sklearn.utils import class_weight

def dm03_训练模型():
    # 1 加载数据集
    train_data = pd.read_csv('./data/红酒品质分类-train.csv')
    test_data = pd.read_csv('./data/红酒品质分类-test.csv')

    # 2 准备数据
    x_train = train_data.iloc[:, :-1]
    y_train = train_data.iloc[:, -1]
    x_test = test_data.iloc[:, :-1]
    y_test = test_data.iloc[:, -1]

    # 2-2 样本不均衡问题处理
    classes_weights = class_weight.compute_sample_weight(
        class_weight='balanced', y=y_train
    )

    # 3 xgb 模型训练
    estimator = xgb.XGBClassifier(
        n_estimators=100, objective='multi:softmax',
        eval_metric='merror', eta=0.1,
        use_label_encoder=False, random_state=22
    )
    # 训练的时候，指定样本的权重
    estimator.fit(x_train, y_train, sample_weight=classes_weights)

    # 4 xgb 模型评估
    y_pred = estimator.predict(x_test)
    print(classification_report(y_true=y_test, y_pred=y_pred))
```

## ━━━━━━━━ 讲解 ━━━━━━━━

### 数据：先看清楚再建模

**基本信息**：
- 11 个化学特征（固定酸度 / 挥发酸度 / 柠檬酸 / 残留糖分 / 氯化物 / 游离二氧化硫 / 总二氧化硫 / 密度 / pH / 硫酸盐 / 酒精度）
- 标签：品质 3-8（PPT 代码里 `-3` 变换为 0-5，便于 XGBoost 多分类）
- 3269 条数据，6 类

**样本不均衡**（这是本案例的核心工程坑）：

```
品质等级分布（示意，真实数据差异更大）
─────────────────────────────────────────
品质 3（极少）  ████  ~30 条
品质 4          ████████  ~200 条
品质 5          ████████████████████  ~1300 条   ← 主力
品质 6          ██████████████████  ~1100 条    ← 主力
品质 7          ████████  ~300 条
品质 8（极少）  ██  ~18 条
```

如果不处理不均衡，模型会学偏——预测"5 或 6"准确率高，但"3 和 8"全猜错，分类报告里 macro avg 严重拉低。

### 完整 Pipeline

整体流程 4 步：

```
dm01: 读数据 + 标签变换 + train/test 分割 + 存 CSV
          ↓
dm02: 基础训练（不处理不均衡）→ 看基线准确率
          ↓
dm03: 加 sample_weight（处理不均衡）→ 改善少数类召回率
          ↓
dm04: GridSearchCV 调参 → 找最优超参数组合
```

#### Step 1：数据预处理

```python
import pandas as pd
from sklearn.model_selection import train_test_split

data = pd.read_csv('./data/红酒品质分类.csv')
X = data.iloc[:, :-1]           # 11 个特征
y = data.iloc[:, -1] - 3        # 标签 3-8 → 0-5（XGBoost 多分类要求从 0 开始）

# stratify=y 确保训练集 / 测试集各类比例一致
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=22
)
print(X_train.shape, y_train.value_counts())
```

**为什么 stratify=y？** 数据极度不均衡时，随机分割可能导致测试集里某个类别样本数为 0，StratifiedSplit 保证每类在训练 / 测试集里的比例稳定。

#### Step 2：基础训练（基线）

```python
from xgboost import XGBClassifier
from sklearn.metrics import classification_report

estimator = XGBClassifier(
    n_estimators=100,
    objective='multi:softmax',   # 多分类：softmax 直接输出类别
    num_class=6,                 # 类别数（0-5）
    eval_metric='merror',        # merror = 多分类错误率
    learning_rate=0.1,           # PPT 用的是 eta=0.1，是 learning_rate 原生别名
    use_label_encoder=False,     # 新版 XGBoost 必须关掉，否则警告
    random_state=22,
    n_jobs=-1,
)
estimator.fit(X_train, y_train)

y_pred = estimator.predict(X_test)
print(classification_report(y_test, y_pred))
```

基线输出大概率显示：品质 5/6 的 precision/recall 很高，品质 3/8 接近 0。

#### Step 3：sample_weight 处理不均衡

```python
from sklearn.utils import class_weight
import numpy as np

# compute_sample_weight 给每个训练样本分配权重
# 少数类样本 → 权重大；多数类样本 → 权重小
weights = class_weight.compute_sample_weight(class_weight='balanced', y=y_train)

estimator.fit(X_train, y_train, sample_weight=weights)
y_pred = estimator.predict(X_test)
print(classification_report(y_test, y_pred))
```

**`compute_sample_weight` 背后的逻辑**：

```
权重 = 总样本数 / (类别数 × 该类样本数)

若品质 5 有 1000 条，品质 3 有 30 条，共 2000 条，6 类：
  w(品质 5) = 2000 / (6 × 1000) ≈ 0.33   ← 权重小（多数类）
  w(品质 3) = 2000 / (6 × 30) ≈ 11.1    ← 权重大（少数类）
```

相当于让少数类样本在损失计算里"占更多话语权"，强迫模型关注它们。

#### Step 4：GridSearchCV 调参

```python
from sklearn.model_selection import GridSearchCV, StratifiedKFold

param_grid = {
    'n_estimators': [100, 200, 300],
    'max_depth':    [3, 5, 7],
    'learning_rate': [0.05, 0.1, 0.2],
}

# 不均衡数据用 StratifiedKFold，保证每折类别比例稳定
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

grid_search = GridSearchCV(
    estimator=XGBClassifier(
        objective='multi:softmax',
        num_class=6,
        eval_metric='merror',
        use_label_encoder=False,
        random_state=22,
        n_jobs=-1,
    ),
    param_grid=param_grid,
    cv=cv,
    scoring='f1_macro',   # 不均衡场景用 macro F1，不用 accuracy
    n_jobs=-1,
    verbose=1,
)

grid_search.fit(X_train, y_train, sample_weight=weights)
print("最优参数:", grid_search.best_params_)
print("最优 CV F1:", grid_search.best_score_)

best_model = grid_search.best_estimator_
y_pred = best_model.predict(X_test)
print(classification_report(y_test, y_pred))
```

**为什么用 f1_macro 而不是 accuracy？**

```
accuracy = (预测对的样本数) / (总样本数)

不均衡时 accuracy 会骗人：
  3269 条样本里品质 5/6 占 70%
  模型全猜"5"→ accuracy ≈ 70%（看起来不错）
  但品质 3/4/7/8 全错

macro F1 = 每类 F1 取简单平均（不按样本数加权）
  品质 3 的 F1 = 0 → macro F1 直接被拉低
  → 强迫模型在少数类上也表现好
```

#### 特征重要性可视化

XGBoost 内置特征重要性，三种计算方式：

```python
import matplotlib.pyplot as plt
import xgboost as xgb

# 方式 1：xgb 原生 plot（快速查看）
xgb.plot_importance(best_model, max_num_features=11, importance_type='gain')
plt.tight_layout()
plt.show()

# 方式 2：从 estimator 取值，自己画（更可控）
importance = pd.Series(
    best_model.feature_importances_,   # 默认 gain
    index=X.columns,
).sort_values(ascending=False)

importance.plot(kind='bar', title='Feature Importance (Gain)', figsize=(10, 4))
plt.tight_layout()
plt.show()
```

**三种 importance_type**：

| 类型 | 含义 | 适用场景 |
|---|---|---|
| `weight`（默认） | 特征被选为分裂点的次数 | 快速看哪个特征被用得最多 |
| `gain` | 使用该特征的平均分裂 Gain | 更能体现特征对目标的真实贡献 |
| `cover` | 该特征分裂覆盖的样本数 | 看哪个特征影响了更多样本 |

工程推荐：用 `gain`，更接近"这个特征真的有用多少"。

### 工程选型指引：XGBoost / LightGBM / CatBoost

三者同属"工业级 Boosting"，粗略指引（具体案例具体分析）：

```
数据特点                          推荐
──────────────────────────────────────────────────────────────
结构化数据 / 特征已编码 / 需要稳健  XGBoost（最成熟，文档全，生态好）

数据量 > 100 万 / 要求速度优先      LightGBM（leaf-wise 生长更快，
                                    直方图桶数可更大，内存更省）

类别特征多 / 不想手动 OHE           CatBoost（原生支持类别特征，
                                    自动处理 target encoding + 防泄露）

要跨语言部署（Go/Java/C++）         XGBoost（save_model → JSON，
                                    官方多语言 API 最完善）

Kaggle 竞赛快速出结果               LightGBM 起步，XGBoost 对比验证
```

**一句话总结**：没想法先 XGBoost；数据量过百万换 LightGBM；类别特征多试 CatBoost。

### 一句话钉板

**红酒案例的核心工程坑是样本不均衡——用 `compute_sample_weight` 校正权重 + `StratifiedKFold` + `f1_macro` 评估，三板斧解决；GridSearch 调 `n_estimators × max_depth × learning_rate`，特征重要性用 `gain` 类型更可靠。**
