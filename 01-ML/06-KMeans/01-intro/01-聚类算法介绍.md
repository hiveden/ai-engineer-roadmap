# 聚类算法介绍

> 维度：概念
> 学习目标：知道什么是聚类

## 底稿

> 【知道】聚类算法介绍

**聚类**（Clustering）：典型的**无监督学习**算法，根据样本之间的相似性自动把相似样本归到同一类别。

与分类的核心区别：分类有标签（监督），聚类只有特征、没有标签——簇的语义靠人事后解读。

相似性度量决定聚类结果——常用**欧氏距离**（Euclidean distance）：距离越小越相似。

## 讲解

### 业务场景：1000 条用户数据，没有标签

后台导出 1000 条用户，只有「年龄、月消费、登录频次」三列，**没人告诉你谁是高价值、谁是流失候选**。运营要你"分一下群"。

这就是聚类的主场。**没有现成答案让模型去学**——只能让数据自己开口，把"长得像的人"圈到一起，再回头看每个圈是谁。

```
监督学习（分类）        无监督学习（聚类）
样本 → 标签              样本 → ?
 x1  → 高价值             x1
 x2  → 流失              x2          → 算法把它们
 x3  → 高价值             x3            分成几堆
 ...                     ...
模型学"标签 ← 特征"       模型学"哪些样本彼此像"
```

### 核心概念扫盲

**无监督学习**（unsupervised learning）：训练数据**只有特征 X，没有标签 y**。算法的任务不是"学映射 X→y"，而是"挖掘 X 自身的结构"。聚类是无监督学习里最常见的一类。

**簇**（cluster，本文同义于"类别"）：算法划出的一个组。**簇的"含义"算法不知道**——它只知道"这堆样本彼此近"。把第 0 簇标成"黄金客户"是**人**做的事，发生在算法跑完之后。

**相似性度量**：两个样本"有多像"的数学定义。最常用的是**欧氏距离**（两点之间的直线距离，初中坐标系那个公式）：

$$d(a, b) = \sqrt{(a_1-b_1)^2 + (a_2-b_2)^2 + \dots}$$

距离越小 → 越像 → 越可能被分到同一簇。

### 直觉锚点：相似性度量是聚类的"灵魂"

**同一份数据，换距离公式 = 换聚类结果。** 这不是 bug，是定义。

举个反直觉的例子：判断两份用户购买清单"相似不相似"。

- 用**欧氏距离**：两人花的钱总额接近就算像 → 月入 5 万的高消费者和月入 5 千的精致穷可能被分一起（消费金额相同）
- 用**余弦相似度**（cosine similarity，看消费**结构**比例）：买的品类比例像就算像 → 上面两人会被分开，"都爱买书 80%+ 咖啡 20%"的两人会被合到一起

**没有"客观正确"的聚类**。距离选错，聚出来的簇就和业务直觉对不上。这是聚类比分类难落地的根本原因——分类有 ground truth 可以判对错，聚类只有"业务方看了点不点头"。

本柱后续都默认用**欧氏距离**（最经典、K-Means 的内置假设），其他度量先知道存在即可。
