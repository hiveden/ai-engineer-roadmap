# 聚类算法在现实中的应用

> 维度：概念
> 学习目标：了解聚类算法的应用场景

## 底稿

> 【了解】聚类算法在现实中的应用

- 基于位置信息的商业推送
- 新闻聚类，筛选排序
- 用户画像，广告推荐
- Data Segmentation
- 搜索引擎的流量推荐
- 恶意流量识别
- 图像分割，降维，识别
- 离群点检测，信用卡异常消费
- 发掘相同功能的基因片段

## 讲解

PPT 列了 9 个场景，看着杂——其实都在干同一件事。先挑 3 个**异质**场景拆开看输入输出，再抽共性。

### 场景一：用户画像（输入是行为，输出是分群标签）

电商有 100 万用户，每人一行特征：[最近一次访问天数, 月均下单数, 客单价, 偏好类目数]。**没有"高价值/低价值"的现成标注**——这恰恰是要算出来的东西。

跑一次聚类（比如 K=5），算法吐回每个用户一个簇编号 0~4。运营拿散点图一看：

```
客单价
  ↑
  │ 簇3 ●●●●  ← 高频高客单 = 黄金客户
  │ 簇2 ●●
  │       ●●● 簇0 ← 沉睡用户
  │ ●● ●●● 簇1 ← 价格敏感型
  │     ●●●● 簇4
  └──────────→ 月均下单数
```

**输出怎么用**：簇 3 推新品 + 会员特权，簇 0 触达召回券，簇 1 推满减——同样的预算，不同的话术。这就是「用户分层」（user segmentation）的工业做法，不靠业务方拍脑袋切年龄段。

### 场景二：信用卡异常消费检测（输出是"不属于任何簇"的孤儿）

输入是每笔交易的特征：[金额, 时间, 商户类型, 与持卡人常驻地距离, 距上一笔时间间隔]。**99% 的交易会聚成几个紧凑的大簇**（早 8 点便利店 30 元、午 12 点公司食堂 25 元、周末超市 200 元……）。

剩下 1% 落到任何簇都很远的点 → 「离群点」（outlier）→ 触发人工复核。**这里聚类不为了"分群"，为了"挑出不合群的"**。同一个算法，目标反过来用。

### 场景三：图像分割（每个像素当成样本）

一张照片是 H×W 个像素，每个像素 3 个特征 [R, G, B]。把所有像素当成样本跑 K-Means（K=5），聚完每个像素被替换成所属簇的中心颜色——原图变成只有 5 种颜色的"色块画"。

```
原图（256³ 种颜色）  →  K-Means 后（5 种颜色）
                        前景物体边界自然浮现
                        ↓
                        进一步做物体识别 / 抠图
```

这是「图像分割」（image segmentation）最朴素的玩法。**输入不是用户、不是交易，是像素**——只要能编码成定长向量，聚类就能跑。

### 9 个场景的共同特征

回头扫一遍 PPT 那 9 条，无论是商业推送、新闻聚类、基因片段还是恶意流量，全部满足两条：

1. **没有现成标签**，或标签太贵 / 太慢拿不到（人工标 100 万用户群？打住）
2. **怀疑数据自己有结构**——业务直觉告诉你"用户应该有几类""新闻应该能归几个话题""正常流量和攻击流量在某些维度上一定不一样"

只要场景同时满足这两条，聚类就值得一试。**反过来——已经有干净标签 / 数据完全均匀没结构，聚类都不是好选择**（前者用监督学习直接学映射；后者聚出来的簇也没业务含义）。
