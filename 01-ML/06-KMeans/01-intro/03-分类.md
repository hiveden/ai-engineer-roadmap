# 聚类算法分类

> 维度：概念
> 学习目标：知道聚类算法的分类

## 底稿

> 【知道】分类

**两个分类视角**：① 不同聚类方法对颗粒度的控制程度不同——粗聚类 / 细聚类；② 根据实现方法分类。

| 类别 | 代表算法 | 思路 |
|---|---|---|
| **划分式聚类** | K-Means | 按照质心分类，用距离计算相似性 |
| **层次聚类** | Agglomerative / Divisive | 对数据进行逐层划分，直到达到聚类的类别个数 |
| **密度聚类** | DBSCAN | 基于密度的聚类算法 Density-Based Spatial Clustering of Applications with Noise |
| **谱聚类** | Spectral Clustering | 基于图论的聚类算法 |

## 讲解

聚类算法很多，PPT 给了**两个正交的分类轴**：颗粒度（粗 / 细）和实现方法（4 类）。先各自讲清楚，再说为什么本柱选 K-Means 入门。

### 轴一：颗粒度——分几个簇？

同样 1000 个用户：

- **粗聚类**：K=2 → 「活跃 vs 沉睡」两大类，结论简单粗暴，运营好执行
- **细聚类**：K=20 → 切到"二线城市 25-30 岁母婴轻度活跃"这种颗粒，精细但每簇人少、策略难规模化

**没有"对的 K"**——业务用途决定颗粒度。给老板看大盘选粗的，给推荐系统打 tag 选细的。后续 04 章模型评估会讲怎么用 SSE / 轮廓系数辅助选 K，但**最终拍板是业务问题**。

### 轴二：实现方法——4 类的直觉差异

**核心区别在"怎么定义一个簇"**。同一份数据丢进去，4 种算法可能给出截然不同的形状。

| 类别 | 簇的定义 | 擅长 | 不擅长 |
|---|---|---|---|
| **K-Means**（划分式） | 离质心最近的点 | 球形、大小相近的簇 | 长条形、嵌套形、噪声多 |
| **层次聚类** | 自底向上合并 / 自顶向下分裂出的树 | 不知道 K、想看分层关系 | 数据量大（O(n²) 起步） |
| **DBSCAN**（密度） | 密度连通的点 | 任意形状 + 自动识别噪声点 | 密度不均的数据 |
| **谱聚类** | 图论降维后再聚 | 高维 / 流形结构数据 | 调参复杂、计算贵 |

术语扫盲：

- **质心**（centroid）：一个簇里所有点的平均位置（坐标平均值）。K-Means 反复算的就是它，详见后续 03 章。
- **密度连通**：两个点附近邻居够多、且能通过"邻居链"连起来，就算同一簇。允许簇是任意弯曲形状。
- **流形**（manifold）：高维空间里弯曲的低维结构，比如三维空间里卷起来的纸面。谱聚类擅长这种。

### 一张图看 4 类的擅长场景

```
                    数据形状
        球形规整 ●●●        长条 / 弯月 / 嵌套
        ┌───────────────┬─────────────────┐
有噪声  │  K-Means OK   │  K-Means 崩      │
        │  DBSCAN 也行  │  DBSCAN 主场     │
        ├───────────────┼─────────────────┤
无噪声  │  K-Means 首选 │  谱聚类 / 层次   │
不知 K  │  + 肘部法选 K │  层次聚类不需 K  │
        └───────────────┴─────────────────┘
```

把这张表反过来读：**数据啥样不知道时先跑 K-Means**——错了再升级，对了赚到。

### 为什么本柱选 K-Means 入门

三个理由，按重要性排：

1. **概念最少**：只需要"距离 + 平均"两个初中数学概念。DBSCAN 要先讲"核心点 / 边界点 / 噪声点 / ε 邻域 / MinPts"五个新词，谱聚类直接进图论，层次聚类要讲连接策略——入门门槛是 K-Means 的 3 倍起。
2. **工业最常用**：sklearn / Spark MLlib 都把 K-Means 摆在最显眼的位置；80% 的"我们做了用户分群"的 PR 稿背后都是 K-Means。
3. **评估指标都为它造的**：04 章要学的 SSE、肘部法、轮廓系数（SC）、CH 指数全部默认拿 K-Means 的输出当输入。先吃透 K-Means，评估指标才有具体抓手。

学完 K-Means + 这套评估方法，**90% 的"先跑个聚类看看"的需求就能交付**。剩下 10% 的怪形状数据再回头看 DBSCAN / 谱聚类不迟。
