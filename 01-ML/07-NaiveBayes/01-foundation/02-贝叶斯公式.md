# 贝叶斯公式

> 维度：**数学**

## 底稿（PPT 原文 + 笔记）

> 【理解】贝叶斯公式

**核心**：用先验 + 似然反推后验。

$$P(C \mid W) = \frac{P(W \mid C) \cdot P(C)}{P(W)}$$

**证据展开**（全概率公式）：

$$P(W) = \sum_{k} P(W \mid C_k) \cdot P(C_k)$$

**术语**（以分类任务为例，$C$ 是类别，$W$ 是特征）：

- **先验** $P(C)$：类别本身的频率（训练集里 spam 占多少）
- **似然** $P(W \mid C)$：给定类别下特征出现的概率
- **证据** $P(W)$：特征的边缘概率（归一化项）
- **后验** $P(C \mid W)$：看到特征 $W$ 后属于类别 $C$ 的概率（**预测目标**）

**分类决策**：对所有候选类别 $C_k$ 算 $P(C_k \mid W)$，取最大者。$P(W)$ 对所有类相同，可省略。

$$\hat{C} = \arg\max_{C_k} P(W \mid C_k) \cdot P(C_k)$$

**女神案例术语对应**

P(C) 表示 C 出现的概率，通常是无条件时 事情发生的概率， 亦被称为 "先验概率"
P(W|C) 表示 C 条件 W 出现的概率
P(W) 表示 W 出现的概率

P(C|W) = P(喜欢|程序员，超重)    已知是超重的程序员，他被女神喜欢上的概率 ， 后验概率，待求
P(W|C) = P(程序员，超重|喜欢)    已知女神喜欢的人，  他是超重程序员的概率
P(C) = P(喜欢)                              女神喜欢上一个人的概率 （先验，无条件时，事情发生的概率）
P(W) = P(程序员，超重)                随机一个人， 是超重 且是程序员的概率

：后验概率，在观察到某些证据（条件）之后，对某个假设或事件发生概率的更新估计。

**贝叶斯公式实算**

根据训练样本估计先验概率P(C)：P(喜欢) = 4/7
根据条件概率P(W|C)调整先验概率：P(程序员,超重|喜欢) = 1/4
3. 那么 P（W）=  P(程序员且超重) = P(程序员) * P(超重|程序员) = 3/7 * 2/3 = 2/7
				= P(超重) * P(程序员|超重) = 3/7 *  2/3 = 2/7

## 讲解

### 业务问题：方向反了的训练数据

回到女神案例。新来一个超重程序员，我们想问的是：

$$P(\text{喜欢} \mid \text{程序员},\text{超重}) \;=\; ?$$

这是「**已知特征，问类别**」——预测的天然方向。

但训练数据告诉我们的是反方向。看 7 行表你能直接数出的是：

- 在「喜欢」的人里，程序员比例多少？ → $P(\text{程序员}\mid\text{喜欢})$
- 在「喜欢」的人里，超重比例多少？ → $P(\text{超重}\mid\text{喜欢})$

这是「**已知类别，问特征**」——训练集的天然方向（按标签分组数特征）。

```
   想要的方向：    特征 ──?──→ 类别       (预测时未知 → 已知)
   能算的方向：    类别 ────→ 特征       (训练集按标签分组就能数)

   贝叶斯公式 = 把箭头掉头的工具
```

整个第 2 节就是把这个「箭头掉头」的机制讲清楚。

### 数学方法

#### 数学符号扫盲（首次出现）

| 符号 | 读法 | 直觉 | 例子 |
|---|---|---|---|
| $\sum_{k}$ | "sum over k" / "对 k 求和" | 把后面的式子按 k 取所有可能值，加起来 | $\sum_{k=1}^{3} k = 1+2+3 = 6$ |
| $\arg\max_{x} f(x)$ | "argmax of f over x" | **使** $f(x)$ **最大的那个 x**（不是最大值本身） | $\arg\max_{x\in\{1,2,3\}} (x-2)^2 = 1\;\text{或}\;3$ |
| $C_k$ | "C 下标 k" | 第 k 个类别（如 $C_1$=喜欢、$C_2$=不喜欢） | 下标用来枚举 |
| $\hat{C}$ | "C hat" | 模型**预测**的类别（带帽子= 估计值） | 区别于真实标签 $C$ |

> 区分：$\max f(x)$ 给你「最高分是多少」，$\arg\max f(x)$ 给你「谁拿了这个最高分」。分类任务关心的是**谁**——所以用 argmax。

#### 公式怎么来的：从联合概率两边除

第 1 节的联合概率有两条等价拆法：

$$P(CW) \;=\; P(W) \cdot P(C \mid W) \;=\; P(C) \cdot P(W \mid C)$$

把后两半拎出来 $P(W) \cdot P(C\mid W) = P(C) \cdot P(W\mid C)$，两边除以 $P(W)$：

$$\boxed{\;P(C \mid W) \;=\; \frac{P(W \mid C) \cdot P(C)}{P(W)}\;}$$

这就是贝叶斯公式。**没有什么神秘的——就是联合概率换了个写法。**

#### 4 个术语，一句白话

| 术语 | 符号 | 白话 | 女神案例 |
|---|---|---|---|
| 先验 | $P(C)$ | 没看任何特征前，这个类别本来多见 | 历史上女神喜欢的比例 = 4/7 |
| 似然 | $P(W \mid C)$ | 这个类别的人，长得像 W 的概率 | 喜欢的人里，是超重程序员的比例 |
| 证据 | $P(W)$ | W 这种特征本身有多常见（归一化） | 总人群中超重程序员的比例 |
| 后验 | $P(C \mid W)$ | 看到特征 W 后，更新过的类别概率 | 看了"超重程序员"后，喜欢的概率 |

**记忆口诀**：「先验 × 似然 ÷ 证据 = 后验」。先验是没看证据前的初始信念，乘以似然（证据对这个类别有多支持），除以证据本身（归一化），得到看了证据后的新信念。

#### 翻转流程图

```
   训练阶段：             预测阶段：
   按标签数特征             ──贝叶斯公式──→     按特征算标签
                                                    ▲
   ┌──────────────┐                                 │
   │ P(C) 先验     │                                 │
   │ ──── 数标签   │ ──┐                             │
   └──────────────┘    │                             │
                       ├──→  P(W|C) · P(C)  ──────→  P(C|W)  后验
   ┌──────────────┐    │              ÷               (新样本归到 argmax)
   │ P(W|C) 似然   │ ──┘            P(W)
   │ ──── 切片数   │
   └──────────────┘
```

#### 全概率公式：把分母切成「各类别加权和」

$P(W)$ 怎么算？常用招数是按类别**切片再加和**：

$$P(W) \;=\; \sum_{k} P(W \mid C_k) \cdot P(C_k)$$

直觉：「W 出现」可以来自任意一个类别。把每个类别的贡献（似然 × 先验）算出来，加起来就是总。

几何上像把饼分成 k 份，每份的"长出 W"的部分加在一起：

```
            类别 C₁          类别 C₂           类别 C₃
         ┌─────────┐       ┌─────────┐        ┌─────────┐
         │ P(C₁)   │       │ P(C₂)   │        │ P(C₃)   │
         │  ┌──W┐  │       │ ┌─W─┐   │        │  ┌W┐    │
         │  └───┘  │       │ └───┘   │        │  └─┘    │
         └─────────┘       └─────────┘        └─────────┘
              ↓                  ↓                  ↓
     P(W|C₁)·P(C₁)  +  P(W|C₂)·P(C₂)  +  P(W|C₃)·P(C₃)   =   P(W)
```

#### 为什么 argmax 决策可以省掉 P(W)

最终预测取「后验最大」的类：

$$\hat{C} \;=\; \arg\max_{C_k} \; \frac{P(W \mid C_k) \cdot P(C_k)}{P(W)}$$

注意 $P(W)$ **不依赖 $C_k$**——对每个候选类别都是同一个分母。比较谁更大时，**共同分母可以约掉**：

$$\hat{C} \;=\; \arg\max_{C_k} \; P(W \mid C_k) \cdot P(C_k)$$

> 类比：比较 $\frac{3}{10}$ 和 $\frac{7}{10}$ 谁大，根本不用看 10——只看分子 3 vs 7。

这一招省下了「算 $P(W)$」这一整步。在朴素贝叶斯实战中（下一节会看到），$P(W)$ 是个连乘求和的麻烦活，能省就省。

#### 实算回看（套到女神案例）

$$P(\text{喜欢}\mid\text{程序员},\text{超重}) \;=\; \frac{P(\text{程序员},\text{超重}\mid\text{喜欢}) \cdot P(\text{喜欢})}{P(\text{程序员},\text{超重})}$$

代入：分子 = $\tfrac{1}{4} \times \tfrac{4}{7} = \tfrac{1}{7}$，分母 = $\tfrac{2}{7}$，相除得 $\tfrac{1}{2} = 0.5$。

50% —— 模糊地带。这暗示：只看两个特征，证据还不够强。下一节看「特征独立假设」怎么把这套机制扩展到多特征场景。
