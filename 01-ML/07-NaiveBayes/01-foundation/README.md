> 【知道】朴素贝叶斯算法

朴素贝叶斯算法 - 利用概率值进行分类的一种机器学习算法

什么是概率？

一件事情发生的可能性，取值在【0,1】之间。比如：抛硬币正面向上的概率、6面骰子抛出5这一面的概率

> 【知道】常见概率公式

**条件概率** $P(A \mid B)$：B 已发生时 A 发生的概率。

$$P(A \mid B) = \frac{P(AB)}{P(B)}$$

**联合概率** $P(AB)$：A、B 同时发生。

$$P(AB) = P(B) \cdot P(A \mid B)$$

**特征独立时**：$P(AB) = P(A) \cdot P(B)$（朴素贝叶斯的基础）。

**联合 + 条件**：

$$P(AB \mid C) = P(A \mid C) \cdot P(B \mid AC)$$

**条件概率实算**

比如：在女神喜欢的条件下，职业是程序员的概率？
- 女神喜欢条件下，有 2、3、4、7号 共 4 个样本
- 4 个样本中，有程序员 3、4号 共 2 个样本
- 则 P(程序员|喜欢) = 2/4 = 0.5

【思考】在女神不喜欢的条件下，职业是程序员的概率是多少？

**联合概率实算**

比如：职业是程序员并且体型匀称的概率？
- 数据集中，共有 7 个样本
- 职业是程序员有 1、3、4号 共 3 个样本，则其概率为：3/7
- 在职业是程序员，体型是匀称有 3号 共 1 个样本，则其概率为：1/3
- 则即是程序员又体型匀称的概率为：3/7 * 1/3 = 1/7

【思考】：体型匀称并且是程序员的概率是多少？ P(B) P(A|B)

> 备注：联合概率 ： 今天下雪天， 而且小明今天穿蓝色棉衣的概率  （同时发生）  =  下雪的概率 *   下雪天小明穿蓝色棉衣的概率

**联合概率 + 条件概率**

比如：在女神喜欢的条件下，职业是程序员、并且超重的概率？ P(程序员, 超重 | 喜欢)
- 在女神喜欢的条件下，有 2、3、4、7 号 共 4 个样本
- 在这 4 个样本中，职业是程序员有 3、4 号共 2 个样本，则其概率为：2/4=0.5
- 在这 2 个样本中，体型超重的有 4号 共 1 个样本，则其概率为：1/2 = 0.5
- 则 P(程序员, 超重|喜欢) = 0.5 * 0.5 = 0.25

事件独立：比如：女神是否喜欢一个人，和 职业选择的概率  或 身材肥胖的概率 ，在没有任何关系
                                       此时 P(B|A)  = P(B)     或 P(A|B)  = P(A)


      那么 联合概率公式退化为 我们所熟悉的 P(A, B) = P(A)P(B)，   此时称事件A与事件B相互独立

$$1.\ P(B|A) = P(B) \rightarrow \text{知道A发生，不影响B发生的概率}$$

$$2.\ P(A|B) = P(A) \rightarrow \text{知道B发生，不影响A发生的概率}$$

总结：
       条件概率：在某些条件已经发生的情况下，计算某些样本的出现的概率，表示为：P(B|A)
        联合概率：多个事件同时发生的概率是多少，表示为： P(AB) = P(A) * P(B|A) = P(B)* P(A|B)

> 【理解】贝叶斯公式

**核心**：用先验 + 似然反推后验。

$$P(C \mid W) = \frac{P(W \mid C) \cdot P(C)}{P(W)}$$

**证据展开**（全概率公式）：

$$P(W) = \sum_{k} P(W \mid C_k) \cdot P(C_k)$$

**术语**（以分类任务为例，$C$ 是类别，$W$ 是特征）：

- **先验** $P(C)$：类别本身的频率（训练集里 spam 占多少）
- **似然** $P(W \mid C)$：给定类别下特征出现的概率
- **证据** $P(W)$：特征的边缘概率（归一化项）
- **后验** $P(C \mid W)$：看到特征 $W$ 后属于类别 $C$ 的概率（**预测目标**）

**分类决策**：对所有候选类别 $C_k$ 算 $P(C_k \mid W)$，取最大者。$P(W)$ 对所有类相同，可省略。

$$\hat{C} = \arg\max_{C_k} P(W \mid C_k) \cdot P(C_k)$$

**女神案例术语对应**

P(C) 表示 C 出现的概率，通常是无条件时 事情发生的概率， 亦被称为 "先验概率"
P(W|C) 表示 C 条件 W 出现的概率
P(W) 表示 W 出现的概率

P(C|W) = P(喜欢|程序员，超重)    已知是超重的程序员，他被女神喜欢上的概率 ， 后验概率，待求
P(W|C) = P(程序员，超重|喜欢)    已知女神喜欢的人，  他是超重程序员的概率
P(C) = P(喜欢)                              女神喜欢上一个人的概率 （先验，无条件时，事情发生的概率）
P(W) = P(程序员，超重)                随机一个人， 是超重 且是程序员的概率

：后验概率，在观察到某些证据（条件）之后，对某个假设或事件发生概率的更新估计。

**贝叶斯公式实算**

根据训练样本估计先验概率P(C)：P(喜欢) = 4/7
根据条件概率P(W|C)调整先验概率：P(程序员,超重|喜欢) = 1/4
3. 那么 P（W）=  P(程序员且超重) = P(程序员) * P(超重|程序员) = 3/7 * 2/3 = 2/7
				= P(超重) * P(程序员|超重) = 3/7 *  2/3 = 2/7

  - 答案 ：  P(喜欢|程序员, 超重) = 0.5

> 【理解】朴素假设

**问题**：直接算 $P(W \mid C) = P(w_1, w_2, \dots, w_n \mid C)$ 需要联合分布，特征多时数据稀疏、参数爆炸。

**朴素**（naive）= 假设特征间**条件独立**：

$$P(w_1, w_2, \dots, w_n \mid C) = \prod_{i=1}^{n} P(w_i \mid C)$$

**朴素贝叶斯分类器（完整式）**：

$$\hat{C} = \arg\max_{C_k} P(C_k) \prod_{i=1}^{n} P(w_i \mid C_k)$$

朴素贝叶斯在贝叶斯基础上增加：特征条件独立假设，即：给定条件下，特征之间是互为独立的。
此时，联合概率的计算即可简化为：
P(程序员,超重|喜欢) = P(程序员|喜欢) * P(超重|喜欢)


                                           适合场景： 投骰子。  对于自然语言处理中：不太适用。
P(程序员,超重) = P(程序员) * P(超重)

> 【知道】拉普拉斯平滑

**问题**：训练集里某个 $P(w_i \mid C) = 0$（词没出现过），整个连乘归零，分类器直接挂掉。

**解法**：分子 +α，分母 +αm。

$$P(w_i \mid C) = \frac{N_i + \alpha}{N + \alpha m}$$

- $\alpha$：平滑系数，通常取 1（拉普拉斯平滑），$0 < \alpha < 1$ 称 Lidstone 平滑
- $N_i$：类别 $C$ 中特征 $w_i$ 出现次数
- $N$：类别 $C$ 下所有特征出现总次数
- $m$：特征取值数（文本场景下即词表大小）

**骰子案例**

你有一个六面骰子，但不知道是否公平。你掷了 5 次，结果是： [1, 1, 2, 3, 1] 你想估计每个点数出现的概率。

朴素估计：
- 点数 1: 3 次 → P(1) = 3/5 = 0.6
- 点数 2: 1 次 → P(2) = 0.2
- 点数 3: 1 次 → P(3) = 0.2
- 点数 4、5、6: 0 次 → P(4) = P(5) = P(6) = 0

骰子并非不能扔出4、5、6点 仅仅是因为样本少而已。

加拉普拉斯平滑（α=1）：

$$P(x) = \frac{\text{count}(x) + 1}{N + M}$$

总观测次数 N = 5；所有可能得特征数 M = 6 (1~6)；P(x) = (count(x)+1)/(5+6)

$$P(1) = (3+1)/11 \approx 0.364$$
$$P(2) = (1+1)/11 \approx 0.182$$
$$P(3) = (1+1)/11 \approx 0.182$$
$$P(4) = (0+1)/11 \approx 0.091$$
$$P(5) = 0.091$$
$$P(6) = 0.091$$

> 【小结】朴素贝叶斯算法

1 朴素贝叶斯算法概念

用概率值进行分类的一种机器学习算法。贝叶斯基础上增加特征条件独立假设。

2 概率数学基础

概率：一件事情发生的可能性。
条件概率：表示事件A在另外一个事件B已经发生条件下的发生概率，P(A|B)
联合概率：表示多个条件同时成立的概率，P(AB) = P(A) * P(B|A) = P(B)* P(A|B)

3 贝叶斯公式

$$P(C \mid W) = \frac{P(W \mid C)\, P(C)}{P(W)}$$

4 拉普拉斯平滑系数

为了避免概率值为 0，我们在分子和分母分别加上一个数值，这就是拉普拉斯平滑系数的作用
