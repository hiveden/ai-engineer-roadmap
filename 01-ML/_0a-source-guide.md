## 1. PPT XML 扫描法（三路提取）

`.pptx` 本质是 zip，解压后从 XML 直接抽文字 / 公式 / 备注，恢复 README 底稿丢失数据。

### 解压 + 顺序映射

```bash
mkdir -p /tmp/pptx_extract && cd /tmp/pptx_extract
unzip -q /path/to/章.pptx

# slide 顺序映射：通常 slideN.xml 编号 = 实际 slide 编号
# 严格起见看 ppt/_rels/presentation.xml.rels 的 sldIdLst 顺序
```

### 三路提取

| 路径 | 提取内容 | XML 标签 |
|---|---|---|
| `ppt/slides/slideN.xml` | slide 主体文字 | `<a:t>` (drawingml) |
| 同 XML | 数学公式 | `<m:oMath>` (MathML) |
| `ppt/notesSlides/notesSlideN.xml` | 讲师备注 | `<a:t>` (drawingml) |

**MathML 渲染要点**（递归处理）：

| 标签 | 渲染为 |
|---|---|
| `<m:sSup>` | `base^{sup}` |
| `<m:sSub>` | `base_{sub}` |
| `<m:f>`（分数） | `(num)/(den)` |
| `<m:rad>`（根号） | `√(...)` 或 `√[deg](...)` |
| `<m:nary>`（大算符 ∑ ∫） | `op_{sub}^{sup}(e)` |
| `<m:d>`（括号） | `(...)` |
| `<m:m>`（矩阵） | `[row1; row2; ...]` |

### 比对 README 找差异

逐条比对，标记三类差异：
- "PPT 原图" / "PPT 原值" / "省略号"占位
- 公式数字与 PPT 不一致（145416 / 281671.6 / 55683.8 等长系数易抄漏）
- slide 备注里的关键解释被遗漏

## 2. ASCII 图代替 PPT 原图

PPT 中的可视化补回 README 时**优先 ASCII 绘制**，不直接嵌图。

### 可行性表

| 图类型 | ASCII 可行 | 例 |
|---|---|---|
| 散点图 + 拟合线 | ✓ | KNN 近邻图 / LR 身高体重 |
| 抛物线 / loss 曲线 | ✓ | y=x²-4x+5 极值 / GD 收敛曲线 |
| 向量箭头 / 几何示意 | ✓ | 用 `╱ → ●` 字符 |
| 矩阵 / 张量概念阶梯 | ✓ | box-drawing 字符 `┌─┐` |
| 切线几何 / 导数图 | ✓ | 03b slide 33 |
| 真实 3D loss 曲面 | ✗ | 留 "PPT 全图占位" + 文字推导 |
| 高密度散点（>20 点）| ✗ | 同上 |

### 为什么不直接嵌图

- **外部资源依赖**（图床消失 / 路径变更）
- **不可 grep / 不可改**
- **ASCII 既保留视觉直觉又是源码**——和 LaTeX 公式同档

### 例：slide 18 候选拟合线

```
体重 Y                                      多条候选拟合线，哪条最优？
  ↑           .       /  ──  红线
  |        .   /  ──
  |     . /─        .
  |   /     ──
  | /───      .
  +────────────────────→ 身高 X
```

## 3. slide → 子文件归属表

**原则**：README 完整持有；子文件按**主题**切片，**不按 slide 编号区段**切。

| slide 类型 | 归属 |
|---|---|
| 章节大纲 / 学习目标 / 课程导览 / 章节衔接 / 本节小结 | **留 README** |
| 知识点核心讲解（概念 / 公式 / 推导）| 主题对应子文件 |
| 实战案例 / 算例 / 应用例 | 主题对应子文件（**即使 slide 编号与上下文不连续**）|
| 自检题 · 同章题 | 对应主题子文件章末 |
| 自检题 · 跨章综合题 | 留 README 章末作综合复习 |
| 自检题 · 多题混合 | 整体归到主题占多数的那个文件 |

### 反例（已修正）

- **03b slide 23-25 实战案例** 最初被遗漏在章 README，后下放到 `01-损失函数.md`
- **03b slide 28/50/52 自检题** 同样按主题下放到 01/03/06
- **04b slide 80 题 3-4**（GD / GD 分类同章）下放到 01/03
- **04b slide 79 题 1-2**（损失函数 / 正规方程跨章）留 README

> **决策口诀**：**同章主题 → 下放；跨章综合 → 留 README**。

## 4. 子文件底稿同步格式

子文件起手必须**完整复制**对应 slide 段（含 ASCII 图 / 公式 / 备注），不用一句话简述。

### 标准模板

```markdown
---
tags: [扁平中文标签, ...]
---

# <主题名>

> 维度：<概念/算法/数学/代码>
> 知识点级别：【掌握/理解/了解/知道/实践】<一句话>
> 章节底稿全文见 [`README.md`](./README.md)（PPT slide X-Y[, Z]）

## ━━━━━━━━ 底稿 ━━━━━━━━

### PPT

> Slide N · <PPT 标题>

<完整复制 README 中对应 slide 段（含 ASCII 图 / 公式 / 备注）>

> Slide N+1 · <PPT 标题>

<...>

### 笔记

> 【掌握/理解/了解/知道/实践】<标题>

<完整复制 README 笔记段中对应主题切片>

---

## ━━━━━━━━ 讲解 ━━━━━━━━

<按 _1-explanation-guide.md 三层结构展开（业务问题 → 数学方法 → 代码落地）>

### <H3 子段 1>
...

> Sources：
> - PPT Slide N-M
> - <其他来源>
```

### 关键纪律

- **底稿 = PPT + 笔记两部分**，都是只读元数据（_架构规则 §5）—— 写完不再改，全量复制 README 对应段
- **H2 横线分隔符** `## ━━━━━━━━ 底稿/讲解 ━━━━━━━━`（粗实线 ━ 字符，每边 8 个）—— 用于在大纲里强分割底稿 / 讲解两段
- **讲解段不跨段引用底稿** —— 讲解段自包含，再次引用 PPT 原文须在讲解段内重抄
- **slide 范围在头部 quote 声明** —— "章节底稿全文见 README（PPT slide X-Y[, Z]）"，让读者一眼看到归属边界
- **讲解段不限字数**（_架构规则 §12）—— 讲清楚是第一原则，超 400 行自检但不强制拆

## 5. PPT 错误处理（忠实誊抄 + ⚠ 警告 + 正确解）

发现 PPT 内部不一致或答案错误时四步走：

1. **忠实誊抄 PPT 原值** —— 不擅自"修复"原文，避免历史溯源失踪
2. **⚠ 警告标注矛盾** —— 明确指出哪两处 / 哪个值不一致
3. **给出"官方答案"** —— 若备注 / 后续 slide 有可自洽版本，明示推荐
4. **若 PPT 答案数学错误** —— 写出**代回验证步骤** + **正确解**

### 反例

**A. 内部数值矛盾**（03b slide 23/24 vs 备注）：

```
PPT 正文 y 值：56.3, 60.6, 65.1, 58.5, 56.3
备注 + slide 56 推导 y 值：56.3, 60.6, 65.1, 68.5, 75
⚠ 标注 + 推荐用备注版本（与最终系数 281671.6 / 145416 / 852 自洽）
```

**B. PPT 答案数学错误**（04a slide 56）：

```
PPT 给：k = 0.0397，b = 60.7615
代回验证：145416·0.0397 + 852·60.7615 ≈ 57541.7 ≠ 55683.8 ✗
消元法正确解：k ≈ 0.9295，b ≈ -93.29，预测 70.3 ✓
```

写在 README 备注块作 ⚠ 警告，**不删 PPT 原值** —— 保留教学溯源价值。

## 6. 收尾自检清单（交付前过一遍）

| 项 | 检验 |
|---|---|
| 所有 slide 都有归属 | `grep "PPT 原图\|PPT 原值"` 应无残留 |
| 子文件底稿与 README 对应段一致 | 同 slide 同段，`diff` 应近零 |
| 跨子文件 slide 编号无重复占用 | 头部 quote 声明的 slide 范围互不冲突 |
| PPT 错误警告都在位 | 已知矛盾点 ⚠ 标注 + 正确解 |
| ASCII 图覆盖所有可视化 | "PPT 全图占位"仅留无法 ASCII 的极少数 |
| 章节级 slide 留 README | 学习目标 / 小结 / 课程导览不下放 |
| 自检题归属正确 | 同章题下放 / 跨章题留 README |

### 自检 grep 套餐

```bash
# 1. 查未补回的占位
grep -rn "PPT 原图\|PPT 原值\|省略号\|⚠️ 待补充" 01-ML/0X-XXX/

# 2. 查子文件 slide 范围声明
grep -rn "对应 slide\|PPT 原文 · slide" 01-ML/0X-XXX/

# 3. 查跨章数据一致性（如 LR 身高 / 体重样本）
grep -rn "176\|170.4\|65.1" 01-ML/02-LR/
```
