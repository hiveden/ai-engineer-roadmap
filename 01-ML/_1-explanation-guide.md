## 1. 概念后置三段式（由浅到深）

每个抽象概念按这个顺序：

```
1. 场景  ──→  2. 机制  ──→  3. 命名
"想象..."   "怎么做"      "这叫 X，英文 ..."
```

| ❌ 反例 | ✅ 正例 |
|---|---|
| "**多数表决** = K 个邻居中票数最多的类别" | "看周围 5 个邻居，3 个三角形 2 个五角星——三角形赢，问号判三角形。**这种'谁多谁赢'叫多数表决 majority vote**" |

**先名字 → 后场景** 是预设观众已知的硬伤。

**章级版本**：每章从**业务场景导入**（先讲"为什么需要" → 再讲"怎么做"），不假设读者有相关背景。这是科普向叙事的基本盘。

**配套铁律**：**不用概念解释概念** —— 讲解段用通俗语言，所有正式定义集中到术语版（§14）。详见 §11 失败模式。

## 2. 直觉锚 = 生活类比

每个抽象概念配**一个**生活类比：

| 概念 | 类比 |
|---|---|
| KNN 整体 | 近朱者赤 |
| K=1 / K=N | 独裁 / 全民投票 |
| 缩放 | 把跑道拉直 / 单位拉齐 |
| 损失函数 | 罚款 |
| 权重 w / 偏置 b | 工资里每件事的"汇率" / 底薪 |
| 梯度下降 | 蒙眼下山 |
| L1 vs L2 正则 | 等量收缩 / 按比例收缩（橡皮筋） |

**硬规则**：

- ❌ 不堆叠（"K=1 像独裁、又像独行侠、还像 X"）
- ❌ 类比无基础（突然蹦出"打开盒子"，前面没建立"盒子"）
- ✅ 类比建立 → 用类比讲机制 → 再命名术语

## 3. 数学和代码后置硬序

```
直觉（白话）  →  视觉（ASCII / 散点 / 表）  →  公式（LaTeX）  →  代码（sklearn）  →  术语版
```

公式是"机制的精确表达"，不是"理解的入口"。代码是"工程落地"，不是"概念解释"。

**直觉版禁用**：

- ❌ 用公式讲公式（公式 + 一堆希腊字母再翻译）
- ❌ 用概念讲概念（"权重就是加权求和的权"——观众不知道"权"是什么）
- ✅ 纯白话故事 / 类比 / 反问对话感

## 4. 视觉锚 = 具体数字

| ❌ 模糊 | ✅ 具体 |
|---|---|
| "K 适中时准确率较高" | "**K=11** 准确率 **89%**" |
| "更多邻居" | "**5 个最近邻**" |
| "近距离" | "**距离 0.30**" |
| "中等学习率" | "**lr = 0.01**" |

观众脑里"K=11 → 89%"是稳定锚点，不是"差不多"的模糊感觉。

## 5. 正例 + 反例双向激活

每个概念给一个正例 + 一个反例：

| 概念 | 正例（建立直觉）| 反例（反推必要性）|
|---|---|---|
| K 选择 | K=5 五张全喜欢 → 预测喜欢 | K=1 噪声带跑 / K=N 多数派绑架 |
| 缩放 | 缩放后预测正确 | 不缩放 → 大量纲特征独裁 |
| 加权 | 近邻权重大 | 噪声近时被放大 |
| 正则化 | L1 把 7 个无用特征归零 | alpha=0 时拟合曲线剧烈抖动 |

只有正例 → 观众不知道"为什么不能省"。
只有反例 → 观众不知道"成功是什么样"。

## 6. 章末"关键启示"金句

每节末尾用一句话总结：

```
**这一节的关键启示**：KNN 不关心数字代表什么（评分 / 主演 / 嵌入），只看数字。任何能用一组数字描述的东西都能用 KNN。
```

设计原则：

- **一句话**（≤ 30 字）
- **可记忆**（押韵 / 对仗 / 反差）
- **可复用**（在 cta / 视频脚本里直接引用）

例：

- "人脑失灵的地方，正是 KNN 接管的地方"
- "核心一个找邻居，变种两个分类投票、回归平均"
- "缩放是 KNN 距离公平的前提"
- "权重决定差距怎么拉开，偏置决定起跑线放多高"

## 7. 数据自洽 + 主线案例延续

### 数据自洽（跨章节）

同一案例的数字必须**全章一致**：

- KNN 章 9 部电影坐标在 02 / 02a / 03 / 04 / 05 等所有章节相同
- LR 章身高体重 5 样本在 01-intro / 02-api / 03b-math / 04a-analytical 等一致
- 一处改 → 全章 grep + 同步

工具：

```bash
grep -rn "(8.4, 9.0)" 01-ML/01-KNN/  # 流浪地球 2 跨章节出现
grep -rn "176" 01-ML/02-LR/          # 小明身高跨章节出现
```

### 主线案例每章只揭一层

同一案例贯穿多章，**每章只揭一层**（不一次倒完）：

- KNN 站豆瓣《流浪地球 3》案例：
  - 02 → 接近程度 + 距离公式
  - 03a/b/c → 过拟合 / 欠拟合 / 调优
  - 04 → 加权投票 / 标准化
  - 05 → 交叉验证 + 网格搜索

→ 案例延续的好处：观众认知连贯（同一组数据 / 同一个虚拟用户），新概念有锚点。
→ 反面：每章把"为什么需要这个概念"重新铺一遍，节奏拖沓。**克制原则**：每章只解决该章特有的问题。

## 8. 主场原则 + 推迟标记 + 跨章引用

### 主场原则

每个概念在它的**主场章节**展开，其他章节只**点名 + 链接**到主场：

- 数学层例外：本章本身是数学物质（如 03b-math）则现在就展开
- **超出本章主场的内容**用 `## 不在本章范围` 段明确划界，**不展开**（避免"啥都讲一点"）

例：KNN 章遇到"标准化"——主场是 04b-scaling，其他章一句话点名 + 链接，不重复讲缩放原理。

### 推迟标记

| 情况 | 处理 |
|---|---|
| 概念主场不在本章 | 标"后续展开 → [`02-X.md`](./02-X.md)" |
| 概念展开太远（>5 章） | 给直觉描述（不展开严格定义），cta 列入"没展开的" |
| 不在本章范围 | 用 `## 不在本章范围` 段明确划界 |

**禁**：跨章 metadata 引用（"假设你已读过 X" / "已在 X 详细展开" / "这里作为基线"）。

## 10. 写作纪律检查清单

写完一节自查 **13 条**：

- [ ] 概念出现前先有场景铺垫（无空降术语）
- [ ] 抽象概念有生活类比（且类比有基础）
- [ ] 数字具体（无"较高 / 适中 / 较多"模糊词）
- [ ] 至少有一个正例 + 一个反例
- [ ] 章末有"关键启示"金句
- [ ] 跨章数字一致（grep 同源数据）
- [ ] 推迟概念明示去向（链接 + 后续展开）
- [ ] 术语版在末尾（不在直觉段引入）
- [ ] **新概念首次出现内联释义**（"鞍点 saddle point" 而非纯"鞍点"）
- [ ] **不超章节定位**（【了解】级不堆完整证明）
- [ ] **从已知迁移而非从零起**（如有读者熟悉的同源概念）
- [ ] **如有 demo 必挂入口**
- [ ] **零造词**（每个汉字搭配能在新华字典 / 现代汉语词典找到，或属公认 ML 行话；详见 §15）

### 自检 grep 套餐

```bash
# 1. 查未填占位
grep -rn "⚠️ 待补充\|TODO\|XXX\|讲解段待填" 01-ML/0X-XXX/

# 2. 查中文夹英文双引号（markdown/Python 字符串冲突高发地）
grep -rn '"[^"]*[\u4e00-\u9fff][^"]*"[^"]*[\u4e00-\u9fff][^"]*"' 01-ML/0X-XXX/

# 3. 查可能的过时 sklearn API

# 4. 查空代码块
grep -B1 -A2 '^```python$' 01-ML/0X-XXX/ | grep -A2 '```python' | grep -B2 '^```$'
```

## 11. 常见失败模式

| 失败 | 教训 |
|---|---|
| "训练前的旋钮"（用未解释术语 + 行话定义新词） | 先讲场景，再命名 |
| "假设你已读过 X" | 跨章 metadata 引用禁止 |
| 一节一个类比堆 3 个 | 一个类比就够 |
| "K 适中" 不给具体数字 | 永远给具体值（K=11） |
| 公式抢戏（直觉前给 LaTeX） | 公式后置 |
| 术语在第一段就轰炸 | 术语版收尾 |
| 反例缺失（只讲"对的怎样"） | 加反例反推必要性 |
| **概念引入断层**（"偏置 b" 首次出现没释义） | 第一次提及加内联释义 + 链回基础章 |
| **过度扩写**（03b 列完整导数表 / 07b 提 Dropout BN） | 砍掉，只保留 LR 用得到的 |
| **中文字符串里夹英文双引号**（如 `"两点之间直线最短"`）| 改用 `「两点之间直线最短」`，避免 markdown 渲染 / Python 字符串冲突 |
| **过时 API 引用**（`squared=False` / `load_boston` / `loss=squared_loss`）| 写文档前查最新 sklearn 文档；同时给"sklearn ≥ X.Y 用 ..." 兼容说明 |
| **学习目标错位**（07a 学习目标列了 07b 的内容）| 章末 review-checker 扫一遍学习目标 vs 实际内容 |
| **造词**（"硬判类别 / 自洽导数 / 把握度 / 不饱死边界"）| 用现成中文词或公认行话；自创复合词等于让读者卡在你的词典里。详见 §15 |

## 12. 跨章节概念迁移（从已知到未知）

读者带着**已有的同源概念**进来——优先用迁移视角，而不是从零讲。

| 新概念 | 已知同源概念 | 迁移技巧 |
|---|---|---|
| LR 的 w / b | 初中 y = kx + b 的 k / b | 一元 k → 多元 w 向量；b 含义不变 |
| KNN 回归 | KNN 分类 | 投票 → 求平均；其他流程不变 |
| L1 正则 | 普通 LR 损失 | 损失函数后加 $\lambda \sum |w_i|$ |
| Adam 优化器 | SGD | 加自适应学习率 + 动量 |
| 决策树 vs 线性模型 | 已知 LR/LogReg 是参数线性 | 树是分段常数（非线性、非参数） |

**迁移段三件套**：

```
1. 「你已经懂 ___」
2. 「现在升级 / 替换 / 扩展」
3. 「具体差在哪」（一张对比表）
```

例（LR 02-线性回归.md 的 "k → w" 段）：

> 你初中已经熟了 y = kx + b。LR 把它升级到多个 x：
> y = w₁·x₁ + w₂·x₂ + ... + b
>
> 一元的 k 在多元时变成向量 w；b 还是 b（含义不变，几何画不出但代数一样）

## 13. 工程化约束（受众视角）

受众是**做工程的人**（开发、架构、转型 AI 工程），不是数学专业。讲解段必须保留工程视角。

| 约束 | 检验 |
|---|---|
| **代码块给可运行片段** | 直接复制到 jupyter 能跑，不要 `...` 省略号 / 不要伪代码 |
| **常见坑独立成段** | sklearn API 的 X 必须 2D / fit_intercept / SVD vs 求逆 等 |
| **demo 链接有就挂** | 顶部表格列出 demo 路径 + 端口 + 启动命令 |
| **跨语言落地视角** | LR 训练后只需保存 w 和 b → 给 Python/Go/JS 推理代码 |
| **生产级提示** | 大数据用 SGDRegressor，小数据用 LinearRegression（含理由 + 阈值）|
| **可解释性** | 强调 LR 的优势：看 coef_ 大小符号能解释 |

> 反例：04b-gd 一开始 agent 把"凸函数"严证一遍，对工程读者无用。改成"碗形：底就一个"+ 一个 ASCII 图直接讲清。

## 14. 术语命名纪律（中英双标）

技术术语**首次出现**给两种叫法：

```
✓ 损失函数（loss function，也叫代价函数 cost / 目标函数 objective）
✓ 偏导（partial derivative，符号 ∂）
✓ 凸函数（convex function）
✗ "正则化" （首次出现没给 regularization）
```

后续出现可以只用中文或只用英文，但**首次必须双标**。

特殊情况——**多名同物**：

```
偏置 bias = 截距 intercept = sklearn 的 intercept_ —— 同一个 b
权重 weight = 系数 coefficient = sklearn 的 coef_ —— 同一个 w
```

第一次出现时列全所有别名，后续可以混用。

## 15. 零造词

讲解段不许造词。**每个汉字搭配必须满足下列任一条**：

1. 现代汉语常用搭配（在新华字典 / 现代汉语词典 / 主流媒体能找到）
2. 公认 ML / 工程行话（教科书 / sklearn 文档 / Wikipedia 中文条目存在）
3. 英文术语的官方中译（如「梯度下降 = gradient descent」）

凡是 Claude 自己拼出来"听起来像术语"的复合词，全部砍掉。

### 造词典型反例（来自 03-LogReg 实战）

| 造词 | 问题 | 修法 |
|---|---|---|
| **硬判类别** | "硬"+"判" 自创动词搭配 | → 输出类别 / 判定类别 |
| **自洽导数** | "自洽"是物理/逻辑学行话，硬塞 ML 语境 | → 导数可由函数自身表达 |
| **把握度** | 名词后缀 "度" 滥用 | → 概率 / 置信度 |
| **不饱死边界** | "饱死" 不是中文词 | → 不撞硬边界 / 渐近不饱和 |
| **数学链路自洽** | "自洽" 行话味重 | → 数学链路对得上 |

### 高发造词模式

- **名词 + 度**：把握度 / 稳定度 / 流畅度（除非在词典里：满意度、清晰度 ✓；自创带"度"⚠️）
- **形容词 + 名词 自创组合**：硬判 / 软切 / 死饱
- **行话错位**：把物理学（自洽）/ 数学（同构）/ 哲学（涌现）的词硬塞 ML 上下文
- **英文直译造词**：把 self-consistent 翻成"自洽" 可以，但 self-referential derivative 别译"自洽导数"——直接用「能用自身表达的导数」

### 例外：公认 ML 行话可用

| 词 | 可用 | 理由 |
|---|---|---|
| 损失函数 / 梯度下降 / 过拟合 | ✓ | 教科书标准译法 |
| 嵌入 embedding | ✓ | sklearn / 中文 ML 教材通用 |
| 后验概率 / 似然函数 | ✓ | 统计学标准译法 |
| **链路对得上 / 链路自洽** | 选前者 | 后者行话味重，前者口语清楚 |

### 自查方法

1. **读出来**：每段大声读一遍，遇到"我自己也觉得别扭"的复合词标黄
2. **百度搜索测试**：把可疑词放百度搜，结果都是技术博客转载/AI 生成 → 大概率自造
3. **替换测试**：能用 5 个口语字解释的就别用 3 字行话

## 16. 数学符号扫盲（首次出现 4 列表）

**前提**：受众久未碰数学是常态。所有数学符号（$\Sigma$ / $\arg\max$ / 下标 / $\partial$ 等）首次出现都需扫盲——不假设读者懂。

**模板**：「数学符号扫盲表」4 列：

| 符号 | 读法 | 直觉 | 例子 |
|---|---|---|---|
| $\sum_{i=1}^{n} x_i$ | 「sigma 从 i=1 到 n 求和 x_i」 | 把所有 $x_i$ 加起来 | $\sum_{i=1}^{3} i = 1+2+3 = 6$ |
| $\partial f / \partial x$ | 「partial f partial x」 | 多元函数对 x 偏导（其他当常数）| $f(x,y) = x^2 + y$，$\partial f / \partial x = 2x$ |
| $\arg\max_i$ | 「argument max」 | 让函数最大的那个 i（不是最大值本身）| $\arg\max_i [3, 5, 1] = 1$（i=1 处值最大）|
| $\lVert \mathbf{w} \rVert_2$ | 「w 的 L2 范数」 | 向量长度（平方和开根）| $\lVert (3,4) \rVert_2 = \sqrt{9+16} = 5$ |

**纪律**：每个符号只扫盲一次（首次出现），后续出现可直接用。

## 17. 讲解段量级 · 讲清楚 > 短

**讲清楚是第一原则**。讲解段不设行数红线——为了凑长度删例子 / 砍实算 / 压缩直觉表，是错误的优化方向。

| 原则 | 内容 |
|---|---|
| **优先** | 直觉锚 / 业务图 / 数学符号扫盲 / 实算演示 / 工程坑提示——能讲透就铺开 |
| **底线** | 不水、不重复、不为了"显得专业"塞冗余术语；每段都为"讲清楚"服务 |
| **超长信号** | 单文件 > 400 行讲解时**自检**——是真的需要这么多，还是能拆独立子文件（按"业务案例 / 练习 / 实算延展"维度），但**不强制拆** |

**对比**：底稿段是元数据只读、章 README 元数据导航部分有 200 行红线（见 [`_架构规则.md`](./_架构规则.md) §10-11）。**讲解段例外**——讲透优先。

## 18. Claude 撸文纪律（Sparring 模式）

> 用户授权"你先动笔"后 Claude 撸文，用户审。撸文时遵守以下纪律。

**PPT 给少则写少**：
- 不主动从 sklearn 官方教程 / Wikipedia / 工程经验拓展
- PPT 只给一句话的概念，讲解段就一句话讲清楚 + 一个直觉锚，**不堆扩展阅读**
- 用了非 PPT 内容（sklearn 文档 / 教学博客 / 国际案例）必须末尾 **Sources 标注**

**数学是工具不是目的**：
- 在算法柱子站讲数学时，公式作为解决问题的工具，**不展开纯数学概念**（单位圆 / 形态光谱 / 度量空间公理 / 完备性等）
- 纯数学概念主场在数学语言层（如 03b-math），其他章只点名 + 链接

**图表主动加**（不等用户提醒）：
- 业务图（痛点）：业务场景遇到什么问题
- 数学图（数轴 / 几何）：公式可视化
- 代码流程图（ASCII）：sklearn 5 步 pipeline 等
- 详见 [`_架构规则.md`](./_架构规则.md) §5 讲解段三层结构

**克制原则**：Claude 不该撸长篇——除非用户明确"先给个初稿我改"。日常协作中 Claude = sparring partner（找图、答疑、校对、反问），不是文章生成器。
