## 1. 立场

- AI 老师做分享，第一人称 **"我们"**
- 每期 cta 末句：**"如果讲得有不对的地方，欢迎在评论区指正"**
- 单位用 **"期"**（不用 今天/这一集/下一集/本集）
- 不端权威，用"想象/假设/注意"引导

## 2. 双源拼接

| 源 | 用途 |
|---|---|
| **A 动画** | 概念铺垫 / 公式定格 / 收束总结 |
| **B 录屏** | 互动 demo / 代码演示 |

重叠优先 B（更具体）。

## 3. 概念引入硬规则

### ❌ 禁

| 反例 | 问题 |
|---|---|
| "**还藏着一个**特殊性质" | 预设观众已知对照系 |
| "打开 sklearn 的**盖子**" | 比喻无基础 |
| "留了两个**钩子**" | 文学化 |
| "fit 和 predict **一字不动**" | 文艺金句 |
| "sklearn **真正的**核心" | "真正"暗示对比 |
| "**调参**细节" | 工程术语未白话 |

### ✅ 必

- **首次术语**：中英标 + 直觉定义/例子（"决策边界 decision boundary：背景色是 KNN 在全平面的预测地图"）
- **数学术语**：信任高中数学（方差/概率不必白话化）
- **工程术语 → 白话**：调参→选数字 / 一字未改→不变 / 黑盒→对外只露 fit 和 predict 的函数包
- **直觉锚**：每个抽象概念配生活类比（K=1 独裁 / K=N 全民投票）

### 推迟规则

教材标"后续展开" → 脚本也推迟。但若推迟章节太远（>5 章），本章给直觉描述，cta 列入"没展开的"。

## 4. CTA 三段结构

```
1. 讲清楚的清单（带视觉锚或核心金句）
2. 提到没展开的清单（推迟章节 + 推迟原因）
3. 下期预告 + cta（"如果讲得有不对的地方，欢迎在评论区指正"）
```

跨期互不重复推迟项。整章末期加"整章一句话总结"。

## 5. 跨期衔接钩

上期 cta 伏笔 → 下期 hook 必须正面回应（关键词级精确）。

## 6. 数据一致性

教材 / demo / 脚本三处数字必须一致。改一处 → 全章 grep + Python 重算。

大改用 Executor + Auditor 双 Agent（先改 + 自测，再独立复核）。

## 7. TTS 字段约束（text）

- ❌ markdown 标记（`**xxx**` / `*xxx*` / `` `xxx` ``）
- ❌ emoji
- ❌ 英文标点夹中文（`,` / `:` / `;` / `?`）→ 用中文 `，：；？`
- ✅ 数字带空格分词（"126 部"、"K=11"、"8.5 分"）
- ✅ 长串数字拆短句（"复仇者 0.32、泰坦尼克 0.50、流浪地球 2 0.54。然后阿凡达 1.22、沙丘 2 1.50。"）
- ✅ 公式细节句号断开（"差是 0.2 和 0.5。平方分别是 0.04 和 0.25，加起来 0.29，开方约等于 0.54。"）

## 8. 段长

| 段长 | 风险 |
|---|---|
| < 150 字（除 hook/cta） | 信息空洞 |
| **250-350 字** | 舒适区 |
| > 450 字 | 信息密度过载 |

## 10. script.json schema

```json
{
  "title": "...",
  "description": "...",
  "style": "tutorial",
  "form": "tutorial",
  "target_duration_s": 1080,
  "segments": [
    {
      "id": 1,
      "type": "hook" | "content" | "cta",
      "topic": "...",
      "text": "TTS 实际输入（无 markdown / 中文标点）",
      "notes": "字数估算 + region 切分（title/body/figure/quote/formula）+ states 清单（B 段）+ cue 词 + [src: 教材路径]"
    }
  ]
}
```

cta type 必须是最后一段；hook 必须是第一段。

## 11. demo 段 notes 模板

```
~XXX字。B demo 录屏段。
demo: 路径/XX.py

states（Playwright 驱动到这些档位）:
  1. intro     — 静帧介绍 UI
  2. baseline  — rating=8.5, lead=9.5, k=5
                 expect: 5/0 喜欢 · 距离 0.30/0.51/...
  3. dislike   — rating=2.9, lead=3.5, k=5
                 expect: 1/4 不喜欢
  4. ...

cue 词（口播 → state 切换时机）:
  "我们把评分调到 8.5"          → state 2
  "现在把新电影拖到上海堡垒那个角落" → state 3
  "再把 k 调到 1"               → state 4
```

## 13. 失败模式（已踩过的坑）

| 失败 | 教训 |
|---|---|
| 凭空写 demo UI（"切换 1D/2D/3D 维度"） | 写 demo 段前必读 demo 代码 |
| K=1 dislike 说"全不喜欢"实际 4/1 | states 必用 Python 实测 |
| 脚本"≈50%" 实际 41.6% | 每个声称数字 Python 重算 |
| K=80 但 demo slider 上限 50 | 跨范围数字必须与 demo 限制对齐 |
| "lazy learner"零 ML 基础观众听不懂 | 预设术语必须前置铺垫 |
| cta 重复中英标 | cta 不读两遍英文 |
| bias-variance 在主段展开 | 主段聚焦核心，扩展推迟到收敛或后续章 |
| "差 0.2 和 0.5,平方是 0.04 和 0.25" 账本式 | 句号断开 + 短句拆解 |
| "钩子 / 盖子 / 一字不改" 修辞预设 | 全部改白话或删 |
| "真正的 / 本质 / 还藏着" 预设对照系 | 直接陈述，不用对照词 |

## 14. Agent 审稿三视角

| 教学叙事 | 观众视角找预设漏洞 / 衔接钩 / 节奏 |
