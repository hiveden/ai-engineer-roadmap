# 评估指标速查表

> 跨 demo 共用的评估指标清单。回归（demo-01）和分类（demo-02）两个家族。
> L3 标准：名字 + 中英 + 一句话含义 = 必会；选型 = 必会；公式 / 数学推导 = 不要求。

---

## 1. demo-01 实际用的 · MAE

demo-01 的 step4 里只用了一个指标：

```python
from sklearn.metrics import mean_absolute_error
mae = mean_absolute_error(y_test, y_pred)
```

- **MAE** = `Mean Absolute Error` = **平均绝对误差**
- 算法：`|预测 - 真实|` 每条算一次，再求平均
- 类比：让新员工估 4128 套房价，**平均每套偏了多少万**

demo-01 看到的 "误差占比 25.9%" = `MAE / 平均房价`。

demo-01 **没有用 MSE**——那是另一个相似指标，以后会碰。

## 2. 评估指标是知识点吗 · 是，L3 必备

对照 `demos/README.md` 的 L3 定义：

| L3 要求       | 落到评估指标上                                     |
| ------------- | -------------------------------------------------- |
| ✅ 能读懂代码 | 看到 `mean_absolute_error` 知道这行在干啥          |
| ✅ 能讲清动机 | 为什么回归用误差类、分类用 P/R                     |

## 3. 最少要记的清单

### 回归家族（demo-01）

| 指标    | 中英                               | 一句话                           |
| ------- | ---------------------------------- | -------------------------------- |
| **MAE** | 平均绝对误差 `Mean Absolute Error` | 平均每条差多少                   |
| **MSE** | 平均平方误差 `Mean Squared Error`  | 先平方再平均（对大误差惩罚更重） |
| **R²**  | 决定系数 `R-squared`               | 0~1，越接近 1 越好               |

### 分类家族（demo-02）

| 指标                 | 中英                        | 一句话                         |
| -------------------- | --------------------------- | ------------------------------ |
| **Accuracy**         | 准确率 `Accuracy`           | 判对几个 / 总共几个            |
| **Precision**        | 精确率 `Precision`          | 说是的里面，真是的几分         |
| **Recall**           | 召回率 `Recall`             | 真是的里面，抓出来几分         |
| **F1**               | F1 分数 `F1 Score`          | Precision 和 Recall 的调和平均 |
| **Confusion Matrix** | 混淆矩阵 `Confusion Matrix` | 看错成哪了（2×2 表）           |

## 4. 选型决策树

### 你的任务是回归还是分类？

- **输出是连续数字**（房价、股价、温度）→ **回归** → 用误差类指标（MAE / MSE / R²）
- **输出是离散类别**（是/否、A/B/C）→ **分类** → 用对错率类（Accuracy / Precision / Recall / F1）

### 回归场景选指标

- **默认先看 MAE**：直观（平均每条差多少单位）
- **对大误差敏感的业务**（金融、医疗剂量）→ **MSE / RMSE**（大误差惩罚重）
- **想知道"模型比瞎猜好多少"** → **R²**（0 = 和猜平均值一样差，1 = 完美）

### 分类场景选指标

按 "漏过 vs 错杀，哪个亏得多" 选：

| 场景           | 漏过代价         | 错杀代价           | 盯谁                          |
| -------------- | ---------------- | ------------------ | ----------------------------- |
| 癌症筛查       | 极高（死人）     | 低（复查）         | **Recall**                    |
| 垃圾邮件过滤   | 低（删一封）     | 高（老板邮件丢了） | **Precision**                 |
| 银行反欺诈拦截 | 高（赔钱）       | 中（好客户冻结）   | 通常 **Recall** 略重          |
| 风控高额贷款   | 高（坏账几十万） | 中（好客户没贷到） | **Precision**（宁少批不错批） |
| 两者都在意     | -                | -                  | **F1**                        |

### 什么时候看混淆矩阵？

**任何分类任务都应该看一次**——它是 4 种情况的总账，Precision/Recall 都是从它推出来的。

## 5. 不用记的（查就行）

- 所有公式（每个指标的数学定义）
- API 参数细节（`average='binary' / 'macro' / 'weighted'` 等）
- AUC / ROC 曲线（更高阶，后续章节再碰）
- 多分类的 `average` 策略区别（二分类不涉及）
- 数学推导（最小二乘怎么推、F1 为什么是调和平均）

**原则**：只记"这是什么、用在哪"；公式、调参、高阶指标用时查 sklearn 文档。

---

## 6. 对应代码入口

两个家族都在 `sklearn.metrics` 里。

**回归**（`from sklearn.metrics import ...`）:

- `mean_absolute_error`
- `mean_squared_error`
- `r2_score`

**分类**（`from sklearn.metrics import ...`）:

- `accuracy_score`
- `precision_score`
- `recall_score`
- `f1_score`
- `classification_report`
- `confusion_matrix`

`classification_report` 一次性打印每个类的 precision/recall/f1——分类任务默认先调它。
