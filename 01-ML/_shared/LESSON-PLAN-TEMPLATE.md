## §8 · 写作规则（本模板的使用指引）

> 这一节给**写 LESSON-PLAN 的人**（Alex + Claude）看。

### 8.1 数据来源三原则（写入 CLAUDE.md 的"课件生产规则"）

每份 LESSON-PLAN 必须来自 3 个信源：

1. **原始培训笔记**（主干）：`assets/source-materials/第4阶段-机器学习/&lt;算法&gt;.md`
2. **第一版改编**（工程师语言参考）：`demos/0X-*.md`（合并后**删原文件**）
3. **联网搜索交叉验证**（补充 + 溯源）：**所有外部信源必须嵌入 URL**

### 8.2 每小节起手必须有"情境锚句"

**为什么**：LESSON-PLAN 本质是给 LLM 查的 RAG 文档，Claude 可能从任意小节被召回。没有情境锚句 = 丢上下文。

**怎么写**：一句话答 "本节讲什么 + 属于哪个更大主题 + 学员/Claude 应带着什么预期读"。见 Anthropic Contextual Retrieval。

### 8.3 先具体后抽象（§1 必须是钩子）

**禁止**：从定义开场（"XX 算法是一种…"）

**强制**：从业务锚 / 可跑 demo 开场。数学/公式延后到 §2.3。

### 8.4 FAQ 散布，不集中

常见追问 **就近放** 在对应术语卡或概念旁边（§4）。**不**建"所有追问集中一章"。RAG 检索按语义相似度跳入，散布才能被精准命中。

### 8.5 显式 active recall

§5 Questionnaire 必填，按 Bloom 三层（记忆 / 应用 / 迁移）各 2-5 题。

### 8.6 Tech Lead 语气 + 工程师语言

参考 `CLAUDE.md` 受众节的 6 条"必须"：
- 工程师语言（微服务、API、GC、零拷贝、ONNX…）
- 跨语言落地（Python 训 → Go/Java/Node 推）
- Software 1.0 vs 2.0 对比
- 高密度、少废话、少形容词
- 修改文件前一句策略
- 术语中英双标 + 业务锚优先

### 8.7 DATA-FLOW / NOTES / EVAL-METRICS 不在本模板

这些是**跑完 demo 后**的临时产出物。起手只写 LESSON-PLAN + `code/step1~N.py`。

### 8.8 备课覆盖全量，不预选

**禁止**在备课时就定死"用哪个数据集 / 上哪种任务类型 / 从哪个入口进"。

**强制**：
- §0.1 用"可支持的任务类型清单"代替单选
- §0.3 列所有候选数据集 + 教学价值对比 + URL
- §1.2 每个候选各写一份最小 demo

数据集选型是**上课时** Claude 根据学员状态的现场决策，不是备课时的预设。

### 8.9 三源定位（不是简单合并）

三份素材分别填不同槽位，缺一源则该节含金量打折：

| 模板章节 | 主要来源 | 补充来源 |
|---|---|---|
| §1 钩子 · 业务锚 | **第一版改编**（已有工程类比） | 联网搜索补缺 |
| §2.1 黑盒视图 | **第一版改编**（工程接口类比） | — |
| §2.2 核心机制 | 原始培训笔记 | — |
| §2.3 数学附录 | **原始培训笔记**（公式/推导） | 联网搜索（验证时效/过时检查） |
| §3 坏→好递进 | **第一版改编**（Software 1.0 vs 2.0） | 联网搜索（现代实践如 HNSW） |
| §4 术语定义 | **原始培训笔记**（严谨中文定义） | 第一版改编（业务锚素材） |
| §4 术语追问答案 | — | **联网搜索**（必带 URL，经得起"你哪看来的"反问） |
| §5.1 记忆层题 | **原始培训笔记**（原课练习题） | — |
| §6 常见坑 | 原始笔记（"注意"提示） + 改编版（跨语言落地坑） | 联网搜索（业界公开翻车） |

**判据**：写完一节，问自己"这节用上了几源"。少于 1 源 = 合并时塌陷；只用 3 源但没分工 = 一锅炖。
