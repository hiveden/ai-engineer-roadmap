# 01-ML 架构规则（v1 试行）

> **状态**：v1 试行期规则。每根柱子复盘后回顾是否升级为《[机器学习学习框架.md](../机器学习学习框架.md)》的固化条款。3 个算法跑完后做一次整体回顾。

## 1. 备课主导权 + Claude 角色

- **讲解文字 Alex 主导写**——不是给学员的讲义，是 Alex 对自己的备课
- **Claude = sparring partner**：
  - 找图 / 动画 / 可视化链接（WebSearch）
  - 答疑（写讲解时遇到的概念追问）
  - 校对（公式 / 术语中英 / 排版）
  - 反问（写完后提"这里别人会问什么"）
  - 补充工程师锚点（Alex 需要时再上）
- **Claude 不该做**：直接撸出长篇讲解段落（除非 Alex 明确点了"先给个初稿我改"）—— 否则又掉回 v0 反模式（AI 写 = 用户没消化）

## 2. 文档结构

- **章 README**：保留 PPT 原文（密度基线），顶部加 blockquote 章节索引
- **知识点子文件**：`# 标题` + `## 课程原文`（PPT 段）+ 讲解段（自由定结构，不预设占位）
- **单文件总字数 ≤ 3000 字**（约 10KB）—— PPT 原文 × 8 + 100~300 图像说明
- 超出按"独有钩子"细切，**不按"知识 vs 案例"切**

## 3. 转写纪律

- 严格按 PPT 原文，**不夹带私料**（不扩展、不预测、不基于 v0 知识印象）
- **排版可整理**：清理 PPT 复制残留空格、全角→半角等
- **内容不增删**：keyword 占位也保留作为内容缺口
- **typo 保留原样**（如"将未知的样本预测的值了"）

## 4. 架构表达

- **不预设元数据**：状态板 / 待填占位 / 横向引用清单 / "v1 试行 · 章节骨架，内容待填"等
- **文件结构 + 内容即架构**——管理成本由 git history 承担
- 加 H2 / 加段落只在有真实内容时

## 5. Tag（Obsidian 风格 + 按需积累）

- **位置**：YAML frontmatter
- **格式**：扁平中文 tag，无层级前缀（例：`[算法/KNN, 数学/距离/欧氏]`）
- **不预设标签词典**——遇到什么打什么

**打 tag 时机**：

| 类别 | 时机 |
|---|---|
| **数学层** | 现在就打（章节本身是数学物质，不需要"重复"信号） |
| **其他横向**（特征工程 / 训练 / 评估） | 等**第二次重复出现**——一次性出现还在算法纵向叙事里 |
| **通用概念**（过拟合 / 欠拟合 / 超参等） | 等**第二根算法柱子**也出现时再打 |

## 6. 横向切分时机

- **第 1 根柱子推进期间**：不切，所有横向内容暂留在柱子内
- **第 1 根柱子完成后**：第一次外循环——把多次出现的横向抽出到顶层（如 `00-math/`）
- **诊断信号**：第 2 根柱子（如 LR）触达"距离 / 缩放 / CV 又要写一遍"时
