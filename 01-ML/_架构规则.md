## 叙事

### 1. 不夹带私料

严格按用户给的 PPT 原话写，不扩展、不预测、不基于历史 v0 知识印象。

- 排版可整理（空格、全角→半角、清理 PPT 复制残留）
- 内容不增删（keyword 占位保留作为内容缺口）
- typo 保留原样（如"将未知的样本预测的值了"）
- **底稿段 = PPT 原话** —— Claude 没拿到的内容用 `〔公式〕` / `〔图〕` 占位
- **Claude 补全独立成段** —— 公式 / 代码 / 实算等补全放讲解层独立段，明示"按 PPT 习惯补全 / sklearn 标准模板补全 / 等"
- **跨章节 metadata 引用不允许** —— 案例延续 OK（"延续 02 豆瓣场景"），但 "已在 X 详细展开 / 这里作为基线 / 假设你已读过 X" 等 metadata 注释不允许
- **Sources 标注** —— 用了非 PPT 内容（sklearn 官方 / Wikipedia / 教学博客 / 国际教学案例等）文末必须列 Sources

### 2. 由浅到深（科普向）

每章从业务场景导入（先讲"为什么需要" → 再讲"怎么做"），不假设读者有相关背景。

### 3. 不用概念解释概念

讲解段用通俗语言，所有概念定义集中到术语版。

> 反例："训练前的旋钮"——用未解释的"训练"+ 行话"旋钮"定义"超参数"。

### 4. 不假设读者懂

遇到陌生符号 / 概念**第一次出现就即时扫盲**。

特别注意：**久未碰数学是常态**，所有数学符号（$\Sigma$ / $\arg\max$ / 下标等）首次出现都需扫盲——用「数学符号扫盲表」（符号 / 读法 / 直觉 / 例子 四列）。

## 结构

### 5. 原文 vs 讲解分层

- **底稿段**：**元数据，只读不改**。包含两部分：
  - **PPT 原话**保真（防止知识点遗漏）
  - **笔记**（PPT Notes 区 / 讲师手稿 / 原始备课文本）保真
- **讲解段**：自包含，Claude/用户在底稿之上**新写**，**不跨段引用底稿**
- **子文件的底稿段**须从章 README 切出 (a) 对应 Slides + (b) 对应笔记片段，**两者都进**
- **章 README 的底稿段**是章级聚合源，永远完整持有 PPT + 笔记，不因子文件吸收而删除

讲解段建议 **三层结构**（每层各自有图）：

- **业务问题**：算法在场景下遇到什么问题（业务图：可视化痛点）
- **数学方法**：公式 + 直觉 + 实算（数学图：数轴 / 几何 / 公式可视化）
- **代码落地**：sklearn 集成完整 demo（代码块 + 流程 ASCII 图）

### 6. 主场原则

每个概念在它的**主场章节**展开，其他章节只点名 + 链接到主场。

- 数学层例外：本章本身是数学物质则现在就打 tag
- **超出本章主场的内容用"## 不在本章范围"段明确划界**，不展开

### 7. 主线案例延续

同一案例贯穿多章（如 KNN 站豆瓣《流浪地球 3》案例），**每章只揭一层**：

- 02 → 接近程度 + 距离公式
- 03a/b/c → 过拟合 / 欠拟合 / 调优
- 04 → 加权投票
- ...

### 8. 内容即架构

- **不预设元数据**：状态板 / 待填占位 / 横向引用清单 / "v1 试行 · 章节骨架待填"等
- 文件结构 + 内容即架构——管理成本由 git history 承担
- 加 H2 / 加段落只在有真实内容时

## 协作

### 9. 角色 + 决策模式

**角色分工**：

- **用户主导写讲解** —— Alex 对自己的备课，不是给学员的讲义
- **Claude = sparring partner**：找图、答疑、校对、反问、补充直觉锚点
- **Claude 不该撸长篇** —— 除非用户明确"先给个初稿我改"
- Sparring 例外**：用户授权"你先动笔"后 Claude 撸文，用户审

**Sparring 撸文纪律**（重要）：

- **PPT 给少则写少** —— 不主动从 sklearn 官方教程 / Wikipedia / 工程经验拓展
- **数学是工具不是目的** —— 在算法柱子站讲数学时，公式作为解决问题的工具，不展开纯数学概念（单位圆 / 形态光谱 / 度量空间公理等）。纯数学概念主场在数学语言层
- **图表主动加** —— 业务图（痛点）/ 数学图（数轴/几何）/ 代码流程图（ASCII），不等用户提醒

**决策模式**：

- 分叉给 2-4 个候选 + 我的推荐 + 用户点位（α / β / γ）
- 不替用户决定，给清单让用户选

## 复习页 + 拆分约束（v1.1 新增）

### 10. 章 README = 复习记忆页

每章 README 持有该章 PPT **完整底稿**（章级聚合）—— 用户可随时打开 README 单文件检索 / 记忆 / 应试。

- **双轨制**：章 README 章级 + 子文件分节，**接受冗余**换扫读性
- **覆盖范围**：学习目标 + 全部 PPT 节标题 + 公式 + 代码片段 + 总结
- 不重复子文件的"讲解段"（业务图 / 实算 / 关键启示等不进 README）

### 11. 章 README 单次阅读可消化

**≤ 200 行红线，底稿不计入**。学习目标 + 子文件索引 + 章节小结这类"元数据 + 导航"部分受红线约束；章级底稿（PPT + 笔记）按双轨制（规则 10）完整持有，超长不削。

### 12. 讲解段不限字数 · 讲清楚是第一原则

**讲清楚 > 短**。讲解段不设行数红线——为了凑 200 行删例子 / 砍实算 / 压缩直觉表，是错误的优化方向。

- **优先**：直觉锚 / 业务图 / 数学符号扫盲 / 实算演示 / 工程坑提示——能讲透就铺开
- **底线**：不水、不重复、不为了"显得专业"塞冗余术语；每段都为"讲清楚"服务
- **超长信号**：单文件 > 400 行讲解时**自检**——是真的需要这么多，还是能拆独立子文件（按"业务案例 / 练习 / 实算延展"维度），但**不强制拆**
- **底稿仍是元数据只读**（规则 5），章 README 元数据导航部分仍受规则 11 约束

### 13. 章按 4 维度拆 a/b/c

维度：**概念 / 算法 / 数学 / 代码**。

章内容混多维度且量超 200 时按维度拆 a/b/c：

- KNN 04 章拆 04b-scaling（数学+代码）/ 04c-iris-case（代码案例）
- LR 03 章拆 03b-math（数学+概念）/ 04a-analytical（数学）/ 04b-gd（算法）

### 14. 每章必须独立成题

1 子文件章如内容薄到无法撑起独立考题（纯概念引子 / PPT 几乎全图无文字），合并到相邻能独立成题的章。

- 反例：KNN 04a-why（"为什么预处理"3 行答案）→ 合到 04b-scaling 作章首引子
- 正例：KNN 04c-iris-case（1 子文件 88 行）能撑"用 KNN+标准化做鸢尾花分类"综合题

### 15. 拆分顺序

**先拆章节（建 README 骨架）→ 再拆子文档（填底稿）→ 最后填讲解段**。

## 过程性规则（试行期通用）

### Tag · Obsidian 风格 + 按需积累

- 位置：YAML frontmatter
- 格式：扁平中文 tag，无层级前缀（例：`[算法/KNN, 数学/距离/欧氏]`）
- 不预设标签词典——遇到什么打什么

**打 tag 时机**：

| 类别 | 时机 |
|---|---|
| **数学层** | 现在就打（章节本身是数学物质，不需要"重复"信号） |
| **其他横向**（特征工程 / 训练 / 评估） | 等**第二次重复出现**——一次性出现还在算法纵向叙事里 |
| **通用概念**（过拟合 / 欠拟合 / 超参等） | 等**第二根算法柱子**也出现时再打 |

### 横向切分时机

- **第 1 根柱子推进期间**：不切，所有横向内容暂留柱子内
- **第 1 根柱子完成后**：第一次外循环——把多次出现的横向抽出到顶层（如 `00-math/`）
- **诊断信号**：第 2 根柱子（如 LR）触达"距离 / 缩放 / CV 又要写一遍"时

## 经验提示（非硬规则，工具箱）

遇到时灵活用，不预先规定：

- **单文件预估 ≤ 3000 字 / 10KB**，超出按"独有钩子"细切（不按"知识 vs 案例"切）
- **公式用 LaTeX**：Typora / GitHub / Obsidian 渲染；绝对值用 `\lvert...\rvert` 避表格 `|` 冲突
- **图像分工**：用户做演示动画，Claude 写文字 + ASCII 草图占位
- **图片 / 外部资源占位**：统一标 `⚠️ 待补充`，不在 md 中嵌入会消失的外部资源（day03 即将删除经验）
- **二维坐标图**：跨章节用同一风格 + 同一数据集（认知连贯）
- **量纲对齐**：案例多特征数据集设计时优先量纲一致，量纲问题留给特征预处理章主场
- **底稿段**：用 blockquote `>` 标识 PPT 节标题
- **H1 简短化**：用文件名简短版（`# 接近程度` 而非"第一个问题：样本的接近程度"）
- **章节索引**：章 README 顶部加 blockquote 章节索引列表
- **数据示例**：教学数据优先用 sklearn 官方 / 国际通用案例，标"示例"+ Sources
