# 05 - 朴素贝叶斯与特征降维 (PCA)：通往大模型时代的序曲

在结束传统机器学习基础阶段之前，我们必须跨越两座大山：**概率模型（朴素贝叶斯）** 和 **维度灾难（特征降维/PCA）**。

传统培训班会教你如何算拉普拉斯平滑系数，如何推导皮尔逊相关系数。作为工程师，你只需知道：**这两门古老的技艺，正是今天统治世界的 Generative AI（大语言模型）和 Vector Search（向量检索引擎）的直系祖先。**

## 1. 朴素贝叶斯（Naive Bayes）：Next Token Prediction 的起源

在 NLP（自然语言处理）领域，早期的情感分析、垃圾邮件分类，都是由朴素贝叶斯完成的。

### 1.1 核心公式的工程翻译：查字典与概率连乘
贝叶斯的核心公式 $P(A|B) = \frac{P(B|A)P(A)}{P(B)}$ 看起来很唬人，但在代码实现上，它就是一个**超大的哈希表（Map/Dict）**。

假设我们要判断一句评论 `[“很”, “难看”, “的”, “电影”]` 是差评还是好评，代码怎么做？
```java
// 极其简化的贝叶斯预测伪代码
float badProb = getPrior("差评"); // 先验概率（历史数据里差评的比例）
for (String word : sentence) {
    // 这句话在“差评”分类下，每个词出现的概率连乘
    badProb = badProb * getWordProbInClass(word, "差评"); 
}

float goodProb = getPrior("好评");
for (String word : sentence) {
    goodProb = goodProb * getWordProbInClass(word, "好评");
}

return badProb > goodProb ? "差评" : "好评";
```
**为什么叫“朴素（Naive）”？**
因为它做了一个非常粗暴的假设：**词与词之间是完全独立的**。“很”和“难看”同时出现的概率，就是它俩单独出现的概率相乘。
但现实中，语言是有上下文关联的（Contextual）。为了解决这个“朴素”带来的低智商问题，人类科学家一路演进：从隐马尔可夫模型（HMM） -> 循环神经网络（RNN / LSTM） -> 最终到达了当今的终极形态：**Transformer 架构（ChatGPT 等）**，彻底解决了上下文长期依赖的难题。

**结论**：不要去纠结贝叶斯的平滑公式。记住，现在所有的 LLM（如 GPT-4）本质上都在做一件事：给定前面的上下文，**计算下一个词（Next Token）出现的条件概率分布，并选择概率最高的那一个。这就是贝叶斯思想在 2024 年的终极形态。**

## 2. 特征降维（PCA）：从内存碎片的灾难到稠密 Embedding

传统的特征工程最爱用 One-Hot 编码（独热编码）。如果你有 10 万个单词的词典，每个单词就会变成一个长度为 10 万的数组，其中只有 1 个位置是 1，剩下 99,999 个位置全是 0。

### 2.1 One-Hot 的物理崩溃与维度灾难
*   **内存浪费**：这种极其**稀疏（Sparse）**的矩阵会浪费海量的内存。
*   **距离失效**：在 10万 维的欧氏空间里，任何两个单词的距离算出来几乎是一模一样的，KNN 等依赖距离的算法会彻底失效。

为了解决高维稀疏特征的灾难，工程师必须使用降维手段。

### 2.2 PCA（主成分分析）：降维打击与数据压缩
PCA 是最经典的数据降维算法。它的数学原理是“正交变换”，但在工程师眼里，它就是一种**有损数据压缩算法（类似 JPEG 压缩图片）**。

*   把一个 10万 维的稀疏矩阵，压缩成一个只有 100 维的稠密（Dense）矩阵。
*   在这个压缩过程中，我们希望尽可能少地丢失原数据的信息（用数学语言说：最大化保留数据方差 Variance）。

### 2.3 通往大模型的桥梁：Embedding 层
传统的 PCA 压缩只能抓住一些统计学上的数字特征，它不懂语言的“语义（Semantics）”。

于是，深度学习登场了。科学家们不再使用死板的数学公式做 PCA 降维，而是直接**用神经网络来学习降维**！
这就是大模型中最伟大的发明之一：**Embedding（词向量/嵌入）**。

*   在 LLM 中，Embedding 层的作用，就是把一个极其稀疏、离散的词 ID，映射（降维）成一个比如长度为 768 或 1536 的**稠密浮点数向量（Dense Vector）**。
*   更神奇的是，由于它是通过神经网络训练出来的，**这些稠密向量带有了强大的“语义距离”**（“国王”-“男人”+“女人”≈“女王”）。

这就是我们在 02 节讲向量数据库（Vector DB）的原因。PCA 只是在做无情的数学压缩，而 Embedding 则将降维升华成了“万物皆可向量化”的思想。

## 阶段一：传统机器学习总结

至此，我们已经拆解了所有核心的传统机器学习算法：
1. **KNN** $\rightarrow$ 演化为 RAG 架构中的 **向量检索（Vector Search）**。
2. **线性回归 / L1&L2 正则化** $\rightarrow$ 神经网络中的基本单元与防崩溃的**系统约束**。
3. **逻辑回归 / 树模型** $\rightarrow$ **风控反欺诈和推荐排序**系统，且不需要归一化。
4. **随机森林 / XGBoost** $\rightarrow$ 分布式架构中的**高可用投票集群与残差补偿链**。
5. **朴素贝叶斯 / PCA** $\rightarrow$ 大语言模型中 **Next Token 生成与稠密 Embedding 层**的思想前置。

我们不再是机械调包的调参侠。带着这份全局的架构视野，让我们准备进入下一阶段的深水区：**深度学习与跨语言部署落地（Deep Learning Engineering）**。
