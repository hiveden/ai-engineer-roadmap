# Demos — 阶段一算法实操

## 学习目标

**阶段一的 10 个算法全部达到 L3。**

## L3 标准（到哪一级才算"学到了"）

借用全栈的类比：**L3 ≈ 后端工程师对 PostgreSQL 的掌握程度** — 能看 SQL 性能问题、能讨论索引策略，但不写 DBA 脚本。

具体到每个算法，L3 意味着：

- ✅ 能**读懂** sklearn 代码（不认识的 API 能查、能讲出这一行在干啥）
- ✅ 能**讲清楚**整个 pipeline 每一步的动机（不是"做了什么"，是"为什么非得有这一步"）
- ✅ **闭卷**能画出这个算法的数据流图 + 用自己话讲一遍
- ✅ 能判断选型：给一个业务场景，能说出"这里适合用它 / 不适合用它"

**不要求**：

- ❌ 不追求闭卷能手写 sklearn 代码
- ❌ 不要求记住 API 参数细节
- ❌ 不要求理解算法背后的数学推导

## L3 达成路径（每个算法两段式）

**第一段 · 输入补足**
- 打开对应 demo，从顶部 docstring 读到最后一行
- 看不懂的贴出来问 Claude（Claude 用工程师语言翻译 + 讲动机）

**第二段 · 闭卷输出**
- 合上代码，画一张数据流图（ASCII 箭头即可）
- 标出每个 step 的输入/输出/做了啥/**为啥非得有这一步**
- 用自己话讲一遍整个 pipeline，Claude 逐句纠错
