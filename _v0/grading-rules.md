## 1. 微观规则 · 单个概念条目评级

任何一个**可打分的概念条目**（如"KNN 五步法"、"距离几何直觉"、"反模式识别"）独立打 L0–L3。

### 1.1 四档通过标准

| 档 | 通过标准（必须**全部**满足） | 典型证据 | 降档信号 |
|---|---|---|---|
| **L0 · 没接触** | 从未听过这个名字 | — | — |
| **L1 · 听过** | 能说出**大概是什么 / 解决什么问题**（一句话）<br>**不要求讲机制** | "KNN 就是找最近邻居投票" | 名字都认不出 → L0 |
| **L2 · 读过 / 跑过** | ① **用自己的话**讲清机制（不照背定义）<br>② 在**已教过的锚定场景**（豆瓣 / Wine / 渔网）独立跑通一次<br>③ 有**一个业务类比**能焊住<br>④ 本概念的**前置基础概念**（如 i.i.d. 之于 `train_test_split`、过拟合之于正则化）已挂到 L1+ | 锚定场景下自述 4/5+ 正确；前置概念至少能一句话说出是什么 | 照背定义换场景即懵 → L1；前置概念悬空 → L1 |
| **L3 · 做过 / 能讲** | ① L2 全部满足<br>② 在**未见过的新业务场景**独立跑通<br>③ 能说出**选型理由**（"为什么这个场景选它而非 X"）<br>④ 能识别**至少一个反模式** | 新场景下决策链 4/5+ 正确；能指出反模式名 + 机制 | 答不出选型理由 → L2 |

### 1.2 评级纪律（强制）

- **禁小数**：不写 L2.5 / L2.7 / "L3 边缘" / "L2 PASS but…"。**一律二选一**。
- **讲不清就降档**：两档之间犹豫 → 往下选。**诚实 > 全面**。
- **L2+ 必须写升级证据**：一行，具体指出触发了 1.1 表里的**哪一条**通过标准。
- **禁用模糊词**：不写"熟悉 / 了解 / 知道 / 会用 / 掌握 / 精通 / 有感觉 / 差不多懂了"。
  - 参考：`~/projects/agent-lab/docs/production-playbook/knowledge-maps/README.md` 的 4 动词纪律。

### 1.3 与 Orientation 第六章 0-3 自评表的合并

原自评表的 0-3 档并入 L 前缀系统。**停用 0-3 写法，统一 L0/L1/L2/L3**。

| 旧 0-3 | 新 L |
|---|---|
| 0 完全没听说过 | L0 没接触 |
| 1 听过但讲不清 | L1 听过 |
| 2 能一句话讲清 | L2 读过 |
| 3 能工程类比 + 知道场景 | L3 做过 |

Orientation 第六章自评表的升级在下一次触及时做（不本次追溯）。

## 2. 宏观规则 · 整个算法/主题评级

整个算法（如"KNN 整体"）由多个**子概念**合成。**不压成单一数字**。

### 2.1 合成方案：分组五元组（强制）

按**六个分组**分别打分，宏观水位写成有序六元组（仍沿用俗称"五元组"以免绕口）：

```
核心机制 / 工程决策 / 工程坑 / 概念对比 / 反面教材 / 架构衔接
```

**示例**：
- KNN 整体 = `核心机制 L2 / 工程决策 L1 / 工程坑 L0 / 概念对比 L1 / 反面教材 L1 / 架构衔接 L0`
- 若某分组本 demo 不涉及，写 `—`（不是 L0——L0 表示"接触了但没学会"）

### 2.2 六个分组的定义

| 分组 | 含义 | 示例子概念（KNN）|
|---|---|---|
| **核心机制** | 算法本体怎么跑的机制层 | 五步法 / 距离几何直觉 / 样本-query-训练数据角色 |
| **工程决策** | 需要自己决定的参数 / 选型 | K 值选择 / 距离度量选型 |
| **工程坑** | 不做会翻车的工程细节 | 标准化必要性 / `fit_transform` vs `transform` 数据泄漏 |
| **概念对比** | 与相邻概念的辨析 | 分类 vs 回归 / lazy vs eager learner |
| **反面教材** | 识别反模式 / 常见陷阱 | 预过滤反模式 / KNN vs K-Means 撞名 |
| **架构衔接** | 与工程主线（RAG / Vector DB / 多 Agent）的连接 | 规模瓶颈 / ANN 三件套 / Vector DB 选型 |

### 2.3 每个 demo 在 LESSON-PLAN 里要列的东西

不在本规则里列具体算法的子概念——那属于 demo 的责任。LESSON-PLAN 应该有一节（建议 §0.X 或 §5.X）列出：

```markdown
## §0.X · 本算法子概念清单（打分对象）

| # | 子概念 | 分组 |
|---|---|---|
| 1 | 五步法 | 核心机制 |
| 2 | 距离几何直觉 | 核心机制 |
| ... | ... | ... |
```

session 打分时**按这份清单逐条打**，不即兴添加。

### 2.4 LESSON-PLAN §0.1 "目标深度" 的书写规范

原来写"目标深度 L3"太粗。按本规则应该写**分组目标**：

```markdown
| 目标深度 | 核心机制 L3 / 工程坑 L2+ / 反面教材 L2+ / 架构衔接 L2+（其余 L1 即可）|
```

每个 demo 按主线自己设定各组目标。
