## 3. Q&A 学习记录

### Q2 spot-check · 小王问"ML testing 和 JUnit 自测有啥本质不同"

**问题原文**：
> 同事小王（只懂传统 Java）问你："你说 ML 里训练完了要测试一下，这个'测试'和我每天写的 JUnit 测试有啥本质不同？" 答 2-3 句话即可。只要抓住一个核心区别就算通关：被测对象不一样。

**用户的回答轨迹**：
- 第一次："这个测试指的是我们 java 开发完的'自测'环节，而不是提测以后的测试环节。"
- 第二次："我们测试的方式是写单元测试，但是模型的测试是数据集。"
- 第三次："不知道，那个叫啥"

**用户在第二次回答里实质上已经到 Level 2**——他知道一边测代码一边测模型，只是在用"方式"层而不是"对象"层描述。Claude 该收的时候没收，硬要追问"那个被测对象的**名字**"，制造了第三次"不知道"。

**揭晓答案后用户说**："模型我知道。但是你的问题让我以为有什么特定的模块、词什么的"——证明用户从一开始就知道答案，是 Claude 的提问方式把他卡在外面。

**这一格的真实评分**：Level 2（用户早就到了）
**Claude 的失误**：用提问 bug 制造了一次伪卡壳

### Q2.5 · 用户主动捕捉到 Claude 的语义重复

用户在 Claude 用"将推理规则固化"做类比时主动追问："这个推理和你说的上线后的推理，这不是重复表述了吗？"

Claude 把"推理"在两个语境里当成两个意思用：
- 名词："推理规则" = 模型的内部映射逻辑
- 动词："上线后的推理" = 跑预测的动作

修正：把"推理规则固化"改写为"模型的内部逻辑固化"。

**这是用户给 Claude 的纠错，不是 Claude 给用户的题**。它揭示的不是用户的盲区，是 Claude 在中文一词多义的术语上没做预先拆解。
