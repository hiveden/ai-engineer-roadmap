## 3. 本次串讲快照（8 站精华保留）

> **用途**：下次复习直接读本节，不用回翻对话。每站保留「一句话锚 + 接收清单 + 核心钩子」。

### 3.1 第一站 · 03-knn（接收 §6 标准化 + §7 欠/过/泛化首焊）

**一句话锚**：KNN = 向量数据库底层的"最近邻检索"（session-11 已焊，本次不重讲）。

**§6 标准化首讲**：
- 量纲吞噬微观现象：`(0.1-0.9)² + (500-510)² = 0.64 + 100 = 100.64` → 特征₁ 被吞
- 业务冲突：你要 (b) 两特征平起平坐，KNN 默认给 (a) 数字大的独裁
- 解法：**特征缩放 = 把所有特征拉到同一把尺子上**
  - 归一化 Normalization / Min-Max：`(x-min)/(max-min)` → `[0, 1]`
  - 标准化 Standardization / Z-score：`(x-mean)/std` → 均值 0、std 1
- 选型：**不知道选啥 → 标准化**（对离群点稳健，工业默认）
- 关键澄清：`[1,2,3,4,5,1000]` → Min-Max 把前 5 个挤成一坨；Z-score 还能看出顺序
- 工程 API：`StandardScaler().fit_transform(X)`

**§7 欠/过/泛化首焊（载体 = K 值）**：
```
K=1   → 过拟合（HashMap 死记，被噪声带偏）
K=5   → ✅ 泛化（甜蜜点）
K=n   → 欠拟合（SELECT AVG 一锅端）
```
- 记忆钩：过拟合 = 记太死 / 欠拟合 = 学太粗 / 泛化 = 考试成绩 ≠ 刷题成绩
- 三件套合体表：训练集超好 + 新数据很差 = 过拟合

### 3.2 第二站 · 04-decision-tree（接收 §6 选择+组合 + §7 加强）

**一句话锚**：决策树 = 机器自动长出来的 Drools 规则引擎（if-else 自动化）。机制（熵/信息增益/基尼）L3 未展开。

**§6 特征选择**：
- `clf.feature_importances_` → 每字段 `[0,1]` 打分，总和 = 1
- 原理直觉：被选得越早/多/分得越干净 = 越重要
- 工程锚：MySQL 建索引看选择度
- 坑挂名：**高基数字段（user_id）有偏**，建议 Permutation Importance / SHAP

**§6 特征组合**：
- SQL 表例：性别、品类单列看不出规律；`CONCAT(性别, '_', 品类)` 后 `男_数码` 高 / `男_美妆` 低一眼出
- 三种拼法：连接 / 相除（心跳频率）/ 相乘（price × qty）
- 工程锚：ETL 衍生字段换个名字

**§7 加强（换锚）**：
```
树深度 = ∞（不剪枝）→ 过拟合 ↔ KNN K=1
树深度 = 合适       → 泛化   ↔ KNN K=中
树深度 = 1          → 欠拟合 ↔ KNN K=n
```
- **剪枝 Pruning** = `max_depth=5` 这类参数
- 回焊高阶直觉：**K 值 和 树深度 都是"模型复杂度旋钮"**

### 3.3 第三站 · 05-random-forest（接收 §6 重要性延续 + §7 偏差方差）

**一句话锚**：RF = 训 N 棵决策树投票 = **群体智慧**。每棵树用不同数据子集 + 特征子集（所以"随机"）。这套叫 **Bagging / 集成学习 Ensemble**。

**§6 特征重要性延续**：
- `rf.feature_importances_` = 100 棵树的 importance 平均 → 比单树稳
- 直觉：一个人看错，100 人平均抵消

**§7 偏差 / 方差**：
- 打靶锚：**偏差 = 瞄得准不准；方差 = 射得稳不稳**
- 映射：高偏差 = 欠拟合；高方差 = 过拟合
- RF 的数学卖点：**Bagging 降方差**（偏差不变 + 方差降低）
- 单树：低偏差 + 高方差（易过拟合）；RF：多树平均抵消抖动

### 3.4 第四站 · 06-xgboost（接收 §7 正则化 L1/L2）

**一句话锚**：XGBoost = 串行训 N 棵决策树，每棵学前一棵的错。
- RF (Bagging)：100 棵并行投票 → 降方差
- XGBoost (Boosting)：100 棵串行纠错 → 降偏差+方差
- 业务位：Kaggle 表格数据王者 / 反欺诈 / 推荐排序生产主力
- 机制（梯度提升）L3 未展开

**§7 正则化 L1/L2**（治过拟合第三招）：
```python
XGBClassifier(reg_alpha=0.1,    # L1
              reg_lambda=1.0)   # L2
```
| | L1 (Lasso) | L2 (Ridge) |
|---|---|---|
| 罚法 | `Σ|w|` 绝对值和 | `Σw²` 平方和 |
| 效果 | 参数直接归零 | 整体变小但都留 |
| 工程锚 | **断舍离**（drop 列） | **限流**（别太大声） |
| 副产物 | 自动特征选择 | 更稳更平滑 |

**三招治过拟合对照**（核心高阶直觉）：
```
KNN     → 选 K（旋复杂度旋钮）
决策树   → 剪枝（砍深度）
XGBoost → 正则化 L1/L2（加罚款）
```
都在干同一件事：限制模型复杂度。

### 3.5 第五站 · 07-1-pca（接收 §6 特征降维）

**一句话锚**：PCA = 数据压缩（像 JPEG/ZIP）—— 100 列压成 10 个合成列，保留 95% 信号。业务锚：彩色→黑白（丢颜色留形状）。机制（方差最大化）L3 未展开。

**§6 特征降维**：
- 动机：KNN 维度诅咒（>50 维崩）/ 768 维 embedding 想进 KNN / 可视化（→2D）
- **特征选择 vs 特征降维**（核心对比）：
  - 选择 = `SELECT a, b, c` → 原字段子集（可解释保留）
  - 降维 = `SELECT weighted_avg(...)` → 合成字段（可解释丢失）
- 场景：embedding 降维 / 可视化 / 训练提速；金融审计别用（合成列法务看不懂）

### 3.6 第六站 · 07-2-kmeans（重锚 §4 无监督）

**一句话锚**：K-Means = 没标签时自动把数据分 K 个簇。业务锚：用户分群 / 冷启动推荐 / 异常检测。

**§4 监督 vs 无监督回焊**：
- 监督（前 6 站所有算法）= X + y 预测
- 无监督（K-Means / PCA）= 只有 X 找结构
- 切换信号：有历史标签 → 监督；只有行为数据想看看长啥样 → 无监督

**⚠ 撞车陷阱（高价值钩子）**：
```
KNN       K = 邻居数（监督分类）
K-Means   K = 簇的数量（无监督聚类）
```
完全不相关，只是名字撞车。

### 3.7 第七站 · 08-naive-bayes（无大纲接收 · L1 最浅）

**一句话锚**：朴素贝叶斯 = 根据词频算概率的分类器。经典场景：垃圾邮件过滤。

**"朴素"含义**：假设所有特征彼此独立（现实通常不成立，但实战效果不差）。工程师锚：一种"粗算"。

**深度配置**：`01-LESSON-PLAN §2.3` 标记 L1——"概念重要，落地场景少"。现代 NLP 被 Transformer/LLM 取代，但小场景、轻量部署仍有用。

### 3.8 第八站 · 09-time-series（反例重锚 §3 IID）

**一句话锚**：时序预测 = 历史趋势预测未来。业务：电力负荷 / 股价 / 销量。模型：ARIMA / Prophet / XGBoost+滑窗。

**§3 IID 回焊（用时序当反例）**：
- 前 7 站算法都**隐含 IID 前提**
- 时序违反 IID：① 今天股价和昨天强相关（不独立） ② 2008 vs 2024 销售不是同一分布
- 典型陷阱：用 `train_test_split(random_state=42)` 切时序 = **特征泄漏**（测试集混着未来，训练集已经看过）
- 正解：按时间切 `df.date < '2024-01-01'` 训练 / `>=` 测试
