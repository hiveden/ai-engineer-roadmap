# Learning Sessions · 学习会话日志

> 这个目录是 ai-engineer-roadmap 项目的**元认知工具**——
> 它存的不是"学了什么"（那是各 stage 章节的事），
> 而是"**怎么学的**" + "**学的过程中我注意到了自己什么**"。

---

## 为什么要写学习会话日志

普通学习者：学完就忘。
写笔记的学习者：知识沉淀，但学习方法不进化。
**写会话日志 + 元反思的学习者**：知识 + 学习方法**双轨进化**。

第三种是少数人，且永远跑得最快。本项目作者主动选择第三种路径。

---

## 文件命名约定

```
learning-sessions/YYYY-MM-DD-session-NN.md
```

- `YYYY-MM-DD`：会话日期
- `NN`：当天的第几次会话（从 01 开始）

例：`2026-04-07-session-01.md` = 2026 年 4 月 7 日的第 1 次会话

---

## 单次会话报告的固定模板

每份会话日志包含 7 个固定字段（前 6 个由 Claude 生成草稿，第 7 个由用户填写）：

1. **元信息**：日期、时长、状态、触及阶段、触及文档
2. **本次涉及的概念**：清单
3. **Q&A 学习记录**：每题包含 第一次回答 → 纠错点 → 最终版本 → 评分
4. **暴露的盲区**：还不知道的部分
5. **自评分变化**：表格对比 before/after（同步到 00-Orientation 第六章）
6. **下次起点**：下次会话从哪里继续
7. **💡 元反思**（**用户填写**）：
   - 今天我注意到自己的学习模式上有什么？
   - 今天最让我意外的发现是什么？
   - 下次我想试着改变什么？

完整模板见 [`_template.md`](./_template.md)。

---

## 触发时机

会话日志**不是为了凑数**，只在以下情况生成：

- 用户主动喊"暂停"或"今天到此"
- 完成一个完整的学习里程碑（比如"问题 N 通关"）
- 注意力窗口结束、需要切换上下文
- 单次对话信息密度过高，需要落地存档

> 不要为了写日志而写日志——只在真有学习内容值得回顾时写。

## 工作流

| 阶段 | 谁做 | 备注 |
|---|---|---|
| 会话进行中 | Claude 正常教学 | 不分心写日志 |
| 会话暂停/结束 | Claude 生成报告草稿（前 6 节） | ⏱ 5-20min |
| 用户接手 | 填写第 7 节「元反思」 | ⏱ 5-10min |
| 收尾 | 提交到 git | 1 commit / session |
