# demo-02 知识点沉淀 · 分类 vs 回归的"置信度"问题

> 对应对话：demo-02 step3 跑完后，分类模型的输出多了一个 `置信度: 87.9%`，demo-01 回归没有。这篇讲为什么。

---

## 一句话

**分类模型能直接吐置信度，回归模型不能。** 因为两种模型输出的东西本来就不是一个类型。

---

## 看代码最清楚

### 分类（demo-02）

```python
model = LogisticRegression()
model.fit(X_train, y_train)

model.predict(X)        # → [1, 0, 1, ...]         给类别：良性/恶性
model.predict_proba(X)  # → [[0.12, 0.88], ...]    给每个类别的概率
```

`predict_proba` 返回一个二维数组：每一行对应一条样本，行里的数字是**这条样本属于每个类别的概率**，加起来等于 1。

```
[0.12, 0.88]
 ^     ^
 |     |
 恶性   良性
 概率   概率
```

**置信度 = 这行里最大的那个数**。所以 `[0.12, 0.88]` 的置信度就是 88%——模型认为"这是良性"，而且有 88% 的把握。

### 回归（demo-01）

```python
model = LinearRegression()
model.fit(X_train, y_train)

model.predict(X)        # → [2.35, 1.87, ...]      给房价预测值
# model.predict_proba(X) 不存在，调了会报错
```

回归模型只有一个 API：给 X，吐一个数字，没了。

---

## 为什么回归不能直接给置信度

因为"置信度"这个概念对"数字"不好定义。

想象你要给 `房价 = 2.35 万` 加一个 `置信度 = 87.9%`——87.9% 指什么？

- 指"房价正好等于 2.35 的概率"？连续数字取任何精确值的概率都是 0，这条死胡同。
- 指"房价在 2.35 附近某个范围的概率"？这个叫**预测区间**（比如"有 87.9% 概率房价在 2.05 ~ 2.65 之间"）。**这个说法成立**，但你要把这个区间算出来，得加额外的技术——不是 LinearRegression 一行代码能出来的。

**结论**：
- 分类的置信度：`predict_proba` 直接给，开箱即用
- 回归的置信度：需要额外技术造（名字叫 bootstrap / quantile regression / conformal prediction，**这些名字你现在不用懂，知道"要额外造"就够了**）

---

## 类比你熟的东西

| 分类模型 | 类比 |
|---|---|
| `predict_proba` 返回概率 | `SELECT COUNT(*)` 返回行数——调了就有 |

| 回归模型 | 类比 |
|---|---|
| 想要置信度要自己造 | "这条数据新鲜不新鲜" Redis 不会告诉你，你得自己存时间戳 |

---

## 为什么这一点要记住

你以后搭系统时，碰到一个节点问"这次预测靠不靠谱"：

- **分类式节点**（判断意图/分拣/打标签）→ 有现成置信度，拿来就用
- **回归式节点**（预测数量/延迟/金额）→ 现成没有，架构设计时要预留"算置信度"的那块工作量

**这一条记住就行，其他名词都先不用管。**

---

## 逻辑回归名字带"回归"但干的是分类

顺带一个坑：`LogisticRegression` 名字里有 "Regression"，但它是**分类模型**。
- `LinearRegression` → 回归（输出数字）
- `LogisticRegression` → 分类（输出类别 + 置信度）

以后翻 sklearn 文档看到 "Regression" 不要直接当回归，看它 `fit` 完用 `predict` 吐的是什么。
