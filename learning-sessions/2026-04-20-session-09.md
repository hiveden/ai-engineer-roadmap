### 主线 Q1 · Recall / Precision 的分母分别是什么？

**第一次回答**：
> Recall 的分母是 实际确诊恶性肿瘤的总人数；Precision 的分母是 实际确诊良性肿瘤的总人数。

**纠错点**：
- ✅ Recall ✔
- ❌ Precision 视角错位——把它当成"实际某一类"，应该是"**模型预测为阳性**的总数"。session-08 标 L2 PASS，本次复查**退回 L1**——暴露了记忆钩子没焊牢。

**补课路径**（非一次讲完，两次换类比才啃下）：
1. 狙击手类比（"开枪次数"）→ Alex 不买账
2. SQL `WHERE 模型预测=恶性 → 返回 40 行`——全栈秒懂但嫌抽象
3. **渔网捕鱼**——Alex 当场 "这个例子好很多。记住"

**最终版本**（Alex 用自己的话复述）：
> Precision 本次撒网的数据，本次撒网拉出来 60/100；实际池塘有鱼 80 拉出真鱼 60，60/80。

**最终评分**：Level 2 PASS（渔网版已焊住；下次再问若还答对即可确认留存）

### 分支 A · "P/R 是不是有数据上的计算关联？"（Alex 自主提问）

**Alex 原话**：
> "我之前总以为他俩是有关系的，但是其实好像没有数据上的计算关联？"

**💎 金句**：这是一个**学生自己挖出来**的高价值元问题——不是 Claude 诱导的。触及"公式看不出的、但实战中天天打架"的真实关系。

**Claude 回答要点**：
- 公式上弱关联：共享分子 TP，分母独立（FP / FN）
- 实战上强关联：通过**决策阈值**这个旋钮，P 和 R 互相挤压（跷跷板）
- 极端推演：阈值 0 → R=100% P→基线；阈值 1 → P→100% R→近 0

**评分**：Level 2+（Alex 能独立发问 → 表明 P/R 的"独立指标 ≠ 独立指标"直觉已形成）

---

### 分支 B · "决策阈值 threshold 又是什么？"

**工程锚**：`if errorRate > 0.05: sendAlert()` 里的 0.05 即阈值。
**demo-02 代码锚**：`model.predict()` ≈ `model.predict_proba()` 取第二列 ≥ 0.5 判阳性——**0.5 是 sklearn 默认阈值，不是定理**。

**Alex 即时复测**（垃圾邮件场景，阈值 0.9）：
> "变脏了，但是不会漏掉重要邮件？"

**纠错点**：
- ✅ 两个判断都对
- ⚠️ "不会漏掉"的"漏"字在中文里默认指 FP（重要邮件被误判进垃圾箱），容易和医院场景的"漏"（FN）混淆。点破了 **"漏"字在不同场景对应不同指标**——永远先问"阳性是谁"。

**评分**：Level 2（阈值、跷跷板、阳性定义三层同时在答案里体现）

---

### 分支 C · "肿瘤和垃圾邮件阈值方向相反，所以业务需求也是相反的？"（Alex 核心困惑）

**💎 金句**：
> "我晕的是 肿瘤诊断 和 垃圾邮件 的业务场景，设定同样的阈值 threshold，但是业务需求是相反的？"

**这是本次会话含金量最高的一题**。Alex 自己把"阈值方向相反"的表面矛盾抽了出来。

**Claude 回答要点**：
- **不矛盾，是一致的**：两场都在做同一件事——**"把更贵的那种错误压到最低"**
- 医院 FN（漏诊）贵 → 保 Recall → 阈值低；邮件 FP（误判）贵 → 保 Precision → 阈值高
- 方向看起来反过来，是因为**贵的错误类型反过来了**

**决策链抽出**：
```
阳性定义 → 错误代价（FN / FP 哪个贵）→ 指标选型（R / P / F1）→ 阈值方向
```

**评分**：Level 3 边缘（能把两个场景的"表面矛盾"抽成"一致框架"——这是**抽象能力**本身，不是单点知识）

### 分支 D · 阈值 vs F1 角色混淆

**Alex 原话**：
> "F1 又是什么，就是这个阈值吗？"

**纠错点**：
- ❌ 混淆"输入"和"输出"：**阈值是你转的旋钮（输入）**；**F1 是仪表合成的综合分（输出）**
- 汽车类比：阈值=油门，P/R=速度表+油耗表，F1=整车评分

**最终版本**：Alex 口头接受"角色不同"，未展开 F1 公式 → 留待下次。

**评分**：Level 1（会区分角色，但 F1 本体没学）
