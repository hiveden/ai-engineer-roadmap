## 3. Claude 自己的错（下次备课弹药库）

### 错误 1 · 数学讲解脱离 KNN 上下文

- **表现**：03-distance 加单位圆 / Lp 形态光谱 / 范数对照表 / 00-向量与范数前置文档 / 00-高斯分布前置文档
- **被点破**："已经完全脱离了业务、直觉、KNN，纯讲数学？"
- **教训**：在算法柱子站讲数学，数学只是 KNN 解决问题的工具，不展开纯数学概念

### 错误 2 · Sparring 撸文从 PPT 标题拓展到 ~2000 字

- **表现**：鸢尾花案例 PPT 给 8 行 6 步标题，我编出完整 sklearn 教程含 fit-transform 铁律 / data leakage / 量纲对比表 / 流程图 / 关键启示
- **被点破**："我只复制给你一点，你哪里拓展这么多？"
- **教训**：PPT 给少则写少，不主动从 sklearn 官方拓展工程经验

### 错误 3 · 底稿段含 Claude 补充

- **表现**：04-平票加权投票.md 底稿段含 3 个我补的 LaTeX 公式 + 衔接句
- **被点破**："找到你自己编的原文内容，清理"
- **教训**：底稿段 = PPT 原话；Claude 补全用 `〔公式〕` 占位 + 独立段明示

### 错误 4 · 跨章节 metadata 引用

- **表现**："已在 02 详细展开（含 1/2/3/N 维推广 + 实算）" / "假设你已读过 02 接近程度（欧氏距离主场）"
- **被点破**："不应该建立这种跨章节的关联关系"
- **教训**：案例延续 OK，metadata 注释（"已在 X 详细展开 / 这里作为基线"）不允许

### 错误 5 · 业务/数学/代码三层混杂

- **表现**：γ 重写时把 KNN 业务 + 数学公式 + sklearn 代码混在一段
- **被点破**："业务、数学、代码全都混一起了？"
- **教训**：三层必须分明，每层各自有图

### 错误 6 · 图表删除 / 缺失

- **表现**：γ 重写删了所有 ASCII 图；03-distance 简单文字版没图
- **被点破**："缺少图表" / "我怎么感觉你在偷懒呢"
- **教训**：每个文件主动加 ASCII 图（业务图 + 数学图 + 流程图），不等提醒

### 错误 7 · 工程师视角锚点

- **表现**："向量 = list / 数组（工程师视角）"
- **被点破**："删掉工程师视角，从入门数学的角度开讲"
- **教训**：数学概念用入门数学语言，不用工程语言"翻译"

### 错误 8 · 超 PPT 主场补充工程经验

- **表现**：鸢尾花案例加 fit-transform 铁律 / data leakage（PPT 没提的工程经验）
- **被点破**：超 PPT 主场
- **教训**：明确"## 不在本章范围"段划界，不补 PPT 之外的工程经验

### 错误 9 · 提前预设元数据

- **表现**：章节骨架文件含状态板 / 待填占位 / 横向引用清单 / "v1 试行 · 章节骨架待填"
- **被点破**："文件结构和内容即架构表达，不要徒增管理成本"
- **教训**：内容即架构，加 H2 / 加段落只在有真实内容时

### 错误 10 · 一次给太多决策候选

- **表现**：一次给 27 条规则候选 / 8 条修改清单等
- **被点破**："过拟合了，抽象规则"
- **教训**：核心抽象优先，具体实现细节下沉到工具箱
