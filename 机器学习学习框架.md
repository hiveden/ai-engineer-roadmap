## 一、需求背景

### 学习者情境

- **学习目标**：机器学习是为后续 NLP 自然语言处理和大模型开发打基础，不是终点。
- **学习方式**：跟随成熟课程体系自学。
- **课件特征**：课件按"算法章节"组织，每章内部把数学概念、数据处理、损失函数、评估指标等内容**穿插**讲解。课程整体策略是"前期把所有概念铺开，后续章节反复复用"。
- **时间约束**：每个算法不可能花 2-3 天精雕，必须有可压缩的节奏，但不能牺牲后期复利。
- **能力前提**：已具备编程基础，第一轮目标是建立可被反复复用的基础，不是一次吃透。

### 核心矛盾

课程**按纵向（算法）组织**讲授，但知识结构本身**是纵横交织的网**——很多关键基础（损失函数、梯度下降、评估指标、特征工程）属于多个算法共用的横向地基，不属于任何单个算法。

如果按课件章节直接做笔记，结果是：
- 同样的概念在每个算法章节里被重新讲一遍、重新记一遍；
- 横向基础始终碎片化，无法沉淀；
- 学到第 N 个算法时仍在原地补基础，速度不会加快；
- 进入深度学习和 NLP 阶段时，机器学习阶段的投入无法转化为复利。

第一轮学习的核心矛盾即：**课件按纵向讲，但知识必须按横向记才能复利**。

### 已排除的备选方案

在框架成型前，以下方案被讨论并排除：

- **先把所有概念过一遍再进入算法**：概念脱离算法上下文过于抽象，记不住、不知重点。
- **完全自下而上（先学完所有数学再学算法）**：周期过长、动机衰减、与课件结构脱节。
- **依赖外部评估者（包括 LLM）判定掌握程度**：LLM 不可作为掌握度的判官，会产生幻觉。掌握度只能由可观测的二元行为（手算结果对错、代码跑通与否）来判定。
- **每个算法去 Kaggle 找真实数据训练**：第一轮能力不足以应对真实数据复杂度，挫败感和沉没成本超过收益。第一轮应使用 sklearn 自带数据集，第二轮再上 Kaggle。

## 二、设计理念

以下五条原则贯穿两个框架的设计，是判断任何具体方案是否合格的标尺。

### 原则 1：对"掌握"的定义必须同时支撑四件事

学、教、测、停四件事（规划路线、给别人讲、出题自测、判断够用）共享同一个底层结构——都需要回答"什么算掌握了某个东西"。

一个真正立得住的"掌握"定义，应当能同时被这四个方向调用而不冲突。本框架中所有判停标准、循环步骤、顺序设计，都必须能在这四个方向上自洽。

### 原则 2：横向地基比纵向柱子更承重

每根纵向算法柱子都同时踩在数学语言、特征工程、模型训练、模型评估这四层横向地基之上。算法章节真正"独有"的内容通常只占 20-30%，其余 70-80% 是对横向基础的调用。

设计上的直接含义：**横向先于纵向、横向独立成章、横向越积越厚、纵向越积越薄**。

### 原则 3：横向不一次性建完，按需随柱子推进

横向四层完整建立周期过长，且脱离算法上下文学习效率低。

正确做法是先建立"够支撑第一根柱子"的最小启动包，剩余横向内容随后续柱子的需求逐步追加。每一层横向基础应在它**第一次被某根柱子需要**之前刚好准备就绪，不提前太多、不滞后一步。

### 原则 4：判停采用二元标准，不打分

掌握度的判定不使用百分制评分，因为分数依赖评估者主观判断，且容易自欺。

所有过关标准必须可还原为**二元行为判定**：手算结果是否与标准答案一致、代码是否跑通、解读方向是否正确。LLM 可用于辅助反馈，但只采信它指出的具体卡点，不采信它给出的总分。

### 原则 5：第一轮目标是建立可被复用的钩子，不是吃透

课程设计本身就是"前期铺开 + 后续反复复用"，所以第一轮的合格标准是"第二轮再次见到时能认出来、能调用"，不是"能从零推导、能证明定理"。

设计上的直接含义：第一轮的过关标准必须明确**到此为止**的边界，避免在某个算法上无限纠缠。

## 三、框架一：知识结构框架

回答"机器学习这门知识应该如何切分"。

### 基本切分：横向 vs 纵向

- **纵向 = 算法**。每个算法是一根独立的柱子，特点是离散、可枚举、可替换。
- **横向 = 算法之间共用的基础设施**。不属于任何单根柱子，特点是连续、累积、不可替换。

### 横向四层结构

横向基础不是平的，它有从底到顶的依赖层级：

| 层级 | 名称 | 内容范畴 |
|------|------|---------|
| 第一层 | 数学语言层 | 线性代数、微积分、概率统计 |
| 第二层 | 特征工程层 | 数据预处理、特征构造、特征选择、特征降维 |
| 第三层 | 模型训练层 | 损失函数、优化方法、训练流程、正则化 |
| 第四层 | 模型评估层 | 评估指标、误差诊断、模型对比 |

每层内部还有子模块。例如"数据预处理"是第二层"特征工程"下的一个子模块，不应与"特征工程"平级并列。

### 纵向柱子结构

纵向柱子按算法依赖关系排序，每根柱子都同时踩在横向四层之上。柱子内容大致分为：

- **复用部分（70-80%）**：调用已建立的横向基础。
- **独有部分（20-30%)**：该算法特有的核心机制。

### 框架检验标准

知识结构框架是否真正建立，有一个客观检验：

> 学到第 N 个算法时，新增笔记的总量是否显著小于第一个算法。

- 若是 → 横向积累在起作用，框架成立。
- 若否 → 仍在按纵向重复学习，框架未建立。

## 四、框架二：学习方法论框架

回答"如何把知识结构框架转化为可执行的学习行为"。

由三个相互独立、相互啮合的子框架组成：

### 子框架 A：顺序框架

回答"什么时候学什么"。

由两个子结构组成：
- **横向推进规则**：横向四层不一次性建完，按"最小启动包 + 随柱子追加"的方式推进。
- **纵向推进规则**：柱子按依赖关系排序，前一根柱子建立的横向积累成为后一根柱子的入场前提。

输出形式：**线性时间轴**。

### 子框架 B：循环框架

回答"每个学习单元怎么走完一圈"。

由两层循环嵌套：
- **内循环**：每根纵向柱子内部的固定步骤，所有算法共用同一套步骤，不为每个算法重新设计。
- **外循环**：每完成若干根柱子后的横向整理回顾，把散落在各章的横向内容合并归位到横向四层中对应的模块。

输出形式：**可重复套用的流程模板**。

### 子框架 C：判停框架

回答"什么时候算学完、什么时候该进入下一单元"。

由三个判停点组成：
- **单元内判停**：内循环走完一圈，是否达到该柱子的过关标准。
- **横向判停**：外循环整理后，横向四层的对应模块是否完成本阶段的厚度积累。
- **阶段判停**：决定是否进入下一阶段（如是否结束第一轮、进入第二轮）。

输出形式：**二元判定标准（过 / 不过），不是分数**。

### 三个子框架的关系

| 子框架 | 决定 | 缺失后果 |
|--------|------|---------|
| 顺序 | 走哪条路 | 知道学什么但不知道怎么学 |
| 循环 | 怎么迈步 | 每个单元都走完流程但互不支撑 |
| 判停 | 何时停何时进 | 永远不知道学完没有 |

三者缺一不可。

## 五、边界声明

### 本文档已包含

- 需求背景与核心矛盾的识别。
- 五条贯穿性设计原则。
- 知识结构框架的层级骨架（横向四层 + 纵向柱子）。
- 学习方法论框架的三个子框架骨架（顺序 / 循环 / 判停）。

### 本文档暂未包含

以下内容属于"框架确认后下一步填充"的范畴，刻意未在本文档中展开：

- 顺序框架的具体时间轴（哪个算法在哪一阶段、横向最小启动包包含什么内容）。
- 循环框架的具体步骤（内循环每一步做什么、外循环整理什么）。
- 判停框架的具体判定项（每个单元、每层横向、每个阶段的具体过关条件）。
- 各横向模块和纵向算法的具体知识点清单。
- 测试题目、测试方式、过关分数线等可执行细节。

### 后续推进方式

框架部分确认无误后，按以下顺序分别填充：

1. 先填**顺序框架**——产出线性时间轴。
2. 再填**循环框架**——产出可套用的流程模板。
3. 最后填**判停框架**——产出每个判停点的二元标准清单。
4. 在三个子框架填充完成后，再回到知识结构框架，把每层横向、每根纵向柱子的具体知识点纳入清单。

填充顺序遵循"先骨架、后血肉，先方法、后内容"的原则，避免在框架未稳时陷入知识点细节。
